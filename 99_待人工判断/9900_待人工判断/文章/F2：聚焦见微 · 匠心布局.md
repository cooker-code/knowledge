> AntV F2 作为移动端专业图表库，始终致力于为用户提供极致的图表体验和强大的数据洞察能力。2025年，我们再次聚焦用户痛点，围绕“数据聚焦、布局优化”做出关键升级，并通过跨端适配与开发者工具提升整体交付效率，让移动可视化既精准又高效。

### 🌟突出数据

#### 堆叠柱状图的数据“聚光灯”

堆叠柱状图在展现整体构成时，往往面临类别间比较困难、信息复杂的问题。我们设计了点击图例，高亮相同类别的矩形。

#### 折线图的“微观世界”

在数据过多的折线图下，常常需要突出或展示某个范围内的走势。类似PC端图表鱼眼、brush的效果。用户可以轻松聚焦任意时间段，深入分析其内部波动，发掘隐藏的规律。

### 🎨 智能布局

随着大模型技术的快速发展，大理财业务积极探索金融图表的智能生成能力——从底层数据挖掘到可视化呈现，全面迈向智能化。然而，数据类型的日益多样与图表样式的高度复杂，给图表布局带来了巨大挑战。为此，我们构建了一套智能图表布局引擎，并制定了多层次的布局优化策略，在运行时动态检测并修复布局不合理问题，将一次生成通过率从60%提升到80%以上，显著提升了图表生成的质量。

#### 饼图标签

对饼图标签进行了全面升级，引入全新的 **「Spider 布局」**。告别传统三段式绘制，现在采用更简洁的二段式设计，并实现了：

* **「自动确定绘制个数：」** 智能判断最佳标签显示数量。
* **「象限智能调整：」** 根据扇区位置自动优化标签方位。
* **「标签上下位置微调：」** 避免重叠，保持清晰。
* **「拐线自动优化：」** 连接更自然，视觉效果更佳。

通过一系列算法，确保饼图在复杂数据场景下依然保持优雅与易读。

#### 坐标轴自适应

当坐标轴文本过长或过多时，遮挡问题常常困扰着用户。今年我们在坐标轴组件中加入了自动检测机制，并提供两种自适应策略：

* **「自动旋转：」** 根据文本高度和间距，自动计算并应用最佳旋转角度，在有限空间内完整显示文本。
* **「自动隐藏：」** 在保证刻度均匀和首尾完整的前提下，自动隐藏部分刻度，确保重要信息不丢失，同时避免拥挤

这大大提升了图表在不同分数据密度下的可读性。

#### 图例紧凑型布局

图例的老布局对于多行的情况下，保证宽度固定，可以更美观。但对于某项文本过长的情况，该布局方式空白过多，占用空间较大，且看起来间隙不同，且容易换行。因此 引入了 `adaptive` 布局，能更合理地利用空间，减少不必要的空白，确保图例在长文本场景下依然美观、紧凑，提升整体布局的协调性。

#### 标注定位

解决了过去堆叠/分组柱状图标注定位不准的问题。通过对分组比例尺调整的精确考量，现在标注能精准定位到数据块的中心，提供更专业、更可靠的数据呈现。并且提供了 `precise` 属性以确保向前兼容性。

### 🚀 跨端

#### 小程序

在小程序生态中，我们新增适配了 WebCanvas，与原有 NativeCanvas 形成“双 Canvas 模式”。

NativeCanvas 虽然 API 对齐 Web 标准，但同层组件的通信耗时在复杂项目中常成为性能瓶颈。WebCanvas 的机制恰好在此类场景下展现出更优的总耗时优势。我们在余额宝首页简单柱状图做以测试：

F2 抹平了两种 Canvas 的 API 差异，确保同一套图表代码能同时运行在 NativeCanvas 和 WebCanvas 上。开发者可根据具体场景，灵活选择最适合的 Canvas 类型，平衡性能需求与开发效率。

#### Cube

在移动端跨平台技术上，除了已经对外的小程序，面向内部区块级，我们还有类 Flutter 蚂蚁内部跨端方案，Cube 卡片。得益于终端团队提供的 CubeCanvas，我们也适配了 F2Cube，让 F2 在 Cube 上也可以绘制图表，进一步拓展了 F2 的应用边界，为更多内部业务场景提供强大的数据可视化能力。
