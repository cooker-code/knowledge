将军赶路不追小兔，目标感，是你和其他人最大的差距。

十年前，我刚从技术骨干转成小组长。

第一次负责一个跨部门项目，我信心满满。结果第一周，我就被各种事情搞得焦头烂额：

* 产品经理需求改了三版，我在群里和他吵了两个小时
* 测试同事提了一个我认为很低级的bug，我私下吐槽他能力不行
* 设计师的稿子延迟了两天，我追着问了三遍为什么
* 甚至前台小姐姐   他攻坚会议室订错了，我都忍不住发了句牢骚

三周后，项目延期了。老板问我原因，我滔滔不绝讲了二十分钟：产品不专业、测试不靠谱、设计拖后腿、行政支持不到位……

老板听完，只问了我一句话：所以，你的目标是什么？

我愣住了。是啊，我的目标是什么？是让项目按时上线，还是证明同事都不够专业？

那一刻我明白了一个道理：将军赶路，不追小兔。

# 二、什么是将军赶路，不追小兔

这句话不是我发明的，是我在一次项目复盘会上，一个老领导说的。

他当时原话是：你们这些人，不是在解决问题，是在解决情绪。将军赶路，哪有功夫追路边的小兔？

我后来想了很久，什么叫小兔？

小兔，就是那些让你分心、让你愤怒、让你想证明自己是对的，但对目标毫无帮助的事。

|  |  |
| --- | --- |
| 你的目标 | 路边的小兔 |
| 项目按时交付 | 产品经理说话态度不好 |
| 团队能力提升 | 某个同事代码风格和你不一样 |
| 拿到晋升名额 | 同事在会议上抢了你的话 |
| 搞定一个难搞的客户 | 客户方对接人学历没你高 |
| 完成年度技术规划 | 老板临时加了个会占用了你两小时 |

小兔的特点是：它真实存在，它让你不舒服，但它和你的目标无关。

追小兔的代价是什么？是你的注意力、你的情绪、你的时间，全部被消耗在无关紧要的事情上。等你回过神来，将军的大部队已经走远了。

# 三、我见过太多人，死在追小兔上

## 案例一：挑刺的技术大牛

我认识一个技术非常厉害的大牛，代码写得漂亮，架构设计一流。但他有个毛病——看不惯任何人。

产品经理不懂技术，他要在群里怼；

测试用例覆盖率低，他要写长文批评；

甚至新来的实习生变量命名不规范，他都要讲半小时。

五年后，他还是高级开发。同期的人，有的成了架构师，有的成了技术负责人。

他很不服气，觉得公司不识货。

但真相是：没人愿意和一个时刻准备挑刺的人合作。

他的技术能力被他的小兔追逐症完全抵消了。

## 案例二：我赢了道理，输了目标。

多年前，我负责一个核心系统重构。项目中期，发现一个兄弟团队提供的接口延迟超标。

我当时的第一反应是愤怒，拉群、抱怨、抄送双方老板，花了整整两天论证这是他们的责任。

结果？接口问题确实在他们那边，但他们修复花了三天。

而我的两天，本来可以用来做兜底方案——加一层缓存、做一个异步降级，任何一件事都能让项目不延期。

我赢了道理，输了目标。

# 四、带着目标做事，到底是什么状态

不是不关注过程，而是过程中的所有因素，都要被放在目标这个天平上称一称。

我现在的做事方式：

第一步：写下来，我的目标是什么

不是脑子里想想，是写下来。一行字，贴在显示器边上。

比如：本周目标：完成订单模块重构，上线预发环境，P0级bug为0。

第二步：遇到任何干扰，先问一句：这和我的目标有关吗

产品改需求了——有关吗？

如果能让目标更好达成，改。如果只是他拍脑袋，拒绝，但不用吵架。

同事不配合——有关吗？

如果不配合会导致目标失败，想办法推动，而不是抱怨他态度不好。

有人质疑我的方案——有关吗？

如果质疑有理，优化方案。如果质疑无理，解释一次，不纠缠，继续赶路。

第三步：每天睡前，复盘今天的时间花在哪了

我会问自己三个问题：

* 今天做的哪件事，离目标最近？
* 今天被哪只小兔追了？花了多少时间？
* 明天如何避开这只小兔？

# 五、将军赶路，不是让你做老好人

很多人误解这句话，觉得不追小兔就是忍气吞声、什么都顺着来。

不是的。

将军不追小兔，是因为小兔不值得追。但如果前面有一只老虎拦路，将军必须拔剑。

什么是老虎？

* 直接影响目标达成的关键障碍
* 团队里真正破坏协作的毒瘤
* 触碰底线的事（诚信、安全、合规）

判断标准：这件事不解决，我的目标就完不成。

如果是老虎，必须打，而且要快、要狠、要一次到位。

如果是小兔，看一眼，继续走。

最难的是：很多人把老虎当成小兔（逃避问题），或者把小兔当成老虎（过度反应）。

# 六、一个习惯

我现在带团队，每周一早上有一个固定动作：

让每个同学用一句话说出本周最重要的目标。

不是todo list，不是我要做三个需求，而是我要达成什么结果。

比如：

不是我要写完登录模块，而是让登录模块通过安全评审，零漏洞

不是我要参加三个会，而是搞定跨部门资源协调，拿到两个开发名额

目标越清晰，小兔越无处藏身。

因为当所有人都知道我们要去哪的时候，追小兔的人会很明显——他总是在讨论无关的事，总是在抱怨，总是在等别人配合而不是自己想办法。

# 七、写在最后

我越来越觉得，人与人之间的差距，不是智商，不是技术能力，而是目标感。

目标感强的人，眼里只有路，没有风景。不是风景不重要，而是他知道，到了目的地，有的是时间看风景。

目标感弱的人，走一步看一步，被路边的每一只小兔吸引，最后忘了自己为什么出发。

将军赶路，不追小兔。

不是小兔不可爱，而是将军有更重要的事要做。

我是无涯，13年技术路，6年管理路，呆过多家大厂，带过多种规模的技术团队，目前就职于一家AI公司任技术负责人。这里记录技术管理者的真实踩坑与成长。既关注技术管理，也关注技术架构，致力于技术管理和技术架构双线发展。
