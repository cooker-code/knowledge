# Google 开源 Magika：用 AI 重新定义文件类型检测

> 一个 1MB 的模型，99% 的准确率，每秒处理数十亿文件——这就是 Google 的答案。

---

## 📌 一个被忽视了 50 年的老问题

你有没有收到过一封"图片.jpg"的附件，打开却是一段恶意脚本？

文件扩展名，从来都不可信。

自计算机诞生以来，准确识别文件的真实类型一直是一个基础却棘手的问题。浏览器需要知道如何渲染文件，邮件系统需要决定是否拦截附件，安全扫描器需要选择正确的分析引擎——这一切，都依赖于一件事：**你得先知道这个文件到底是什么**。

传统的做法有两种：

* • **靠扩展名**：`.pdf` 就当 PDF 处理。简单，但极易伪造，几乎形同虚设。
* • **靠"魔法字节"（Magic Bytes）**：检查文件头部的特征字节序列。比扩展名可靠，但面对混淆、加密、多格式嵌套文件时，依然力不从心——尤其是纯文本类文件（代码、配置文件），几乎没有可靠的字节特征可用。

Linux 系统自带的 `libmagic` 和 `file` 命令，用了超过 50 年，依然是业界事实标准。**但在文本类文件的识别上，它表现平平。**

这是 Google 内部也面临的真实问题。Gmail、Google Drive、Google Safe Browsing 每周需要处理数千亿个文件，将它们分发给正确的安全策略扫描器。分错了，轻则体验变差，重则出现安全漏洞。

## 🔍 Magika 是什么？

**Magika** 是 Google 于 2024 年初开源的 AI 驱动文件类型检测工具，并于 2025 年在软件工程顶会 ICSE 上发表了对应研究论文。

它的核心思路非常直接：**既然规则写不好，就让神经网络来学习。**

Magika 在底层使用了一个用 Keras 训练、以 ONNX 作为推理引擎的深度学习模型。这个模型：

* • **体积极小**：仅约 1MB，无需 GPU，普通 CPU 即可运行
* • **速度极快**：模型加载后，每个文件推理时间约 **5ms**，且与文件大小几乎无关（它只读取文件的一小部分内容）
* • **精度极高**：在约 **1 亿个样本、200+ 种文件类型**的训练集上训练，测试集上平均精确率和召回率达到 **~99%**
* • **对文本类文件尤为出色**：相比传统工具，在代码文件、配置文件等纯文本格式上的识别准确率提升约 **20%**

目前，Magika 已被 Google 内部用于 Gmail、Drive 和 Safe Browsing 的文件路由，并与 VirusTotal、abuse.ch 等安全平台完成了集成。

## 🛠️ 怎么用？

Magika 提供多种使用方式，覆盖命令行、Python、JavaScript/TypeScript、Rust 和 GoLang（实验中）。

### 方式一：命令行工具

**安装：**

```
pip install magika
```

**常用命令：**

```
# 检测单个文件  
magika 某个文件.bin  
  
# 递归检测目录  
magika -r ./uploads/  
  
# 输出 MIME 类型  
magika -i 某个文件  
  
# 输出 JSON 格式  
magika --json 某个文件  
  
# 显示置信度分数  
magika -s 某个文件
```

**示例输出：**

```
payload.bin: ELF executable (elf)  
report.pdf: PDF document (pdf)  
config.txt: Python source (python)
```

> 💡 即使文件扩展名是 `.txt`，Magika 也能识别出其实际内容是 Python 代码。

---

### 方式二：Python API

```
from magika import Magika  
  
m = Magika()  
  
# 传入文件路径  
result = m.identify_path("某个文件.bin")  
print(result.output.label)       # 文件类型标签，如 "python"  
print(result.output.mime_type)   # MIME 类型，如 "text/x-python"  
print(result.output.score)       # 置信度分数，0~1  
  
# 也可以直接传入字节内容  
with open("某个文件.bin", "rb") as f:  
    data = f.read()  
result = m.identify_bytes(data)  
print(result.output.label)
```

---

### 方式三：JavaScript / 浏览器端

```
npm install magika  
  
import { readFile } from "fs/promises";  
import { MagikaNode as Magika } from "magika/node";  
  
const data = await readFile("某个文件");  
const magika = await Magika().create();  
const prediction = await magika.identifyBytes(data);  
console.log(prediction);
```

> 浏览器端也可以直接运行！官方提供了完全在本地执行的 Web Demo，拖入文件即可检测，无需安装任何东西。

---

### 置信度模式

Magika 支持三种置信度模式，可根据业务需要灵活调整：

| 模式 | 说明 |
| --- | --- |
| `high-confidence` | 只在非常确定时给出具体类型，否则返回通用标签 |
| `medium-confidence` | 默认模式，平衡准确性与覆盖率 |
| `best-guess` | 始终给出最可能的类型，即使置信度较低 |

## 💡 典型应用场景

* • **安全系统**：邮件附件过滤、恶意文件检测的前置分类器
* • **Web 应用**：用户上传文件的真实类型校验，防止类型伪造攻击
* • **代码编辑器**：根据文件内容自动切换语法高亮模式
* • **数据处理管道**：大规模文件分类、归档与内容路由
* • **云存储服务**：对存储内容进行智能分类和策略匹配

## 📊 总结

| 维度 | Magika | 传统工具（libmagic 等） |
| --- | --- | --- |
| 技术方案 | 深度学习模型 | 规则 / 魔法字节匹配 |
| 文本类文件准确率 | 显著领先（+~20%） | 较弱 |
| 模型大小 | ~1MB | 无模型 |
| 推理时间 | ~5ms/文件（CPU） | 极快（无模型加载） |
| 支持文件类型数 | 200+ | 较多，但文本类弱 |
| 多语言支持 | Python / Rust / JS / Go | C / Python 绑定等 |
| 开源协议 | Apache 2.0 | LGPL 等 |

Magika 不是要取代所有现有工具，而是用 AI 的方式**补上了传统方案在文本类文件检测上最大的短板**。对于需要处理用户上传内容、做安全扫描或文件路由的系统来说，它几乎是一个开箱即用的升级选项。

如果你的项目里有类似需求，不妨一试——毕竟，一个 `pip install magika` 换来 99% 的准确率，这笔交易挺划算的。
