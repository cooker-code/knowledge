**上一篇文章提到了，任何生意的底层公式是 LTV 和 CAC 的比较，而 LTV 就是用户生命周期的价值，也有人叫 CLV（Customer Lifetime Value）。**

先说几个商业世界的常用概:

* 用户生命周期价值, LTV（LifeTime Value）
* 平均每月每用户收入, ARPU（Average Revenue Per User）
* 月留存率, r（retention）
* 平均用户生命周期, LT（LifeTime)
* 获客成本, CAC（Customer Acquisition Cost）

商业世界的基本公式

商业世界的基本公式是：

 LTV > CAC

当然，如果你能够持续免费获客，那么 LTV > 0 也是可以的，但这种情况在现实世界中极少。 

ARPU这个词最早是在通信运营商领域使用的，因为运营商的生意模式非常简单，假设你是中国移动的用户，你的月租是200元，合同期限是3年，那么你的生命周期价值就是 600元，如果中国移动获取一个用户的成本低于600元，那么生意模式就没大问题。

现实中不同的生意的用户生命周期的价值不同，获客成本也不同。房产中介的中介费一般是0.6%，300万的房产中介费就要1.8万，所以房产中介才能帮用户打车、给用户买咖啡，背后的逻辑用户喝咖啡怎么也喝不了1万块钱；又假如你是街边卖烤冷面的，你的用户生命周期价值可能就8块钱（算毛利的话也可能就6块钱），这样的生意是永远不可能请用户喝咖啡的。

用户生命周期价值

用户生命周期的价值（LTV） = 用户生命周期（LT） \* ARPU

当然，ARPU值的计算有时间比较 tricky，尤其是对于通过商业化变现的公司和亏损型的业务周期，这个需要根据 LTV 的具体应用场景而选取合适的计算口径，例如我们之前就用过用户产生的GMV来代替用户的ARPU，但其实这不是公司从用户身上赚取的真正收益。

实际计算过程中也会把 ARPU 拆成固定周期的用户消费频次和客单价、以及毛利率，所以 LTV 的公式会进化成下面这种

LTV = LT \* 消费频次 \* 客单价 \* 毛利率

用户生命周期（LT）

任何一个产品或服务，用户都会有生命周期，即使像QQ这种国民级产品，N多年之后也会被微信这样的新时代的社交产品取代，所以用户运营的基本目标是在用户的生命周期里面最大化产品对用户的价值，以换取用户对公司的价值。

那么问题来了，用户的生命周期到底是多久呢？是一天？一个月？一年？而且每个用户的生命周期都不一样，有的人体验了一天觉得产品不好用，马上就流失了，有的人用了十年了还在用，如何去计算用户的平均生命周期呢？这个就是今天主要讨论的内容。

一般来说，计算用户的平均生命周期，主要有三种方法：求和法、倒数法、函数法。

无论哪一种方法，其核心都使用一个指标——用户留存率，因为要知道用户的平均生命周期，你得知道用户存活在每一个周期的概率是多少，例如有多大概率用户存活1个月、2个月、12个月、36个月，这样才能计算平均的期望值。

求和法

求和法是最简单的方法，就是对用户各个周期的留存率进行求和，算出来就是用户的平均LT，用公式来表示就是

LT = 1 + R(1) + R(2) + ... + R(n)

其中R(n)就是第n个周期的留存率。当然你不能无限制地计算下去，所以一般选取一个符合业务的最长周期进行截断即可，例如计算到第36个月，36个月以后的就计为0不在计算了。

这里主要说一下这种算法背后的逻辑，因为这个逻辑代表了用户留存和LT之前的关系的本质。

单用户的LT很难算，但是转化一下思路，单用户的LT就是一群用户的LT的加总除以期初的用户数。那么一群用户的LT的加总，其实等于存续了一个月的用户数 + 再存续了一个月的用户数 + 存续了又一个月的用户数 + ...

所以 总的LT 本质上 

LT = n(t1) + n(t2) + n(t3) + ... + n(ts)

其中 n (t1) 就是t1时刻活跃或交易的用户数。

如果两边同时除以期初的用户数，那公式变成了上面求和法里面的公式。

LT = 1 + R(1) + R(2) + ... + R(n)

那为什么最终会变成一个用户流失率的倒数呢？因为上面的公式，其实相当于对用户的留存率曲线下的面积求和，也就是相当于对留存率曲线求积分

假如用户的留存率是一个稳定的值，那么

LT = 1 + r + r2 + r3 + ... + rn

函数法

之前的两种方法都需要用户经历过很长生命周期以后根据数据回算用户的LT，但现实中用户的留存率可能不是稳定的、或者很难在用户已经过了12个月再去看这批用户的LT是多少，更多的时候我们需要根据用户早期的一些表现来“预测”用户的LT，这样才能更快地调整拉新策略或者运营策略，这时候就涉及到了函数法。

函数法的基本逻辑是用一个函数来预测用户的留存率曲线，因为LT本质上是对留存率曲线进行求积分，所以一旦我们能够根据用户早期的留存情况拟合一个用户留存曲线出来，那么我们在用户生命周期的早期就能“预测”这批用户的LT，所以说函数法是真正用来做LTV预测的方法。
