**Hazelcast 是一个开源的分布式数据平台**，它提供了高性能的内存计算和数据存储解决方案。它允许企业实时处理大量数据，并且能够快速地从数据中获得洞察力。Hazelcast 特别适用于需要高并发处理、低延迟和高吞吐量场景的应用程序。

Hazelcast 的一些关键特性包括：

1. IMDG（In-Memory Data Grid）: Hazelcast 提供了一个内存中的数据网格，它允许数据被分布式地存储在多个节点上，从而提供快速的访问速度和高可用性。

2. 流处理: Hazelcast 支持实时数据流处理，允许用户对数据进行实时分析和处理，以便快速做出决策。

3. 分布式算法: 它实现了多种分布式算法，用于负载均衡、故障转移和数据分区。

4. 易于集成: Hazelcast 可以轻松地与现有的应用程序和数据存储系统集成，支持多种编程语言和数据格式。

5. 安全性: Hazelcast 提供了安全性的支持，包括身份验证、授权和加密通信。

6. 可扩展性: 它支持水平扩展，允许通过添加更多的节点来增加处理能力和存储容量。

7. 灵活性: Hazelcast 提供了多种部署选项，可以在云中、本地或混合环境中运行。

Hazelcast 的官方网址是 www.hazelcast.com，在那里可以找到关于产品的详细信息、文档、示例代码和社区支持资源。此外，Hazelcast 还提供了免费的社区版和商业版，商业版提供了额外的支持和功能。

Hazelcast 提供了一个可以处理多种类型的工作负载以构建实时应用程序的平台。

* 对流数据或静态数据进行状态数据处理
* 直接使用SQL查询流式和批量数据源
* 通过连接器库摄取数据并使用低延迟 SQL 查询提供数据
* 根据事件推送更新到应用程序
* 基于低延迟队列或发布-订阅消息传递
* 通过读/写和后写等缓存模式快速访问上下文和事务数据
* 微服务的分布式协调
* 将数据从一个区域复制到另一个区域或同一区域的数据中心之间

### 运营数据存储

Hazelcast 提供可分区、可复制和可查询的分布式内存数据结构。Hazelcast 的主要用例之一是存储一*组工作*数据以便快速查询和访问。

Hazelcast 底层的主要数据结构称为`IMap`，是一个键值存储，具有丰富的功能，包括：

* 与数据源 一次性集成或持续摄取
* 直读和直写 缓存模式
* 通过SQL建立索引和查询
* 处理原子更新的条目
* 根据 TTL 或上次访问时间等特定条件自动使项目过期
* 近端缓存 用于在客户端缓存条目
* 用于向客户端推送更改的侦听器
* 数据中心之间的数据复制 （仅限企业版）
* 磁盘上数据的持久性 （仅限企业版）

Hazelcast 将数据存储在 分区中，这些分区分布到所有节点。您可以通过添加额外的节点来增加存储容量，如果其中一个节点出现故障，数据会自动从备份副本中恢复。

您可以使用 SQL 或您选择的编程语言客户端与地图进行交互。您可以按如下方式创建地图并与之交互：

```
CREATE MAPPING myMap (name varchar EXTERNAL NAME "__key", age INT EXTERNAL NAME "this")   
TYPE IMap  
OPTIONS ('keyFormat'='varchar','valueFormat'='int');  
INSERT INTO myMap VALUES('Jake', 29);  
SELECT * FROM myMap;
```

可以使用受支持的编程语言之一以编程方式完成相同的操作，如下所示。以下是 Java 和 Python 中的一些示例：

```
var hz = HazelcastClient.newHazelcastClient();  
IMap<String, Integer> map = hz.getMap("myMap");  
map.set(Alice, 25);
```

```
import hazelcast  

client = hazelcast.HazelcastClient()  
my_map = client.get_map("myMap")  
age = my_map.get("Alice").result()
```

其他支持的编程语言有 C#、 C++、 Node.js和 Go。

或者，您可以使用 SQL 直接从支持的许多源获取数据 ：

```
CREATE MAPPING csv_ages (name VARCHAR, age INT)  
TYPE File  
OPTIONS ('format'='csv',  
    'path'='/data', 'glob'='data.csv');  
SINK INTO myMap  
SELECT name, age FROM csv_ages;
```

Hazelcast 还提供了额外的数据结构，例如 ReplicatedMap、Set、MultiMap 和 List。有关完整列表，请参阅 文档的分布式数据结构部分。

### 状态数据处理

Hazelcast 有一个内置的数据处理引擎，称为 Jet。Jet 可用于构建弹性的流式和批处理数据管道。您可以使用它来处理大量实时事件或大批量静态数据集。为了给人一种规模感，Hazelcast 的单个节点已被证明每秒聚合 1000 万个事件，延迟低于 10 毫秒。Hazelcast 节点集群每秒可以处理十亿个事件。
