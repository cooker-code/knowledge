# 优质 Skills 推荐：25 个营销与增长 Skills 实战指南（三）

前两篇介绍了 25 个营销与增长 Skills 的速查表和工作流，这篇带来 3 个可复现的实战打法，以及边界条件和失败经验。

## 实战案例：3 个看起来亮眼、但能复现的打法

下面每个案例都按"目标/准备/步骤/产出"来写。你只要把数据入口和素材补齐，照着跑一遍就能看到结果差在哪。

### 案例一：用 pSEO 做出第一波自然流量（目录页 + 结构化数据）

**目标**：在 4 周内做出一批能被收录、能带来注册的 SEO 页面

**准备**：

* • 1 个关键词模式
* • 1 份可持续更新的数据
* • 1 套模板组件
* • GSC/GA4 访问权限

**步骤**：

1. 1. 用 **programmatic-seo** 选 playbook（比如 Directory/Integrations/Comparisons）并写模板结构
2. 2. 把"每页独特价值"做成硬规则：独特 intro、数据差异、条件渲染模块、相关页面内链
3. 3. **schema-markup** 给关键页面加 JSON-LD（例如 FAQ/HowTo），上线前用结构化数据测试工具校验
4. 4. 上线后用 **seo-audit** 做"索引与可抓取"检查，保证 sitemap、canonical、robots 没坑
5. 5. 用 **analytics-tracking** 给目录页补充关键事件：cta\_clicked / signup\_started / signup\_completed

**产出**：

* • 1 份 playbook 选择与页面信息架构文档
* • 1 套模板组件清单（哪些是可复用模块、哪些必须页面级独特）
* • 1 份结构化数据实施清单 + 校验截图
* • 1 份上线后 7 天的索引监控指标表

**我会把这条当成硬约束：先做 20 个高质量页面，再讨论规模。**

### 案例二：把"页面不转化"拆成可执行清单（CRO + 实验 + 埋点）

**目标**：把落地页注册转化率提升到"可以谈优化"的水平（至少能稳定收敛到一个漏斗模型）

**准备**：

* • 落地页 URL
* • 主要流量来源
* • 现有事件列表（如果没有就先补）

**步骤**：

1. 1. **page-cro** 先做"5 秒清晰度"检查：价值主张、headline、CTA 层级、信任背书
2. 2. **signup-flow-cro** 只盯注册漏斗：字段取舍、错误提示、加载性能、表单摩擦
3. 3. **analytics-tracking** 写出事件表，确保每个漏斗节点都有事件
4. 4. **ab-test-setup** 把 2-3 个争议点写成假设，定义指标与运行规则（不要一次测太多变量）

**产出**：

* • Quick wins 清单（当天就能改）
* • 高影响改动清单（需要设计/前端配合）
* • 3 个可跑的实验假设（包含指标、触发人群、预期变化方向）
* • 事件表（Event/Properties/Trigger）

**这套组合我最喜欢的一点是：把"我觉得"变成"我们测一下"。**

ab-test-setup 会逼你把因果写清楚：为什么改这个、怎么衡量、什么叫成功。

### 案例三：工程化营销的"免费工具"打法（获客 + 培育 + 变现）

**目标**：用一个免费工具做 lead magnet，顺带做 SEO 入口

**准备**：

* • 一个明确的用户痛点
* • 可在 1-2 周做完的 MVP
* • 明确的导流路径（工具 → 邮件 → 试用/咨询）

**步骤**：

1. 1. **free-tool-strategy** 先做选题评估：自然搜索需求、产出价值是否可衡量、维护成本
2. 2. **content-strategy** 围绕工具做 hub & spoke：教程/案例/模板/对比页
3. 3. **email-sequence** 做"工具用户 7 封培育"：每封只有一个目标，逐步引导到产品价值
4. 4. **pricing-strategy** 把"免费工具用户"的升级路径写清楚：什么情况下该付费、买哪个档
5. 5. **referral-program** 加一个轻量分享激励（但先保证工具真好用）

**产出**：

* • 工具 MVP 需求清单 + 上线版本节奏
* • 内容矩阵与发布节奏
* • 邮件序列草稿（目标/主题/CTA）
* • 定价包装对照表（让用户能自助选择）

**这条路径的逻辑也很朴素：**

* • 免费工具把线索带进来
* • 邮件把意图养起来
* • 定价把愿意付费的人接住

## 边界条件 / 取舍 / 失败经验

### pSEO 最容易死在"薄内容 + 过度规模"

最常见的翻车路径大概是这样：

* • 你做了 3000 个模板页
* • 每页差异只有标题里换了城市/关键词
* • 内链和 sitemap 都上了
* • 结果：收录少、排名差，甚至被判定为低质量页面

**我一般用这几条卡住自己：**

* • programmatic-seo 已经把原则写得很直白：每页必须有独特价值、质量优先于数量、避免 doorway pages
* • 先做"少量高质量"验证：20-100 页能稳定收录、能带来行为事件，再扩规模
* • 把"独特性"做成工程约束：每页必须至少满足 2 个差异模块（数据差异 + 条件渲染 + 独特 intro/洞察）

### CRO 的取舍：表单字段越少不一定越好

你可能想"字段砍到极致"。但如果你做的是 B2B 线索，字段太少会导致：

* • 线索质量下降（销售跟进成本上升）
* • 误触发大量垃圾注册

**我会这样取舍：**

* • form-cro/signup-flow-cro 的优化要配合风控、合规、线索评分策略一起做
* • 先把"必须字段"和"可推断字段"分开（例如可以从邮箱域名推断公司类型）

## 最小落地清单（跑一遍就能动起来）

* • **先写 product-marketing-context**：把定位/ICP/价值主张写成可复用文档
* • **再写 analytics-tracking**：事件命名 + 事件表 + UTM 规则（没有这一步，后面很容易靠感觉争）
* • **同步跑 content-strategy + copywriting**：先把"能讲清楚"做出来
* • **流量起来后再做**：seo-audit → page-cro → signup-flow-cro → ab-test-setup
* • **想做规模化增长，再引入**：programmatic-seo / schema-markup / competitor-alternatives / free-tool-strategy

## 总结

这 25 个营销与增长 Skills 的核心价值在于：

| 价值点 | 说明 |
| --- | --- |
| **标准化** | 每个 skill 都有固定输入、步骤和输出格式 |
| **可执行** | 产出是文档产物，不是口头建议 |
| **可复用** | 同一套框架可以反复用于不同项目 |
| **降低门槛** | 技术团队不用成为营销专家也能做增长 |

对于技术团队来说，这套工具可以把模糊的"增长需求"拆解成可执行的任务，让每个人都清楚要做什么、怎么做、产出是什么。
