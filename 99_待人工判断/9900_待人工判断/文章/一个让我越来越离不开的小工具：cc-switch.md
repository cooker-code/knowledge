# 分享一个我最近一直在用的小工具，叫`cc-switch`。

项目地址：https://github.com/farion1231/cc-switch 

下载地址：https://github.com/farion1231/cc-switch/releases 直接安装下载即可。

我自己平时主要会在 `Claude Code`、`Codex` 这些工具之间切换。真正麻烦的是我会频繁切不同的 Provider，现在是公益站和付费线路混着用，一天切好多次。刚开始还觉得手改配置也能接受，但 Provider、MCP、Skill 一多，整个环境就会越来越碎，切一次、改一次，还得去翻笔记，不然 URL、token、MCP 信息根本记不住，时间就都消耗在这些琐事上了。

`cc-switch` 对我来说，最大的意义不是多了什么全新的能力，而是把这些原本很零散的事情，尽量收到了一个地方统一管理。

像这些事，它都能帮上忙：

* 管理不同 AI CLI 的配置
* 切换不同 Provider
* 统一管理 MCP
* 管理常用 Prompt
* 管理 Skills
* 还能做反代，不过我个人不太推荐，感觉有封号风险，我自己就封过几个 Google 账号

以前切个环境，经常要先想半天：这个工具的配置文件在哪？我上次是怎么改的？这个 Provider 对应的是哪套配置？这个 MCP 到底配在谁那边了？用它之后，这些事基本点几下就能完成。

## 我比较常用的几个功能

这是我用得最多的功能。如果你平时不只连一个服务，而是会在不同 Provider之间切换，这块会非常顺手。

我自己的用法很简单：

* 名字起清楚一点
* 后面按场景直接切

### 2. MCP 管理

这个功能我自己挺看重的。现在很多 AI CLI 都越来越依赖 MCP，但 MCP 一多，配置维护就很容易开始乱。有些是本地服务，有些是远程服务，有些是多个工具共用，有些又只在某个工具里用。

以前这类配置往往分散在不同地方，后面你自己都不一定记得清。`cc-switch` 把它们统一收口之后，最大的帮助就是管理成本明显降下来了。

我平时的用法大概是：

* 先把常用的 MCP 都整理进去
* 名字标清楚
* 按需要同步到不同应用

这样后面不管是加新工具，还是调整现有工作流，都不用从头再折腾一遍。

### 3. Prompt 管理

如果你平时已经有一些固定在用的 Prompt，这个功能会特别舒服。很多人其实都有自己的“常用模板”，只是平时散在不同地方。

* 写代码时的一套规则
* 做代码审查时的一套规则
* 写文档时的一套规则
* 做分析、总结、翻译时的不同模板

这些东西如果一直靠自己复制粘贴，时间久了肯定会乱。`cc-switch` 把这块收起来之后，维护起来会轻松很多。

我自己的习惯是，不会存特别多，只保留真正高频会用的几套。名字尽量直白一点，按场景分。这样到了实际使用的时候，不用临时重新组织语言，直接调用已经打磨好的模板就行。

具体使用方式其实都差不多，我这里就不一一截图了。Skill 和 Prompt 都在 MCP 左侧，用起来不难，主要还是想推荐这个工具本身，实际上手门槛并不高。

### 4. Skills 管理

如果你平时也会用 Skills，这块也挺方便。以前装 Skills、删 Skills、同步 Skills，其实都带一点“手工维护”的感觉。仓库一多、来源一多，慢慢就会忘记自己装过什么，哪些还在用，哪些已经可以清掉了。

`cc-switch` 把这件事做顺之后，一个很明显的变化就是，你会更愿意去整理自己的技能库，而不是一股脑全堆着。

* 留下真正常用的
* 不常用的及时清掉
* 看到适合自己的再补进去

这样整个环境会干净很多，用起来也更稳。

### 5. 会话管理

说实话，这个功能我自己用得不算多，主要是拿来看一些对话历史，或者删掉不再需要的会话记录。
