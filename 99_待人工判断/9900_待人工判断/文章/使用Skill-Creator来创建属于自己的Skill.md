你有没有遇到过这样的场景——让 AI 帮你写周报，每次都要重新解释格式要求、汇报对象、内容来源？如果能把这些要求”打包”成一份可复用的指南，下次直接调用就好了。

这就是 **Skill** 的核心思路：给 AI 一份写好的操作指南，让它按照固定的方法和步骤来处理特定任务。不用每次从头教，效果更稳定，也更省时间。

那么问题来了——这份指南本身该怎么写？Anthropic 官方提供了一个专门帮你创建 Skill 的工具，叫做 **Skill-Creator**。你只需要用对话描述需求，它就能帮你一步步生成一个完整的 Skill。

接下来，我们通过一个完整的实操流程来了解 Skill-Creator 的工作方式。

别被下面的步骤数量吓到——实际操作非常简单：**你只需要说出想做什么，然后跟着 Claude 的引导回答问题就好**。整个过程就像一次普通对话，Claude 会自动完成草稿生成、测试、评审等所有技术性工作。

### 前置准备

Skill-Creator 的获取非常简单。现在流行的 AI 编程工具（Claude Code、Codex、Cursor 等）大多都支持这个 Skill。

如果你使用的是 **Claude Code**，且你有海外网络环境，可以直接通过官方插件市场安装——Claude Code 中使用 `/plugin` 命令打开插件商店，搜索 `skill-creator` 一键安装

而通常，我建议使用对话方式安装。这样无论你是哪一个工具，只要有海外网络环境，就可以通过 官方 GitHub 仓库 下载安装——在项目中，对 AI 输入【使用curl的方式下载这个skill：https://github.com/anthropics/skills/tree/main/skills/skill-creator】

如果你没有海外网络环境，你可以使用**SkillTruck**的skill-creator镜像来进行安装

安装完成后，当你在对话中表达"我想创建一个新 Skill"之类的意图时，Claude 会自动识别并启用 Skill-Creator 来引导你完成整个流程。

安装成功后**重启一下对话**即可使用。

### 小结

回顾整个流程，Skill-Creator 的设计理念可以归纳为两点：

1. **迭代式开发**

   ：不追求一步到位，而是通过"草稿 → 测试 → 反馈 → 改进"的循环来逐步打磨。这与现代软件工程中的敏捷开发理念一脉相承。

1. **人机协作**

   ：Claude 负责草稿生成、测试执行和测试评估等重复性工作，而人类负责提供需求、审查输出和判断质量。每一轮迭代都有人类在回路中（Human-in-the-Loop），确保最终产物真正符合需求。
