如果你是后端开发人员，那么你应该知道，SQL是数据分析领域的利器。在信息技术飞速发展的今天，数据已成为企业最宝贵的资产之一。这个分析利器是一款功能强大的数据处理命令行工具，它提供了丰富的SQL查询和分析功能，能帮助用户快速从比如：Excel、JSON、CSV中提取有价值的信息。无论是简单的数据查询，还是复杂的数据分析，它都能轻松应对，让你的数据处理工作事半功倍。这个分析工具就是——dsq。例子在后面，可以先简单领略下这个工具的强大之处！

**它支持分析的文件类型有：**

CSV、TSV、JSON、Newline-delimited JSON、Concatenated JSON、ORC、Parquet、Avro、YAML、Excel、ODS、Apache Error logs、Apache Access Logs、Nginx Access Logs、LogFmt Logs

它相对于市面上其它分析工具来说，是非常具有优势的：

**特色:**

支持不同的格式文件，比如：Excel、JSON、CSV、XML、YAML、Markdown 和 Text等等

采用 SQL 查询语句的方式进行操作，只要你是后端开发人员，可以说是学习0成本

它支持多种高级的功能，比如说正则表达式、缓存、函数和脚本什么的

它的性能非常的优异，底层库高效，分析速度非常的快

最重要的它开源免费，可自由使用和修改。

**举例说明：**

```
#查询x字段大于10的记录$ dsq data.json "SELECT * FROM {} WHERE x > 10"#或者把某个数据分组dsq data.ndjson "SELECT name, AVG(time) FROM {} GROUP BY name ORDER BY AVG(time) DESC"#如果你要把查询结果使用json展示$ dsq data.json 'select count(*) from {}' | jq[  {    "count(*)": 1000  }]#又或者，可以在dsq中使用-p或--prely进行漂亮的打印$ dsq --pretty data.json 'select count(*) from {}'+----------+| count(*) |+----------+|     1000 |+----------+
```

还可以实用管道模式进行数据操作

```
$ cat data.csv | dsq -s csv "SELECT * FROM {} LIMIT 1"
```

你还可以将多个文件传递给DSQ。只要它们是支持的有效格式的数据文件，就可以将所有文件作为表运行SQL。每个表都可以由字符串{N}访问，其中N是在命令行上传递的文件列表中文件的基于0的索引。

```
$ dsq users.csv ages.json \  "select {0}.name, {1}.age from {0} join {1} on {0}.id = {1}.id"[{"age":88,"name":"Ted"},{"age":56,"name":"Marjory"},{"  
#可以给文件定义别名进行使用$ dsq users.csv ages.json \  "select u.name, a.age from {0} u join {1} a on u.id = a.id"[{"age":88,"name":"Ted"},{"age":56,"name":"Marjory"},{"age":33,"name":"Micah"}]
```
