是不是群太多，但又没时间去一一查看他们到底在聊什么？每天几千几千聊天信息。wechat radar  就是专门对付「群太多、真正有用的消息被淹没」这件事。

加过十几个技术群、AI 群的人应该都有同感。每天几百条消息滚过去，里面肯定有好东西——谁分享了个新工具、哪个话题讨论得特别深、有人 @ 你问了个事——但这些东西全混在「收到」「哈哈哈」「+1」里头。翻吧，浪费时间；不翻吧，又怕错过什么。

  

所以 WeChat Radar 干的事让我特别舒服。它不让你翻聊天记录，它直接给你一张「今日简报」。

  

奶白色界面，深色模式也支持，打开就是一张情报驾驶舱。活跃群数、总消息数、文章链接、工具链接、@ 我、高信号人物，一屏全收。下面还有个「今日优先看」分区，把最值得看的内容顶上来。

我认真扒了一下它到底能干嘛，发现四个能力，每个都踩在我想解决但一直没找到顺手工具的痛点上。

  

第一个是话题雷达。你有没有经历过这种情况：好几个群在同一天聊同一个东西，但你要一个一个点开才知道，哦，原来大家说的是一件事。话题雷达做的事就是跨群聚合，把同一天的热门话题收在一起。不用来回切群，看一眼就知道今天圈子里在讨论什么。

  

第二个是链接情报。技术群、开源群里每天被丢进来的链接太多了，同一个好工具可能被不同人往不同群里扔了七八遍。它会自动把文章、GitHub 链接、小工具链接去重，还帮你整理出可读标题。你再也不用点进去才发现「这个我昨天已经看过了」。

  

第三个最妙，叫群日报。它会把当天群聊内容做一键摘要，然后你可以把这份摘要复制出来，丢给 ChatGPT、Claude、Kimi，随便哪个你顺手的 AI。让 AI 帮你写大纲、列工具试用清单、甚至直接生成周报。以前是人翻消息，现在是机器翻消息，人看结论。

  

第四个让我安心：本地存储。所有数据都在你电脑的 ~/.wechat-radar/ 目录下，用的 SQLite，默认不上传云端。谁聊了什么、哪些话题热、哪个群最活跃，全在你自己的硬盘上。

  

这项目 5 月 24 号刚开源，到今天已经连更好几版。很新，但已经能完整跑通了。

轻松上手，四条命令的事：

```
git clone https://github.com/zjp1997720/wechat-radar.gitcd wechat-radar pnpm install pnpm rebuild better-sqlite3 pnpm dev
```

编辑 config.yaml 的时候填上你的微信数据路径和想监控的群名就行。作者还很贴心，怕你一开始不敢用真数据，设了个demo模式。你可以先勾上 demo，或者跑之前设个环境变量 WECHAT\_RADAR\_DEMO=1，先用模拟数据看看界面长什么样。看顺眼了再接真实数据，像试衣服一样，先试试合不合身。

跑起来之后，浏览器打开本地 7860 端口，那个情报看板就在那儿等你了。

  

说真的，微信群里聊了什么这件事，以前基本靠运气和眼力。你刚好在某个时间点点进去，刚好看到那条消息，才算没白过。其他时候，那些好链接、好讨论、别人认真 @ 你的事，就这么沉下去了。

WeChat Radar 的思路挺对我的胃口：我不需要你盯着看，我替你看，看完给你一张纸。你花五分钟扫一眼，再用两分钟丢给 AI 处理一下，一天群聊里真正有价值的东西就到手了。
