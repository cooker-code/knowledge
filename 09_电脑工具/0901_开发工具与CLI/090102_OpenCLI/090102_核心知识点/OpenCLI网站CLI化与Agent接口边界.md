## 核心问题
网站 CLI 化的价值不在“能调网页”，而在把登录态、页面操作、结构化输出和错误处理变成 Agent 可复用的接口契约。

## 判断准则
- 适合沉淀的 OpenCLI 文章必须说明安装方式、适配器来源、登录态/Token、输出格式和失败信号。
- CLI-first 对 Agent 有价值，是因为命令可记录、可组合、可审计；如果只列热门站点示例，不能单独成为知识点。
- OpenCLI、bb-browser、浏览器 MCP 不是同一层：OpenCLI 偏稳定命令契约，bb-browser 偏适配器集合，浏览器 MCP 偏实时页面操作。

## 认知偏差
| 常见错误认知 | 正确理解 |
|---|---|
| 网站 CLI 化就是把网页变成 API | 真正边界是登录态、权限、反爬、输出结构和错误恢复 |
| Adapter 越多越好 | 对用户更重要的是常用站点是否稳定、字段是否可靠、失败是否可诊断 |
