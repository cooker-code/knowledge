## 文章进入时的判断表

| 判断项 | 必答问题 |
|---|---|
| 流程节点 | 文章补的是 OCLI00-OCLI06 的哪个环节 |
| 接入对象 | 是公开 API、登录态网站、桌面应用、本地 CLI，还是工具注册表 |
| 可运行 | 是否能复现最小命令、adapter 或浏览器连接 |
| 可验证 | 是否能检查输出字段、退出码、空结果、登录态和回放结果 |
| 可排障 | 是否提供 daemon、浏览器扩展、代理、权限、反爬、字段变化的定位方式 |
| 可迁移 | 是否能迁移到用户的 Agent、知识库、日报、社媒搜索或本机自动化场景 |

## 认知校准点

| 校准点 | 当前处理 |
|---|---|
| CLI-first 对 Agent 有价值，但不是所有网页都适合 CLI 化 | 按任务频率、结构化程度、登录态风险和验证成本判断 |
| 结构化输出优于 DOM，不代表输出天然可信 | 必须补字段契约、错误契约和来源锚点 |
| OpenCLI 与 bb-browser 的覆盖数、性能和稳定性结论都可能过时 | 标为待补证，不写成稳定事实 |
| 桌面应用 CLI 化比网站查询风险更高 | 单独纳入安全与隐私节点，不默认进入自动化 |
