## 核心问题
Playwright CLI 的长期价值在固定流程、可复现浏览器操作和低上下文消耗，而不是把所有网页任务都交给自然语言浏览。

## 判断准则
- 固定、重复、低分支流程适合脚本化；高变化、高权限、需要人工判断的流程必须保留审批。
- 0 Token 方案的本质是把浏览器步骤封装成工具，但仍要维护选择器、登录态、等待条件和失败截图。
- 如果目标是测试前端回归，应归前端测试；如果目标是让 Agent 操作网页，应归工具调用；如果目标是本机固定流程，留在 09。

## 认知偏差
| 常见错误认知 | 正确理解 |
|---|---|
| 自然语言控制浏览器就能省掉脚本 | 稳定自动化仍依赖明确选择器、等待和错误处理 |
| 0 Token 就没有成本 | 维护脚本、登录态和失败恢复同样有成本 |
