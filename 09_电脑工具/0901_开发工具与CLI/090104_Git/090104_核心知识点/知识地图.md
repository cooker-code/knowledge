## 认知校准点

| 校准点 | 当前处理 |
|---|---|
| Git 动作必须先看真实状态 | 把 branch/status/diff/log/remote 作为破坏性动作前置检查 |
| worktree 是隔离机制，不是自动解决合并和冲突 | 重点沉淀基线同步、分支命名、清理和收口 |
| commit 校验价值在可执行协作规则 | 不把规范写成格式审美，而写成质量门和审计入口 |
| gh/GitButler/worktree CLI 是上层工具 | 只吸收它们对 Git 工作流的可验证增量 |
