# 今天，Obsidian 官方正式上线了 **Obsidian Community**——一个全新的插件和主题目录 + 开发者控制面板。（地址放在了文末）

Obsidian 从 2020 年开放 API 以来，社区已经累计创建了超过 **4000 个插件和主题**，总下载量突破了 **1.2 亿次**。但很长一段时间里，这些插件的发现、管理、审核机制都相当原始——基本上靠 GitHub 提交、人工审核、用户自己辨别安全性。

这个局面，今天终于改变了。

---

## 先说说这个新平台能做什么

### 1. 全新社区网站：终于能好好「逛」插件了

之前的插件发现体验说实话比较糟糕。新的 Community site 提供了：

* **按类别浏览**：集成、Bases、图表等数十个分类
* **多维度排序**：按名称、下载量、流行度、发布日期、更新日期
* **每个插件有详情页**：截图、详细介绍、安全评分卡
* **付费插件和官方集成打上了新标签**

开发者还能自定义个人主页，添加赞助链接、社交媒体。

这基本上就是 Obsidian 终于有了自己的「应用商店」——虽然还不支持直接安装，但发现和筛选体验已经是质的飞跃。

### 2. 开发者仪表板：提交、管理、追踪状态一体化

所有现有的插件、主题，以及之前在 GitHub 排队的提交都已经**自动迁移**过来了。

开发者登录社区网站、连接 GitHub 账户后，就可以管理现有项目、提交新项目、编辑个人资料。不再需要手动往 GitHub 仓库提 PR 了。

### 3. 自动化审核系统：这次是动真格的

**这是我认为最关键的变化。**

之前 Obsidian 团队靠人工审核插件，但随着插件数量爆发式增长，根本审不过来——而且**后续版本根本没有审核**。很多插件上线时是安全的，但更新之后就没人管了。

新的自动化审核系统：

* **每次版本更新都会扫描**，不只是首次提交
* 检查是否违反开发者政策
* 检查源代码是否遵循最佳实践
* 检测已知安全漏洞和恶意代码

官方还特别提到——**随着 AI  coding agent 加速插件生成，提交量只会越来越大**，所以自动化是必然选择。

好消息是：**之前积压的 2300+ 个待审核提交，这几天全部处理完了。**

### 4. 插件安全：透明度大幅提升

新系统引入了两个重要安全机制：

* **自动扫描**：每次版本更新都检查代码质量和安全漏洞，包括恶意软件扫描
* **评分卡（Scorecards）**：每个插件页面显示自动化检查的状态，用户安装前就能看到

未来几个月还会进一步增加透明度：

* **披露机制**：插件会声明它访问了哪些能力（网络、文件系统、剪贴板等），用户安装前就能看到
* **验证作者标签**：通过额外验证的可信开发者会打上标签

### 5. 团队工具：企业级管控要来了

Obsidian 本身已经支持团队部署安全管控，接下来要简化团队对社区插件的管控，并支持**分发私有插件**给团队成员。

官方插件还可以申请 **Official badge**。

## 我的几点看法

**1.2 亿下载量是什么概念？**

这个数字说明 Obsidian 的插件生态已经是一个不容忽视的独立王国了。4000+ 插件，覆盖了笔记、数据库、图表、日程、写作、阅读几乎所有能想到的场景。很多用户留在 Obsidian，就是因为某个插件。

**AI 加速插件开发的警示**

官方特别提到 AI coding agent 加速了插件提交——这其实是整个行业都在面临的问题。代码生成门槛降低，但代码质量、安全风险反而更高。Obsidian 选择用自动化审核来应对，而不是简单地提高准入门槛，这个思路是务实的。

**旧插件的「豁免」只是暂时的**

官方说已存在的插件如果不符合新审核标准，会有一个「暂时豁免」期，但最终还是要逐步淘汰。这是一个明确的信号：**插件生态不是在原地踏步，安全性会越来越严格。**

**这个平台对普通用户意味着什么？**

简单说：**以后找插件更方便了，安装插件更安全了**。评分卡 + 披露机制 + 自动化扫描，至少在安全性上不再是「裸奔」状态。

---

## 下一步是什么

官方说这只是更大一系列举措的开始。接下来 Obsidian 还会对 App 和 API 本身做改进，进一步提升发现和安全性。

社区生态是 Obsidian 最有趣也最强大的部分。搭建好基础设施，生态才能持续繁荣。

Obsidian Community地址：https://community.obsidian.md/
