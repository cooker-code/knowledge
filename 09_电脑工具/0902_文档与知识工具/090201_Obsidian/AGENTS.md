## 技术定位

| 项 | 内容 |
|---|---|
| 工具名 | Obsidian |
| 一级类目 | 电脑工具 |
| 二级类目 | 文档与知识工具 |
| 技术本体 | 本地 Markdown 知识库工具，通过文件、插件、URI/CLI、同步和图谱能力支撑个人知识工作流 |
| 全局位置 | 位于阅读、摘录、笔记、知识整理、图表表达、AI/CLI 自动入库之间 |

| 主要产出 | Markdown 笔记、链接关系、图表、插件工作流、自动入库结果 |

## 排重准则

问题指纹：

```text
Obsidian + 工作流节点 + 插件/CLI/文件机制 + 解决问题 + 同步/验证/排障边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是插件推荐 | 只有插件清单不新建笔记，合并到插件选型边界 |
| 都是 CLI 调用 | 按启动、写入、搜索、同步、失败恢复拆分 |
| 都是 Web Clipper | 按采集、清洗、模板、入库、去重拆分 |
| 都是 Mermaid | 区分 Mermaid 语法、Obsidian 渲染、图表知识表达 |
| 涉及 Agent 自动入库 | 必须记录权限、路径、重复写入和验证动作 |

## 后续追查

- Obsidian CLI 的稳定调用方式、失败模式和替代方案。
- Web Clipper 的模板、清洗、图片保留和去重规则。
- Obsidian 与本仓库 `knowledge` 的边界：原文锚点、正式知识点和长期规则不要混写。
- 后续新增 Obsidian 文章前，先做路由表，不直接扩写核心知识点；低置信文章只记录判断，不创建临时目录。
