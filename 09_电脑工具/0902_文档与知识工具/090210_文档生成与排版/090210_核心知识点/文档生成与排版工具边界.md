## 核心问题
文档生成与排版工具要解决的是可交付形态，不是普通 Markdown 写作；核心验收是布局、分页、样式、字体、图片和导出稳定性。

## 判断准则
- Quarkdown 类工具扩展 Markdown 表达能力，适合从文本走向排版，但要检查语法锁定和渲染兼容。
- Ironpress 类 HTML/CSS 到 PDF 工具强调布局引擎和性能，适合批量生成，但必须验证 CSS 支持、分页和字体。
- 如果主问题是 PDF/扫描件解析，应转到文档解析与 OCR；如果主问题是生成 PDF/网页，留在本节点。

## 认知偏差
| 常见错误认知 | 正确理解 |
|---|---|
| Markdown 能写文档就不需要排版工具 | 当交付物要求分页、字体、版式和 PDF 稳定输出时，需要独立工具链 |
| HTML 转 PDF 快就足够 | 可交付质量还包括 CSS 兼容、分页控制和跨平台字体 |
