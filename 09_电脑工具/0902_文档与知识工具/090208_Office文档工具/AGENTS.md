## 文章处理 SOP

1. 先读正文第一屏和结论段，确认主问题，不按标题词硬归类。
2. 只有能补充安装配置、工作流节点、验证方式、排障信号、权限/安全边界或横向对标的文章才保留。
3. 纯工具推荐、营销话术、资料领取、重复基础清单、跨域但不能服务本节点的文章，移出本节点，不保留 `done-*`。
4. 保留文章必须改为 `done-*`，并在文章顶部写 `已吸收至` 反链。

## 排重判断

```text
工具名/主题 + 工作流节点 + 输入输出 + 自动化方式 + 验证/排障边界 + 对用户的认知增量
```

| 判断项 | 处理 |
|---|---|
| 只介绍工具功能 | 没有边界或验证动作时删除 |
| 只列命令/插件清单 | 能补充 SOP 时合并到已有知识点，否则删除 |
| 有新失败模式或权限边界 | 追加到核心知识点并保留来源 |
| 与相邻类目冲突 | 按主问题转出或删除，不在本节点硬留 |
