# 文档与知识工具

## 知识点入口

- 本模块先看宏观流程，再看文章：[知识地图](090200_核心知识点/知识地图.md)。
- 新文章必须先归入流程节点，再判断是补充、冲突、不同层次还是降权。
- `文章/` 只保留原文锚点，长期知识必须沉淀到 `090200_核心知识点/`。

## 类目定位

| 项 | 内容 |
|---|---|
| 一级类目 | 电脑工具 |
| 二级类目 | 文档与知识工具 |
| 核心问题 | 文档、图表、知识库和内容生产工具如何支撑个人知识整理与交付 |

## 排重准则

问题指纹：

```text
工具名 + 内容对象 + 生成/整理机制 + 工作流节点 + 输出验证 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是 Markdown 工具 | 按写作、排版、发布、知识入库、图表渲染拆分 |
| 都是代码文档生成 | 区分代码仓库理解、文档站生成、Wiki 维护、Agent 上下文供给 |
| 都是图表工具 | 区分 Mermaid 语法、渲染引擎、流程图工作流、交付排版 |
| 只有“神器”式推荐 | 降权，只保留可证据化部分 |
| 涉及 Agent 自动维护知识库 | 优先判断是否应进入 `02_Agent与AI工程/0203_RAG与知识库` |

## 文章进入规则

| 情况 | 处理 |
|---|---|
| 能形成文档/知识工作流、模板、入库规则或输出验证 | 精读候选 |
| 只是工具介绍或截图演示 | 略读 |
| 主问题是知识库架构、检索增强、记忆系统 | 转到 Agent 与 AI 工程 |
| 主问题是具体工具长期使用 | 抽离到具体工具目录 |
