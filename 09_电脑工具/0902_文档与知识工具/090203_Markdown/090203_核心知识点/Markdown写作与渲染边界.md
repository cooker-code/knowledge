## 核心问题
Markdown 的关键不是记住所有语法，而是在不同渲染器、发布平台和知识库里保持结构稳定、链接可用、代码块正确。

## 判断准则
- 长期知识优先使用标题、列表、表格、代码块、引用和 wiki 链接，少依赖平台私有样式。
- 公众号吸粉类排版经验不等于 Markdown 知识；只有能改善结构、可迁移和渲染一致性才保留。
- 复杂图表、排版和 PDF 输出应转入 Mermaid 或文档生成与排版节点。

## 认知偏差
| 常见错误认知 | 正确理解 |
|---|---|
| Markdown 是简单语法不用管 | 不同平台的链接、表格、HTML、代码高亮和换行规则会影响交付 |
| 排版好看就是知识点 | 长期知识优先结构和可维护性，不追求营销样式 |
