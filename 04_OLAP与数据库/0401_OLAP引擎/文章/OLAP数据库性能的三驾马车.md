## **01 MPP**

MPP（Massively Parallel Processing）架构是一种大规模并行处理架构，通常用于处理海量数据和复杂查询任务。MPP架构通过将数据分布到多个独立的节点上，每个节点独立地执行查询计算，从而实现高效的数据处理和分析。

MPP 执行框架中，一条查询请求会被拆分成多个物理计算单元，在多机并行执行。每个执行节点拥有独享的资源（CPU、内存）。MPP 执行框架能够使得单个查询请求可以充分利用所有执行节点的资源，所以单个查询的性能可以随着集群的水平扩展而不断提升。

一个查询在逻辑上切分为多个 Query Fragment（查询片段），每个 Query Fragment 可以有一个或者多个 Fragment 执行实例，每个 Fragment 执行实例会被调度到集群某个计算节点上执行。一个 Fragment 可以包括一个或者多个 Operator执行算子，Fragment 包括了Scan、Filter、Aggregate。每个 Fragment 可以有不同的并行度，提到并行度那么就不得不提pipeline。

数据库 **Pipeline 执行引擎**基于 **多核 CPU** 的特点主要体现在并行处理、任务分配、内存管理和优化数据访问等方面。数据库系统的 Pipeline 执行引擎通常负责处理查询的各个阶段（如选择、过滤、连接、聚合等），而多核 CPU 的引入使得这些操作可以更高效地进行并行化和优化。

总结下来，Pipeline 调度的目的包括三点： 

1. 降低计算节点的任务调度代价，将传统 Pull 拉取的逻辑驱动的执行流程改造为 Push 模型的数据驱动的执行引擎。
2. 阻塞操作异步化，减少了线程切换，线程阻塞导致的执行开销，对于 CPU 的利用更为高效，提升 CPU 利用率。
3. 通过时间片的切换的控制，在混合负载的场景中，减少大查询对于小查询的资源挤占问题。充分利用多核计算能力，提升查询性能；控制了执行线程的数目或者自动设置并行度来消除人为设置并行度的不准确性。

## **03 向量化引擎**

**OLAP数据库的向量化引擎**，顾名思义，是通过向量化（Vectorization）技术来加速数据处理和查询的一种执行引擎。向量化是指将操作向量化，以便一次处理一批数据，而不是像传统的标量处理那样逐个处理单个数据项。这种方式特别适合 OLAP 查询，它通常涉及大量的数值计算、聚合和过滤等操作，能够显著提升查询性能。

在 OLAP 数据库中，向量化引擎通过将多个数据元素包装在一个向量中，并且通过 SIMD（单指令多数据）指令在 CPU 中并行处理这些数据元素，极大地提升了处理效率。向量化引擎的设计能够有效利用现代 CPU 的并行处理能力，提高了数据扫描、计算、聚合和排序等操作的速度。

更多细节可以参看之前的文章《[OLAP数据库之向量化执行引擎](https://mp.weixin.qq.com/s?__biz=MzI2ODYzMjkwMg==&mid=2247484286&idx=1&sn=aea0fedae31120e8bd83622b222780d1&scene=21#wechat_redirect)》。

总之，MPP解决了多个计算节点间的分治并行调用能力；Pipeline 解决的是一个计算节点的多CPU等资源高效利用；向量化提高了现代 CPU 的并行处理能力。有了这三种的架构能力，OLAP数据库才能将硬件资源利用到极致，所以也称之为OLAP数据库性能的三驾马车。
