# Elasticsearch

## 技术定位

| 项 | 内容 |
|---|---|
| 技术名 | Elasticsearch |
| 一级类目 | OLAP 与数据库 |
| 二级类目 | 搜索与索引 |
| 技术本体 | 分布式搜索与分析引擎，核心是倒排索引、文档写入、查询 DSL、分片副本和近实时检索 |
| 全局架构位置 | 搜索服务层，通常接在业务库、对象存储、日志系统或数据同步链路之后 |
| 主要使用者 | 后端工程师、搜索平台工程师、日志/观测平台工程师、数据平台工程师 |
| 主要产出 | 索引、文档、查询结果、聚合结果、搜索平台服务 |

## 核心模块

| 模块 | 职责 | 重点问题 |
|---|---|---|
| Mapping / Analyzer | 字段建模、分词、类型管理 | 中文分词、字段类型、动态映射失控 |
| 倒排索引 / Segment | 文档到词项的索引结构 | 写入可见性、段合并、存储放大 |
| Query DSL | 查询、过滤、聚合、排序 | 相关性、分页、聚合内存、查询超时 |
| Shard / Replica | 分布式扩展与容错 | 分片数量、热点、复制延迟、恢复成本 |
| Cluster Coordination | 集群元数据、主节点选举 | 脑裂、master 压力、集群状态膨胀 |

## 横向对标

| 对标对象 | 对标点 | 使用判断 |
|---|---|---|
| 关系数据库索引 | 精确过滤、事务查询 | 事务一致性和强关系查询选关系库；全文检索、相关性排序、日志搜索选 ES |
| OLAP 引擎 | 聚合分析 | 大宽表聚合、报表并发选 OLAP；文档检索、文本相关性、复杂过滤选 ES |
| RAG 向量库 | 召回方式 | 关键词/结构化过滤强时 ES 有优势；语义相似召回强时看向量库或混合检索 |
| Lucene / OpenSearch | 底层或替代实现 | Lucene 是底层库；OpenSearch 是 ES 生态替代；选型要看协议、插件和运维生态 |
