## 用户认知重点

| 项 | 内容 |
|---|---|
| 已知基础 | 用户知道 ES 是搜索引擎，可做日志/文件/全文检索 |
| 待补边界 | 倒排索引、分片副本、写入可见性、Geo 索引；与数据库索引、RAG 召回、代码搜索工具的差异 |
| 易偏差点 | 把 RAG 召回、Agent 搜索、代码搜索工具混入；用 ES 套关系库索引经验 |
| 优先抽取 | 倒排索引、Segment、Refresh/Translog、Shard/Replica、选主、查询 DSL |

## 排重指纹

```text
搜索技术本体 + 索引/写入/查询/分布式模块 + 核心机制 + 解决问题 + 适用边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都讲 ES 基础概念 | 比较是否新增倒排索引、写入链路、一致性、分片副本、选主或空间索引边界 |
| 只是接入案例 | 没有新增索引设计、查询建模、失败模式或验证指标时只保留路由锚点 |
| 和 RAG 混淆 | RAG 召回策略归 Agent 与 AI 工程；底层搜索引擎机制留本目录 |
| 和数据库索引混淆 | 单机 B+Tree、执行计划索引选择归关系数据库或查询优化 |

## 已覆盖问题（按三级节点）

| 节点 | 已覆盖 | 还缺 |
|---|---|---|
| Elasticsearch | 初始承接搜索引擎本体、空间索引、文件检索集成案例 | 倒排索引、Segment、Refresh、Translog、Shard/Replica、Master 选举、Geo 索引完整沉淀 |
| Pizza Engine | 轻量/嵌入式搜索引擎候选承接 | 开源状态、底层索引结构、浏览器内运行模型、与 Lucene/Tantivy/SQLite FTS 的差异 |

| 高 | ES Shard/Replica 与选主 | 分布式一致性与可用性的关键 |
| 中 | Pizza Engine 底层索引结构 | 候选目录已建，需补证 |
| 中 | Lucene / Tantivy / Meilisearch / OpenSearch | 有明确文章来源时再建目录 |
