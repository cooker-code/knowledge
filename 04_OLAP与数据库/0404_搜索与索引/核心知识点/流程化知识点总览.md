# 搜索与索引流程化知识点总览

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 04_OLAP与数据库 / 04_搜索与索引 |
| 解决的问题 | 搜索引擎如何把非结构化或半结构化数据建成可检索索引，并提供查询、写入、扩展和运维能力 |
| 典型输入 | 文档、日志、文件内容、地理位置、知识片段、业务对象 |
| 典型产出 | 搜索索引、查询 DSL、排序与过滤结果、分片副本、搜索平台服务 |
| 用户阅读目标 | 判断文章是在讲搜索引擎机制，还是把“搜索”当作 Agent/RAG/开发工具里的普通能力 |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| S00 场景建模 | 搜索对象是文档、日志、文件、地理空间还是本地知识片段 | 搜索引擎选型先看数据形态、查询模式和实时性 | 讲搜索服务边界、ES/OpenSearch/Lucene/Tantivy/Pizza Engine 对标的文章进入 |
| S01 索引建模 | 文档如何切分字段、分析、分词、建倒排索引或空间索引 | 索引设计决定召回、过滤、排序和写入成本 | 倒排索引、Analyzer、Mapping、Geo 索引、Segment 文章进入 |
| S02 写入与可见性 | 文档写入后如何 refresh、持久化、复制和可见 | 搜索写入不是事务库写入，要关注近实时、Translog、Segment 合并 | ES 写入链路、一致性、副本、并发控制文章进入 |
| S03 查询与排序 | 查询 DSL、过滤、相关性、聚合和分页如何工作 | 查询效果取决于索引结构、查询表达和排序代价 | 查询 DSL、相关性、聚合、分页、空间查询文章进入 |
| S04 分布式与运维 | 分片、副本、选主、扩缩容、集群故障如何治理 | 搜索平台风险在 shard 规划、脑裂、热点、段合并和资源治理 | 分片副本、master 选举、脑裂、搜索平台实践文章进入 |

## 待补缺口

## 使用方式

- 新文章先问：主问题是搜索引擎本体，还是 Agent/RAG/工具中的搜索能力。
- 搜索引擎本体才进入本目录；其他文章按技术本体移出。
- 技术目录优先于二级目录大池，`文章/` 目录只作为临时入口。
