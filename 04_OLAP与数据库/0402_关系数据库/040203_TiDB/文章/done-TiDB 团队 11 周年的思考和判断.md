4 月 8 日，TiDB 迎来成立 11 周年。今年的周年主题被定为——Connection（连接）。

在这一节点上，CEO 刘奇与 CTO 黄东旭分别发布了全员信。这两封信共同指向了一个更大的命题：当 AI 正在重塑软件世界，数据库将走向何处？

刘奇在信中给出了一个极具冲击力的判断——数据库的主要使用者，正在从“人类开发者”转向“AI Agent”；而数据库的角色，也将从“记录系统”演变为“思考系统”。这不仅意味着技术架构的升级，更意味着整个软件价值链的重构。

在黄东旭的视角中，“Connection”并不仅仅是一个技术词汇。它既是分布式系统连接机器的核心能力，也是开源社区连接开发者、软件连接用户的底层逻辑。在 AI 放大一切的时代，连接的广度与深度，正在决定新的可能性边界。

当“连接”从技术能力变成一种时代隐喻，TiDB 的第 11 年，也不再只是一个时间节点，而更像是一次面向未来的方向确认。

**/**01 数据库，已经不再服务人类了**/**

这是一个有点“反直觉”的判断，但它正在发生。

在 CEO 刘奇的全员信中，有一个细节非常关键：在一些生产环境中，超过 95% 的数据库实例，不再由人创建。 

它们来自：

> * 自动化流水线
> * AI Coding Agent
> * 多 Agent 协作系统

人类还在，但已经不是主角。

这意味着一件事：数据库的“用户”，正在从人，变成 AI。而一旦用户变了，整个行业就必须重写。

**/**02 我们熟悉的软件世界，正在被“压平”**/**

你可能已经感受到变化，但还没意识到它有多彻底。

过去的软件是这样的：

> * 有 UI
> * 有菜单
> * 有报表
> * 有“人类友好”的交互

但现在，正在发生一件更激进的事——软件，正在被 Agent 吃掉。

在现场分享中，刘奇讲得很直接：当你通过 AI 使用软件时，你不再打开 UI，而是直接调用 API。 

你想看报表？AI 帮你生成。

你想做分析？AI 直接跑。

你甚至不再关心软件长什么样。

于是——SaaS 被压平了。

UI 消失了，入口变成了 Agent。

**/**03 问题来了：数据库准备好了吗？**/**

如果软件入口变了，那底层最关键的东西是什么？

答案其实很清楚：数据。

但问题在于——AI 使用数据的方式，和人类完全不同。

过去：

> * 人写 SQL
> * 查固定结构
> * 访问节奏可预测

现在：

> * Agent 自动生成请求
> * 上下文动态变化
> * 查询规模指数级增长

甚至，一个 AI 创业者在和刘奇交流时，已经不再用“表”和“索引”思考问题，而是：

> * 任务（Task）
> * 记忆（Memory）
> * 上下文（Context）

这不是优化问题，这是范式变化。

**/**04 数据库，从“存储系统”变成“思考系统”**/**

过去，数据库是干什么的？

存数据。

但在 AI 时代，它正在变成另一种东西：System of Thought（思考系统） 

什么意思？简单说就是——数据库不只是“记住发生过什么”，还要支持 AI 去“理解、推理、决策”。

换句话说：数据库，正在成为 AI 的“中枢神经系统”。

**/**05 不是“AI + 数据库”，而是“Agent Native 数据库”**/**

很多公司已经在做一件事：给数据库加 AI 功能。

但刘奇的判断很明确：这不够。

就像当年：

* 把应用搬到云上 ≠ Cloud Native

今天：

* 接入一个模型 ≠ Agent Native

真正的变化是：数据库必须围绕 Agent 重新设计。

包括：

> * 如何支撑海量实例生命周期
> * 如何处理高并发、上下文驱动的请求
> * 如何管理“记忆型数据”
> * 如何与 Agent 编排系统协同

这不是加一个功能，而是换一套底层逻辑。

**/**06 TiDB 团队的选择：直接重写边界**/**

在这个判断之下，TiDB 团队没有选择“微调”，而是直接改定义。

从： HTAP 数据库

走向：AI 统一数据底座 

核心三块：

> * Database
> * File System
> * Agent Memory

再往上：

> * 支持分析（OLAP）
> * 支持数据上的计算
> * 支持 Agent 运行

本质上，它在做一件事：

让数据库，变成 AI 可以直接使用的基础设施。

**/**07 为什么是现在？为什么是 TiDB？**/**

这波机会，有一个非常残酷的特征：时间窗口极短。

刘奇给出的判断是：只有 36 个月。

三年之内，这个赛道会完成：

> * 定义
> * 爆发
> * 收敛

那为什么 TiDB 团队有机会？

因为“提前做对了一些难的事”：

> * 分布式架构（天然扩展）
> * 实时能力（适配 AI）
> * 大规模验证（不是 PPT）

这些，在过去是“优势”，在今天变成了“入场券”。

**/**08 最后一个问题：技术的尽头是什么？**/**

在 CTO 黄东旭的内部公开信里，他讲了一个词：Connection（连接） 

> * 机器之间的连接，构成系统
> * 人与人之间的连接，构成组织
> * AI，让连接的规模和速度被无限放大

所以这场变革的本质，不只是技术跃迁。而是——人类能力，被重新放大的一次过程。

**/**结尾**/**

过去，数据库始终是为人服务的系统。

而今天，这个前提正在被改变。

当软件被 AI 重写，Agent 成为新的入口，数据库不再只是“记录数据”，而开始参与“思考”。

在这样的转折点上，TiDB 团队在 11 周年给出的选择很明确——从数据库，走向面向 AI 的统一存储底座。

这不仅是一次产品演进，更是一种方向判断。

未来的分水岭，或许只剩一个问题：

你的数据基础设施，是否已经为 AI 准备好。
