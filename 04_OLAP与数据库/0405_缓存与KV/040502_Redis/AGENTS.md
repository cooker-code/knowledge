# Redis

## 技术定位

| 项 | 内容 |
|---|---|
| 技术名 | Redis |
| 一级类目 | OLAP 与数据库 |
| 二级类目 | 缓存与 KV |
| 技术本体 | 内存 KV 数据库，提供低延迟访问、多种数据结构、持久化、复制和分布式缓存能力 |
| 全局架构位置 | 应用服务和数据库之间的低延迟数据服务层，也可承担分布式锁、消息流、实时计数等短状态能力 |
| 主要使用者 | 后端工程师、平台工程师、数据服务工程师 |
| 主要产出 | Key/Value、缓存命中、排行榜、计数、锁、Stream、持久化快照和 AOF |

## 核心模块

| 模块 | 职责 | 重点问题 |
|---|---|---|
| 数据结构 | String、Hash、List、Set、ZSet、Stream、Bitmap、HLL、Geo | 业务问题到数据结构的映射、内存成本和精度 |
| 协议与访问路径 | RESP、连接、Pipeline、Lua | 网络往返、批处理、原子性和脚本风险 |
| 持久化 | RDB、AOF、混合持久化 | 丢失窗口、fork 成本、rewrite、恢复速度 |
| 高可用 | 主从、Sentinel、Cluster、双活 | 复制延迟、故障切换、脑裂、跨地域一致性 |
| 可观测性 | 慢日志、延迟、命令统计、内存、连接 | 高时延定位、热点 key、阻塞命令、网络问题 |

## 横向对标

| 对标对象 | 对标点 | 使用判断 |
|---|---|---|
| MySQL/PostgreSQL | 持久化事务库 | 强事务、复杂查询选关系库；低延迟短状态选 Redis |
| Caffeine/Guava | 本地缓存 | 单进程热点缓存选本地缓存；多实例共享状态选 Redis |
| RocksDB/Kvrocks | 持久化 KV | 海量持久 KV 看 RocksDB/Kvrocks；低延迟内存数据结构看 Redis |
| Kafka | 消息流 | 强消息队列、回放和消费组治理选 Kafka；轻量 Stream 可用 Redis Stream |
