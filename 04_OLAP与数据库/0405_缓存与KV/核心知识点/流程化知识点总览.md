# 缓存与 KV 流程化知识点总览

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 04_OLAP与数据库 / 05_缓存与KV |
| 解决的问题 | 缓存/KV 系统如何用低延迟访问、数据结构、持久化、高可用和客户端抽象支撑应用或数据服务 |
| 典型输入 | Key/Value、计数、排行榜、分布式锁、会话、热点数据、短期状态 |
| 典型产出 | 缓存命中结果、KV 查询结果、分布式对象、锁、消息流、统计数据结构 |
| 用户阅读目标 | 判断文章是在讲 Redis/KV 机制，还是普通后端缓存策略 |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| K00 场景建模 | 是缓存、计数、排行榜、消息流、分布式锁还是持久化 KV | Redis 不是通用数据库，先明确它承担的数据服务角色 | 数据结构选型、缓存边界、Redis/RocksDB 对比文章进入 |
| K01 访问路径 | 请求如何通过客户端、连接、协议、Pipeline 和 Lua 到达服务端 | 网络往返、原子性和批处理决定延迟上限 | Pipeline、Lua、客户端封装、Redisson 文章进入 |
| K02 数据结构 | String、Hash、ZSet、Stream、Bitmap、HLL、Geo 如何映射业务问题 | 数据结构决定内存成本、精度、查询方式和失效策略 | 高级数据类型、统计、排行榜、地理位置文章进入 |
| K03 持久化与恢复 | RDB、AOF、混合持久化、复制和持久化 KV 如何保障恢复 | 缓存系统也要明确丢失窗口、恢复成本和写放大 | Redis 持久化、Kvrocks、Redis/RocksDB 对比文章进入 |
| K04 高可用与排障 | 双活、延迟、热点、超时、缓存穿透/击穿/雪崩如何定位 | 生产价值来自可观测信号和失败闭环 | 双活、延迟排查、可观测性、Redisson 锁风险文章进入 |

## 使用方式

- 新文章先问：主问题是 Redis/KV 机制，还是后端应用缓存代码。
- 技术机制留在本目录；应用层缓存策略移到工程与架构。
- Redis 文章进入后先归到 K00-K04 节点，再决定是否写核心知识点。
