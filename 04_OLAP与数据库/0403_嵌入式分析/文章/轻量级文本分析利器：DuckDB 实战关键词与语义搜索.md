在自然语言处理（NLP）和数据分析场景中，关键词匹配、全文搜索与语义相似度计算是常见而关键的任务。传统方案如 Elasticsearch、FAISS、向量数据库等虽功能强大，但往往部署复杂、依赖多、资源消耗大，不利于快速迭代或轻量化场景的应用。

DuckDB 是一款嵌入式、零依赖的高性能列式数据库，近年来在数据分析领域崭露头角。除了高效处理结构化数据，它也能胜任许多文本分析任务。在本篇文章中，我们将展示如何借助DuckDB，快速实现从关键词匹配、全文检索到语义嵌入向量相似度搜索的一整套轻量级文本分析流程，无需引入复杂的外部服务，适合嵌入到日常数据分析、原型验证和低资源环境中。

# 关键词搜索(**Keyword Search**)

关键词搜索是最基本的文本检索方式，它通过 SQL 条件（如 `CONTAINS`、`ILIKE` 或其他 DuckDB 提供的文本函数）来匹配文本字段中的确切单词或短语。这种方法速度快、无需预处理，适用于结构化查询场景，例如日志过滤、标签匹配或商品名搜索等。

文本处理中一个常见的步骤是将文本拆分为词元（关键词），也就是将原始文本分解为更小的单元（通常是单词），以便后续分析或索引。这个过程被称为**分词（tokenization）**，它有助于将非结构化的文本转换为适合关键词检索的结构化形式。

在 DuckDB 中，可以使用 `regexp_split_to_table` 函数实现这个过程。该函数会根据提供的正则表达式拆分文本，并将每个关键词作为一行返回。

在分词（tokenization）步骤中，我们通常会排除一些常见词（例如 and、the），这些词被称为**停用词（stopwords）**。在 DuckDB 中，我们可以通过对一个托管在 GitHub 上的停用词 CSV 文件执行 **ANTI JOIN** 来实现排除操作。

现在我们已经完成了文本的分词和清洗，可以通过使用相似度函数（例如 Jaccard 指数）对匹配结果进行排序，从而实现关键词搜索。

我们还可以通过可视化数据来获取更多洞察。一个简单而有效的方法是绘制使用频率最高的词。通过统计整个数据集中词元的出现次数，并将其展示在气泡图中，我们能够快速识别出主要主题、重复出现的关键词或文本中的异常模式。

# 全文搜索(**Full-Text Search**)

DuckDB 的全文搜索（Full-Text Search，FTS）扩展是一个实验性功能，主要实现了两项核心全文搜索功能：

* `stem` 函数，用于提取单词词干；
* `match_bm25` 函数，用于计算最佳匹配得分（Best Match score）。

虽然 `stem` 函数可以单独使用，但 `match_bm25` 函数需要先构建全文搜索（FTS）索引，这是一种特殊的索引，通过对列中的单词（词元）进行索引，实现快速高效的文本搜索。

# 语义搜索(**Semantic Search**)

# 相似性连接(Similarity Joins)

向量嵌入最常见于搜索引擎，但也适用于多种文本分析场景，例如主题分组、文本分类或文档间的语义匹配。VSS扩展提供的向量相似性连接功能，正可用于此类分析。

例如，下方的热力图展示了不同情绪标签组合对应的文本数量分布：

* **横轴**：文本与情绪标签的语义匹配度
* **纵轴**：算法分类的情绪标签
* **颜色深浅**：各组合的文本数量

值得注意的是，在6种情绪中，仅"悲伤"标签的文本与其语义匹配高度一致。与全文搜索类似，语义搜索的效果受文本长度差异影响（本例中表现为情绪关键词与完整文本的对比）。

我们展示了如何通过结合关键词搜索、全文检索和语义搜索技术，利用DuckDB进行文本分析。通过实验性的fts和vss扩展与sentence-transformers库的配合使用，我们论证了DuckDB既能支持传统文本分析流程，也能胜任现代文本分析任务。
