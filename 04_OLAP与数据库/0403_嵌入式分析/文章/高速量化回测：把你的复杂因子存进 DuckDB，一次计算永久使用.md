随着大家对量化交易研究的深入，策略的复杂度也在不断提升。我们不再满足于简单的均线策略，而是开始引入各种复杂的指标：比如基于大盘情绪的择时因子、基于基本面数据的财务指标、甚至是利用机器学习模型生成的预测分数。

这些指标往往计算逻辑复杂，或者依赖的数据源非常庞大。如果在回测过程中，每一根 K 线都实时计算一遍，不仅会消耗大量的 CPU 资源，还会让回测速度变得难以忍受。试想一下，如果你有一个全市场选股策略，每次回测都要跑上几个小时，那参数优化的效率得有多低？

**“算一次，反复用”**，这是数据工程中非常重要的理念。

既然我们已经在 V3.2 版本中引入了高性能的 DuckDB 数据库，为什么不把这些“昂贵”的指标提前计算好，直接存入数据库呢？这样在回测时，策略只需要做一个简单的“读取”动作，就能大大提升效率。

今天这篇文章，就带大家深入了解如何利用 DuckDB 实现自定义指标的**写入**与**高效读取**，彻底打通数据与策略的最后一公里。

## 一、 为什么要进行指标的“预计算”与“持久化”？

在传统的量化平台中，数据和计算往往是绑定的。但在 KhQuant 的架构设计中，我们将**数据存储**和**策略计算**进行了彻底的解耦。这种设计带来了两个巨大的优势：

**1. 极大幅度提升回测效率**

对于全市场 5000 只股票的回测，如果每个时间步都重复计算复杂的因子，耗时可能是数小时。而如果直接读取预计算好的字段，耗时可能缩短到几分钟。特别是对于那些计算量大、但结果固定的指标（如复杂的波动率模型、甚至神经网络输出），预计算是最优的解决方案。

**2. 沉淀专属的数据资产**

你的量化系统不应该只是一个“代码运行器”，更应该是一个“数据资产库”。你可以将自己独有的因子、情绪指标、甚至爬虫获取的非量价数据，统一标准化存入 DuckDB。这些数据将成为你最宝贵的量化资产，随时调用，永不丢失。

## 二、 核心工具：如何向数据库“灌入”数据？

要实现指标的持久化，我们需要用到两个核心的工具函数：\*\*`khDuckDB`\*\*（读取）和 \*\*`khDuckWrite`\*\*（写入）。

这两个函数就像是数据库的“进出水管”，配合使用可以满足我们几乎所有的数据处理需求。根据数据来源的不同，通常有两种典型的使用场景：

### 场景一：基于现有行情计算衍生指标（“内循环”模式）

这是最常见的场景。比如你想计算 MACD、RSI 或者某种复杂的波动率指标。这些指标的计算依赖于数据库里已有的基础行情数据（如开高低收）。

本地计算MACD指标后写入数据库

**流程如下：**

1. **读取**：使用 `khDuckDB` 函数，从数据库中读出指定股票的原始行情数据（Open, High, Low, Close）。
2. **计算**：在 Python 中利用 Pandas、Talib 等库计算出你需要的指标（如 DIF, DEA, MACD）。
3. **写入**：使用 `khDuckWrite` 函数，将计算好的新指标列，按时间对齐写回数据库。

### 场景二：外部数据导入（“外挂”模式）

有时候，你的指标数据并不是算出来的，而是“拿”来的。比如你从某个财经网站爬取了“每日北向资金流向”，或者从 Excel 里整理了一份“自选股评分表”。

**流程如下：**

1. **读取**：直接使用 Python 的文件读取功能（如 `pd.read_csv` 或 `pd.read_excel`），获取外部数据。
2. **写入**：使用 `khDuckWrite` 函数，将这些数据直接导入 DuckDB 中。

这种模式下，数据完全来源于外部，我们只需要做一步简单的格式转换，就可以将非量价数据（如宏观数据、财报数据）与标准的行情数据无缝融合。

通过这两种模式的组合，无论是依赖内部行情的复杂指标，还是纯外部导入的数据源，我们都能轻松搞定。

## 三、 推荐方案：使用 khIndex 高效读取

指标写进去了，策略里怎么用呢？

为了达到极致的读取效率，我强烈推荐使用 **`khIndex`** 配合 **`khAddExtraFields`** 的组合拳。

这套方案的设计核心是**“内存优先”**。

### 1. 预加载机制：`khAddExtraFields`

在策略初始化阶段，我们需要明确告诉回测引擎：“我要用到这几个额外的字段”。这通常通过 `khAddExtraFields` 函数在 `init` 阶段完成。

这一步非常关键！当你声明了这些字段后，底层引擎会在回测开始前，一次性将这些列从 DuckDB 加载到内存中。这就意味着在后续每一次 `khHandlebar` 的循环中，我们访问这些数据时完全不需要再进行磁盘 IO 操作，而是直接从内存中极速读取。

### 2. 极速读取：`khIndex`

在策略的主循环 `khHandlebar` 中，我们使用 `khIndex` 函数来获取当前时间步的指标值。

由于数据已经在内存中就绪，`khIndex` 的读取速度几乎等同于直接访问 Python 字典或数组。对于那些需要频繁调用指标的高频策略，或者全市场选股策略，这种**零 IO 开销**的优势是巨大的。

## 四、 灵活方案：使用 khDuckDB 动态读取

虽然我首推上面的 `khIndex` 方案，但在某些特殊场景下，你可能需要更灵活的数据读取方式。这时，\*\*`khDuckDB`\*\* 函数就派上用场了。

### 1. 它的定位：备选与补充

我要特别强调一点：\*\*`khDuckDB` 并不是用来替代 `khIndex` 的高性能方案，而是一个功能强大的“万能钥匙”。\*\*

如果你在策略的主循环中频繁调用 `khDuckDB`，由于每次调用都会触发一次磁盘 IO 和数据库查询，虽然省去了指标计算的时间，但可能会因为频繁的 IO 操作导致整体回测速度并没有显著提升，甚至更慢。

### 2. 它的优势：极致灵活

既然 IO 开销大，为什么还要用它？因为它能做到 `khIndex` 做不到的事情：

* **跨周期与任意时段查询**：`khIndex` 只能获取当前时间点的数据，而 `khDuckDB` 可以让你随心所欲地获取过去任意一段时间的历史数据。比如你需要回溯过去 100 天的收盘价来计算某种复杂的形态，或者统计过去一年的涨跌幅分布。
* **跨标的查询**：在处理某一只股票时，你需要参考大盘指数或者板块龙头的数据，`khDuckDB` 可以随时跨标的查询。
* **无需预加载**：有些临时性的数据需求，不想在初始化阶段占用大量内存，可以用它即查即用。

### 3. 适用场景

所以，我的建议是：**核心交易逻辑尽量用 `khIndex`，只有在低频触发（如每天收盘后做一次复盘分析）或复杂的历史形态分析时，才考虑在策略中直接调用 `khDuckDB`。**

## 附录：核心函数详解

### 1. khDuckWrite（写入数据）

`khDuckWrite` 是将数据持久化到本地 DuckDB 数据库的核心函数。它不仅负责写入，还会自动处理表结构变更（如自动添加新列）。

```
def khDuckWrite(  
    stock_list,         # 股票代码或列表，如 '000001.SZ'  
    period,             # 周期类型，如 '1d', '1m'  
    data,               # 数据源 (dict 或 DataFrame)  
    fields=None,        # 要写入的字段名列表  
    time_col="time",    # 时间列名，默认 'time'  
    duckdb_path=None,   # 数据库路径，默认自动寻找  
    field_types=None,   # 字段类型映射，如 {'ma5': 'DOUBLE'}  
    insert_missing=False, # 是否插入不存在的时间点，默认 False  
    update_time=True    # 是否更新修改时间，默认 True  
):  
    """  
    功能:  
        - 按股票/周期定位表并写入字段值  
        - 缺字段时自动 ALTER TABLE 添加  
        - 默认仅更新已有 time 记录，避免插入空行  
        - 可选插入不存在的 time 记录  
  
    返回:  
        dict: {stock_code: {'updated': int, 'inserted': int, 'rows': int}}  
    """
```

### 2. khDuckDB（读取数据）

`khDuckDB` 是通用的数据读取接口，支持灵活的参数配置，可以获取任意历史时段的数据。

```
def khDuckDB(  
    stock_list,         # 股票代码列表  
    period,             # 周期类型  
    fields=None,        # 字段列表，None 表示返回所有字段  
    start_time=None,    # 开始时间 'YYYYMMDD'  
    end_time=None,      # 结束时间 'YYYYMMDD'  
    dividend_type='none', # 复权类型: 'none', 'front', 'back'  
    duckdb_path=None,   # 数据库路径  
    return_format='dict' # 返回格式，目前仅支持 'dict'  
):  
    """  
    从 DuckDB 获取历史数据，支持复权处理。  
  
    Returns:  
        dict: {股票代码: DataFrame}  
    """
```

### 3. khIndex（策略中高效读取）

`khIndex` 是专门为策略回测设计的轻量级读取函数，配合预加载机制实现零 IO 开销。

```
def khIndex(data: Dict, stock_code: str, field: str) -> float:  
    """  
    获取股票指标数据的便捷函数  
      
    语义上区分于 khPrice，专门用于获取自定义的指标（如 MA, RSI, MACD 等）。  
    底层直接包装调用 khPrice，共享其处理逻辑和容错机制。  
      
    Args:  
        data: 策略数据字典  
        stock_code: 股票代码  
        field: 指标字段名（例如 'MA_5', 'RSI'）  
          
    Returns:  
        float: 指标数值，如果获取失败或数据无效返回 0.0  
    """
```

### 4. khAddExtraFields（策略初始化注入）

`khAddExtraFields` 用于在策略初始化阶段声明需要预加载的字段。

```

    Args:  
        context: 策略的 context 字典  
        fields: 需要额外加载的字段列表，如 ['RSI_14', 'MA_5']  
    """
```
