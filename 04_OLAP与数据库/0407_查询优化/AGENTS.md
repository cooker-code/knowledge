## 排重指纹

```text
查询优化主题 + 统计信息/基数估算/代价模型/计划搜索/计划表示模块 + 核心机制 + 解决问题 + 适用边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都讲慢 SQL | 不能抽象为优化器机制时移到具体技术目录 |
| 都讲 AI 优化 | 比较是计划表示、成本估算、Join Order、鲁棒性还是自动索引 |
| 只有性能数字 | 没有查询结构、基线、执行计划和数据规模时降权 |
| 和 OLAP 引擎混淆 | 主角是 Doris/ClickHouse/StarRocks 实现时放对应引擎目录 |
