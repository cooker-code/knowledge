# 查询优化器

## 技术定位

| 项 | 内容 |
|---|---|
| 技术/主题 | 查询优化器 |
| 一级类目 | OLAP 与数据库 |
| 二级类目 | 查询优化 |
| 技术本体 | 数据库把 SQL 转换为执行计划并选择较低成本计划的一组规则、统计、搜索和代价模型机制 |
| 全局架构位置 | SQL 解析之后、执行引擎之前 |
| 主要使用者 | 数据库内核工程师、DBA、数据平台工程师、高级 SQL 使用者 |
| 主要产出 | 逻辑计划、物理计划、Join 顺序、访问路径、代价估算、优化建议 |

## 核心模块

| 模块 | 职责 | 重点问题 |
|---|---|---|
| 统计信息 | 表行数、NDV、直方图、列相关性 | 统计不准会导致基数估算错误 |
| 基数估算 | 估算过滤和 Join 后的中间结果规模 | 复杂谓词、非主外键 Join、字符串条件容易失效 |
| 计划搜索 | 枚举 Join 顺序和物理算子 | 搜索空间爆炸，需要剪枝和启发式 |
| 代价模型 | 估算 CPU、IO、网络、内存成本 | 公式假设可能无法覆盖真实工作负载 |
| 学习型优化 | 用模型学习计划成本或表示执行计划 | 关键在训练数据、计划表示、泛化和线上验证 |

| 对标对象 | 对标点 | 使用判断 |
|---|---|---|
| 手写 SQL 调优 | 人工改写与索引调整 | 单次业务慢 SQL 可先手工调优；反复误判需要理解优化器机制 |
