如果一句话总结这个版本：**它不是只加了一个新功能，而是在全屏交互、Remote Control、插件/MCP 稳定性、会话恢复和安全性上，做了一次比较完整的增强。**

对于高频使用 Claude Code 的用户来说，这版的实际价值不低。

---

## 这次更新最值得关注的 5 个点

### 1. `/tui` 来了，全屏模式更稳定

2.1.110 新增了 **`/tui` 命令** 和 `tui` 设置。

现在可以在同一会话里通过下面的命令切换到全屏渲染模式：

```
/tui fullscreen
```

官方给出的重点是：这个模式会带来更稳定、**无闪烁**的全屏渲染体验。

这意味着什么？

对长时间在终端里和 Claude Code 交互的人来说，最直观的提升就是界面更稳、阅读更舒服，频繁刷新带来的干扰会更少。

---

### 2. Remote Control 继续补能力，跨设备体验更完整

这版对 Remote Control 的补强也很明显。

主要包括：

* 新增推送通知工具
* 在开启 Remote Control 且配置“由 Claude 决定时推送”后，Claude 可以发送移动端推送通知
* `/autocompact`、`/context`、`/exit`、`/reload-plugins` 这些命令现在也能在 Remote Control 客户端使用
* `--resume` 和 `--continue` 现在会恢复尚未过期的计划任务

这些变化放在一起看，重点其实很明确：**Claude Code 正在把“离开当前终端后还能继续工作”这条链路补完整。**

对于需要电脑和手机来回切换的人，这些增强会很实用。

---

### 3. 焦点模式和快捷键职责更清晰了

2.1.110 还调整了交互逻辑：

* `Ctrl+O` 现在只在普通转录和详细转录之间切换
* 焦点视图改成通过新的 **`/focus` 命令** 单独切换

同时还有两个细节增强：

* 新增 `autoScrollEnabled` 配置，可以在全屏模式下关闭自动滚动
* `Ctrl+G` 外部编辑器新增选项，可显示 Claude 上一次回复，作为带注释的上下文

这类更新看起来小，但对日常使用影响很大。

因为它不是单纯“多了个开关”，而是在让交互逻辑更清楚，避免一个快捷键承担过多职责。

---

### 4. 插件安装和管理体验继续优化

插件相关也有一批实用更新。

`/plugin` 的 Installed 页面现在做了这些调整：

* 需要处理的项目和收藏项会置顶
* 禁用项默认折叠隐藏
* 支持按 `f` 收藏

另外还修复了一个更关键的问题：

* 插件安装会按照 `plugin.json` 中声明的依赖自动处理，并在安装时列出依赖

这比界面优化更重要。

因为插件系统一旦依赖没有处理好，后面遇到的问题通常不是“不方便”，而是“装上了但跑不起来”。这次算是把实际可用性又往前推了一步。

---

### 5. MCP、Hook、工具执行层稳定性提升很多

这版更新里，底层稳定性修复占了很大比例，尤其集中在 MCP、hook 和工具执行层。

比如：

* 修复 MCP 工具在 SSE/HTTP 传输中断时可能无限挂起的问题
* `/doctor` 现在会在同一 MCP server 被多个配置作用域定义、且端点不一致时发出警告
* 写入工具会在你手动修改 IDE diff 后提醒模型，避免模型误判当前提议内容
* Bash 工具现在会严格执行文档里定义的最大超时时间
* 修复 `PreToolUse` hook 的 `additionalContext` 在工具调用失败时丢失的问题

如果你只是轻度使用 Claude Code，可能不会第一时间感知这些变化。

但如果你已经在接入 MCP、插件、hook、自定义工作流，这些修复非常重要，因为它们直接关系到：**工具会不会卡住、上下文会不会丢、行为会不会和预期不一致。**

---

## 这次还修了哪些值得注意的问题？

除了上面这些主要改进，2.1.110 还有不少修复值得单独提一下。

### 安全相关

* 修复 “Open in editor” 在不受信任文件名场景下存在的命令注入风险
* 修复 `PermissionRequest` hook 返回 `updatedInput` 后，没有重新检查 `permissions.deny` 规则的问题
* 修复 `setMode:'bypassPermissions'` 更新没有尊重 `disableBypassPermissionsMode` 的问题

这部分虽然不一定每个人都会遇到，但它很重要，因为这类问题影响的是系统边界，而不是单次体验。

---

### 会话恢复与状态一致性

* 修复 `--resume` 在会话仍在运行或异常退出时，有时显示首个提示而不是 `/rename` 名称的问题
* 修复会话清理时没有删除完整会话目录的问题
* 修复 Remote Control 会话过旧时只显示通用错误，而不是提示重新登录
* 修复 claude.ai 改名后，Remote Control 会话标题没有同步到本地 CLI 会话的问题

这类问题的核心价值是：**减少“看起来恢复了，其实状态不对”的情况。**

---

### 性能与显示问题

* 修复全屏模式下工具运行时，选中文本导致 CPU 占用过高
* 修复 piped 非 TTY 输出中单行过宽可能造成的过度内存分配
* 修复焦点模式里 session recap、slash 命令输出和其他系统状态行不显示的问题
* 修复 `/skills` 菜单在全屏模式下内容溢出时无法滚动
* 修复 macOS Terminal.app 等不支持同步输出的终端启动时渲染乱码

这部分说明官方不仅在加功能，也在补“重度使用后才会暴露出来”的边界问题。

## 这版更新最大的用户价值是什么？

我觉得可以归纳成 4 点。

### 第一，更适合长时间使用了

`/tui fullscreen`、自动滚动开关、焦点模式拆分，这些都在提升高频交互场景下的可用性。

Claude Code 的终端体验，正在越来越像一个可以长期驻留的工作界面，而不只是一个偶尔调用的命令行工具。

### 第二，远程协作链路更完整了

推送通知、Remote Control 命令增强、任务恢复，这些更新组合起来，明显是在强化跨设备工作体验。

### 第三，高级工作流更稳了

插件、MCP、hook、写入工具、Bash 超时控制，这些都是高级用户高频踩到的区域。

这版的价值，不只是“新增了什么”，更是“减少了多少莫名其妙的问题”。

### 第四，安全边界继续补强

命令注入、权限检查、bypass 模式控制，这些修复说明官方在持续完善 Claude Code 的安全边界。

对个人用户来说，这提升的是安心程度；对团队使用来说，这提升的是可控性。

---

## 怎么看 Claude Code 2.1.110？

如果说前几个版本更多是在补体验细节和稳定性， 那么 **2.1.110 更像是一次围绕“长期可用性”做的集中打磨。**

它没有只追求新功能数量，而是在下面几条关键链路上同时推进：

* 全屏终端交互
* Remote Control
* 插件系统
* MCP / hook 稳定性
* 会话恢复
* 安全边界

对普通用户来说，这版会更顺手。

对重度用户来说，这版会更可靠。

而这类更新，往往比新增一个表面上更显眼的功能更有价值。

## 官方已确认的其他修复点

除上文重点提到的内容外，官方 changelog 还列出了这些修复：

* 修复 API 不可达时，非流式 fallback 重试可能卡住数分钟的问题
* 修复插件安装未按 `plugin.json` 中声明的依赖自动处理的问题
* 修复 `disable-model-invocation: true` 的 skills 在通过 `/<skill>` 调用时失败的问题
* 修复多工具调用轮次中，排队消息短暂重复出现的问题
* 修复 CLI 重启后按键丢失的问题，例如 `/tui`、provider 设置向导
* 修复 stdio MCP server 输出 stray 非 JSON 行时，首个异常行就被断开连接的问题
* 修复 headless / SDK 会话的自动标题在某些环境下会多触发一次 Haiku 请求
* 修复 Remote Control 会话标题与本地 CLI 会话不同步的问题
