## 技巧 1：跳过权限模式 —— 告别重复确认

### 痛点

刚开始使用 Claude Code 时，你会发现每执行一个操作都需要确认权限，频繁的授权提示会严重影响工作流畅度。

### 解决方案

启动时使用"危险模式"（其实一点也不危险）：

```
claude --dangerously-skip-permissions
```

**使用后：** Claude 直接执行，流畅到飞起！类似 Cursor 的 YOLO 模式，但更可控。

---

## 技巧 2：图片输入 —— 充分利用视觉理解能力

### 你可能不知道的事

Claude Code 不仅能理解代码，还能"看懂"图片！这是很多人忽略的强大功能。

### 使用方法

* **Windows/Linux:** `Ctrl+V` 粘贴图片
* **Mac:** `Cmd+V` 粘贴图片

### 为什么有用？

"一图胜千言"，在编程世界同样适用。用视觉信息沟通，比文字描述清晰 100 倍。

---

## 技巧 3：深度思考模式 —— 解决复杂问题的秘密武器

### 什么是深度思考模式？

Claude 有多个"思考档位"，可以根据问题复杂度调用不同级别的计算资源。

### 四个思考级别

只需在提示词中包含关键词即可激活： （中文的思考关键词也会激活）

```
think → think hard → think harder → ultrathink
```

**使用方法：**

```
 think 分析这个分布式系统的性能瓶颈
```

## 技巧 4：Plan Mode —— 最重要的功能！

> **如果只记住一个技巧，请记住这个：规划的时间越多，成功的可能性越大。**

### 激活方式

连续按两次 `Shift + Tab`

### 工作原理

Plan Mode 不会修改任何内容，你可以放心的讨论清楚后，再执行修改

### 官方统计

使用 Plan Mode 的成功率比直接执行高 **40%** 以上！

## 技巧 5：及早且频繁纠偏 —— 保持正确方向

发现 Claude 的方向不对时，不需要停止任务，直接打字告诉它调整即可。Claude 会接受最新的指令并修正方向。

**示例：**

```
等等，这个方案不太合适，我们用 GraphQL 而不是 REST API
```

这比等到最后再返工要高效得多，及早调整能节省大量时间。

## 技巧 6：模型切换策略 —— 平衡质量与速度

Claude Code 提供两个模型：Opus 4.0（质量高、成本高）和 Sonnet 4.5（速度快、性价比高）。

### 推荐策略

* **规划阶段** → Opus（深度分析、架构设计）
* **执行阶段** → Sonnet（代码实现、文档编写）
* **遇到难题** → 切回 Opus

### 切换方法

输入 `/model` 然后选择想要的模型。

混合使用可节省约 50% 成本，同时保持 95% 的质量。

---

## 技巧 7：规范开发流程 —— Spec → Todo → Code

### 三步法则

很多人直接说"帮我实现 XXX"就让 Claude 写代码，容易需求偏差、遗漏功能。建议使用三步法：

**Step 1: 先写需求文档和技术文档**

**Step 2: 让claude code生成 Todo**

**Step 3: 逐项实现**

### 为什么有效？

* ✅ 思路清晰，不遗漏
* ✅ 可视化进度

## 技巧 8：TDD 测试驱动开发 —— 对抗 AI 幻觉

### 为什么重要？

AI 最大的问题是"幻觉"（生成看似正确但有 bug 的代码）。TDD 是最有效的对抗手段。

## 技巧 9：控制上下文 —— 节省Token，也加快执行速度

### 问题

对话历史会累积消耗 token，成本呈指数增长：

### 两个命令

**`/clear` - 清空历史**

* 开始新功能时用

**`/compact` - 压缩历史**

* 需要保留部分历史时用
* 长时间对话需要"瘦身"时用

## 技巧 10：Memory 记忆管理 —— 让 Claude 记住你的偏好

通过创建 `CLAUDE.md` 文件，让 Claude 永久记住你的编码规范和偏好，避免每次重复说明。

### 两种级别

* **项目级：** `./CLAUDE.md` 或 `./.claude/CLAUDE.md` - 当前项目的特定规范
* **全局级：** `~/.claude/CLAUDE.md` - 所有项目通用偏好

### 实用命令

```
/memory      # 查看当前加载的所有 memory 文件/init        # 为项目自动生成 CLAUDE.md 模板
```

Claude Code 会从当前目录向上递归查找并自动加载所有 CLAUDE.md 文件。
