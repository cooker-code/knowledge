Claude Code 更新了一个小功能：`/loop`。

 

一句话说就是，你可以让 Claude 定时执行一段指令。每 5 分钟检查一次任务状态、每小时采集一次数据、2 小时后提醒你开会——都行。

 

听起来没什么大不了的。但用了之后，我发现它改变的不只是一个功能。

# 一个场景讲透

我们在做自媒体，多个平台同时发内容。公众号、小红书、B站——每个平台都有后台数据。

 

有两个选择：

 

**选择一：自己写个定时脚本。** 每天自动抓数据，存到表里。定时没问题，但它就是个脚本——抓什么字段你得提前写死，数据拿回来你还得自己看。它用不了 Claude 的 Skills，跑不了 Agent team，更不会帮你分析"为什么这篇涨了"。

 

**选择二：手动问 Claude。** “帮我看看昨天的数据怎么样。” Claude 分析得挺好，能调 Skills、能跑 Agent team、能给你结论。但问题是——**你得记得问。** 忙起来就忘了。等你想起来，已经过了三天，趋势断了。

 

一边是定时但没脑子，一边是有脑子但不定时。

**`/loop` 把两边的好处合在一起了。** 定时执行，同时拥有 Agent 的全部能力——Skills、Agent team、推理判断，一个都不少。

 

这也是为什么 `/loop` 必须跑在 Claude Code 里。它不是一个通用的定时工具，它的价值来自于 Agent 的全部能力：

* **调 Skills**：你积累的写作 Skill、数据分析 Skill、排版 Skill，定时任务里全能用
* **跑 Agent team**：一个定时任务可以同时派多个 Agent 并行干活
* **用 MCP 工具**：连数据库、调 API、操作浏览器——你手动能用的工具，定时任务里都能用
* **读项目上下文**：它知道你的项目结构、知道 CLAUDE.md 里的规则、知道之前的对话历史
* **自然语言指令**：不用写代码，用中文描述你要干什么就行

 

不是"定时执行一段脚本"，是"定时唤醒一个了解你项目的 Agent，让它用全部能力帮你干活"。

# 实际效果

回到数据分析的例子。你只需要一行：

```
/loop 1d 获取昨天各平台的内容数据，分析哪篇涨了，为什么，有什么规律
```

Claude 每天自动执行一次。因为跑的是 Agent，它能调用 Skills 采集数据、跑 Agent team 做多维分析、用推理能力找规律——然后把结论整理好等你看。

 

不是一堆数字扔给你，是"昨天那篇小红书涨了 300%，标题用了XX关键词，建议下次继续用"。

 

它不会打断你——调度器在后台跑，只在你当前对话结束的间隙才插入。你正在跟 Claude 聊别的事？它会等你聊完再说。

 

而且，平台改版了？接口变了？脚本会挂，Agent 不会——它发现结构变了，会自己调整解析逻辑，继续跑。出了 bug 它能自己修。

# 语法指南

## 命令结构

`/loop` 是一个内置 Skill，不是底层工具。你输入的一行命令，其实分两部分给了两个不同的角色：

`/loop` 只认英文间隔——它把 `1d` 转成 cron 表达式 `0 0 */1 * *`（每天0点），然后把提示词原封不动交给调度器。到点了，调度器把提示词发给 Claude，Claude 理解中文、调 Skills、跑分析。

 

所以：**时间间隔必须英文**（`5m`、`2h`、`1d`），**提示词随便用中文**。

 

完整的流程长这样：

你输入 `/loop 5m 帮我检查任务完成了没`，`/loop`（Skill）解析出时间间隔和提示词，调用 `CronCreate` 创建定时任务。调度器每秒检查，到点了把提示词发给 Claude，Claude 理解意图、调用工具、返回结果。管理任务用 `CronList` 查看、`CronDelete` 删除。

## 三种写法

间隔写前面：

```
/loop 5m 帮我检查任务状态
```

间隔写后面：

```
/loop 帮我检查任务状态 every 2h
```

不写间隔，默认每 10 分钟：

```
/loop 帮我检查任务状态
```

## 管理任务

查看现在有哪些定时任务：

```
我现在有哪些定时任务？
```

取消某个任务：

```
取消那个采集数据的任务
```

直接用中文说就行，不用记命令。

## Cron 表达式速查

大部分时候你用 `/loop 5m` 就够了。但如果你想精确控制触发时间，可以直接写 cron 表达式。

Cron 表达式就是 5 个字段，分别代表什么时候触发：

| 表达式 | 意思 |
| --- | --- |
| `*/5 * * * *` | 每 5 分钟 |
| `0 * * * *` | 每小时整点 |
| `0 9 * * *` | 每天早上 9 点 |
| `0 9 * * 1-5` | 工作日早上 9 点 |
| `0 9,18 * * *` | 每天早 9 晚 6 |

`*` 是"任意"，`*/5` 是"每隔5"，`1-5` 是"从1到5"，`9,18` 是"9和18"。记住这四个写法就够用了。

# 边界

最后说几个限制。

**关掉窗口就没了。** 所有定时任务只存在于当前 Claude Code 会话里。关掉终端，任务全部消失。不跨重启。

 

**3 天自动过期。** 循环任务创建 3 天后自动删除。防止被遗忘的任务一直跑下去。

 

**不补执行。** 如果 Claude 正忙，任务到期了会等它空了再触发一次。但不会补上中间错过的。
