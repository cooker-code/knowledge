相信很多同学在使用claude code的时候，面对众多命令和使用技巧，可能都有点手足无措，来来去去也只能记得其中的几个，特别是对于非技术出来的人，github上各类学习仓库也太多。

大多数人的路径是：先用 `/help` 看一眼有哪些命令，然后关掉，然后只会用最基础的几个操作。剩下那些高级功能——Hooks、多智能体协调、MCP 集成——听起来很厉害，但不知道从哪里开始学，也不知道自己用没用好。

这个问题，Claude Code 上周直接出了一个命令来解决：**`/powerup。`**

---

## /powerup 是什么

Claude Code v2.1.90（2026年4月1日）新增的交互式学习系统命令。

在 Claude Code 会话里直接输入 `/powerup，`它会在终端内启动一个带动画演示的教程菜单。不用打开浏览器，不用翻文档，就在你工作的那个窗口里学。

这个命令想解决的问题很具体：**你不知道自己不知道什么。**

Claude Code 的功能分了好几层：

* 基础：提问、生成代码、改 bug
* 中级：上下文管理、CLAUDE.md、自定义命令
* 高级：多智能体协调、Hooks 自动化、MCP 集成、代码库级别重构

从基础到高级，中间断层很大。大多数人卡在中间，不是因为能力不够，是因为根本不知道那条路怎么走。

`/powerup` 专门补这个断层。

---

## 怎么用

在 Claude Code 会话中直接输入：

```
/powerup
```

进入交互菜单后，用方向键选课程，Enter 进入，跟着动画演示一步步操作。

全程不需要离开终端。

---

## 10 个核心课程讲了什么

目前 `/powerup` 提供 10-18 个结构化课程，从上下文管理一路覆盖到多智能体协调。核心的 10 个：

**1. 上下文管理**

Claude 怎么读取你的代码库？`@` 提及和 `.claudeignore` 分别什么时候用？

很多人以为上下文越多越好，结果把大量无关文件喂进去，反而降低了回答质量。这门课讲怎么精准控制 Claude 能看到什么。

**2. CLAUDE.md 项目记忆**

如何配置"项目记忆"，让 Claude 在每次新对话里都自动知道你的项目规则、技术栈、工作习惯。

从"每次开场都要重新解释"到"一次写好永久生效"，这是其中最关键的一步。

**3. 计划模式（Plan Mode）**

用 `Shift+Tab` 切换计划阶段——Claude 先生成多步骤路线图，你确认之后再执行。

处理复杂任务时，这能避免跑了一半才发现方向跑偏。

**4. Hooks 与自动化**

在命令执行前后设置触发器，构建自定义工作流。

比如：每次 Claude 改完代码就自动跑测试；每次对话开始时自动加载特定上下文。把 Claude Code 从"交互工具"变成"自动化流水线"，Hooks 是入口。

**5. 智能体与子智能体**

生成并行子智能体同时处理多个任务。

一个大任务拆成三个部分，三个子智能体并行，最后合并——比单线程逐步走快很多，处理大型任务的正确姿势。

**6. MCP 集成**

把 Claude 连接到外部工具和数据库：Notion、Slack、数据库、自定义 API，都可以通过 MCP 接入，让 Claude 直接调用。

**7. 自定义斜杠命令**

从 Markdown 文件构建自己的 `/命令，`把常用工作流打包成命令复用。

**8. 多智能体协调**

分层系统：一个主智能体协调，多个专业子智能体并行执行。或者对等团队：多个智能体独立工作，最后汇总结果。大规模工程任务的高级玩法。

**9. 代码库级别重构**

6 阶段重构模式 + Git Worktree 隔离：把大规模重构拆成安全可逆的步骤，独立工作区保护主分支。

**10. 工作流编排**

链接多个工具调用，设置自主循环执行，处理需要多步骤、跨工具的复杂工作流。

## 配套命令：/insights

`/powerup` 有个搭档：**`/insights。`**

两个命令的区别是：

* `/powerup：`

  这里有所有课程，你来选
* `/insights：`

  系统分析了你的实际用法，这是你应该先学的

`/insights` 会分析你的历史使用记录，生成个性化的反馈——哪里用得好，哪里有盲区——然后针对性推荐课程。

举个例子：如果你一直在每次对话开头手动粘贴项目背景，但从来没用过 CLAUDE.md，`/insights` 会发现这个模式，推荐你去学 CLAUDE.md 课程。

**推荐顺序：**

1. 先跑 `/insights，`找到自己的盲区
2. 再跑 `/powerup，`针对性学推荐的课程
3. 用到实际项目里，然后再跑一次 `/insights` 看变化

---

## 和查文档有什么区别

文档的问题是你得知道搜什么才能搜到答案。

`/powerup` 的设计不一样：

* 动画演示，不是干读文字
* 在终端里学，学完直接用，零切换成本
* 结构化进阶路径，不需要自己规划怎么学
* `/insights`

  找你的具体盲区，不是让你从头学一遍

更接近"交互式培训"，而不是"查手册"。
