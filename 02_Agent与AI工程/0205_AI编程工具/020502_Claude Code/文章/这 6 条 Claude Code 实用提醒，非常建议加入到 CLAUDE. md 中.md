1. 1. 在编写任何代码前，先描述你的方法并等待批准
2. 2. 如果我给出的需求模糊，请在编写代码前提出澄清问题
3. 3. 完成任何代码编写后，列出边缘案例并建议覆盖它们的测试用例
4. 4. 如果任务需要修改超过 3 个文件，先停止并将其拆分成更小的任务
5. 5. 出现 bug 时，先编写能重现该 bug 的测试，再修复直到测试通过
6. 6. 每次我纠正你时，反思你做错了什么，并制定永不再犯的计划
