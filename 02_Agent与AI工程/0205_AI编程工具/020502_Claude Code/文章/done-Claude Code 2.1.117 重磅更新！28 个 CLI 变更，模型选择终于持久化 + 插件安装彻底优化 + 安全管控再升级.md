**大家好，我是你的 AI 工具观察员。就在刚刚，Anthropic 的 Claude Code CLI 悄然推送了 2.1.117 版本。虽然官方没有大张旗鼓，但更新内容相当扎实：整整 28 项 CLI 调整，精准击中了我们这些长期泡在终端里的开发者最隐秘的痛点。**

如果你每天都在用 Claude Code 跑任务、写代码，或者把 CLI 当作主力编码伙伴，这波更新非常值得立刻执行 `claude update`。因为它把模型切换迷糊、插件安装麻烦、安全管控薄弱、各种小卡顿等问题，一一给解决了。

### 一、三大核心亮点

1. 1. **模型选择终于聪明了**
   以前项目里 pinned 一个模型，启动时经常被覆盖。现在启动时会明确告诉你当前使用的模型，以及它是来自项目设置还是 managed-settings。切换模型再也不用迷糊，体验大幅提升。
2. 2. **插件安装体验大升级**
   已经安装过的插件再次 `install` 时，会自动补齐缺失的依赖，避免手动折腾。依赖报错也会直接给出安装提示，`claude plugin marketplace add` 还能自动从已配置的市场解析依赖，省心不少。
3. 3. **安全管控更严格也更可靠**
   `managed-settings` 中的 `blockedMarketplaces` 和 `strictKnownMarketplaces` 现在在 `install`、`update`、`refresh`、`autoupdate` 全流程都生效，强制只允许使用已批准的来源，安全感直接拉满。

### 二、其他重要修复与优化（28 项 CLI 变更精选）

本次更新还干掉了大量日常使用中的老大难问题：

* • OAuth token 过期时自动刷新，不会再突然提示 “Please run /login”
* • WebFetch 处理超大 HTML 页面不再卡死
* • Proxy 返回 204 时不再崩溃，而是给出清晰错误
* • NO\_PROXY 环境变量终于被正确尊重（Bun 环境下也生效）
* • Undo（Ctrl+\_）现在正常工作，不会跳步或失效
* • Opus 4.7 的 context 计算 bug 修复（之前按 200K 计算，现在正确识别 1M）
* • `/resume` 命令会主动询问是否先 summarize 大 session，避免长时间卡顿
* • macOS 和 Linux 原生构建搜索速度大幅提升（Glob/Grep 换成嵌入式 bfs + ugrep）
* • 启动时本地和远程 MCP server 并行连接，默认更快
* • 插件依赖处理、智能清理、闲置渲染循环等问题均得到优化

整体来看，这次更新包体仅增加 0.2%，提示 token 几乎无变化，却在稳定性和流畅度上带来了明显进步。

### 三、我的真实感受：技术满分，策略却让人复杂

作为从 Claude Code 早期就一直使用的老用户，我必须说：**这次更新在技术层面绝对是满分**。团队把日常最难受的那些小痛点，一个个都认真修复了，用起来确实更丝滑、更舒服。

但与此同时，打开 Claude 的定价页却让我心情有些复杂——**Claude Code 这一核心功能在 Pro 计划里已被标记为 ❌**，只有 Max 5x / Max 20x 计划才能继续使用。

我理解 Anthropic 需要控制成本、推动更高阶订阅，但先是大幅收紧使用限额和 compute，再把 Pro 用户常用的 CLI 功能直接砍掉，这一步确实让很多老用户感到突然。

功能越来越强、体验越来越好，可“好用”却正在和“负担得起”慢慢脱钩。这让不少原本铁杆的 Pro 用户，开始认真思考：是继续 Pro、咬牙升级 Max，还是寻找其他更接地气的替代方案？

### 四、总结

Claude Code 2.1.117 是一次非常务实、非常“懂开发者”的更新。它没有堆新模型、没有炫酷大功能，却把终端使用的舒适度和可靠性拉到了新高度。

**技术上值得点赞，产品策略上却值得我们认真思考。**
