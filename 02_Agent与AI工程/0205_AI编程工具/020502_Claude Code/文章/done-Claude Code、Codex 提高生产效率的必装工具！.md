**导读：** 这个刚刚开源的AgentBro 必装！

如果你现在已经在用 Claude Code、Codex、Gemini CLI，可能会遇到同一个问题：Agent 很能干，但它经常在你没注意的时候停住。一会儿要权限，一会儿问你选项，一会儿写完计划等你批准；你刚切到浏览器查资料，它又在终端里等你。

AgentBro 做的事情很直接：给这些 AI Agent 一个桌面灵动岛，把最容易被忽略、但又需要你处理的时刻收进一个小浮窗里。项目已经在 GitHub 开源，如果你觉得这个方向有用，欢迎先点个 Star，也欢迎一起来提 Issue、交 PR。

## 我为什么想推荐它

这几年 AI 编程工具的主战场，一直在模型能力、上下文长度、IDE 插件和 CLI 体验上。但真正日常用起来，很多人的痛点没那么宏大。

你开着编辑器，旁边跑一个 Claude Code；另一个窗口里还有 Codex；远程机器上可能还挂着一个 Gemini CLI。它们不是一直需要你盯着，但一旦需要你，它们通常卡在最不方便的地方：终端里、tmux 里、SSH 里，或者某个你已经切走的窗口里。

AgentBro 的思路不是再做一个巨大的控制台，而是先做一个很小、很准的入口。

它作为桌面浮层展示 Agent 当前状态。需要你处理的东西，比如权限、提问、计划审批、直接回复、完成提醒，都可以在这个浮窗里完成。

我喜欢这个Bro，是因为它没有试图替代终端和编辑器。

它更像是给 Agent 加了一层“桌面通知 + 即时处理”的外壳。终端继续做终端的事，编辑器继续做编辑器的事，AgentBro 负责把那些需要你伸手按一下的时刻，送到你眼前。

一个好用的 Agent 工具，不一定要替你写更多代码；更重要的是，每次该你审批、该你留意的时候，它都能稳稳叫到你。

## 它到底解决什么问题

AgentBro 官方 README 里给它的定义很清楚：面向 AI Agent 的桌面灵动岛。它会观察 Claude Code、Codex、Gemini CLI 等会话状态，把权限请求、问题、计划、工具调用、快速回复、远程会话和完成提醒，收进一个轻巧的桌面浮窗。

说人话就是：你不用到处找 Agent 卡在哪了。

现在这类 CLI Agent 的交互，往往有几个典型卡点：

| 卡点 | 以前的体验 | AgentBro 的处理 |
| --- | --- | --- |
| 权限审批 | 切回终端看 diff | 浮窗里直接批准或拒绝 |
| 提问选择 | 找到对应会话再回答 | 单选、多选、自由输入都在卡片里 |
| 计划审批 | 终端里读长 Markdown | 浮层展示计划、任务和权限 |
| 会话定位 | 忘了 Agent 在哪个窗口 | 一键跳转回对应终端或客户端 |
| 对话详情 | 切窗口翻上下文 | 直接在浮窗里查看详情 |
| 完成提醒 | 靠终端输出或系统通知 | 灵动岛里展示结果和后续回复 |
| 多会话并行 | 靠窗口记忆 | 会话列表统一收纳 |

会话列表

这里展示所有你正在工作的Agent，你可以清楚的看到所有的工作状态，并且展示了所有需要你审批处理的会话

查看会话详情

当收到通知的时候，可以快速回复，快速查看上下文，你需要查看历史会话记录，直接点击详情就可以查看，偷偷告诉你，在详情里面可以直接对话哦

这不是“更炫的通知中心”。核心差别在于：AgentBro 不只是告诉你“有事发生”，它把处理动作也放在同一个地方。

多个提问选择

Edit授权

比如 Edit 权限，不只是弹一个“是否允许”。它会把工具参数、文件路径、diff 和操作按钮一起展示出来。你能快速判断这次改动是不是合理，然后选择拒绝、允许一次、始终允许，或者自动批准。

一键跳转

它也支持一键跳转。你从灵动岛看到某个 Agent 卡住了，不需要先想“这是哪个终端、哪个 tab、哪个客户端”，直接跳回对应会话就行。这个能力听起来小，但多 Agent并行时很省脑子。

这类细节对重度用户很关键。因为 Agent 真正进入日常工作后，最烦的不是它不会做事，而是它在正确的时间点找不到你，或者你找不到它。

授权通知

当Agent在自行运转的时候，需要问你要授权的话，如果你在做别的事情，这个时候你很可能就忘记了，然后任务一直就无法推进下去，你需要一个授权通知卡片，并且一键授权

Plan计划授权

当Agent在工作的时候，会有计划模式，等规划好了之后会让你查看并且授权是否继续执行

一键回复

会话太多，看不过来，每次有消息通知你都要切过去回复，其实你并不需要这样，AgentBro支持你快速回复，让你更聚焦当下任务，不被打扰

Subagent展示并且查看

当Agent在工作的时候，会派生出多个子任务来执行，在Agentbro里面你可以详情的看到每个Subagent的执行状态

移动到任意为止

因为刚好MacOS的灵动岛跟AgentBro的灵动岛可能会重合，所以你可以把Agentbro移动到任意的位置

除了上面这些功能还有很多等你去发掘的小心思小功能~

## 最该看的，是这些真实演示

这次官网演示有个很加分的点：它不是画了一个假 UI 动画，而是直接复用了 AgentBro 应用里的真实组件。

从代码里能看到，官网 demo 引入的是 AgentBro 应用内部的 `NotchPanel`、`SessionStore` 和主题系统。也就是说，页面上看到的审批卡、提问卡、计划审批、会话详情，并不是营销图，而是生产组件用 fixture 数据跑出来的状态。

这点很重要。

很多工具官网看起来很完整，装完才发现真实应用是另一回事。AgentBro 现在把真实交互直接摆在首屏上，反而降低了理解成本：你一眼就能知道它会在什么时候出现、会让你做什么、会不会挡住主工作区。

直接回复也很实用。Agent 写完一段回复后，你不一定想切回终端继续追问。AgentBro 可以在浮窗里给出回复卡片和输入框，让你把下一句补上。

它还可以直接查看对话详情。也就是说，你看到一个提醒，不必立刻切到对应终端或客户端，只要在这个轻量浮窗里展开，就能看上下文、看结果，甚至继续回复。这才是我觉得它顺手的地方：不是把所有东西都搬进一个大后台，而是在你需要的那一下，把最小必要信息和动作摆出来。

这对“多 Agent 并行”的工作方式尤其友好。你可以让一个 Agent 跑测试，让另一个 Agent 看文档，让第三个 Agent 做代码审查。只要它们需要你，AgentBro 就把关键节点收回来。

## 兼容的 Agent 已经不少

AgentBro 现在刚开源，第一版先把灵动岛、Hook 接入、快速处理和远程会话这些核心链路跑扎实。官网写的是 17+ Agent，README 里列出的适配器已经覆盖 Claude Code、Codex、Gemini CLI、Cursor / Cursor CLI、GitHub Copilot、Cline、Trae、Qwen、Kimi、OpenCode、Factory、StepFun、AntiGravity、WorkBuddy、Hermes、Pi、Kiro 等一长串。

这里我最关心的不是数量，而是它对“不同入口”的兼容。

现在很多人的 AI 编程工作流并不只在一个工具里发生。Claude Code 有一套习惯，Codex 有一套习惯，Cursor 又是另一套。AgentBro 的价值，是在这些工具上面放一层统一的桌面交互，把“等你确认”的时刻统一收口。

它还支持 SSH Remote，把远端机器上的 Agent 事件接回本机灵动岛。对经常在服务器、远程开发机、tmux 里跑 Agent 的人，这个功能比看起来更重要。

比如你在远程服务器上跑 Claude Code、Codex 或其他 Agent，本地其实很难一直盯着那个 SSH 窗口。你希望远端 Agent 要权限、问问题、跑完任务时，本地能收到提示，就可以通过 SSH Remote 把这条链路接回来：远程继续跑任务，本机负责接住关键提醒和操作。

因为真正的远程开发痛点，常常不是算力不够，而是你不知道远端那个 Agent 什么时候停下来等你。

## 怎么上手

AgentBro 的上手路径也比较清楚：

直接去官网  https://www.agentbro.net 下载免费使用即可

如果你习惯用命令行，可以把它理解成这样一条链路：

💡 PROMPT

上手步骤：
1. 从 GitHub Releases 下载 AgentBro 当前可用版本
2. 安装后打开 AgentBro
3. 进入 Settings -> Island -> Integration
4. 运行 Hook Doctor
5. 点击 Install All Hooks，或只安装你正在使用的 Agent Hook
6. 重启 Claude Code / Codex / Gemini CLI 等会话

截至我写这篇时，GitHub Releases 页面能看到 `v0.3.0`。开发者可以一起来共建”这款工具

README 明确写了，第一个开源版本聚焦灵动岛模块。Agent Monitor、Agent Switch、Skills 管理等更大的模块，后续会逐步开放。

平台支持上，AgentBro 当前从 macOS 开始，Windows 和 Linux 都在后续路线里。也正因为刚开源，很多适配、体验细节和路线优先级，都可以被社区一起影响。

## 还有一点：它不是只能黑底黄字

这种常驻桌面的工具，视觉风格其实很影响使用意愿。

AgentBro 内置了多套主题，包括午夜、AgentBro 经典、磨砂玻璃、苹果、烟灰、海雾、暖纸、柔薰衣草，以及跟随系统。官网演示里也接入了同一套主题系统，可以直接切换看效果。



我个人更喜欢它没有把自己做成一个特别“赛博”的大面板。

AgentBro 这个产品如果做重了，反而会输。因为它服务的是心流，不是存在感。它应该在需要时出现，在不需要时安静待着；它应该让 Agent 更顺手，而不是成为另一个要管理的系统。

从现在的设计看，它在这个分寸上是有意识的。

## 适合谁，不适合谁

我会优先推荐给三类人：

第一类，是已经在日常用 Claude Code、Codex、Gemini CLI 的开发者。只要你每天有多个 Agent 会话，AgentBro 就很容易提供价值。

第二类，是经常让 Agent 跑长任务的人。比如重构、测试、文档整理、批量修改、远程机器任务。你不需要一直盯着它，但你需要它卡住时能马上叫到你。

第三类，是喜欢本地优先工具的人。AgentBro 的 Hook Server 默认跑在本机，支持 `/tmp/agentbro.sock` 或 `127.0.0.1:17892`，整体方向更接近桌面工作流增强，而不是把所有东西都搬到云端。

但如果你只是偶尔在网页上问问 ChatGPT，或者只在 IDE 里轻度用补全，那 AgentBro 现在未必是刚需。它真正解决的是“AI Agent 已经成为日常协作者之后，怎么把它们管理得更顺手”。

我的判断很简单：AgentBro 不是在抢 IDE 的位置，也不是在抢 CLI 的位置。它抢的是“该你留意的那一秒”。

如果你已经开始同时使用多个 AI 编程 Agent，这一秒很值钱。
