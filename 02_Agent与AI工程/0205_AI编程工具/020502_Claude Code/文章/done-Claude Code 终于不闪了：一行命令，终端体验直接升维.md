如果你用过 Claude Code 跑过长对话——我是说真正的那种，几百轮来回、多个 Agent 并行跑任务——你肯定遇到过这事儿。

屏幕闪烁。

不是偶尔闪一下，是那种让你的眼睛怀疑"我是不是该换显示器了"的持续闪烁。内容跳跃。刚看到一句话想细读，啪，整个画面重绘了，找不到刚才在哪。

我几乎每次使用 Claude Code 都会遇到这个问题，一旦交互轮次稍微多一点，重新切回对话框时，界面就会不受控制的上下乱跳，只有手动去拖滚动条才能解决问题。

说实话，挺烦的。

## 这个问题有多普遍？

翻了一圈 GitHub Issue 和社区讨论，发现不是个例——简直是一片哀嚎。

有人在 Hacker News 上专门开了帖子讨论这事儿——"Claude Chill: Fix Claude Code's flickering in terminal"。底下回复炸了，各种终端用户都在吐槽： kitty 、 WezTerm 、 iTerm2 ，基本全覆盖。有人甚至说"我怀疑是我的显卡坏了"——结果不是，就是 Claude Code 的渲染问题。

问题根源其实挺技术性的：传统终端渲染在长对话或多 Agent 运行时，会进行全屏重绘。每次 Claude Code 有新输出，整个终端画面都被清空再重画一遍。这事儿做得挺蠢的，说实话。

**画面闪烁**——因为重绘不是瞬时的，你能看到"旧画面消失 → 新画面出现"的过程。

**资源消耗随对话长度持续上升**——内存和 CPU 使用率跟对话轮次正相关。跑久了不是卡就是崩。

我之前试过一个笨办法：每跑一段时间就新建一个会话。有点用，但治标不治本。而且当你真正进入心流状态的时候，切会话这动作特别打断节奏——说实话挺搞心态的。

更扯的是，有时候你忘了切，跑了大半天，终端突然卡住。那种感觉……怎么形容呢，就像你写了两个小时代码没保存， IDE 崩了。想砸键盘。

直到昨天。

## NO\_FLICKER ：一行命令，问题没了

Claude Code 在 v2.1.88 版本悄悄加了一个实验性功能——NO\_FLICKER 模式。

开启方式简单到离谱：

```
CLAUDE_CODE_NO_FLICKER=1 claude
```

或者直接写进 shell 配置文件永久生效：

```
echo 'export CLAUDE_CODE_NO_FLICKER=1' >> ~/.zshrc
source ~/.zshrc
```

原理是这样的：它切换到备用屏幕缓冲区——跟 vim 或 htop 全屏接管终端的方式一样。只渲染当前可见视口的内容，不是每次都重绘整个历史记录。

效果呢？

不闪了。是真的不闪了。

长对话往上翻看历史记录，流畅。多 Agent 并行跑任务，稳。资源消耗曲线从"一直往上爬"变成了"基本平着走"。

## 意外惊喜：鼠标也能用了

这个我没想到。

NO\_FLICKER 模式带来的不只是消除闪烁，还有完整的鼠标交互支持。这事儿说起来可能不算大事，但用起来——真香。

**光标定位**：直接点击输入框的任意位置，光标就过去了。不用再按几十下方向键调整光标位置。

**交互元素可点击**：折叠的工具调用结果，点一下展开/收起； URL 直接点击在浏览器打开；文件路径点击后在默认编辑器打开。

**智能选中复制**：拖拽选中文字，松开鼠标自动复制到剪贴板。双击选词，三击选行。在支持 Kitty 键盘协议的终端（ kitty 、 WezTerm 、 Ghostty 、 iTerm2 ）里，选中状态下 Ctrl+C 执行复制，不会打断选择。

**滚轮翻看历史**：就……正常滚动。不用按 Page Up 了。

怎么说呢，有点像从命令行界面突然有了图形界面的感觉。不是那种花里胡哨的 GUI ，而是"本该如此"的顺畅。

## 当然，也有代价

新渲染器不是完美无缺的。

原生 Cmd+F 搜索可能没法直接用了——这事儿挺烦的。需要通过内置快捷键（ Ctrl+O 后输入 /）进行搜索。部分终端行为也需要重新适应。

说实话，刚开始适应那会儿我有点不爽。用了两年的 Cmd+F 突然不灵了，肌肉记忆全废。但——算了，也不是不能克服。

但——

根据内部用户反馈，多数人表示更偏好这一模式。我自己的体验也是：回不去——呃，或者说，不想回去了。

一旦习惯了不闪烁、能用鼠标的终端，再切回旧模式会觉得"怎么这么难受"。

## 值不值得开？

如果你是重度 Claude Code 用户——每天用它写代码、跑任务、做 Agent 协作——我的建议是：

**开**。

一行命令的事，试完不喜欢随时能关回来，而且就我目前的体验来看是好处多于坏处的。

如果你只是偶尔用用，或者主要用短对话，那可能感知不强。毕竟闪烁问题在短对话里不明显。

但有一个判断标准我觉得挺准的：如果你曾经因为 Claude Code 的终端体验感到过烦躁——哪怕只有一次——那就值得试试这个功能。

因为那个烦躁不会自己消失。它只会随着你用得越来越多，变得越来越难以忍受。
