# 020504_Hermes知识地图

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 02_Agent与AI工程 / 0205_AI编程工具 / Hermes |
| 解决的问题 | 把 Hermes 作为具体 AI 编程工具来理解，而不是把所有 Hermes 文章混进 Agent 框架或电脑工具 |
| 典型输入 | 用户任务、项目目录、Profile、SOUL.md、Skill、知识库、Webhook、外部工具 |
| 典型产出 | 可执行任务、代码/文档产物、看板状态、Skill、记忆沉淀、自动化结果 |
| 用户阅读目标 | 判断一篇 Hermes 文章补的是入口、配置、协作、记忆、Skill、集成还是稳定性 |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| H00 入口与安装 | Hermes 怎么启动、以什么界面使用 | CLI、Web UI、Desktop 是入口差异，不是能力本体 | 安装清单只入路由表 |
| H01 Profile 与 SOUL | 如何定义角色、规则和协作习惯 | Profile/SOUL 是 Hermes 的项目人格和协作边界 | 能说明角色边界和冲突处理才精读 |
| H02 任务运行时 | 长任务如何保存状态、恢复和续航 | 运行时要看状态持久化、Checkpoint、Judge 闭环 | 长任务、恢复、稳定性文章进入 |
| H03 Kanban 协作 | 多 Agent 如何拆分、排队、交接和合并 | Kanban 是状态管理，不只是看板 UI | Hermes Kanban、多 Profile 协作文章进入 |
| H04 Skill 与 Learning Loop | 经验如何变成可复用 Skill | Skill 沉淀要有触发、版本、审查和回滚 | Hermes Skill、Learning Loop 文章进入 |
| H05 记忆与知识库 | Hermes 记住什么、如何检索和更新 | 记忆要区分事实、经验、偏好和任务状态 | Hermes 记忆、Obsidian、知识库文章进入 |
| H06 外部集成 | 如何连接浏览器、CLI、Webhook、IMA、Obsidian | 集成价值在于减少人工转译和保存证据 | Webhook、Playwright、AutoCLI、IMA 文章进入 |
| H07 质量与边界 | Hermes 什么场景不该用 | 需要区分工具能力、用户配置和模型能力 | 踩坑、稳定性、路线对比文章进入 |
