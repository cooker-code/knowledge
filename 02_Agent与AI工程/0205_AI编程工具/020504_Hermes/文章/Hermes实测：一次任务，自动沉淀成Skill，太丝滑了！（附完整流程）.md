1 Hermes智能体是什么？

直接了解Hermes官网，一句话总结Hermes是什么：

它是一个**真正的自主智能体（Autonomous Agent）**，并且运行时间越长，能力越强。自主智能体，就是能自己思考、自己做事、自己执行完整流程的AI.

Hermes最大特色，在使用过程中会把经验变成技能，不断优化，并持续记住和理解用户，从而越用越强，学术和工业界首先做到这点的。

**所以这篇文章的重点，就是探究它是如何把经验自动转化为技能（Skill）的，并通过一个例子来展示这个过程。**

# 2 从零安装 Hermes

# 

我们先从最基础的一步开始，把 Hermes 真正安装并跑起来。

安装本身并不复杂，打开终端之后，执行下面这行命令即可：

```
curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash
```

这一步做完，Hermes 只是被装到了你的系统里，还不算真正可以使用。真正关键的是后面的初始化配置，因为这一步决定了它有没有接上模型、能不能正常工作。

接下来执行：

```
hermes setup
```

这一过程里，

你需要给它配置模型提供商、填写对应的 API key，再选择具体使用的模型，比如 qwen3.5-plus：

国内千问Base URL为：

```
https://dashscope.aliyuncs.com/compatible-mode/v1
```

这里特别值得提醒一句，这一步非常容易踩坑。很多人看到报错会以为是 Hermes 有问题，实际上大多数情况都不是。真正的问题通常出在配置没有对上。API key、provider、base URL，这三者必须严格对应。

你用阿里的 key，就要接阿里的 endpoint；这个我刚开始填错了，导致来回折腾了几遍才发现问题出在这里，看到这里的读者可以小心一些，直接复制过去。只要这里不一致，就很容易出现 401 这种认证失败的报错。

4 Hermes技能Skill创建

对于其他Agent 来说（包括龙虾🦞），一次任务做完就结束了，下次再来类似需求，仍然是重新思考、重新规划、重新执行，整个过程像临时发挥一样，每次都是从零开始。

但 Hermes 不一样。第一次帮你完成“写文章并生成 Word”这件事时，这对它来说是一段经验，而它会进一步把这段经验抽象成 Skill。

最后总结一下

Hermes最强大的地方，就是会把提问丝滑创建为Skill，而Hermes的经验还不是一次性的，而是会被沉淀；沉淀下来的内容又不是静止不变的，而是会在继续使用中被优化。

随着使用越来越深入，你会慢慢感受到一个变化：你不再只是“用一个工具”，而更像是在“带一个会成长的智能助理”。
