01 > 让AI成为自己的生产力

如何让AI真正意义上地成为我们自己的生产力？相信大家跟我有同样的困惑。那么，接下来我就用自己最近3个月的尝试和获得的成果来跟大家分享一下。

最终花了3个月的业余时间，大约平均每天2个小时的投入（也就是利用每天下班后的2-3个小时的时间），做出了以下这个工具站：

https://tooli.fun/

虽然有点四不像，但是至少真正意义上经历了一个产品从无到有，从1.0到2.0的心路历程。

从思路，到产品规划，到开发、测试、部署、运维，我都自己做了一遍。有些环节事情也是挺多的，例如部署阶段服务器购买，服务安装，域名购买和工信备案、公网安备之类，很繁琐。

很多以前不熟悉的领域，也是通过AI边学边做。以下我会把每个过程的细节都进行分享。

02 > 作为普通人，我们如何借助AI成就自己

文章内容较长，有些细节较琐碎，适合作为教程或故事阅读。我把每个阶段遇到过什么问题，如何处理，以及使用的AI工具和的Prompt技巧等内容都记录下来。相信这段经历能为每个想用好AI的人带来收获。

首先我这样思考，作为一个10年数据人，我一直以来都是为企业进行数据治理，不论是搭建数据框架，还是跟进数据挖掘分析。似乎所有掌握的技能都无法支撑我自己去做一些ToC的产品。当然也有可能是我自身存在局限。

如果我想自己做产品，不管是ToC，还是ToB，那我要怎么做？我能怎么做？是不是只能做数据产品？是不是需要搭建团队？是不是需要掌握更多更全面的技能？

想得太多，反而会把自己困住。因为有太多的疑问，太多的未知需要去找答案。何不随意找个方向尝试，并在尝试过程中根据数据来调整方向。

在AI的浪潮里，不要止步于自我的无知。于是，说干就干。没有思路，那就让AI给我思路。☞

从AI给出来的建议里，我选择了 [小众工具型应用]，因为它难度较小。其他的需要体系化构建，而且标准难统一。甚至需要搭建一支具备一定战斗力的团队。而这明显不符合我的现状。

而工具站类型的也是我曾经想要尝试做的，现在契机正好。

于是我拓宽了思路，不把自己局限在数据类的工具，而是有需求的或者是我能做的任何工具。

当然，能在某个垂直领域做深做精才是更好的选择，但我本着探索的目的，就不给自己设限。

因为我使用过一些工具站，解决一些小需求。一些小工具即开即用，很方便。比如时间戳转换，编码转换...

于是我决策结果是，做一个工具站，没有特定方向。有用的，或者有趣的，或者好玩的，我能想到的，我能做出来的，都会尝试加入到工具站里。

于是乎，后面做着做着又多出了资源分享模块，继续做着做着又多出了导航链接模块...

回到正题，万事开头难。第一步，选型。

刚开始我不确定所有代码从0到1全部自己写或者用AI写能否完成。要不就找个开源的工具站模板做。

于是去GitHub上翻了一圈。结果没找到心仪的。

后来在网络上看到几个工具站界面挺好看，想着要不直接抄算了。于是找了几个能拷贝网站的AI，例如：

https://copyweb.ai/  
https://same.new/

结果得到的东西一言难尽。困惑了2天，仔细想了下，如果要符合自己期待的样子，最好还是得自己从头开始做。

前后端分离架构的暂时不考虑，因为短时间内自己在纯前端这块现有能力真跟不上。而自己本来还是懂一点HTML+CSS+JS, 那么可以采用MVC的架构来做。

于是我有了基础的思路：使用Python的Django框架+HTML。于是我需要2个助手，一个开发Django，一个开发HTML。当然这两个角色肯定还得AI来扮演。

最初选用Django时，我还有个疑虑，虽然我用Python多年，但是基本都是数据处理相关的，框架也仅仅用过Fastapi，没用过Django和Flask之类的web框架。

同样也是让AI结合我的现状在Django和Flask做个对比选择。

最终我采纳了AI的建议，选择Django作为后端框架，然后以Django展开，开始学习怎么上手并做出首页。

先让AI给出Django的学习资料，然后得到一堆的学习资源，刚好周末花了2天左右的时间，把Django的基础语法和框架用法大体看了一遍。

当下也有很多生成网站的AI工具，号称一句话就能生成一个网站，基本各个大厂都推出了类似的工具。

但从实际测试效果看，大部分结果也是一言难尽。小部分前端效果还行，但是基本属于全静态页面。

而要能够全面自定义的功能，还是需要把页面拆开来优化和集成。与其忍受各种限制，我还不如自己来定义生成每个页面。

于是我就组建了一支AI团队。

1️⃣ 我的前端工程师

有个工具叫deepsite，huggingface上的地址为：

https://huggingface.co/spaces/enzostvs/deepsite

该工具能够根据指令生成一个完整的且精美的HTML网页。

例如我输入要求：生成一个工具网站的首页

对应的HTML代码我就不贴上来了，因为没意义，读者可以自己尝试生成。以下是生成的结果预览：

值得一提的是，GitHub上有个大神对该工具进行魔改后开源了：

https://github.com/kiritoko1029/deepsite

因此，借助该开源工具，我能够在本地无拘无束地生成网页，调试每个网页的布局和样式。（借此也向这些开源大神们致敬🫡）

直接clone代码后，修改`docker-compose.yml` 文件中模型的API地址和Key即可。

```
version: '3.8'  
services:  web:    image: 195658/deepsite:latest    ports:      - "3000:3000"    environment:      - OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxx      - OPENAI_MODEL=deepseek-chat      - OPENAI_BASE_URL=https://api.deepseek.com      - APP_PORT=3000    restart: always
```

然后运行下载并启动容器

```
docker-compose up -d
```

访问：http://localhost:3000/

 于是，我现在拥有了以为专业的前端工程师了。

生成工具站首页，并经过各个细节的调优后，得到以下的可用的页面。并将html放到项目中，于是网站的第一个页面就完成了。

有时生成的页面结构布局或者样式达不到一致的风格，那么可以先把首页的页面放入代码框，再重新输入Prompt按照首页的风格生成新的页面。

这样每个页面都能承接上个页面的样式风格。最终保持整个网站的风格一致。

在刚开始的那2周里，每天几乎就是在生成和调试各个页面。最后优化好各个页面布局和样式，得到以下各个可用的基础页面。（不包含工具页面）

所有HTML页面可以在这里下载：

https://www.tooli.fun/resource\_detail?resource\_id=227

另外，关于HTML页面生成还有一个技巧，就是我们可以参考Deepsite工具的Prompt去用AI生成：

```
{  model: 'deepseek-chat',  messages: [    {      role: 'system',      content: `ONLY USE HTML, CSS AND JAVASCRIPT. If you want to use ICON make sure to import the library first. Try to create the best UI possible by using only HTML, CSS and JAVASCRIPT. Please write clear and helpful comments in your code to explain the structure and functionality. Use as much as you can TailwindCSS for the CSS, if you can't do something with TailwindCSS, then use custom CSS (make sure to import <script src="https://cdn.jsdelivr.net/npm/@tailwindcss/browser@4"></script> in the head). Also, try to ellaborate as much as you can, to create something unique. ALWAYS GIVE THE RESPONSE INTO A SINGLE HTML FILE 请使用中文编写所有的注释。`    },    { role: 'user', content: '生成一个工具网站' }  ],  stream: true,  max_tokens: 8000,  temperature: 0}
```

翻译系统的Prompt：

```
仅使用HTML、CSS和JAVASCRIPT。如果你需要使用ICON，一定要先导入相关库。尽量只使用HTML、CSS和JAVASCRIPT来创建最好的UI。请在代码中写下清晰有用的注释来解释结构和功能。尽可能多地使用TailwindCSS作为CSS，如果你无法使用TailwindCSS，那么就使用自定义CSS（确保导入<script src="https://cdn.jsdelivr.net/npm/@顺风/browser@4"></script>在头部）。此外，尽你所能做到精致，以创建独特的东西。始终将响应放入一个HTML文件中。请使用中文编写所有的注释。
```

实际使用时可以将

`<script src="https://cdn.jsdelivr.net/npm/@顺风/browser@4"></script>`

替换成为：

`<script src="https://cdn.tailwindcss.com/3.4.16"></script>`

因为前者加载太慢，并且我在使用过程中经常出现加载不出来的情况。

使用该提示词在Deepseek官网或者元宝里使用Deepseek模型，也能和deepsite工具达到相似的结果。

后续在遇到超长文件时也需要使用这种方式，因为deepsite的大小限制会导致它能实现的功能有限，有些复杂的样式逻辑总是生成一半就结束了。因此遇到这种情况需要使用Deepseek官网或者元宝等客户端。

后续有时间再单独展开讲讲这块。

总结下来有一点最重要，就是在每次生成页面的提示里，都要要求AI使用TailwindCSS，这样不仅有用现代化的美观样式，还能确保整个站点的样式一致性。

2️⃣ 我的后端工程师

前面介绍了我的前端工程师，那么接下来介绍下我的后端工程师。

相比于前端工程师，我的后端工程师极其简单。因为仅仅是一句Prompt。

```
你是一个资深的web全栈开发专家，精通Python的Django开发。用户正在开发一个在线工具站点，该站点包含生活实用工具，职场效率工具，儿童教育工具，和集成AI工具等各种类型的工具集合。用户将会使用Django作为开发框架，结合现有的TailWindCSS和HTML的模板作为前端。你的目标是和用户一起将这个网站产品开发并部署落地。并且你需要根据一个产品从开发到部署各个阶段遇到的任何问题给出解决方案与合理的建议。你将自行从用户提出的问题中判断用户进展到某个阶段。用户预计会经历的阶段有：1：学习Django框架，完成基础框架搭建。2：完成基础页面，以及用户注册登录功能，开发积分管理体系。3：完成相应在线工具的开发并集成到网站中。4：网站部署上线，并不断集成新的工具。5：开发管理后台，用于管理用户信息，工具信息等。当用户提出问题时，你作为全栈助理，需要根据用户的需求进行计划拆解，功能开发，流程优化和产品建议等。
```

实际使用过程中，并不总是需要用这个Prompt，该Prompt更适合工具的system的Prompt。

我使用大模型客户端，往往直接将问题输入，并给出对应有问题的部分资料，例如代码块等。AI就能给出答案。大部分情况下问题都能顺利得到解决。

3️⃣ 我的网站运营专员

这个角色跟后端工程师角色一样，都是实用简单的Prompt来扮演。

```
你是一个资深的网站运营，现在需要你去给一个网站做推广，主要从SEO到自媒体等途径。没有经费预算，全部用免费的方案。以下是网站的基本信息：网站首页地址是：https://www.tooli.fun/web网站的主要内容是：工具类网站，包含生活实用工具，职场效率工具，教育教学工具等。同时还有一些资料下载，导航链接等模块。网站主要的用户：中国用户。你需要根据网站实际情况，给出实用的推广或优化方案。并且考虑到网站的盈利问题，需要给出长期建设方面的建议和可落地的操作。
```

AI给出的也都是正常可执行的方案，没有特别高的难度，只需跟着步骤推进即可。

当遇到某个步骤不懂或者不清晰时，直接拿该步骤深入问答。让AI不断拆解，细分。最终都是可以细化到可执行步骤去推进。

按照AI给出来的SEO策略，很快就能让Google收录，但是百度的却一直都是0收录。待有空还得研究下百度的SEO策略。

05> 最后说点鸡汤

---

现在AI越来越强大，身边很多同事，不管是以前共事的老同事，还是当下团队的小伙伴们。都多多少少有些焦虑。日后会不会被AI取代。是不是要去多学点技能提升自己的竞争力。

我个人认为，AI只是一把武器，只是它在不同人的手里有不同形态。在有的人手里是把水果刀。但是在有的人手里是把加特林。前提是我得认为AI能成为加特林并且想着让它成为加特林。

奥特曼说过，取代工作岗位的不是AI，而是用AI的人。

所以通过这几个月的尝试，我也更加深刻地理解了这个问题。AI可以补足我们的短板，我们要做的，是付出所有时间和精力构建我们的长板。

如果想做一件事情，不要被未知和疑惑困住，先做起来。一切未知的困难都是没遇到火的纸老虎。而你一旦点燃了心里那团火。自然无需为他所困。
