## 四步完整流程

▲ 从数据采集到报告生成，四步全自动

### STEP 1：数据采集

WorkBuddy 可以调用 Python 的 `requests` 或 `Selenium` 自动爬取数据。典型用法：

```
帮我爬取 XXX 网站的商品价格数据，
包含商品名称、当前价格、销量、评分，
保存为 data.csv
```

WorkBuddy 会自动写爬虫脚本、处理分页、识别反爬机制，直到数据采集完整为止。

若目标网站需要登录，可以先打开浏览器手动登录，再让 WorkBuddy 接管已登录的 session 继续爬取。

### STEP 2：数据清洗

原始数据往往有缺失值、重复行、格式不统一。只需要说：

```
读取 data.csv，清洗数据：
- 去除重复行
- 价格列去掉「元」字符，转为数字
- 缺失的评分用该类目均值填充
- 保存为 data_clean.csv，输出数据摘要
```

WorkBuddy 会用 Pandas 完成清洗，输出清洗前后的数据量对比、各字段的空值率等摘要信息供你确认。

### STEP 3：可视化

清洗完成后，告诉 WorkBuddy 你想看什么图：

```
基于 data_clean.csv：
1. 画过去12个月的销售额折线趋势图
2. 画各渠道用户占比柱状图
3. 画价格区间和销量的相关性散点图
所有图保存为 PNG，宽1200px，背景白色
```

WorkBuddy 会调用 Matplotlib / Seaborn 生成高清图表，参数全部自动调整，颜色搭配合理，图例完整。

▲ WorkBuddy 自动生成的可视化图表示例

### STEP 4：AI 生成分析报告

这一步是 WorkBuddy 区别于普通 Python 脚本的核心环节。它不只产出数据，还会读懂数据：

```
分析上述图表和数据，生成一份分析报告：
- 指出3个关键发现
- 标注任何异常数据点
- 给出2个可操作的业务建议
- 保存为 report.md
```

WorkBuddy 会综合数据统计结果和图表信息，生成一篇有观点、有逻辑的分析报告——而不只是描述数据本身。

## 6大实用场景

▲ 6个真实业务场景，WorkBuddy 全部支持

### 场景1：竞品价格监控

每天早上自动爬取竞品的商品价格，和昨天的数据对比，价格有变动时自动发邮件告警。

以前需要专门的监控工具或运营人员每天手动检查，现在一次配置好，WorkBuddy 定时自动跑。

### 场景2：用户行为分析

把用户日志文件（CSV 或 JSON）扔给 WorkBuddy，让它自动计算各步骤的转化率、绘制漏斗图，找出流失率最高的环节。

一段通用的 prompt 模板：

```
读取 user_log.csv，这是用户行为日志数据，
字段包含 user_id, event, timestamp。
帮我分析用户从「注册」到「首次购买」的转化漏斗，
找出流失最多的步骤，出一张漏斗图。
```

### 场景3：财务数据汇总

多个部门提交了不同格式的 Excel，WorkBuddy 可以自动读取所有文件、按统一格式合并，计算各部门的实际 vs 预算差异，生成汇总 Excel 报告，节省财务人员每月末最痛苦的那半天。

### 场景4：社媒舆情分析

爬取微博、知乎、抖音评论区的用户评论，自动做情感分析（正面/负面/中性），生成词云图和情绪趋势折线图。品牌公关、产品团队可以实时掌握用户口碑变化。

### 场景5：A/B 测试分析

提供 A 组和 B 组的实验数据，WorkBuddy 自动计算 p 值、置信区间，告诉你实验结果是否显著、推荐使用哪个方案——不需要你自己搞统计学。

### 场景6：周报自动生成

这个最省心：结合上文说的 Automation 定时任务，设置每周一早 8 点自动运行，WorkBuddy 读取数据库数据、跑分析脚本、生成周报文档，推送到钉钉/微信，全程无需人工介入。

## 实战示例：用 WorkBuddy 分析公众号数据

下面是一个完整的真实案例演示——分析公众号近30篇文章的阅读数据，找出高互动选题规律。

第一步，导出数据：登录微信公众平台，在「内容分析」里导出近30篇的数据 CSV（包含标题、发布时间、阅读量、点赞数、分享数）。

第二步，把文件拖到 WorkBuddy 工作目录，然后输入：

```
读取 article_data.csv，这是我公众号近30篇文章的数据，
字段包含：标题、发布日期、阅读量、点赞、分享。
请帮我分析：
1. 哪些标题关键词与高阅读量相关？
2. 一周中哪几天发布的文章互动更好？
3. 阅读量前5名和后5名的文章有什么规律差异？
4. 给我3条选题建议
所有分析结果输出为 report.md，包含对应的可视化图表。
```

几分钟后，WorkBuddy 会输出：

* 关键词热力图（哪些词出现在高阅读量文章标题里）
* 发布时间与互动率的折线图
* 高低表现文章的对比分析
* 3条具体的选题方向建议（带数据支撑）

这个分析如果手工做，要写 Python、跑模型、想结论，少说3~4小时。WorkBuddy 一次对话，完整跑完。

## 几个提升效果的 Prompt 技巧

**1. 告诉 WorkBuddy 你的角色和目的**

不要只说「分析这份数据」，要说「我是电商运营，想找出高转化的商品特征，用于选品决策」。背景越清楚，输出越有针对性。

**2. 要求输出具体文件**

明确说「保存为 report.md」「图片保存到 charts/ 目录」，WorkBuddy 会自动创建文件，方便你后续使用。

**3. 让 WorkBuddy 主动发现异常**

在 prompt 里加上「如果发现数据异常或有趣的规律，主动指出来」——这一句可以让分析质量提高一个台阶。

**4. 迭代式分析**

先让它出一份粗略分析，看完之后再说「重点深入分析第3点，用户留存的下滑原因」，一步步挖深，比一次性给长 prompt 效果更好。
