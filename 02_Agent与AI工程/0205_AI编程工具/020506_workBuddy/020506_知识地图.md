# workBuddy 知识地图

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 02_Agent与AI工程 / 0205_AI编程工具 / workBuddy |
| 解决的问题 | 把 workBuddy 作为具体 AI 编程/自动化工具来理解，而不是按使用场景散落到数据分析、效率工具或 Agent 框架 |
| 典型输入 | 用户任务、项目/文件上下文、网页或聊天记录、数据文件、模型和工具配置 |
| 典型产出 | 自动化执行结果、代码或文档产物、数据报告、摘要、知识沉淀和任务记录 |
| 用户阅读目标 | 判断一篇 workBuddy 文章补的是入口、设置、工具调用、记忆、数据分析、长任务还是质量边界 |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| W00 入口与配置 | workBuddy 如何启动、选择模型、连接账号和本地环境 | 入口配置只决定可用性，不等于工程能力 | 基础教程只入路由表，除非补权限或环境边界 |
| W01 任务建模 | 用户任务如何被拆成可执行步骤 | 任务要有输入、输出、停止条件和人工接管点 | 场景文章只有说明可复用任务结构才精读 |
| W02 工具调用 | 如何控制浏览器、文件、聊天、网页或外部服务 | 工具能力必须看权限、审计和失败恢复 | 浏览器控制、微信摘要、网页操作文章进入 |
| W03 上下文与记忆 | workBuddy 记住什么，如何复用历史和知识 | 记忆要区分事实、偏好、任务状态和经验 | 长记忆、知识闭环文章进入，但要和 Memory Management 排重 |
| W04 编程与协作 | 如何和 Codex、DeepSeek、CodeBuddy、Cursor 等协作 | 编程协作价值在于上下文交接、验证和可回滚 | AI 编程、模型/工具切换、代码产物文章进入 |
| W05 数据分析任务 | 如何完成爬取、清洗、可视化和报告 | 数据分析文章必须有输入、过程、结果和验收 | 只有图表展示或效率体验时降权 |
| W06 质量与边界 | 什么场景不该用或需要人工验收 | 自动化能力不能替代验证、权限和可追责 | 踩坑、失败、对照和权限文章优先吸收 |

## 待补缺口

| 流程节点 | 缺什么 | 后续文章进入时怎么判断 |
|---|---|---|
| W01 任务建模 | 缺少 workBuddy 如何拆任务的真实例子 | 看是否给出输入、步骤、状态和退出条件 |
| W02 工具调用 | 缺少权限、失败恢复和审计说明 | 看是否能复现浏览器/微信/网页控制链路 |
| W03 上下文与记忆 | 缺少记忆写入、检索、更新和冲突处理证据 | 看是否区分事实、偏好、经验和任务状态 |
| W04 编程与协作 | 缺少与 Codex、Cursor、CodeBuddy 等同任务对照 | 看是否有同任务、同验收标准和产物差异 |
| W06 质量与边界 | 缺少不可用场景和人工验收门槛 | 看是否列出失败案例、权限风险和回滚方式 |
