### 一、Cursor Rules：AI编程行为的"交通规则"

Cursor Rules本质上是一套用于自定义AI在Cursor中行为的指令系统，可以理解为对大型语言模型(LLM)的高级提示词。

它解决了AI编程中常见的"乱改代码"问题，让开发者能够：

1. 统一团队编码风格
2. 保持项目一致性
3. 减少代码审查工作量
4. 提高AI生成代码的可用性

Cursor支持两种规则设置方式：

#### 1. 全局规则

位置：Cursor设置中的`Rules > User Rules`特点：适用于所有项目，适合个人开发者设置通用偏好

#### 2. 项目规则

实现方式：

* `.cursor/rules`目录中的`.mdc`文件（推荐）
* 项目根目录的`.cursorrules`文件（即将废弃）

#### 3. 文档规范规则

文件路径：`.cursor/rules.document.mdc`

应包含：

* Markdown编写规范
* API文档模板
* 版本变更记录格式
* 技术文档结构要求

#### 4. 版本控制规则

文件路径：`.cursor/rules.git.mdc`

关键内容：

* Git提交消息格式
* 分支命名规范
* Merge Request流程
* 代码审查要点
* 敏感信息处理

### 三、高级使用技巧与注意事项

1. **规则优先级**：项目规则 > 全局规则
2. **版本控制**：建议将规则文件纳入Git管理
3. **规则调试**：使用命令查看生效规则
4. **性能优化**：避免设置过多复杂规则
5. **团队协作**：定期Review和更新规则

常见问题解决方案：

* AI偶尔不遵守规则：尝试简化规则描述
* 规则冲突：检查不同类型规则的优先级
* 新规则不生效：重启Cursor或清理缓存
