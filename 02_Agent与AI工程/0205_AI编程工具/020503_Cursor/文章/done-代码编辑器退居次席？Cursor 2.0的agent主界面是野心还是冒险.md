老金我跟Cursor有段不解之缘。

之前前前后后推了好久的的Cursor教程，写了不少文章。

那时候Cursor还是IDE的代表，AI编程的首选，也是老金作为一个不懂代码的人第一个接触的编程软件。

后来老金我发现，Claude Code这个CLI工具功能更强大，就转投了CLI的阵营。

但这不代表我不关注IDE的发展了。

今天看到Cursor 2.0发布，老金我这个老用户还是激动了。

从1.x到2.0，这是Cursor历史上最大的版本更新。

很多人还没意识到，这可能代表了AI编程的一条新路线。

在老金我看来，从ChatGPT 3.5开始，AI创作经历了三代进化：

第一代：结构化提示词

第二代：低代码编排平台（n8n、dify、coze）

第三代：AI编程

当然，这只是老金我的个人看法，不一定对。

今天Cursor 2.0的发布，让第三代AI编程有了新玩法。

## 很多人为什么还在怀疑AI编程

老金我知道，很多人对AI编程还有顾虑。

代码补全老是猜错要写啥。

给出的建议驴唇不对马嘴。

改bug的时候更离谱，有时候改出更多bug。

很多人就觉得，这玩意儿不行。

但老金我一直相信这只是工具问题，不是方向问题。

直到最近看到一份研究报告，验证了我的想法。

METR找了16个资深开发者，记录了146小时的编程录像。

这些人都是大型开源项目的主力，平均有多年开发经验。

结果发现：资深开发者用AI编程，反而慢19%。

当然，这个研究样本很小，而且是在高质量开源项目中。

在其他场景下，AI工具可能完全不一样。

但这个研究至少说明了一个问题：

AI工具不是万能的，需要学习和适应。

这时候我才明白。

不是AI编程不行，是我没用对工具。

## 今天发布的Cursor 2.0到底牛在哪

今天Cursor发布了2.0版本。

老金我第一时间看了官方发布，直接惊了。

### Composer模型，快得离谱

Cursor自己搞了个AI模型，叫Composer。

专门为编程设计的，速度快了4倍。

这里注意，如果想切回老界面，在左上角进行切换。

什么概念？

官方数据显示：大部分任务30秒内完成。

这是Cursor首席研究员Sasha Rush在X上说的：

"Composer是我们用RL训练的大型MoE模型，专门用于真实世界编程，而且非常快。"

如果对你有帮助，记得关注一波~

###

### 多智能体并行工作

这是最震撼的功能。

可以同时运行多个AI智能体，各自干活，互不干扰。

就像一个团队，有人写后端，有人写前端，有人写测试。

每个智能体都有自己的专长，但能完美协作。

但Hacker News上有用户提出了质疑：

"我不理解现在推多智能体。让单个智能体可靠工作都是问题，让多个同时运行，还可能有重叠的diff，这看起来不可能。"

这个担心很有道理，但老金我觉得技术会慢慢成熟的。

### 应用内浏览器正式发布

AI写完代码，可以直接在编辑器里测试。

不用来回切换窗口，不用自己复制粘贴。

AI一边写，一边在浏览器里跑，看到不对立马改。

这个功能在1.7版本是beta，2.0正式上线了。

### 语音输入编程

对着电脑说话就能写代码！ 但它没有触发词概念，你只能找个安静的地方自己去和Cursor说话。

Cursor官方在X上宣布："用语音模式将语音转换为代码。"

老金我觉得这个功能适合写一些简单的页面。

复杂逻辑还是得手动敲，但交互方式很新颖。

## 踩坑指南

基于官方文档和社区反馈，老金我提醒大家注意这些坑。

### 坑1：别完全依赖AI

AI写的代码一定要看，一定要测试。

Reddit上用户分享过：

"你需要对编程语言有扎实理解，因为AI生成的代码不一定准确，需要手动修复调整。"

AI是助手，不是替代品。

### 坑2：注意成本控制

Cursor最近涨价引发了很多不满。

Hacker News用户抱怨：

"以前25个工具调用算一次请求，现在每个单独计费，配额用得快多了。"

如果你要用，一定盯紧用量。

### 坑3：大项目要小心

Reddit用户反馈：

"Cursor在大型项目上表现不佳，特别是涉及多个文件时，经常修改你没要求改的部分。\*\*

如果你的项目很大，建议先在小型项目上测试。

## 为什么老金我一直推AI编程

老金我从AI编程刚开始就一直在推。

很多人觉得还早，可以等等。

老金我告诉你们：现在已经晚了。

看看这些数据：

* X上 200万 人观看 8岁女孩 用AI做聊天机器人的视频
* Hacker News上Cursor 2.0的讨论帖已经有几百个回复
* 官方宣布Composer模型完成大部分任务只要 30秒

这不是趋势，这是现实。

就像15年前，有人觉得用智能手机不是必需品。

现在你敢说不用智能手机吗？

在老金我看来，这就是普通人成为更自由创造者的路径。

第一代：学会写提示词，让AI按你的想法干活。

第二代：用编排平台，把不同AI工具串联起来。

第三代：直接编程，让AI成为你的技术合伙人。

老金我觉得每进一步，创作的自由度就提升十倍。

当然，这只是我的个人观察。

现在开始学第三代，还能赶上早班车。

再等两年，可能就跟不上时代了。

## 老金为什么关注这次更新

Cursor的这次2.0更新，老金我实测下来，其实有着很多的BUG与不舒服的体验，但经历过的都知道，Cursor几乎是一天一个小版本，天天更新到你崩溃。

话说回来，我在这次的更新上看到了未来的有一条路。

Cursor 2.0最大的改变是什么？

老金我觉得，是把 agent页面作为了主界面 。

以前IDE的核心是代码编辑器，现在变成了智能体。

代码编辑器从主座退到了次席。

这个变化背后，是Cursor对AI编程未来的判断：

AI编程的核心不是写代码，而是指挥AI写代码。

所以界面要围绕着"指挥AI"来设计。

老金我觉得这个思路很有意思。

虽然我现在主力用Claude Code CLI，但作为IDE，Cursor 2.0确实走在了一条新路上。

这条路能不能走通，老金我不好说。

但至少，他们敢于尝试，值得尊重。

对自己好一点，关注一下这次更新吧。

虽然老金我现在主力用Claude Code CLI。

但作为曾经的Cursor老用户，我真心觉得2.0的这次更新值得关注。

工具这东西，没有绝对的最好。

CLI有CLI的优势，IDE有IDE的用途。

重要的是保持开放的心态，拥抱好的变化。

这样，不管技术怎么发展，咱们都能跟上节奏。
