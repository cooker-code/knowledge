基础原理

**AI 模型**

我们先看下面的图片，宏观理解一下我们 AI 模型，以及用 AI 来写代码的感受。

> Vibe: 氛围

来源：https://baoyu.io/blog/slot-machine-vibe-coding-comparison

为什么会这么比喻，让我们深入今天世面上的模型工作原理，因为底层就是概率。大家不要急着关掉，放心，这里没有让人头痛的公式。这里只有大白话。今天世面流行的模型都是一句朴素的定律：大数定律。怎么理解？模型已经学会海量信息，其内部分布已经“拟合”信息的真实分布，你输入模型的每一句话，模型其实都“知道”，它会找到最相似的几个，然后输入其中一个。随着你输入的字符越多，那么其输出某些特定字符的概率就越大。给人一种必然发生，或者说模型理解你的输入的感觉。

> 大数定理：在数学与统计学中，大数定律又称大数法则、大数律，是描述相当多次数重复实验的结果的定律。根据这个定律知道，样本数量越多，则其算术平均值就有越高的概率接近期望。 大数定律很重要，因为它“说明”了一些随机事件的均值的长期稳定性。

> 编者注：训练过程就像在统计投掷无数次公平🎲之后，每个数字出现的概率。模型学会了这个公平🎲的概率分布，即每个数字出现的概率为 1/6。

没错，就是这样，和你不会做选择题一样，通过扔骰子来决定选择选项是一回事。

但是，和你纯粹扔骰子，有个区别，你可以控制概率，相信聪明的你已经发现，将问题尽可能描述清楚，说白了，就是把问题描述详细！详细！详细！

当然，怎么详细？产生了提示词工程（角色、背景、目标、要求、约束、例子、魔法词 step by step等），当然现在叫做上下文工程了。

**Agent**

明白了基本原理，让我们理解另一个名词，Agent。

> Agent: 代理人

来源：https://bytebytego.com/guides/what-is-an-ai-agent/

```
# 过程用户输入 - AI 模型理解（“匹配”） - loop:调用工具，根据新信息进一步理解 - 输出答案  
# 工具- 规划(Todo List)- 读文件、搜索、访问网站- 写文件、记录笔记- 执行命令  
# 扩展- 调用 MCP（一种将工具外置的扩展协议）- 委托子 Agent
```

Multi Agent，在 Single Agent 之后必然的、朴素的想法。

来源：https://www.anthropic.com/engineering/multi-agent-research-system
