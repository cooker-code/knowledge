声明：本篇文章，除了封面使用了Midjourney，其余文字均为原创。

最近终于把Open Code装好了，也开始跟上时代步伐，尝试使用各种 skill 了。

在下载了不少官方 skill 之后，我发现了一个问题：

就是这些 skill 存下来之后，成为这样一个个文件夹，我很难去记住每个 skill 它到底是干什么的，而且每个 skill 都是英语，看得人非常头大。

所以在下载了七八个 skill 之后，我就产生了一个需求：

我需要有一个 skill，专门可以对新增的这些 skill 进行管理，我希望能把它们的信息同步到一个云表格进行管理，而且要让我可以一眼就看出来，这些 skill 到底每个功能是什么。

这样我在调用 skill 的时候，我可以很方便地根据这些中文去找到我需要用的 skill，让Open code去使用。

所以我第一个手搓的，属于自己的专属 skill，就是对这些 skill 进行管理，并同步到飞书多维表格的 skill。

做完以后效果是这样的，下面的表格完全是Open code去使用这个skill完成的，可以看到效果非常棒。

有每个 skill 的名称，每个 skill 的用途描述，它存放的路径是什么，它是哪一天被我下载或者创造出来的，它所属的技能分类是什么？目前的状态是已同步还是已更新的？还有它占的大小是多少？

看起来非常一目了然。

这些信息如果你手工填进去，其实是蛮累的一件事，特别是技能描述。

因为有些 skill 的技能描述还是英语的，我是让它直接进行了翻译，包括路径的粘贴也挺费劲的。

而现在，我只要和 Open Code 说一下，帮我更新一下新的 skill ，这一行的记录就可以自动上去了，我觉得非常方便。

当然手搓 skill 之前需要下载一个神器，是专门帮助你进行定制 skill 的 skill。

今天这个文章写得有点拗口，管理 skill 的 skill，定制 skill 的 skill，就有点好玩儿，是吧？

这个 skill 的名称叫 skill-creator，也是 Anthropic 官方出的一个。下载之后，你就可以根据自己的需求去定制任何一个属于你的 skill 了，我觉得是最方便的一个技能。

当然 Anthropic 官方还有不少好用的通用技能，像 docx，PDF，pptx，xlsx，都可以顺便下了。

地址在这里：

https://github.com/anthropics/skills/tree/main/skills

我也没有自己下载，就是直接跟Open Code说，到这个地址去下载 skill-creator 就好了，它自己会做，下载到哪里也不用你管，它都自己知道。

接下来，我就说一下，我是怎么通过 Open Code来手搓出这样一个技能的，因为我觉得这个技能还是比较通用的，大家以后在自己的电脑上，如果 skill 多了需要管理，也完全可以通过同样的方法来手搓出和我一样的 skill。

下面就是我第一次和 Open Code 描述这个技能的时候，我给它的一段话。可以看到，我用的也完全是自然语言，没有任何编程术语在，只是把我的需求很明确地跟它说明白了而已。

原文：

我现在需要做一个skill，现在技能越来越多了，我觉得没有办法好好地记住它，使用的时候也容易出问题。我想要做一个技能，就是在新创技能，或者新下载技能后，可以调用它，帮我在飞书的多维表格里，自动把新增的技能填写进去。当然，目前存量的技能，我也需要做一次统一更新，作为多维表格的初始化。这样的技能能做吗？

这个时候 Open Code 就调用skill-creator的技能开始理解这个技能需要怎么做。

它首先告诉我，需要到飞书的应用中心（https://open.feishu.cn/app），先手动创建一个飞书应用。

这个还是要你手工做一下，然后登录你的飞书账号，点击创建应用，再选择下面的创建企业自建应用。不用担心，虽然你是个人用户，但是你也可以创建企业自建应用的。

接下来就填写应用名称和应用描述，你们可以参考我的填法，最后点击创建，这个应用就创建好了。

* 应用名称：Skill Tracker
* 应用描述：管理技能元数据

创建好了以后，你需要进到这个应用里，把应用凭证里面的APP ID 和 APP Secret 复制出来，这个是让 AI可以直接写入飞书多维表格，它需要的信息。

然后你还要给应用开通相应的权限，权限管理你可以参照我下面的截图，主要就是把编辑权限给这个应用。

我手搓这个 skill 的时候，第一次还失败了。在给完权限之后，Open Code开始帮我搓技能的其他部分，最后测试总是不成功。

所以这个 skill 其实我搓到一半，我是给它放在一边的，我也不知道怎么办，因为怎么也没法解决。

后来我看了另外一个老师写的文章，说他做了一个skill，是把飞书多维表格的官方文档所有的链接都放在一起，然后搓了一个可以指导他做复杂多维表格的skill，让 skill 可以去学习官方文档的所有内容。

我看了以后非常受启发，我就在想，我这个技能之所以做到一半没成功，很有可能是 AI 在帮我做的时候，它使用的不是飞书官方文档的知识，而是它自己编的一套知识，所以看似像模像样帮我做了，实际就成功不了。

我在想飞书的应用中心肯定也有相关的文档，我只要把相应的创建多维表格的文档找出来，让AI先学习一下，它就可以按照正确的知识来帮助我操作了。

所以我就找到了飞书的开发文档，里面有一个自建应用开发流程，从这里点进去。

https://open.feishu.cn/document/home/index

文档往下拖一点，你可以找到一个叫“快速调用服务端API（创建一个多维表格）”，这个正好是我需要的，所以点击这个，就是它官方文档所对应的网址链接了。

https://open.feishu.cn/document/introduction-2

我把这个网址链接复制下来，然后回到 Open Code ，重新去开发这个skill。

当然我是开了一个新的窗口，节约上下文token嘛，下面是我跟它说的原话，大家可以看一下，我唤醒了一下它的记忆，告诉它，我们以前在做skill-tracker这个skill，但是没成功，现在我有了开发文档，让它再学习一下，然后再去使用 skill-creator 的技能，重新去完善我们之前在做的 skill-tracker 的 skill。

原文：

之前我们做了一个skill-tracker的skill，不太成功，没有办法直接操作飞书多维表格。

现在我找到了飞书开发者文档，怎么通过API建立多维表格的说明，你学习一下：https://open.feishu.cn/document/introduction-2

  

然后再使用skill-creator技能，重新完善skill-tracker技能

它这次学习以后，就成功地把相应的数据写进了我的多维表格。

大家可以看下面，就是第一次的成品。

接下来就是一系列的微调，比如说，第一次我发现成品里，它前4列是系统默认的列，是空白的。我并不需要这些。我就把上图丢给它，让它调整掉：

前面4列不需要的，你创建的时候能不能不要搞出空白的4列？

第二版已经改得很好了，像模像样的。

我又发现技能描述这个时候是全英文的，我又看不懂；第二，显示的日期也是不对的，全都是1970年；第三，文件大小这里，它纯告诉我数字，我又不知道什么意思；第四，状态列没有信息。

所以下一稿一共是做了4个微调。

同时我让它调整完以后，一定要记得去更新自己skill里面的记录，否则它下次用的时候又不对。这个要求，主要是为了防止事情只在一个窗口做对了。

因为这次我给它有要求，但是它又没有写在真正的 skill 里面，那以后换一个窗口，就会出同样的问题，所以一定要额外提醒它一下。我给它提这样的要求，就是为了让它把这些要调整的东西固化在 skill-tracker 这个技能里面。以后无论换多少个窗口，它都可以看到，就不会犯同样的错误。

接下来我也提醒它，我说多维表格的路径，你一定要放在skill-tracker技能里面的，否则它换了一个窗口，也不记得我这个路径了。

所以除了上面这个截图，我的原文要求是这么写的：

大体不错了，现在做一下微调：

1、技能描述需要用中文，否则我看不懂

2、最后更新时间现在不对，怎么可能是1970年呢？

3、文件大小，需要告诉我是多少MB，或者多少KB

4、状态未更新

你调整完这些后，不要忘记更新skill-tracker技能，否则下次用又不对了。要让skill-tracker能做到：

以后但凡我新增了技能，跟你说一下，使用这个skill-tracker技能去更新飞书，它就能把我新的技能的详细信息，更新到这个多维表格里面的。

所以你skill-tracker技能里面，也要把目前这个飞书多维表格的路径放着的。

改好的样子是这样的：

大家可以看到，其他效果都好了，但最后的更新时间还是不对的，所以我又让它去微调了一下：

我看了，其他都ok了，最后更新时间还是不行，最后更新时间你不用搞得很复杂。你只要告诉我是几几年几月几日更新就可以了，不要细化到时间戳的。

调完这一轮以后，在整个显示的层面，它就已经完全过关了。

接下来，是因为我发现它在每次做调整的时候，都是对我电脑里面所有的skill都进行了一遍全量的覆盖，然后把新的数据重新给贴到多维表格上。

也就是说，它并没有达到我要求的，下载一个新的，更新一条，这样的效果。

所以我又给它做了一个小优化，我让它去区分出两个使用场景。

一个是第一次用这个 skill 的时候，的确是要对我电脑里所有的 skill 都进行一次遍历，然后全量更新到多维表格去的。

第二个，但凡我已经进行过初始化之后，以后我如果新搓了一个 skill ，或者下载了一个 skill ，像这种情况，我就只要更新这一条技能就ok了。

下面是我给它的原文：

现在skill tracker技能是对我电脑里面的所有skill进行遍历，然后全部更新吗？

你要区分两个使用场景：

  

一个是第一次用skill tracker的时候，的确是要对电脑里面所有的skill进行遍历，然后全量更新的。

第二个是进行过初始化以后再新增的技能，就直接跟你说，我要更新这条技能的数据，你只把这条技能相应的信息更新在多维表格的最后一行。

你检查一下这个skill目前是不是这样的功能？不是的话要优化一下。

所以它又帮我优化了一下最终的效果。

大家可以看到，这个是我后面自己又手搓的一个飞书的 skill。

搓完以后，我其实在聊天窗口，我直接跟它说，使用skill-tracker来更新一下飞书的这个新skill，它立马就使用了我这次手搓的skill-tracker，把飞书新skill的信息同步到了多维表格的最后一行。

原文：

使用skill-tracker,更新一下这个技能：feishu-bases-guide

可以看到，这个效果真的是非常棒的。

大家可以按照我的操作步骤去做，提示词也可以直接copy我的，自己做下微调就行。

但我现在没有办法把这个 skill 打包放在 Github 上，因为涉及到比较多的测试环节，我实在是没有时间去搞这一块。

而且，因为它是要跟飞书的应用做一个 API连接的，所以我的 skill 里面，有我自己飞书多维表格的路径，包括我飞书应用里面，APP ID还有APP Secret，就比较敏感。

所以我觉得我现在没有办法搞定分享的环节，我不知道怎么把这些信息给脱敏出来，给大家去使用。所以我这次就把我从头到尾手搓的过程教给大家，大家可以复制我的提示词，去跟自己的 Open Code也好，Trae也好，去进行一个交互。

我这次全程手搓，都是通过智谱的4.7 模型来做的。我是买了智谱的包年套餐，之前在搞活动，当时是173块入手的。我觉得智谱的模型在做这种工作的时候已经绰绰有余了，大家不用去使用国外的模型，国外的模型毕竟贵。

为什么我每篇文章，都非常详细地告诉大家，我到底是怎么把 skill 搓出来的？

我不想只告诉大家我最终的结果，做了一个很牛逼，很有用的功能，然后想得到大家的点赞，不是这样的。

因为我也是从小白做起的，所以我非常明白，每个小白在刚开始手工创造属于自己的 skill 的时候，会发生什么问题。

可能就是，在需求描述的地方，都不太清楚怎么去和AI进行交互。

所以，我是把自己和AI交互的整体的过程，都拿出来和大家做分享。

大家就可以看到，我是怎么从小白开始，一步一步去把看似有点复杂的功能给做出来的。

其实就是不要放弃。有的时候第一次没做成功，你给它稍微放一会儿，你可能过了一阵子，有了一些新的思路，那就重新再做一下。

毕竟你只要给AI提需求就可以了，它会库库帮你做的。你所要做的，只是把它需要的那些文档找到，告诉它，然后你给它不断地提优化的需求就可以了。

这是每一个普通人，只要你能够克服自己内心的胆怯，你不怕失败，你就可以完成的一件事情。

之所以把整个过程写出来，大家也可以看到，我这个过程并不是非常顺畅，一次性完成的。但我觉得，这就是咱们普通人在和AI做交互的时候，真正会碰到的过程。

大部分人都不是程序员出身，也不是产品经理出身，你很难一次性把自己的需求非常完美地和AI交代清楚。 AI需要的那些知识，你也很可能不知道它需要这些知识。

你得在碰壁了以后，可能过了一阵子，你突然意识到，原来当时它是缺这一部分知识，所以它做不下去。

做任何功能，产品，都是有这样的一个不断打磨的过程在的。

只要你有这样的一个不怕报错，不断实验的精神，我想，我们和真正高级的功能之间的差距，也并没有那么大。

只要你有自己的想法，你就可以通过不断试验，去把它做成的。

看着看着，你们自己也会有感觉，你们自己也会敢于去尝试。我比较想起到这样一个桥梁的作用。
