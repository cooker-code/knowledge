# 021000_安全与权限知识地图

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 02_Agent与AI工程 / 0210_安全与权限 |
| 解决的问题 | 让 Agent 在工具调用、文件访问、命令执行、联网、凭证使用和外部系统操作中可隔离、可审批、可审计 |
| 典型输入 | 工具清单、Agent 角色、外部入口、凭证、敏感路径、沙盒策略、审批规则、审计日志 |
| 典型产出 | 威胁模型、权限矩阵、沙盒策略、审批规则、凭证保护规则、审计字段、红队验证用例 |
| 用户阅读目标 | 读到安全文章时，先判断它补的是哪一个控制面，而不是只记“安全很重要” |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| S00 资产与入口边界 | Agent 能触达哪些数据、工具、系统和入口 | 安全先画清资产面和入口面 | 有工具清单、入口来源、敏感资源边界的文章进入 |
| S01 威胁建模 | 要防幻觉、Prompt Injection、恶意 Skill、越权入口还是本地攻击者 | 不同威胁对应不同控制面 | 只喊风险、不拆威胁类型的文章降权 |
| S02 权限分层 | 工具、角色、MCP Server、Skill、浏览器和文件权限如何最小化 | 权限要按角色和动作拆分，不给全能工具 | 有 allow/deny、角色矩阵、策略默认值的文章优先吸收 |
| S03 执行隔离 | 命令、文件、网络、进程和浏览器登录态如何隔离 | 沙盒是执行控制面，不是完整安全方案 | Docker、sandbox-exec、容器、内核隔离和浏览器隔离文章进入 |
| S04 凭证与数据防护 | Token、密钥、环境变量、敏感路径和日志如何保护 | 凭证不能跟随 Agent 任意下沉到工具环境 | 有脱敏、隔离、出站限制和敏感路径规则的文章进入 |
| S05 审批审计回滚 | 高风险动作如何审批、记录、追责和回滚 | 自动化不等于无审批，可追责才能生产使用 | 有审批流、审计字段、变更回滚和失败复盘的文章进入 |
| S06 安全验证 | 如何用坏例、红队、回归集验证安全策略 | 安全要用攻击样本和失败模式验证 | 有 Prompt Injection、恶意工具、越权测试样例的文章进入 |

## 文章进入时的对比规则

| 判断项 | 处理方式 |
|---|---|
| 补充 | 能补威胁类型、控制面、默认策略、审计字段、失败模式或验证样例时，写入对应流程节点 |
| 冲突 | 如果文章把“沙盒”“权限”“审批”“审计”混成一个万能方案，先记录边界，不直接吸收为结论 |
| 不同层次 | 产品发布、配置实操、底层隔离机制、生产安全流程要分层，不放在同一个判断里 |
| 降权 | 只有安全口号、产品宣传、无基线收益、无权限默认值、无审计字段时，不进入核心知识点 |

## 待补缺口

| 流程节点 | 缺什么 | 后续文章进入时怎么判断 |
|---|---|---|
| S00 资产与入口边界 | 缺少完整 Agent 资产清单模板 | 是否能列清文件、网络、浏览器登录态、外部 API、MCP Server 和 Skill |
| S01 威胁建模 | 缺少可复用威胁分类表 | 是否区分 Prompt Injection、恶意工具、未授权入口、凭证泄漏和本地攻击者 |
| S02 权限分层 | 缺少 Agent 工具权限矩阵 | 是否能产出角色、动作、资源、默认拒绝和审批条件 |
| S03 执行隔离 | 缺少 sandbox-exec、Docker、微虚拟机、浏览器隔离的横向对标 | 是否讲清隔离对象、兼容性、冷启动、性能和降级 |
| S04 凭证与数据防护 | 缺少凭证隔离和日志脱敏细则 | 是否有环境变量、token、敏感路径、日志字段和出站限制 |
| S05 审批审计回滚 | 缺少审计字段和回滚模板 | 是否能给出可检索事件、审批原因、执行结果和变更回滚路径 |
| S06 安全验证 | 缺少红队样本和回归集 | 是否提供可复现的攻击输入、预期阻断结果和回归检查 |

## 使用方式

- 先读本文件的安全流程，再读具体主题知识点。
- 新文章不要单独写摘要，先归入 S00-S06 节点，再判断补充、冲突、不同层次或降权。
- 如果某个节点已有多篇文章，后续应拆成更细的 `021000_核心知识点/<主题>.md`，而不是继续扩展文章列表。
