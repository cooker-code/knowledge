## 用户认知重点

| 认知项 | 当前假设 | 后续整理策略 |
|---|---|---|
| 已知基础 | 用户知道 MCP/Skill/工具调用会带来权限风险 | 不重复讲“安全很重要” |
| 待补边界 | Prompt Injection、恶意工具、未授权入口、凭证外泄、破坏性命令的不同控制面 | 每篇文章必须定位到威胁类型和控制面 |
| 易偏差点 | 安全文容易只列原则，不给可验证配置和审计事件 | 优先抽取默认策略、拒绝列表、审批流和日志字段 |

## 排重准则

问题指纹：

```text
Agent 安全对象 + 威胁类型 + 控制面 + 核心机制 + 解决问题 + 可验证边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是沙盒 | 按文件、网络、进程、凭证、降级策略拆分 |
| 都是工具权限 | 按 Tool Policy、MCP Server 权限、Skill 安全、浏览器工具权限拆分 |
| 都是审批 | 按命令审批、危险操作审批、人工确认、超时和审计拆分 |
| 只有原则 | 没有配置、日志、流程或威胁映射时只略读 |
| 有审计字段和默认值 | 优先沉淀为工程化知识点 |
