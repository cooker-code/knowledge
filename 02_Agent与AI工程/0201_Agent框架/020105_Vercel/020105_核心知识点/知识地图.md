## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| V00 应用边界 | 文章讲 Web AI SDK、Agent 平台、Skills 分发还是 Vercel 部署 | 不把所有 Vercel 平台文章都塞进 Agent 框架 | 只讲部署、托管、前端工程的文章转 0702/0703 |
| V01 模型抽象 | 如何统一 OpenAI、Anthropic、Google、OpenAI 兼容和社区 provider | Provider 抽象降低模型切换成本 | 能讲 provider、core API、结构化输出的文章进入 |
| V02 交互与工具 | 如何支持流式 UI、聊天状态、工具调用和对象生成 | Web AI 应用需要模型调用和 UI 状态一起设计 | 只讲 UI 美化的文章降权或转前端工程 |
| V03 Skills 分发 | agent-skills、add-skill、skills.sh 如何管理能力包 | Skills 从单机提示词转向可安装、可排行、可复用的工作流包 | 能讲跨工具路径、版本、质量和安全的文章进入 |
| V04 执行与浏览器 | agent-browser 如何让 Agent 操作浏览器和网页 | 浏览器执行必须考虑权限、会话、证据和可观测性 | 只提 Vercel 名字但主问题是 Playwright 的文章转工具调用 |
| V05 云端 Agent 平台 | Open Agents 如何组织 Web、workflow、sandbox、GitHub | 控制面与执行面分离是云端 coding agent 的关键 | 能讲持久运行、沙箱、OAuth、GitHub App 的文章进入 |
