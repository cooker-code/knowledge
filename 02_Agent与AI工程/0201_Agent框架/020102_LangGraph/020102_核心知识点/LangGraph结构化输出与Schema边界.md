# LangGraph 结构化输出与 Schema 边界

## 来源
- [LangGraph结构化输出详解：让智能体返回格式化数据](<../文章/done-LangGraph结构化输出详解：让智能体返回格式化数据.md>)

## 核心问题
结构化输出解决的是 Agent 输出契约问题，不是事实正确性问题。LangGraph 中的 Pydantic/schema 约束适合让下游节点稳定消费结果，但仍要配合校验、重试和人工验收。

## 判断准则
- 把结构化输出放在节点边界，而不是把整条链路都变成 JSON 搬运。
- Schema 字段要体现业务验收项；只约束格式不能证明内容为真。
- 失败处理至少包含解析失败、字段缺失、模型拒答和下游消费失败四类。

## 认知偏差
| 常见错误认知 | 正确理解 |
|---|---|
| 结构化输出能解决幻觉 | 结构化输出只能约束格式，事实仍要由工具、数据或评估器验证 |
| Pydantic 写得越复杂越好 | 复杂 schema 会抬高模型负担，应该从下游真正需要的字段开始 |
