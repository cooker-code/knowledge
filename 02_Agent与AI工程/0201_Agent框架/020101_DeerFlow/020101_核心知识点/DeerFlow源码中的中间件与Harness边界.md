| 维度 | 判断 |

| 本文位置 | 源码结构和工程边界，不是完整使用教程 |
| 核心机制 | 中间件管道、延迟工具发现、harness/app 边界、边界测试 |
| 使用链路 | 创建任务 -> Gateway 转发 -> Runtime 执行 -> 中间件注入能力 -> 子代理/工具执行 -> 流式返回 |
| 前置条件 | 有清晰模块边界、日志、测试和配置开关 |
| 边界 | 小项目可能不值得引入复杂中间件；中间件顺序会增加调试成本 |

## 横向对标

| 对标技术 | 实现方式 | 优势 | 劣势 | 适合场景 |
|---|---|---|---|---|
| Demo 式 Agent | 主函数里串工具和 prompt | 上手快 | 横切能力会污染主流程 | 一次性验证 |
| DeerFlow 中间件 | 横切能力按顺序挂载 | 可演进、可配置 | 顺序依赖和调试成本高 | 持续演进的 Agent 应用 |
| MCP 直接暴露工具 | 全部工具入上下文 | 简单直观 | 上下文和选择成本高 | 工具少 |
| tool_search 延迟发现 | 先找工具再调用 | 控上下文、适合大工具集 | 多一次检索链路 | 工具多、权限复杂 |
