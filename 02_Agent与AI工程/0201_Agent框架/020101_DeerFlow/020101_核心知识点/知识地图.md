# 020101_DeerFlow知识地图

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 02_Agent与AI工程 / 0201_Agent框架 / DeerFlow |
| 解决的问题 | 先定义 DeerFlow 类研究型多智能体框架的应用流程，再把文章吸收到流程节点 |
| 典型输入 | 复杂研究问题、产物生成需求、本地文章、源码分析和后续官方补证 |
| 典型产出 | 计划、子任务、搜索证据、代码/文件产物、报告/PPT/网页和运行边界 |
| 用户阅读目标 | 读 DeerFlow 文章时，判断它补的是任务编排、上下文、工具沙箱、产物交付还是成本治理 |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| D00 任务边界 | 任务是否真的需要深度研究、多轮搜索和多产物生成 | DeerFlow 适合多步研究和产物交付，不适合单次问答 | 只讲“效率翻倍”的宣传文降权 |
| D01 计划与状态 | Coordinator、Planner、Todo、Plan Mode 如何拆任务和修订计划 | 计划修订要有轮次上限、人工确认点和退出条件 | 能讲计划、状态和失败恢复的文章优先吸收 |
| D02 上下文与工具 | 文件、记忆、摘要、MCP、搜索和沙箱如何进入 Agent | 工具不应一次性全暴露，长上下文要压缩和卸载 | 能讲延迟工具发现、摘要、文件系统和权限的文章进入 |
| D03 子代理执行 | Researcher、Coder、Reporter、PPT 等子代理如何并发和隔离 | 子代理价值在独立上下文、专业工具和失败隔离 | 只说“AI 团队”但无契约和边界的文章降权 |
| D04 交付与验证 | 报告、网页、PPT、图表如何验收和回滚 | 产物必须可检查、可复跑、可标注缺口 | 有验收指标、日志和产物链路的文章优先 |
| D05 成本与安全 | Token、搜索轮次、沙箱、人工等待和并发限制如何治理 | 成本瓶颈常在 Researcher 搜索，安全瓶颈在工具和沙箱 | 有成本拆解、超时、重试和安全边界的内容写入核心知识点 |
