**LangChain 官方最新发布了年度总结报告 ——《State of Agent Engineering》 ，该报告基于对 1300+ 从业者（包括工程师、产品经理、业务负责人等角色）深度调研，系统地梳理了当前 Agent 工程的现状，很有参考价值，这里总结核心内容分享给大家。**

# 核心观点——TLDR

# 

* **AI Agent 已经大规模进入生产环境，真正的难点不在“能不能做”，而在“能不能长期稳定、安全、可控地跑”。**
* **Agent 已经不是概念，而是“生产现实”。**
* **导致 Agent 失败的最大杀手不是成本，而是「质量」。**
* **Agent Engineering 并不是 Prompt 技巧，是一门系统化工程。**
* **可观测性（Observability）是刚需，不是加分项。**
* **评测（Evaluation）在追赶，但还没跟上。**
* **多模型接入是常态，没有人 All-in 一家。**

# 

# 先搞清楚一个词：什么是 Agent Engineering？

文章里给 Agent Engineering 下了一个很关键的定义：

翻成大白话就是：

* 大模型本身 **不确定、会飘、会幻觉**
* Agent 又是 **多步调用 + 工具 + 决策**
* 所以你不能“一次写好就完事”
* 而是要：**观测 → 调整 → 再观测 → 再调整**

**Agent Engineering 更像“系统工程”，而不是 Prompt 技巧。**

# 

# Agent 不再是炒概念，而是真的在上生产

# 

* **57% 的公司已经有 Agent 在生产环境**
* 去年只有 51%，明显在增长
* 还有 **30% 正在认真开发，准备上线**

## 

## 这说明什么？

## 

**Agent 这件事，已经过了“要不要做”的阶段，大家都已经做起来了，而且现在最关心的是：“怎么才能别翻车？”**

# 

# 公司越大，越在积极推进 Agent

# 

文章对比了公司规模：

* **1 万人以上公司**

+ **67% 已经在生产**

* **100 人以下公司**

+ **50% 在生产**
+ **更多还停留在开发阶段**

## 

## 通俗理解

## 

大公司更容易做 Agent，不是因为更聪明，而是因为：

* 有 **平台团队**
* 有 **安全、合规、监控**
* 能承受 **试错成本**

这也解释了：**Agent 真正难的不是“写出来”，而是“撑得住”。**

# Agent 主要被用来干什么？

# 

## 第一名：客服（26.5%）

* 面向真实用户
* 直接影响体验和成本

## 

## 第二名：研究 / 数据分析（24.4%）

* 查资料
* 总结报告
* 跨文档推理

## 

## 第三名：内部流程自动化（18%）

* 提效
* 少写脚本
* 少对人

一个重要变化是：**Agent 正在从“内部助手”，走向“直接面对客户”。**

# 

# 最大的坑是什么？（非常重要）

# 

## 第一大阻碍：**质量（32%）**

## 

* LLM 会不会胡说？
* 回答稳不稳？
* 语气对不对？
* 会不会越权？
* 多轮下来会不会跑偏？

**质量 = Agent 是否“可信”**

## 第二大阻碍：**延迟（20%）**

## 

* Agent 越聪明 → 步骤越多
* 步骤越多 → 越慢

很多团队在 **“更聪明 vs 更快”** 之间拉扯

## 有意思的点：**成本不再是头号问题**

* 模型变便宜了
* 大家更关心：

**别翻车、别出事、别让用户不爽。**

# 

# 为什么大家都在疯狂卷 Observability

# 

## 

## 数据很夸张

* **89% 的团队已经做了 Agent 观测**
* 有 Agent 在生产的团队中：

+ **94% 有观测**
+ **71% 能看到每一步推理和工具调用**

## 

## 通俗说就是：

**如果你看不到 Agent 在“脑子里干了什么”，你就永远修不好它！！**

这包括：

* 每一步想了什么，如何规划？
* 为什么选这个工具，调用参数是什么？
* 哪一步开始跑偏

**没有 Observability，就没有 Agent Engineering。**

# 

# Evaluation 反而没那么普及

# 

* **只有 52% 做离线评测**
* **37% 做在线评测**

为什么？

* 评测比打日志更麻烦
* 很多团队先解决“能看见”，再解决“好不好”

## 

## 评估方式也很现实：

## 

* **人审（最靠谱，但贵）**
* **LLM 当裁判（可规模化）**
* **传统指标（BLEU、ROUGE）几乎没人用**

**因为 Agent 没有标准答案！**

# 模型路由：没有人 All-in 一家

# 

## 关键结论

* OpenAI 用得最多
* **但 75% 的团队用不止一个模型**

大家开始：

* 简单任务 → 便宜模型
* 复杂推理 → 强模型
* 对延迟 / 成本做路由

**Agent 本身，正在成为“模型调度器”。**
