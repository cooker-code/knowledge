这件事挺有意思的。

A2A协议，全称Agent-to-Agent Protocol，去年4月由Google发布，一年时间里累积了150多家组织的支持，GitHub上拿了22000多个Star，Google云、微软Azure、AWS都做了原生集成。

然后你去看今年最热闹的那几个Agent Team产品，Claude Code、Cursor、Devin。

内部通信全是私有协议。

没有一个用A2A。

这个现象让我想了挺久，觉得背后有些东西值得聊一聊。

先说说A2A是什么

A2A要解决的问题，用一句话说，就是"不同家的AI agent怎么互相说话"。

想象一个场景，你的公司用了Salesforce的销售agent，供应商那边用的是SAP的库存agent。你想让这两个agent直接对话、协调订单。问题是，Salesforce和SAP是两个完全独立的系统，它们互相不认识，没有共同的语言。

A2A想做的就是那个"共同语言"，一套标准的agent发现机制、任务委托格式和结果返回规范，让不同厂商的agent能互通。

它和另一个你可能听过的协议MCP有个很清晰的分工。MCP解决的是"agent怎么用工具"，A2A解决的是"agent怎么找同事"。一个往下连接工具，一个往旁边连接其他agent。

两个放在一起，理论上就能搭出一套完整的多agent生态。

一年发生了什么

A2A这一年的发展速度，说出来挺令人意外的。

去年6月，Google把A2A捐给了Linux Foundation，变成了社区治理的中立标准，而不是Google自己的私有项目。这一步很关键，意味着Google愿意让出控制权，推动行业采用。

去年12月，更大的事情发生了。Anthropic、OpenAI、Block三家联合在Linux Foundation下成立了Agentic AI Foundation，A2A、MCP、还有IBM主导的ACP协议全部纳入统一治理。

这件事的分量很重。这是AI圈里少有的几家主要竞争对手公开坐到一张桌子上，说"我们要一起把这个标准推起来"。协议战争在这一刻基本宣告结束。

今年3月，A2A v1.0正式发布，加入了签名Agent Card（每个agent有加密身份证）、企业多租户支持、异步HTTP传输等关键特性。

从生态建设的角度看，这个成绩在一年内能做到，相当快了。

那为什么主流产品没用它

这才是最有意思的地方。

Claude Code今年2月上线的Agent Teams，16个Claude并行工作，靠的是git锁协调文件访问，加上自定义的进程间通信，不是A2A。Cursor的Background Agents，每个agent跑在独立的git worktree里，互相之间基本不需要通信，更谈不上A2A。Devin的Managed Devins，协调者和子agent之间走的是Cognition AI自己的私有API。

这不是这些公司不知道A2A的存在，也不是故意抵制。原因其实很直接。

**A2A设计的使用场景，和这些产品内部agent团队的使用场景，根本就不是同一件事。**

A2A是为"你不知道对方是谁、用什么模型、什么框架"这种场景设计的，核心价值是跨厂商互通。就像HTTP统一了Web，你不需要知道对面服务器跑的是Java还是Python，照样能发请求拿到响应。

但Claude Code里的16个agent，全都是同一个模型、同一套工具、跑在同一个代码库里。它们对彼此了如指掌，不需要"先发现对方是谁"这一步。用A2A来协调它们，就像你家四个孩子开家庭会议要先互相出示护照才能说话。

这事搞清楚之后，"叫好不叫座"这个判断本身就需要修正一下，不是叫好不叫座，而是这把锁还没找到对的门。

A2A真正的战场在哪里

Google的Gemini Enterprise是目前A2A落地最真实的产品，因为它本来就是A2A的发起者。Gemini Enterprise的Agent Registry和Agent Gateway，是A2A Agent Card机制的商业化实现，Salesforce、SAP、ServiceNow等partner的agent之间的任务路由，走的就是A2A。

这才是A2A的主场，企业内部跨系统的agent互通、供应链上下游的agent协作、不同SaaS平台之间的自动化流程。

拿一个具体场景举例，你的公司的财务agent发现某个合同金额超标，需要通知法务agent做合规审查，法务agent审完了通知采购agent修改条款。这三个agent可能来自三家不同厂商的系统，A2A在这里的价值才真正体现出来。

个人开发者用Claude Code或者Cursor写代码，可能一辈子都不会直接感受到A2A的存在。但企业里负责AI基础设施的人，这一年可能已经悄悄把A2A纳入了技术选型。

两个协议，两个层面

我觉得现在理解这个格局最清晰的方式，是把它想象成两层基础设施：

MCP是纵向的，往下连，给agent装"手"，让它能用工具、访问数据库、调用API。MCP现在已经有超过18000个社区服务，SDK每月下载量以千万计，可以说是事实上的agent-to-tool标准。

A2A是横向的，往旁边连，给agent找"同事"，让它能委托任务、交换结果、跨系统协作。A2A的企业落地正在发生，只是节奏比MCP慢一些，因为企业级互通本来就是一件比工具集成更复杂的事。

这两层放在一起，是目前想象中最完整的多agent基础设施。至于什么时候真正变成每个产品都默认支持的底层，我估计还需要一两年。

但治理层已经想清楚了，Anthropic、OpenAI、Google坐在一起维护同一套标准，这在之前是不可想象的。剩下的只是时间问题。

协议战争结束了，但协议普及才刚刚开始。这件事的最终形态，可能比我们现在想到的还要大。
