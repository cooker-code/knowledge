## 核心问题
A2A 的核心看点是 Agent 发现、能力描述、任务协作和安全边界；风险在于标准热度不等于真实生产互操作。

## 判断准则
- 协议评估要同时看发现机制、任务状态、认证授权、审计和失败恢复。
- 跨组织协作必须把权限、数据泄露和责任边界前置。
- 如果没有真实双端实现，先按候选标准处理。

## 认知偏差
| 常见错误认知 | 正确理解 |
|---|---|
| A2A 已经成为事实标准 | 需要看实际互操作和生产采用，不只看支持名单 |
| Agent 之间天然可以共享上下文 | 上下文共享是高风险动作，需要最小必要原则 |
