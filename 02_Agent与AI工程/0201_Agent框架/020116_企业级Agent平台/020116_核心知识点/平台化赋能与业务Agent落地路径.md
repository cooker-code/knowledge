## 核心问题
业务 Agent 落地通常从单场景助手开始，逐步沉淀通用能力：知识接入、工具编排、权限、评估、运营和模板化交付。

## 判断准则
- 先选业务闭环短、工具边界清、结果可验收的场景。
- 平台化前要证明至少两个场景复用同一能力。
- 答疑/诊断类 Agent 要明确知识库版本、工具权限和人工升级路径。

## 认知偏差
| 常见错误认知 | 正确理解 |
|---|---|
| 一个成功业务助手就该平台化 | 平台化需要复用证据和运营机制 |
| 开源项目可以直接企业落地 | 还要补权限、审计、部署、知识更新和责任边界 |
