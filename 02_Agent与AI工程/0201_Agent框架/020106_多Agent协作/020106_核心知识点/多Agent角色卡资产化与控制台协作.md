## 核心问题
角色卡只有进入版本管理、目录结构、共享标准和验收流程后，才从提示词变成工程资产；控制台的价值是让任务、上下文、产物和冲突可见。

## 判断准则
- 角色卡要包含职责、输入、输出、禁止行为和验收标准。
- 共享目录、命名规范和交接协议比“角色人设”更重要。
- 多 Agent 控制台要暴露任务状态、证据来源、冲突和人工接管点。

## 认知偏差
| 常见错误认知 | 正确理解 |
|---|---|
| 角色卡就是提示词模板 | 角色卡要能被复用、评审、版本化和审计 |
| 控制台只是 UI | 控制台是协作状态和质量门禁的可视化入口 |
