如何让 Agent 在跨越多个上下文窗口时仍能保持高效协作？

Anthropic 再发神作，《长效运行 Agent 的
有效架构方案》，链接见文尾。

文章其实就在讲一件事：如何让 AI 像人类程序员一样，能够长期、稳定地完成一个庞大的编程项目，而不是写着写着就“断片儿”或者“烂尾”了。

如果你之前看过我的专栏文章：[项目改崩N次后，我总结出大型项目重构人机协同SOP](https://mp.weixin.qq.com/s?__biz=MzIzMzQyMzUzNw==&mid=2247508353&idx=1&sn=28daad5834505afbf0c4b9372ea704e8&scene=21#wechat_redirect)就会发现结构观点和实施完全一样。

AI容易断片儿，原因很简单。它的上下文窗口是有限的，目前主流的差不多 256K ，类似Gemini也就2M token，当它做大项目时，不得不分几次做。

这就好比两班倒，上午的员工走了，下午的员工来了，如果没交接好，下午的员工根本不知道上午干了啥，还得重新研究。

眼高手低这种情况就更常见了！比如你让它做一个淘宝出来，它可以一口气完成，但是走到一半就直接上下文爆炸了，给你看一个登录页，就说自己完工了。

那这些问题如何解决呢？和专栏文章结构讲得大差不差。

Anthropic发现，要解决这个问题，不能只用一个 AI 从头干到尾。

而是要分工！

**第一步，做好初始化**（下面的图都是nano画的）

上来第一步绝对不是写代码，首要的任务是搭脚手架和列清单。

把你的要求拆解成个细小精确的功能点。

比如不做“做一个微信”，而是拆成“1.要有输入框”、“2.按回车能发送”、“3.发送后能看到气泡”等。

然后建档案，建一个实施日志，后面谁干了啥都要记下来。

最后每实施一步，存进度，使用Git代码版本管理，准备好保存进度。

**第二步：实施编码**

为了保持长时长效的运行，每一步的具体的实施编码都要遵守一套严格的交接班手续。

进场先看日志看清单看代码历史Git Log。

然后每次任务只聚焦到一个任务点，也就是只挑清单里的一项任务做，**绝对不准**试图一口气做完所有事。

做完后进行检查，不光是代码写，还要模拟真人去点击测试，确保功能真的能用。

这点其实Anti就做得很好，是一个真正负责任完成编程任务闭环讲武德的AI Agent，详情可以看这里，没有对比就没有伤害：[Trae SOLO免费了，但是不建议用！](https://mp.weixin.qq.com/s?__biz=MzIzMzQyMzUzNw==&mid=2247509073&idx=1&sn=691bedf5ff36b62790723b254f751373&scene=21#wechat_redirect)

最后一步再交接：把代码存盘（Git Commit），在日志里写清楚做了啥，测试结果是啥然后把清单里的那一项打钩。[AI前端小课从零到全栈：Git](https://mp.weixin.qq.com/s?__biz=MzIzMzQyMzUzNw==&mid=2247503873&idx=1&sn=75d3dd7d0e9c1f9f4f8b17cabacac2e1&payreadticket=HClrExTVMlf15-r62cT328hRCZrsPVKe-scNnlRvex1u1pVQ_dg6gn4pIeFoBXO_9z0b1JU&scene=21#wechat_redirect)

可问题是到底应该怎么做呢？其实说白了就这一个动作：写文档!

这个世界整个宇宙都在维持着一种精确的平衡中，你的代码量少了，文档量肯定就得上升，不然这工资就这么好得，摸着鱼钱就到手了，哪有那么好的事哟！

而写文档最关键的问题是如何写，如何组织，其实专栏里面也写过了：

官方在这里给了三个参考文件：

任务清单feature\_list.json

这个主要是防止 AI 偷懒嘴硬，清单上有 200 个钩要打，它就不敢做完 3 个就说“我搞完了”。

日志claude-progress.txt

防止 AI 失忆，新任务一看日志就知道：“哦，上个人刚修好了登录页，那我现在该去修注册页了。”

真实模拟测试

AI 经常觉得自己代码写对了就是对的。现在强制它必须像用户一样打开浏览器点一点，看看到底能不能跑，单元测试接口测试到底能不能过，还要写详情的模拟测试报告。

其实除了官方上面的讲这些，还有一很重要的点，那就是交叉验证。可以让不同模型不同工具来对比测试。

这个文章讲的内容很浅白，但是真的实用，核心观点就一个：

别指望 AI 像个天才一样挥挥手就搞定大项目。要把 AI 当成一群只有短期记忆的实习生，给他们准备好详细的待办清单、强制他们写工作日报、并且干完每一点活都要存盘。

AI才能接力完成一个复杂的软件开发！
