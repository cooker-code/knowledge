## 来源

## 核心问题
Ralph Loop 类循环强调持续执行、判断和修正；可吸收的是“自我推进需要外部目标和 Judge”，不是无限循环。

## 判断准则
- 循环必须有 Judge、停止条件、最大轮次和失败升级路径。
- 每轮只允许基于新增证据调整计划，不能凭空重写目标。
- 适合可验证产物，不适合开放式聊天和无验收任务。

## 认知偏差
| 常见错误认知 | 正确理解 |
|---|---|
| 循环越多质量越高 | 无 Judge 的循环只是成本放大器 |
| Agent 自己干到完不需要人 | 风险、权限和目标变更仍需要人工门禁 |
