## 核心问题
长任务 Agent 的核心不是“跑得久”，而是目标状态、检查点、恢复路径、预算和验收条件能否持续约束执行。

## 判断准则
- 任务开始前写清目标状态、完成定义、预算和禁止动作。
- 运行中必须记录检查点、证据、产物和未决问题。
- 恢复时优先读状态摘要、当前计划和最近失败，而不是回放全量上下文。

## 认知偏差
| 常见错误认知 | 正确理解 |
|---|---|
| 长上下文就能跑长任务 | 长任务依赖状态管理、恢复和验收，不只是上下文窗口 |
| 自动运行越久越好 | 越久越需要预算、门禁和人工接管 |
