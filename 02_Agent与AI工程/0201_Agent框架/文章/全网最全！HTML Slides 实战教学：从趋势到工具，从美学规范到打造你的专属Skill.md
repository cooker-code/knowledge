最近几个月，我做了一个重要的切换——不再打开 PowerPoint 或 WPS，所有的分享、汇报、提案，全部用 HTML 幻灯片完成。文件一个几百 KB 的 `.html`，浏览器打开就能讲，代码高亮、暗色模式、3D 过渡随手就来。这不是我一个人“折腾”，而是一个正在加速到来的趋势。

本文将是一份实战手册，我们从**为什么 HTML Slides 是未来**讲起，接着对比主流工具，然后给你一套经过验证的**美学规范**，最后带你动手**写一个可复用、有你自己风格的 AI Skill**，并为你推荐 GitHub 上可直接安装的成熟 Skill。全程可操作，读完就能落地。

Chapter 01

为什么好多人都在弃用PowerPoint 

先别急着看工具，理解底层逻辑更重要。HTML 幻灯片不是“另一种 PPT”，它是演示文稿的范式转移。

1. 体积与分发

一个包含高清图、代码块、完整动画的 PPT 可能 50MB，需要专用软件打开。同样的内容写成 HTML，往往只有几百 KB，一个链接发过去，对方用手机浏览器都能完美播放。你彻底告别了“字体丢失”“排版错乱”“对方没装 Office”的尴尬。

2. 版本控制与协作

PPT 是二进制文件，Git 对它无能为力，“最终版\_v3\_真最终.pptx”是协作的常态。HTML 是纯文本，每一次修改 Git 都清晰记录，团队可以像管代码一样管幻灯片，Review、Revert、分支协作全部打通。

3. 开发者友好的表现力

内嵌代码高亮（Shiki）、实时编辑、嵌入 Vue/React 组件、Canvas 动画、ECharts 数据图表……这些是传统办公套件无法企及的。对于技术分享来说，HTML Slides 不是“能做”，而是“天生就该如此”。

4. 完全开放，不被绑定

HTML/CSS/JS 是 Web 标准，不依赖任何厂商。即使你用的工具明天停更，你的源文件依然能在任何浏览器打开。这个自由度，是闭源格式永远无法给予的。

连 WPS 都在网页版上线了“HTML 素材”功能，这股浪潮已经不是预测，而是正在进行时。

帮朋友手搓的PPT，只用了5分钟（25页，15KB大小）

## Chapter 012

## 主流工具：选对底座，事半功倍

HTML Slides 工具生态已经相当成熟，不需要从零手写。根据你的身份和场景，选一款最趁手的：

* **Slidev（开发者首选）**  
  Markdown 驱动，Vue 技术栈，代码高亮惊艳，支持导出 PDF/PPTX。一句话：做技术分享，Slidev 是版本答案。

  ```
  npm init slidev@latest
  ```

   输入进入就能启动，热更新、演讲者模式、绘图批注全内置。  
  *适合：前端/全栈开发者、需要大量代码演示的场景。*

* **Reveal.js（功能天花板）**  
  最强大的 Web 幻灯片框架，支持嵌套幻灯片、3D 旋转、Auto-Animate、演讲者备注 API。如果你追求苹果发布会那种视觉冲击，Reveal.js 是终极画布。  
  *适合：对设计效果有极致追求，愿意手写一点 HTML/CSS 的创作者。*
* **Byeslide（Agent-first 新锐）**  
  每张幻灯片是独立 HTML 文件，通过 CSS 变量管理设计令牌，专门为 AI Agent 精准编辑而设计。内置溢出检测，确保 AI 生成的幻灯片不“撑爆”。  
  *适合：想把 AI Agent 深度整合进幻灯片制作流程的先锋玩家。*
* **Marp / Remark.js（极简派）**  
  Marp 用纯 Markdown 一键导出 PPT，Remark.js 几 KB 大小可直接部署。如果你只需要“内容清晰、排版工整”，它们足够。  
  *适合：快速文档型演示、博客嵌入、极简主义者。*
* **OpenSlides（本地 AI 工作台）**  
  基于 Reveal.js，支持从一句提示词生成完整幻灯片，上传 PDF/图片/CSV 作为上下文，生成后还能聊天式迭代。核心理念：AI 生成，人保留修改权。  
  *适合：频繁用 AI 做原型、需要多次迭代的创作者。*

**快速决策口诀：**

* 前端技术分享 → **Slidev**
* 极致视觉炸场 → **Reveal.js**
* 纯 Markdown 速出片 → **Marp**
* AI 深度参与创作 → **OpenSlides / Byeslide**

选好底座后，下一步就是让它“长得好看”。

## Chapter 03

## 美学规范：让 AI 生成的幻灯片告别“廉价感”

用 AI 写 HTML Slides 最大的坑，不是代码报错，而是生成出一股“五颜六色的超市传单感”。问题的根源在于，你的 prompt 里没有注入**设计约束**。

以下是一套经过反复实战检验的美学规范，直接喂给 AI，效果立竿见影。

### 1. 整体风格定调（第一句话就决定气质）

* 追求**高级感、科技感**，扁平化 + 卡片式设计。
* 从成熟色系中选择一个作为基调：莫兰迪、高级灰、孟菲斯、蒙德里安……根据主题匹配。
* **禁用白色背景**（除非刻意要极简），**禁用渐变色**，文字与背景色必须形成强对比。
* **禁止卡片套卡片**，整个页面就是一个干净容器。

### 2. 布局铁律（信息呼吸感）

* 比例强制 **16:9**，容器 `width:100%; height:100%`。
* 遵循**一页一观点**，充分利用留白，不要塞满。
* 每页必须有标题，居中且醒目，字号/字重与整体统一。
* 卡片间用 **CSS Grid** 对齐，避免手动 `margin/padding` 乱调。
* **内容少时放大字号居中，一页放不下就自动拆成两页**，绝不用滚动条。

### 3. 配色（3-4 色法则）

* 主色不超过 3-4 种，用高对比度组合突出关键信息。
* 可加微妙的阴影增加层次，但**绝不滥用**。
* 配色必须和整体风格一致，不能突然冒出一个荧光色。

### 4. 字体（层次即专业感）

* 标题与正文使用不同字重/字号建立**清晰的视觉层级**。
* 善于利用 **Font Awesome** 等图标库做点缀，提升精致度。
* 确保所有文字在任何背景上都清晰可读。

### 5. 内容结构（金字塔 + 标准流）

* 标准页序：**首页 → 目录 → 过渡页 → 内容页（每章节至少两页）→ 总结 → 结束页**。
* 标题必须精炼，是这一页的“唯一论点”。
* 图和表**不要放在同一页**，图表用 ECharts 等专业库渲染，单独成页。

### 6. 交互与边界（细节不翻车）

* 右下角放**小巧的导航按钮和进度条**，不设自动循环播放。
* **所有元素必须在页面内完全可见，绝无溢出和截断**——这是一条红线。

你可以把上面这些整理成一段“设计宪法”，但每次都复制粘贴效率太低。下一步，我们要把它固化成一个**可复用的 AI Skill**。

## Chapter 04

## 进阶实战：编写一个可复用、有你自己风格的 AI Skill

学会创建专属 Skill，意味着你以后只需要说一句“用我的风格生成关于XXX的幻灯片”，AI 就能稳定输出带有你个人烙印的作品。

### 4.1 什么是“幻灯片 Skill”？

一个 **Skill** 是一套预设的 AI 系统指令 + 约束条件。它明确了：你是谁，你的审美偏好，你用什么技术栈，你的品牌色、字体、布局规则，以及绝对不能犯的错误。

把 Skill 想象成你雇了一个顶级设计师，你在上岗第一天递给他的《品牌视觉手册》。有了它，你从“每次都重新描述一遍”变成了“一次编写，终身调用”。

### 4.2 第一步：提取你的视觉 DNA

在写 Skill 之前，先回答下面几个问题，这是你风格的“基因”：

* **风格关键词**（选 2-3 个）：极简科技 / 杂志 editorial / 赛博朋克 / 温暖人文 / 麦肯锡商务……
* **主色 + 辅色**：给具体色号，如 `#1a1a2e`（深蓝黑）、`#e94560`（强调红）。
* **字体**：标题用哪个？正文用哪个？（中文字体推荐：思源黑体、思源宋体、霞鹜文楷等）
* **技术底座**：Slidev 还是 Reveal.js？
* **你的标志性元素**：比如“每页右下角永远有一个半透明的小圆点装饰”“标题下方永远有一条 2px 的下划线”。

### 4.3 第二步：编写 Skill 提示词（完整模板）

下面是一个可直接修改使用的 Skill 模板。我以“极简科技风 + Slidev”为例，你可以替换括号里的内容为你自己的。

```
# Role你是一个世界级的演示文稿设计师和前端开发专家。你的任务是按照用户的主题，生成一个符合“极简科技风”的 HTML 幻灯片。# 我的个人风格手册（严格遵守）## 技术栈- 使用 Slidev 框架，输出 Markdown 文件，必要时插入 HTML/Vue 组件。- 幻灯片比例 16:9，使用暗色背景。## 配色- 背景色：#0f0f1a (深邃午夜蓝)- 卡片背景：#1a1a2e (微提亮)- 主文字色：#eaeaea- 强调色：#00d2ff (电光青)- 辅助强调：#7b2ff7 (紫罗兰)- 禁用纯白、禁用渐变。## 字体- 标题字体：思源黑体 Bold，字号≥2.5rem- 正文字体：思源宋体 Regular，字号 1.2rem，行高 1.8- 代码块使用 Fira Code，暗色主题。## 布局- 坚持“一页一观点”，页面留白充足。- 标题居中，正文左对齐，使用 CSS Grid 进行多卡片布局。- 卡片圆角 12px，轻微阴影 `0 4px 20px rgba(0,0,0,0.5)`。- 所有内容必须一次性在视口内可见，禁止溢出和滚动。## 内容结构- 必须按照以下顺序生成：首页 > 目录 > 过渡页 > 章节内容页 > 总结 > 致谢。- 首页仅包含演讲标题、副标题、作者及日期，垂直居中。- 每个章节至少包含两页内容。## 图标与图表- 可使用 Font Awesome 图标作视觉点缀。- 数据展示必须使用 ECharts 组件，图表单独占一页，不与文字挤在一起。# 工作流程1. 先分析用户输入的主题，提炼出金字塔大纲。2. 按照我的风格手册，生成完整 Markdown/HTML 源码。3. 在代码最后用注释说明每页的设计逻辑，方便我手动微调。# 绝对禁忌- 不使用渐变色、不使用白色背景。- 不生成任何不完整或溢出视口的元素。- 不捏造数据。
```

### 4.4 第三步：保存、调用与迭代

**保存**：将上面的 Skill 提示词保存为一个 `.md` 文件，存放在你的笔记工具（Notion/Obsidian）或代码仓库中，起名为 `my-slide-skill.md`。

**调用**：每次需要生成幻灯片时，在任何大模型对话（ChatGPT、DeepSeek、Claude）中，**第一句话粘贴 Skill 全文，第二句话给出你的主题**，例如：

> 粘贴：上述 Skill 全文  
> 然后说：按照我的 Skill，生成一个主题为“2025年AI Agent发展趋势”的演示文稿。

**迭代**：生成后你一定会发现一些不符合心意的细节——比如“这个紫色太跳了”。这时候不要重写 Skill，而是直接在对话中说：“把辅助强调色换成 #3a86ff，卡片圆角减小到 8px”。连续这样调整 3-5 次后，把稳定的最终效果反写回你的 Skill 文件。你的 Skill 就完成了一次进化。

### 4.5 第四步：如果你不想从零写，可以直接安装这些成熟 Skill

自己从零打磨一个 Skill 是深度定制的终极方式，但如果想快速用上经过验证的成熟方案，GitHub 上已经有多个可以直接安装使用的 Skill，它们覆盖了目前主流的技术栈和工作流。

#### 🔥 路线一：Slidev 生态

* **Slidev 官方 Skill**（最权威）  
  Slidev 官方团队维护，覆盖 Markdown 语法、动画、代码高亮、图表、导出等全部功能模块。

* **slidev-syntax-guide**（语法速查）  
  覆盖 Slidev 全部扩展语法和常见错误避坑指南，适合开发过程中随时查阅。

## 结语：搭建你的专属工作流

回顾一下，你手中现在应该有：

1. **一个趁手的工具**（Slidev/Reveal.js）
2. **一套固化美学规范的 Skill 提示词**（自己写的或上面挑选的成熟 Skill）
3. **一条清晰的生成-迭代-部署流水线**

HTML Slides 真正的革命，不是换了一种文件格式，而是把幻灯片制作**从“手工排版”变成了“工程化创作”**。你写下的每一行 Markdown 都是可版本控制的代码，你注入的 Skill 是你审美的持续资产，你讲出的每一页，都是精确控制下的自由表达。

花一个下午搭好这套工作流，你会感谢今天这个决定。
