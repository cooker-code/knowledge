**一、AI****Agent的特征：**

1. 可与环境交互、并自动作出主动决策（脱离人工监督）
2. 可调用工具并操作（联动外部程序API）

**二、AI Agent实现智能的步骤：**

1. 先感知环境（与环境交互，并记住“当前环境和历史环境”）
2. 触发决策（前一步的“感知环境”来决定Agent什么时机触发）
3. 主动做出行为决策（“生成内容”算其中一个主要的行为决策）
4. 根据行为决策，执行适当行动（如：在对话中回复出“生成内容”、联动外部程序并操作）
5. 反馈决策和行动的结果（同时变为“历史环境数据”的一部分）

**三、AI Agent的2种场景：**

1. 单一AI Agent：针对不同领域和场景的特定任务；
2. 多个AI Agent合作：由不同角色的AI Agent合作完成任务（通常是第一个智能体的响应可以作为输入，传递给第二个智能体）；

**四、AI Agent任务执行流程实现的3种方式：**

* **方式一：用户完全自定义编排流程**

即“用户定义任务”：用户可以提前自定义编排流程，可灵活修改编排；

****1️⃣**MetaGPT**（软件自动开发的Agent框架，多个Agent协作）

MetaGPT是一个多智能体开源框架，主要用于解决大模型落地应用问题，它可以通过模拟软件公司多角色（产品经理、架构师、项目经理、开发工程师、测试工程师），输出产品设计、架构图、代码repo等内容。它将会完成编程的自动化SOP流程，完整利用AgentOS能力，完成Agent的大规模生产。

MetaGPT 会收到用户输入的需求，就会有智能体作为产品经理进行需求和可行性分析，再由充当架构师、项目经理、工程师的智能体按顺序完成软件开发。最后还有智能体负责对软件进行全面的测试。整个过程很好地模拟了现实世界的开发过程。

23年10月，MetaGPT在开源社区举办的Mini Hackathon中，有3个团队分别仅用了3周时间，基于MetaGPT框架实现了Minecraft、狼人杀、斯坦福虚拟小镇等游戏智能体的复刻。这里面就有UC Berkeley、Northwestern University、墨尔本大学、香港大学、北京大学等多个学校学生，和腾讯、阿里、智谱等多家公司工程师的贡献。

**2️⃣ 亚马逊 Alexa 电子商务助手**（电商业务型Agent）

亚马逊的Alexa是一个AI智能体，可以推荐产品，下订单和跟综发货。可以为用户提供个性化推荐，改善供应链管理，增强客户体验。

**3️⃣ 微软 AutoGen**（办公任务型Agent框架）

**Autogen是一个由Microsoft推出的框架，它允许用户创建和管理多个****自主代理****，以协同完成复杂的任务。这个框架的灵活性极高，你可以根据自己的需求定义不同的代理和它们的角色，然后让它们一起工作。**

**微软**AutoGen允许 **多个****LLM****智能体****通过聊天来解决任务。**

AutoGen背后的基本理念是，创建智能体，即由LLM（如GPT-4）驱动的编程模块。这些智能体通过自然语言交互，以完成各种任务。智能体可以通过提示工具，和外部工具对进行定制和增强，使其能够检索信息或执行代码。

LLM智能体可以扮演各种角色，如程序员、设计师，或者是各种角色的组合，对话过程就把任务解决了。这与MetaGPT不同，MetaGPT的角色模型是被定义好的，而AutoGen可以让开发者自己定义Agent，还可以让他们相互对话。

论文地址：https://arxiv.org/pdf/2308.08155.pdf

开源地址：https://github.com/microsoft/autogen

4️⃣ MiAO  LARP 开放世界游戏角色的语言**智能体**

游戏智能体：类似Generative Agents 斯坦福“AI小镇” （带角色+场景的AutoGPT的特例）

MiAO希望以游戏为切入点，为玩家和创作者建立共生的平台。让用户在享受游戏这一娱乐形式的同时，探索更多将游戏与社交互动相结合的可能性。创造一个让数十亿用户满意的世界。这是MiAO的使命和愿景。

引入了**LARP（Language-Agent Role Play for open-world games）**来实现实现能灵活适应复杂环境并持续保持长期记忆以确保连贯行动的智能体。包括一个包含记忆处理和决策助理的认知架构，一个具有反馈驱动的可学习动作的环境交互模块空间，以及促进各种个性对齐的后处理方法。LARP 框架完善了用户和代理之间的交互，预定义了独特的背景和个性，最终增强了开放世界环境中的游戏体验。此外，它还强调了语言模型在娱乐、教育和各种模拟场景等一系列领域的多样化用途。

即“Agent定义任务”：脱离人类监督，完全让模型自己决定Agent下一步的流程；

**1️⃣ AutoGPT**

AutoGPT是一个开源的AI代理Python应用程序，由开发人员Significant Ggravitas最早在23年4月发布在GitHub上。几天内就获得了超过27,000个星标，并在一个月内增长至超过42,000个。截止至2023年4月16日，Auto-GPT项目在GitHub上共斩获了72.8k Star，吸引了全世界关注着AIGC的人的关注。

它使用GPT-3.5+GPT-4作为驱动基础，一个全自动可联网的AI机器人，只需给它设定一个或多个目标，它就会自动拆解成相对应的任务，并派出分身执行任务直到目标达成，完全无需用户提示每个操作。30分钟内就可以完成设置，以设置和入门的简便性在用户中大受欢迎。

AutoGPT可以通过主任务生成子任务（LLM自主产生Prompt重新作为输入），并且通过Google搜索和Python脚本来实现。Auto-GPT就会自己将主任务生成子任务，再把子任务列表拆解成某些具体的任务项，并生成后续的任务执行列表Task List，再自动依次执行Task List，自己给自己提问并进行回答。每一步可以是执行Google搜索、生成Python脚本、存储生成文本、执行脚本文件等等。

Auto-GPT案例的内容包括有商业调查、无代码生成APP或网页、自动化办公和文本生成等等。在加入了Stable Diffusion后甚至可以拥有图片的生成能力。

### **五、从 Agent 到 AGI**

AGI通用人工智能，最重要的一个特点就是脱离人工监督可自行完成任务。但当前Agent技术的能力仍有局限性（**仍依赖特定任务优化、**缺乏自主意识与长期规划、**协作仍受限于任务设定）******，未达到通用人工智能（AGI）的目标，差距在于：**通用性不足、**自我改进能力有限、**缺乏真正的理解与创造力。******

******Agent ****未来****进化的方向可能是：******

* **强化学习与多智能体融合**：强化学习RL介入，优化Agent的任务执行能力。
* 人机协作增强：周鸿祎提出的“人在回路”（Human-in-the-loop）模式可能成为过渡方案，人类监督智能体的执行，逐步提升其自主性。
* 更强大的基座模型：GPT-5等下一代模型可能进一步缩小与AGI的差距。
