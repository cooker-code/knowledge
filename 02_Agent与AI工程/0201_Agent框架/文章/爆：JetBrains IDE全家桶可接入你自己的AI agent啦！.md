### 前言

JetBrains 始终乐于为用户带来市场上最优质、最受欢迎的工具。

几十年来，传统开发工具如编译器、调试器、构建工具等一直如此。JetBrains几乎为各语言的编程构建了一套生态：IDEA、PyCharm、GoLand、WebStorm、PHPStorm等等。

过去几年，JetBrains 同样的方式发布了所有最新的大型语言模型，从AI Assistant 到Junie。

现在，JetBrains 再次推出开发代理客户协议 （ACP）。同时，在JetBrains 最新的候选版本中已支持其测试版！

### 什么是 ACP？

T he Agent Client Protocol（代理客户端协议）规范了代码集成开发环境（IDE）与编码代理之间的通信，类似于 LSP 为语言服务器所做的那样。

用户可以在 IDE 中选择任何他们喜欢的代理，代理开发者专注于核心功能而非集成，IDE 作者也能更快地获得新功能。

我们认为编码代理和编辑之间的共同标准实际上是非常自然的分界点。人类和人工智能在同一环境中协作，解决相同的问题，因此双方说共享的语言是合理的。

编辑器供应商和 AI 代理开发者有不同的优势和专业领域，统一的标准帮助它们在不互相干扰的情况下达成平衡。

这样的协议提供了核心基础，使访问变得可预测并简化分发流程，让每个人都能更快行动。

### 演示

最新的 25.3 候选版本，将测试版的 ACP 支持带入了我们的统一 AI 聊天中。欢迎在我们的 IDE 中使用任何兼容 ACP 的代理 。

目前还处于测试阶段。因此，JetBrains假设你在将ACP代理添加到 IDE 之前，已经登录了你的 CLI 代理。之后，你只需要调整一个配置文件，就可以开始了。

### What’s next?

以下是我们接下来重点关注的关键方向。首先，关于产品的用户体验：在应用标准的同时，我们会交换一些定制功能，并希望将其中一些带回协议中。其次，我们开始着手代理分发，并致力于构建一个经过策划的 ACP 代理注册库。第三，我们希望将同一标准用于远程服务器上的代理，而不仅仅是本地机器上。最后，我们希望提供集成开发环境中的 MCP 工具链，从而提升代理本身的质量。

### 最后

在结束之前，我想先分享一下 JetBrains 是如何加入 ACP 的幕后背景。

今年夏天，我们开始着手将 Junie（由 JetBrains 开发的编码代理）引入我们为 AI 聊天设计的同一个界面。

我们最后构建了一个协议，允许我们在 IDE 与 Junie 的本地及远程版本之间通信。我们正准备将这项工作发布到互联网，但几乎同时，Zed 宣布了自己的协议。

我们与 Zed 团队交流，达成共识，不再将两个竞争标准推向市场，而是共同打造一个统一的标准。

统一的标准将更加健全。

从用户角度来看，我们看到现在有许多具备 ACP 能力的代理，未来还会有更多。

从技术上讲，该协议易于实施 ，同时为产品提供了强大且灵活的用户体验。

从业务角度看，我们现在与多个合作伙伴建立了良好的关系。
