## 文章处理 SOP

1. 先看正文第一屏和结论段，确定主问题，不按标题关键词归类。
2. 判断是否补充机制、边界、失败模式、版本差异、生产案例或验证动作。
3. 有贡献文章写入核心知识点、知识地图、版本记录或本文件后，改为 `done-`。
4. 只有营销、热榜、安装清单、泛入门且无新增准则的文章直接删除。
5. 版本、发布、API、兼容性结论只写入版本记录的待核验区，官方补证后再转为结论。

## 排重判断

```text
Pi + 具体模块/机制 + 解决问题 + 适用边界 + 失败模式/验证动作 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 同一框架同一机制 | 无新增边界或证据时不新建 |
| 只有案例不同 | 案例不改变准则时只保留一个来源或删除 |
| 有失败模式 | 优先追加到已有知识点 |
| 有错放迹象 | 按正文主问题迁到正确节点 |
