# 020104_Pi知识地图

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 02_Agent与AI工程 / 0201_Agent框架 / Pi |
| 解决的问题 | 先定义 Pi 作为终端 coding agent runtime 的应用流程，再把文章吸收到流程节点 |
| 典型输入 | 编程任务、项目文件、模型 provider、工具调用、事件日志、session 历史和上层应用封装需求 |
| 典型产出 | 文件修改、命令执行结果、事件流、turn/session 状态、可嵌入 SDK/RPC 输出 |
| 用户阅读目标 | 读 Pi 文章时，判断它补的是 CLI 使用、runtime loop、工具权限、会话恢复、扩展嵌入还是上层封装 |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| P00 技术边界 | 文章是在讲 Pi 本体、Pi CLI 体验，还是上层套壳产品 | Pi 是 runtime / CLI / SDK 组合，不等于所有 AI 编程工具 | 只讲工具体验的文章降权，能讲 runtime 和 SDK 的文章优先 |
| P01 上下文与 Provider | 项目目录、模型、认证和资源如何进入会话 | Provider 抽象降低切换成本，但不能消除模型差异 | 能说明 ModelRegistry、ResourceLoader、认证和 provider 差异的内容进入 |
| P02 工具执行面 | read/write/edit/bash 如何声明、执行和返回结果 | 四工具极简降低表面复杂度，但权限和审计要外接治理 | 只夸“工具少”不够，必须说明执行边界、风险和替代方案 |
| P03 Runtime 事件流 | Agent loop 如何推进模型响应、工具调用和事件输出 | turn、message、tool result、agent_end 是可观察和恢复的基础 | 能给出事件类型、字段和复现步骤的内容优先吸收 |
| P04 会话与恢复 | session、fork、clone、resume 如何支撑探索式编程 | 会话树让复杂任务可回溯，不把对话锁成单线历史 | 能讲 session 存储、checkpoint、恢复边界和失败模式的内容进入 |
| P05 扩展与嵌入 | Pi 如何被 OpenClaw、GUI、agent-wrapper 等上层系统调用 | Pi 可以作为被封装的 provider/runtime，但上层治理不是 Pi 本体 | 上层封装文章标为不同层次，避免混同本体机制 |
| P06 安全与成本 | Bash、文件写入、订阅登录和 token 预算怎么治理 | 极简不等于安全，生产要补审批、沙箱、预算和审计 | 有权限、预算、沙箱、日志证据的内容优先，宣传数字降权 |
