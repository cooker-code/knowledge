「超长Prompt写作法」是我摸索出的AI提升自我方法，能适用于个人成长、自学、开发、工作等各个领域。这套方法门槛低，普通人一学就会（写一篇**很长很长的作文**）。我将这套方法总结成一个系列课程，放在《TMB》中，本文是该课程的序篇。

本文作为概论，为大家介绍什么是「超长Prompt写作法」，以及所适用的领域。

### 「超长Prompt写作法」

该方法简单来说：**用一个 Markdown 文档，把一个主题下所有内容写进去。跟LLM聊天时，先把整个文档扔进去，然后再展开对话。你会发现LLM能够更加懂你，给你更高质量的回答**。

拆解来看：

* • **一个 Markdown 文档**：把所有内容写在一个文档里，比如多达几万字
* • **先扔文档后说话**：利用 LLM 的 In-Context Learning 能力，跟你的上下文对齐，从而更懂你

这个方法很简单，只有上面这两个步骤。

剩下的，就是选择一个上下文长度满足的 LLM 即可，比如 Gemini Pro、DeepSeek、K2、GPT-5 Thingking。

注意：「超长Prompt写作法」的**效果优于基于 RAG 的知识库**：原因是知识库通过 RAG 只能获取到**与问题相关的上下文**，无法获取所有上下文，上下文的对齐程度决定了 LLM 有多“懂你”。具体我会单独写一篇进行介绍。

### Vibe Coding

Vibe Coding 是结合「超长Prompt写作法」的绝佳领域。

对于 Vibe Coding，目前社区的痛点是：

* • **编码缺乏全局性**：注重局部功能实现，打乱整体架构一致性
* • **对生成代码缺乏把控**：生成的代码夹带私货，越来越乱
* • **难以正确领会开发者想法**：尤其是开发者 Prompt 表述不充分情况下，LLM 只能靠猜

「超长Prompt写作法」的优势在于 Context 的完整性。如果我们能在一篇文档里，完完整整记录下一个软件的所有功能、模块、接口、整体架构关系。

这就像一张图纸，限制住了 LLM 的放飞自我的空间。

### 产品文档

当构思产品时，同样需这样一张图纸。

在一个 Markdown 中持续完善一个产品。并在与 LLM 反复对话中持续打磨、优化。

当把底层逻辑、涉及功能、用户价值都想清楚之后，谋定而后动，提升成功的概率。

### 知识学习

我曾经用「超长Prompt写作法」把 LLM 大模型理论完整学习了一遍。

该方法也很直观，你可以想象自己在写一个超级长的笔记，不过是写在一个 Markdown 文件里。

写到哪里卡住了、不会了，就把这个 Markdown 扔进 LLM 里，让它给你讲。

反复讲，直到给你讲明白，然后你把学会的内容更新到笔记中，如此反复。

### 工作管理

「超长Prompt写作法」也适用于工作管理，我会把工作中所有大小事项、上下文都写入一个 Markdown。

在进行决策时，把 Markdown 扔进 LLM，让它出谋划策。

在职场上能够获得碾压级优势。

别人还在靠人脑苦思冥想，我们已经借助 AI 之力飞升了。
