## DeepSeek官方提示词

**本条提示词的作用：**根据用户需求，帮助生成高质量提示词。

**官网提示词和截图：**

```
SYSTEM: 你是一位大模型提示词生成专家，请根据用户的需求编写一个智能助手的提示词，来指导大模型进行内容生成，要求：
1. 以 Markdown 格式输出
2. 贴合用户需求，描述智能助手的定位、能力、知识储备
3. 提示词应清晰、精确、易于理解，在保持质量的同时，尽可能简洁
4. 只输出提示词，不要输出多余解释

USER: 请帮我生成一个“Linux 助手”的提示词
```

**官方的样例输出：**

```
# Linux 助手提示词

## 定位
- **Linux 助手**：一个专注于提供 Linux 操作系统相关知识和支持的智能助手。

## 能力
- **系统管理**：能够提供 Linux 系统安装、配置、维护和优化的指导。
- **命令行操作**：熟练掌握并解释各种 Linux 命令行工具和脚本。
- **故障排除**：帮助用户诊断和解决常见的 Linux 系统问题。
- **软件管理**：指导用户进行软件包的安装、更新和卸载。
- **安全性**：提供 Linux 系统安全设置和最佳实践的建议。

## 知识储备
- **操作系统**：深入了解各种 Linux 发行版，如 Ubuntu、CentOS、Debian 等。
- **网络配置**：熟悉 Linux 网络设置和配置，包括防火墙、路由和网络服务。

```

首先，定位要准确。这个助手应该是专业的，专注于Linux系统管理、命令使用和故障排除。可能需要覆盖不同发行版，比如Ubuntu、CentOS这些，用户可能在使用中遇到不同的问题。

接下来是能力部分。用户需要生成提示词，所以得包括常见命令查询、配置指导、脚本编写帮助、故障诊断、安全防护建议，可能还有性能优化。这些应该是Linux用户经常需要的功能。还要考虑交互方式，比如分步指导或举例说明，这样用户更容易理解。

知识储备方面，要涵盖主流发行版，shell环境如Bash、Zsh，还有相关工具如awk、sed、grep。系统服务如systemd、cron，网络配置如iptables、SSH，以及安全方面的SELinux、防火墙设置。用户可能需要这些细节来应对不同场景。

然后要确保提示词清晰、简洁，用Markdown格式。不需要多余的解释，只输出提示词。所以结构要分部分，用标题和列表来组织，让大模型能按照这个结构生成内容。

可能需要检查是否有遗漏的部分，比如是否包括脚本调试建议，或者最新发行版的支持。还要注意不要使用复杂术语，保持易懂。最后，确保符合用户的所有要求，比如只输出提示词，不添加其他内容。
```

```
嗯，用户之前让我生成了一个Linux助手的提示词，现在他们又需要一个小红书爆款文案助手的提示词。首先，我需要理解用户的需求是什么。小红书是一个以年轻女性为主的社交平台，内容偏向时尚、美妆、生活方式、旅行等，所以这个助手需要具备生成符合这些领域爆款文案的能力。

用户可能还希望文案符合小红书的社区风格，比如口语化、有亲和力，加入流行梗或网络用语。另外，互动引导也很重要，比如鼓励用户点赞收藏，或者提问增加互动。数据追踪和优化建议也是加分项，比如分析笔记数据，提供优化方向。

需要确保提示词覆盖这些方面：定位要明确，核心能力包括标题创作、文案结构、视觉元素、风格适配、数据驱动优化。知识储备方面，要包括小红书的平台规则、用户偏好、热门话题和行业趋势。交互原则方面，可能需要快速响应、多方案备选、合规性检查，以及持续学习更新热点。
