**01 前言**

无论是与大模型对话，还是开发AI Agent，提示词都是我们与AI协同的核心桥梁。模型的表现，直接受限于提示词的质量。正因如此，各家厂商无不将优化提示词视为关键实践。

当然，世上没有“万能”的提示词。任务类型、模型架构与参数规模均有不同，这意味着最优解离不开持续的测试与迭代——唯有通过亲手实践，才能找到那个最令人满意的答案。

**02 提示词策略**

**清晰的说明**

提示词需要清晰地表达背景、任务和目标，避免让模型猜测。如果我们自己都觉得晦涩难懂，那么模型大概率会胡说八道。

1、明确细节

不要让模型揣测，你关心的细节要提供给你模型，让它知道。

**例如**：你想让模型输出Markdown格式，请通过提示词告诉模型。你想让模型最多输出多少字符，请告诉模型。

2、使用分隔符

对于提示词内容比较多的时候，使用分隔符（三重引号/XML标签/章节标题等定界符）划分不同的文本部分。

**例如**："""

这里是用户的历史对话。

"""

**提供参考文本**

可以通过提示词提供给模型参考内容，如参考专业知识进行回答、参考风格示例、参考回复的内容话术。

1、参考专业知识进行回答

这个类似RAG，通过提供给模型不知道的内容或想圈定回复的内容，让模型基于我们提供的内容范围来回复问题。

**例如**：参考知识库"""这里是知识库内容"""回答用户问题。

2、参考示例回复

如果你希望模型回复一种特定的风格（语气、格式等），而这种风格又难以直接描述时，就可以通过提供示例的方式来说明。这被称为**“少样本”提示**。

**例如**：提示词中增加："""

以下是几个例子，请学习并模仿其处理逻辑和输出格式： 

 示例 1： 

输入： “[示例输入1]” 

输出： “[示例输出1]”

"""

3、参考回复的内容话术

在特定情况下，让模型回复特定的内容。

**例如**：如果用户提问敏感问题，让模型直接回复“涉及敏感话题，无法回答您”。

**让模型思考**

让模型思考有助于回答推理类的问题，即使会增加模型的响应时间，效果（质量）仍是考虑的第一要素。

1、一步一步推理

要求模型输出其思考过程，能将内部的推理逻辑显式化。这种做法不仅能提升模型回答的透明度，其分步推导的模式，甚至在特定任务上能帮助低参数模型达到与高参数模型相仿的效果。

**例如**：对于算数题，告诉模型，“请一步一步推导过程”或者“请自我验证结果的正确性”来让模型进行思考。

2、参考处理问题的思考步骤

在提示词中写出具体的步骤来完成某类任务，写出步骤可以让模型更容易遵循，比较适合流程相对明确的任务。

**例如**："""请使用如下步骤响应用户问题

第一步，提取用户问题的一个核心关键词

第二步，通过该关键词为用户写一首诗。"""

**提示词之外**

除了提示词本身，从效果和目标出发，可以让模型额外使用工具等外部能力，或者直接拆分任务和提示词。

1、使用外部工具

模型可以调用计算、搜索、制图等工具，将“模型生成”与“工具执行”结合。（当然模型本身不会调用工具，需要我们解析模型的输出，并编写程序和制定流程完成工具的调用）

**例如：**当用户问到“生成某股票的最新走势”时，可以引导模型调用搜索工具，获取最新股票数据。如告诉模型："""首先调用搜索工具获取某股票最新数据"""

2、任务拆分

复杂任务不必一次完成，可以通过拆分任务，编写多个提示词逐步完成。

**例如**：例如让模型基于主题写一篇博文，先让模型生成章节目录，再根据目录以递归的方式让模型完成每个章节内容输出并进行拼接，最后再交由一个模型检查文章，如连贯性等。

不管用什么策略和技巧，都要不断打磨提示词，使其可以稳定输出预期的结果，并能应对一些“边界”提问。

**03 乔哈里视窗**

针对日常的模型提问场景，我们可以类比乔哈里视窗模型。

乔哈里视窗模型是有美国心理学家1955年提出的一种用于分析人际沟通和信息流动的模型。它是一个2x2的矩阵，将人的认知信息分为四个象限：

|  |  |  |
| --- | --- | --- |
| \_ | 自己知道 | 自己不知道 |
| 他人知道 | 1、开放区  信息公开，是有效沟通的基础 | 2、盲区  别人知识，自己没意识到的信息 |
| 他人不知道 | 3、隐藏区  自己知道，但选择不告诉别人的隐私 | 4、未知区  双方都不知道的潜能或者信息 |

**该模型的核心思想****是**： 通过有效的沟通，扩大“开放区”，缩小其它三个区域，从而增进理解、提升沟通效率和信任。

将该模型用于提示词编写上，同样分为**模型知道**、**模型不知道**、**自己知道**、**自己不知道**，组成一个四象限，在不同象限使用不同的提示词策略，目标是最大化开放区，让模型回复符合预期：

例如：

1、对于AI知道且我们知道的一些概念，位于第一象限，就可以直接提问AI，假设我们想知道像素的定义，直接提问“什么是像素”。

2、对于AI知道，我们不知道的内容，可能我们都不知道该如何提问模型，就需要根据模型的回复，进行扩展提示词。如：让AI“写点有趣的东西”，但我们不知道什么对AI来说是“有趣”的（这是我们的盲区），AI可能会给出一个平庸的结果。此时通过反馈缩小盲区：AI给出了一个平淡的笑话，我们意识到问题，于是修改提示词：让AI“写一个关于程序员的双关语笑话。” ，这下就清晰了。

**04 最后**

主要介绍了些关于文本生成模型的提示词策略和技巧。对于一些多模态的模型不一定适用，就像文生图模型提示词，如果能很好的描述出图片的风格、比例、画面、动作等等细节或专业的概念会更有帮助。

总之，提示词是我们与模型对话的媒介和桥梁，需要不断打磨来达到理想的效果，同时减少幻觉的出现。
