## 💡 使用场景

### 场景一：需求分析

**以前怎么做**：拿到需求文档，自己想破脑袋设计测试场景，边界情况经常漏 **现在怎么做**：

```
1. 复制需求分析提示词  
2. 附加需求文档  
3. AI自动生成全维度测试场景  
4. 包含正向路径、异常路径、边界值分析
```

### 场景二：写测试用例

**以前怎么做**：手工一条条写，格式乱，质量也不稳定 **现在怎么做**：

```
1. 使用测试用例编写提示词  
2. 输入测试场景  
3. AI生成标准化测试用例  
4. 包含前置条件、测试步骤、预期结果
```

### 场景三：性能测试

**以前怎么做**：不知道从哪下手，测试指标也不知道该看啥 **现在怎么做**：

```
1. 使用性能测试提示词  
2. 描述系统特点  
3. AI生成完整性能测试方案  
4. 包含负载、压力、容量测试
```

## 🛠️ 如何使用

### 步骤1：访问项目

🌐 **在线文档**：https://naodeng.github.io/awesome-qa-prompt/ 📦 **GitHub仓库**：https://github.com/naodeng/awesome-qa-prompt

### 步骤2：选择提示词

根据测试任务选择对应的提示词文件：

* 中文项目 → 使用中文提示词
* 英文项目 → 使用英文提示词（\_EN.md）
* 快速上手 → 使用精简版（\_Lite.md）

### 步骤3：复制使用

1. 复制提示词内容到AI助手（ChatGPT/Claude/Cursor AI）
2. 附加你的输入材料（需求文档/测试场景等）
3. AI生成结构化的测试文档
4. 根据需要调整完善
