# 020700_Prompt Engineering知识地图

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 02_Agent与AI工程 / 0207_Prompt Engineering |
| 解决的问题 | 把 Prompt 从临时话术变成可复用、可评审、可测试的 Agent 指令接口 |
| 典型输入 | 系统规则、用户任务、few-shot 示例、反例、输出格式、工具说明、安全策略 |
| 典型产出 | System Prompt、User Prompt、模板、示例库、反例库、Prompt 版本和测试用例 |
| 用户阅读目标 | 判断文章补的是提示词层级、结构、示例、输出契约、安全边界还是治理方式 |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| P00 任务建模 | Prompt 要控制什么任务 | 没有任务类型就没有通用 Prompt | Agent Prompt、AI Coding Prompt 文章进入 |
| P01 层级设计 | system、developer、user 各放什么 | 层级决定优先级和可复用性 | System/User Prompt 文章进入 |
| P02 示例与反例 | few-shot 和反例如何约束输出 | 示例要覆盖正常、边界、失败场景 | 模板、案例、Prompt 库文章进入 |
| P02b 推理范式 | CoT、ToT、ReAct 如何控制推理路径 | 推理范式必须服务任务边界和验证闭环 | CoT、ToT、ReAct、Self-Consistency、Reflection 文章进入 |
| P03 输出契约 | 输出格式如何可解析、可验证 | Prompt 必须服务后续执行和校验 | JSON、表格、代码生成文章进入 |
| P04 安全边界 | 用户输入和工具结果如何防注入 | Prompt 注入是接口安全问题 | 注入攻击、防御、拼接风险文章进入 |
| P05 版本治理 | Prompt 怎么沉淀、评审、回滚 | 团队 Prompt 是资产，不是聊天记录 | PromptHub、管理器、Skill 转化文章进入 |
| P06 失效诊断 | 什么时候该换 Context/Harness/Skill | Prompt 不是所有问题的解法 | “Prompt 不够用”类文章进入 |
