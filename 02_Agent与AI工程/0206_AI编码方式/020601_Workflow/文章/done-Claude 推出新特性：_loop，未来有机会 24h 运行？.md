大家好，我是煎鱼。

不知道大家最近还有没有关注 A 社持续推出的新特性？

最近他们出了个新特性 —— 内置的 `/loop` 定时任务功能 (Scheduled tasks)，这一出不仅在官方文档里占了重要版面，在社区里更是引起了不小的讨论。

## 背景

日常写代码，这个诉求挺常见：我们经常需要在终端里跑一些耗时的任务。

举几个大家都经历过的例子：

* 你刚提交了一段代码，需要等待 CI/CD 管道跑完部署流程。
* 本地运行一个巨无霸项目的集成测试，一跑就是十几二十分钟。
* 你眼巴巴地盯着一个 PR ，等着你的同事来 Review 或者等某个自动化检查通过。

很无奈的是，以前我们用各种 AI 工具，你给它一个指令，它干完一件事就停了。

你要是想让它帮你盯着某个任务的进度，你得不停地手动输入：

* “部署完了吗？”
* “测试过了吗？”
* “那个 PR 有人回没？”

这就会导致你的聊天对话框里有一堆的手动检查，开发者们苦不堪言。 大家以前是怎么解决的呢？

要么写个 `watch` 命令死死盯住终端，要么手写一堆复杂的 Bash 脚本加上 `crontab` 来轮询。还是比较麻烦的。

## 新特性 /loop 介绍

为了解决这个痛点，Claude Code 官方团队开发了个新特性，推出了 `/loop` 定时任务功能（Scheduled tasks）。

简单来说，就是允许你在当前的 Claude Code 会话中，设定一个智能定时器，让 Claude 在后台自动、定期地执行某个自然语言 Prompt。

它的灵感显然来源于 Unix 系统的 Cron 机制，以及现代开发中越来越火的“代理自动循环 (Agentic Loop) ”概念。

他解决了一个关键问题：如何把自然语言交互和周期性任务完美结合。

## /loop 使用例子

我们来看下具体的代码和命令示例。咱们循序渐进，从最简单的例子看起。

以前的人工写法，是每隔五分钟你都要手动敲一次这行命令：

```
check if the deployment finished
```

现在的写法非常直观，直接上 `/loop` 加时间间隔：

```
/loop 5m check if the deployment finished and tell me what happened
```

如果你觉得把时间放前面别扭，时间甚至可以放在句末，像平时说话一样：

```
/loop check the build every 2 hours
```

不仅如此，如果你只想要一个单次的提醒 (One-time reminders) ，也可以不用重复循环，像设定闹钟一样搞定：

```
/loop in 45 minutes check whether the integration tests passed
```

甚至，我们还可以更进一步，让它去执行你封装好的 subagent 或者 skiils。

比如你之前写好了一个专门用来 Review 代码的自定义 skills `/review-pr` ，你可以直接这样组合：

```
/loop every 20m /review-pr 1234
```

这样一来，Claude 就会每 20 分钟自动去抓取 PR 1234 的最新代码并进行 Review。

## 社区观点

这个特性一出，社区可是炸开了锅。有的人觉得这功能简直是神器，彻底解放了双手。也有觉得他这样就是 OpenClaw。

新特性 `/loop` 的核心价值在于上下文感知 (Context-aware) 。传统的定时脚本只能死板地执行命令、吐出日志。而 Claude Code 是带着你的项目上下文去检查报错、分析日志的。

一旦你的 CI 测试在 `/loop` 期间跑挂了，Claude 不仅能发现失败，还能根据它的代码库上下文，直接告诉你哪里报错了，甚至顺手帮你生成一段修复代码。这两者的体验能一样吗？

这对于我们日常开发，尤其是在管理复杂项目或者是跨语言项目，又或是长周期运行任务时，有着非常直观的落地价值。

## 注意事项

实现逻辑上，本质是 Claude 接收到指令后，会在后台解析你输入的自然语言时间间隔，把它转换成底层的 Cron 表达式，调度这个 Job 并生成一个任务 ID。

现阶段有两个关键的点要注意：

1. **低优先级队列**：Claude 调度器会每秒钟检查一次到期任务，并将它们放入低优先级队列。这些定时任务是 Session-scoped (会话级别) 的。只要你不退出当前的 Claude Code 进程，它就会一直默默在后台帮你打工。
2. **单终端且 3 天有效期**：目前的 `/loop` 还是绑定在单个终端 Session 里的，退出进程就失效了，并且每个会话最多只能跑 50 个任务，最长有效期是 3 天。

## 总结

可以看到 Claude Code 这次推出的 `/loop` 定时任务功能，能切切实实地在解决开发者在终端里被打断工作流的痛点。

它巧妙地把时间调度器和 LLM (大语言模型) 的推理能力结合在了同一个代码上下文里。

遥想一下，他们能不能把这种轻量级的会话任务，进一步扩展到无缝对接后端的持久化队列里？

甚至让 Agent 在云端代替你 24 小时监控项目库，再做一些运营类的事情，那将会进一步打破边界！
