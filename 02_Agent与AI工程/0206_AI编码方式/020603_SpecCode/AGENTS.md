## 文章处理 SOP

1. 先判断文章主问题，不按标题里的工具名直接归类。
2. 只保留能说明编码方式、流程边界、失败模式、验证方式或迁移代价的内容。
3. 工具安装、Git/worktree、企业协作 CLI 等本机工具用法转到 `09_电脑工具`；具体工具机制转回 `0205_AI编程工具`。
4. 新文章保留时必须进入 `文章/` 并被知识地图、核心知识点或本文件引用。

## 排重判断

```text
编码方式 + 流程节点 + 关键机制 + 解决问题 + 验证/维护边界 + 对用户的认知增量
```

| 判断项 | 处理 |
|---|---|
| 规格先行 | 复杂需求先写目标、边界、验收和任务，不直接进入编码 |
| 文档可执行 | PRD、Spec、AGENTS、Rules 只有能约束输出和验证时才有价值 |
| 偏离治理 | 代码偏离规格时要有修正、回滚和变更记录 |
| 成本意识 | 规格过重会拖慢原型阶段，必须按任务风险选择强度 |
