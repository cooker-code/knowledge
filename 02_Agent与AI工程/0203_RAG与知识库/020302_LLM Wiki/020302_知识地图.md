# 020302_LLM Wiki知识地图

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 02_Agent与AI工程 / 0203_RAG与知识库 / LLM Wiki |
| 解决的问题 | 先定义 `LLM Wiki` 的应用流程，再把文章吸收到流程节点 |
| 典型输入 | 本地文章、已有 AGENTS、已有核心知识点、后续新增资料 |
| 典型产出 | 流程节点知识、判断准则、风险边界、待补证据 |
| 用户阅读目标 | 读到一篇文章时，先知道它优化哪个环节，再决定是否吸收 |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| W00 原始资料接入 | raw 资料如何保存来源、结构和不可变锚点 | 原文不是知识点，先保留可追溯来源 | 抓取、归档、原文锚点、图片本地化文章进入 |
| W01 知识预编译 | 如何从资料中抽出概念、实体、对比和判断准则 | 页面应服务复用判断，不是单篇摘要 | Ingest、概念抽取、比较页、实体页文章进入 |
| W02 互链与索引 | 页面之间如何形成 wikilink、目录和查询入口 | 链接质量决定 Agent 检索和人读路径 | index、AGENTS、wikilink、导航文章进入 |
| W03 查询使用 | Agent 如何通过索引查询、引用和追溯证据 | 查询入口要减少全文重读并保留证据链 | Query、上下文注入、知识调用文章进入 |
| W04 Lint 治理 | 如何检查断链、孤立页、重复、过期和矛盾 | Git + Lint 是 LLM Wiki 区别于普通文档库的闭环 | lint、CI、成熟度、引用追踪文章进入 |

## 文章进入时的对比规则

| 判断项 | 处理方式 |
|---|---|
| 补充 | 文章提供新的模块边界、失败模式、实现步骤、验证指标或工程权衡时，写入对应流程节点 |
| 冲突 | 文章观点和当前节点准则不一致时，先记录冲突原因，不直接覆盖已有沉淀 |
| 不同层次 | 入门教程、源码机制、生产实践、产品发布分属不同层次，必须标注层次后再吸收 |
| 降权 | 只有标题党、工具清单、无证据收益、泛泛最佳实践时，不进入核心知识点 |

## 使用方式

- 先读本文件的宏观流程，再读具体主题知识点。
- 新文章不要单独写摘要，先归入流程节点，再判断补充、冲突、不同层次或降权。
- 如果某个节点已有多篇文章，后续应拆成更细的 `<主题>.md`（保留在 `020302_核心知识点/` 内），而不是继续扩展文章列表。
