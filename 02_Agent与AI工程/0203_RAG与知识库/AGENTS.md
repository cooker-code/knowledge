| 项 | 内容 |

## 用户认知重点

| 认知项 | 当前假设 | 后续整理策略 |
|---|---|---|
| 已知基础 | RAG 是外部知识增强，不是模型训练 | 不重复讲“向量库是什么” |

| 易偏差点 | 工具清单文章容易把“支持格式多”当成质量高 | 优先抽取可验证的解析质量、召回质量和维护成本 |

## 排重准则

问题指纹：

```text
知识系统对象 + 链路节点 + 核心机制 + 解决问题 + 质量/成本/维护边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是文档解析 | 按 PDF、HTML、表格、知识图谱、多模态拆分 |
| 都是 RAG 优化 | 区分解析、切分、召回、重排、生成、引用、评估 |
| 都是知识库产品 | 先判断是 RAG 系统、文档管理、AI 工作流还是个人知识管理 |
| 只有工具清单 | 没有选择准则和质量指标时只略读 |
| 与 LLM Wiki 对比 | 必须说清是检索系统、预编译知识库，还是个人知识沉淀流程 |
