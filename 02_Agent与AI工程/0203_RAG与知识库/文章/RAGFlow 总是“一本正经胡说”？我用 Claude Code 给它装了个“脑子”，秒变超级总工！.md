RAGFlow 进阶打法

双路检索 + 二次回溯

让 AI 变聪明的避坑指南

// 前言

前段时间我写了一篇关于 RAGFlow 的文章，很多朋友体验后跟我反馈："这东西解析确实强，但有时候回答问题还是会'飘'，不够精准。

"确实，我也遇到了这个问题。但有一说一，在开源界，RAGFlow 依然是目前解析能力（DeepDoc）最强的知识库，没有之一。

既然它"底子"这么好，我们能不能别光盯着它的缺点，而是想办法帮它一把？

所以我最近就在琢磨一个新思路：能不能把我现在最顺手的"最强大脑" Claude Code，和 RAGFlow 通过 API 强强联合？ 既然 RAGFlow 偶尔会"眼瞎"，那我就让 Claude Code 去当它的"监理"。

经过这几天的 API 深度研究和暴力测试，我摸索出了一套"双路检索 + 二次回溯"的进阶打法。虽然还在验证阶段，但我先把这套"让 AI 变聪明"的避坑指南分享出来，大家一起探讨。

// 一、 架构设计：让"脑子"和"手"分家

现在的 RAG（文档问答）工具，大多是"前店后厂"混在一起。我的搞法是 Headless 模式（去头模式），简单说就是把脑子和手分开：

Claude Code（大脑/项目经理）：

负责听你的指挥，写代码调用 API，做决策，做对比。

RAGFlow（手/资料员）：

闭嘴干活，只负责扫描图纸、通过 API 返回"东西在哪"。

为什么要死磕 API？因为通过 Web 界面点点点是没有灵魂的。我们要让 Claude Code 学会自己写 Python 脚本去调用 RAGFlow 的 API（让CLAUDE CODE自己指挥RAGFLOW）。比如 Claude 发现资料不够，它能自动通过 API 告诉 RAGFlow："再去把那本《混凝土规范》重新解析一遍"，这才是真正的自动化。

// 二、 进场清理：垃圾进，垃圾出

这一条是铁律：垃圾进，垃圾出。如果喂给 RAGFlow 的数据不干净，出来的答案肯定也是废的。我在用 Claude Code 处理数据时，发现 Word/Excel 和 PDF 必须区别对待，这里有两个史诗级的大坑：

1. Word/Excel：大刀阔斧，手术级改造

Word 和 Excel 是"软装修"，Claude Code 可以直接读取内部结构，想怎么改就怎么改。

痛点：表格里经常写"本项目"、"该指标"。切成碎片后，AI 根本不知道这个"该指标"属于混凝土还是钢筋。

✅ 专家搞法：让 Claude 遍历文档，把所有的"它"、"该数值"直接替换成全称（比如把"它"改成"C30 混凝土强度"）；把重复的废话行直接删掉。

结论：对于 Office 文档，改得越透，检索越准。

2. PDF：不能重写，只能"涂改"

PDF 是"硬装修"，一旦你把文字提取出来重写，排版就炸了，页码全错。保留原件的视觉定位，是工程人的底线。

❌ 错误搞法：转成 TXT 清洗后再转回 PDF —— 格式全废。

✅ 专家搞法（涂改法）：我不动原文结构。我让 Claude 写脚本算出页眉页脚的坐标，然后画一个白色的矩形框盖上去。妙处：就像用"修正带"一样。图纸还是那张图纸，页码没变，但 AI 扫描的时候看不到那些干扰噪点了。

// 三、 动态分拣：看菜下碟

RAGFlow 有好几种解析模式：按法律切、按论文切、按表格切。以前得手动选，现在我通过 API 让 Claude 实现了自动分拣。

Claude 在上传文件前，先"看一眼"特征：

看到"第 1.0.1 条" →→ 这是规范，API 传参指定 laws 模式（防条款粘连）。

看到 XML 里全是表格 →→这是台账，API 传参指定 table 模式。

看到摘要、参考文献 →→这是论文，API 传参指定 paper 模式。

这就是用 API 实现的自动化：用通用的脑子，去指挥专用的工具。

// 四、 终极杀招：双路检索 + 二次回溯

这是最硬核的一步，专治 AI 的"断章取义"和"胡言乱语"。

普通的 RAG 是单行道：搜到啥就是啥。我设计的这套是闭环侦探模式，流程如下：

第一步：双管齐下（同步搜索）

A 路（云端问询）：Claude 通过 API 问 RAGFlow："地下室防水等级是多少？" RAGFlow 快速返回："一级防水（来自图纸说明 第3页）"。

B 路（本地排查）：Claude 同步在本地文件系统中，利用关键词（Grep）或本地索引，定位到相关文档的对应章节。

第二步：交叉对证（找茬）

Claude 会把 A 路和 B 路的结果拿来对比：Claude 思考："RAGFlow 说是一级防水，但我看本地文档的备注里写着'除配电间外'。这里有矛盾，信息不全。"

第三步：二次回溯（追查到底）

一旦发现矛盾或模糊点，Claude 不会直接回答你，而是触发二次检索：

生成新问题：Claude 自动生成一个更精准的关键词——"配电间 防水等级 特殊要求"。

再查一遍：拿着这个新词，再去 RAGFlow 或本地文档里扫一圈。

第四步：结案陈词

最后，Claude 会告诉你："根据图纸说明，大面是一级防水，但是，根据第 15 页详图备注，配电间区域需加强处理..."

这才是真正的智能。它不是鹦鹉学舌，而是像老工程师一样：先看目录，再看图纸，发现对不上，再去查变更单，最后才敢签字。

// 五、 写在最后

这套"API 自动化 + 清洗分拣 + 双路回溯"的思路，目前我正在把它封装成 Claude Code 的一个 Skill（技能包）。

我现在每天都在和 AI 一起啃 RAGFlow 的 API 文档，试图挖掘更多参数，让 Claude 能更灵活地控制这个"资料员"。这也是抛砖引玉，既然 RAGFlow 底子好，那我们就用更好的思路去用好它，而不是一味抱怨它不准。

最后必须得提一嘴，这种"查一遍-对比-再查一遍"的玩法极其费 Token（算力）。为了测试这个双路逻辑，我让 AI 把几百页的规范反复读、反复洗。得亏我买了 GLM-4.6 的 CODE MAX 套餐，无限 Token 随便造。

那种开着推土机平推 bug、不用心疼油钱的感觉，确实是太爽了。等我把这个 Skill 彻底跑通了，再来给各位汇报！
