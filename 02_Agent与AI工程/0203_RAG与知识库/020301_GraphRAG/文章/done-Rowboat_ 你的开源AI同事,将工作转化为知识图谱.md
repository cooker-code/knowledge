# Rowboat: 你的开源AI同事,将工作转化为知识图谱

在信息爆炸的时代,我们每天都在处理海量的邮件、会议记录和文档。如何有效地管理这些知识,让AI真正成为我们的智能助手,而不是简单的问答机器人?今天给大家介绍一个革命性的开源项目 - **Rowboat**,一个本地优先的AI同事,它不仅能记住你的工作上下文,还能主动帮你完成各种任务。

## 什么是Rowboat?

Rowboat是一个开源的AI同事系统,它的核心理念是**将工作转化为知识图谱并基于此采取行动**。与传统的AI助手不同,Rowboat会构建一个长期存在的知识图谱,积累你的邮件、会议记录和工作笔记中的关键信息,形成一张相互关联的知识网络。

想象一下这样的场景:你即将参加一个重要会议,只需要对Rowboat说"帮我准备与Alex的会议",它就能自动从知识图谱中提取过往决策、未解决问题和相关讨论,生成一份简洁的会议简报。这不是简单的搜索,而是基于深度理解的智能整合。

## 核心特性一览

### 1. 长期知识记忆

Rowboat最独特的地方在于它的**长期知识记忆**能力。它会:

* •从Gmail、Google Calendar、Fireflies.ai、Granola等数据源自动同步信息
* •使用主题提取和实体识别技术构建Obsidian兼容的Markdown知识库
* •维护实体之间的双向链接关系,形成真正的知识图谱
* •所有数据都存储在本地,保证隐私安全

### 2. 智能助手能力

基于构建的知识图谱,Rowboat可以帮你完成各种工作:

* •**生成演示文稿**: "帮我制作下季度路线图演示文稿" → 自动从知识图谱提取信息生成PDF
* •**会议准备**: 自动汇总相关人员的背景信息、历史决策和待办事项
* •**邮件起草**: 基于上下文智能起草邮件回复
* •**语音备忘录**: 录制语音后自动提取关键要点并更新知识图谱

### 3. 可视化的知识图谱

Rowboat提供了一个交互式的知识图谱可视化界面,你可以:

* •查看知识节点之间的关联关系
* •点击节点查看详细的Markdown笔记
* •手动编辑和补充知识内容
* •搜索和过滤特定主题

## 技术架构解析

Rowboat采用现代化的技术栈,基于Electron构建桌面应用,确保跨平台兼容性。

### 系统架构

整个系统分为几个核心模块:

1. 1. **Electron主进程**: 负责窗口管理、系统级服务初始化和IPC通信
2. 2. **渲染进程**: 基于React + Vite构建的现代化UI界面
3. 3. **核心服务层**: 包含知识图谱构建器、Agent运行时、MCP集成等
4. 4. **数据源适配器**: 对接Gmail、Calendar、Fireflies等外部服务
5. 5. **本地存储**: Obsidian Vault格式的Markdown文件 + SQLite状态管理

### 知识图谱构建流程

知识图谱的构建是一个自动化的流水线过程:

1. 1. **数据采集**: 定期从各数据源同步原始内容
2. 2. **内容处理**: 对邮件、会议记录等进行预处理和格式化
3. 3. **实体提取**: 使用AI模型识别人名、项目、决策等关键实体
4. 4. **关系建立**: 分析实体间的关联,生成双向链接的Markdown笔记
5. 5. **索引构建**: 建立知识索引,为AI助手提供快速查询能力
6. 6. **应用消费**: AI助手基于知识图谱提供上下文感知的智能服务

## 本地优先的隐私保护

在当今AI应用泛滥的时代,数据隐私是一个不可忽视的问题。Rowboat坚持**本地优先**原则:

* •所有知识数据存储在本地Markdown文件中
* •使用Obsidian兼容的格式,你可以随时用任何文本编辑器查看和编辑
* •只有在执行AI任务时才与外部模型API通信
* •支持本地LLM部署,实现完全离线的AI能力

## 如何开始使用?

### 安装步骤

1. 1. 从GitHub Releases下载最新版本(支持Mac/Windows/Linux)
2. 2. 配置Google API密钥以连接Gmail和Calendar
3. 3. (可选)配置Deepgram API密钥启用语音备忘录功能
4. 4. (可选)配置Brave或Exa API密钥启用网络搜索功能

### Google服务配置

Rowboat需要访问你的Gmail、Calendar和Drive来构建知识图谱。配置过程包括:

* •在Google Cloud Console创建OAuth 2.0客户端
* •启用Gmail、Calendar和Drive API
* •将客户端ID配置到Rowboat中

详细的配置指南可以参考项目文档中的google-setup.md文件。
