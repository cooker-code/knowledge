## 类目定位

| 项 | 内容 |
|---|---|
| 一级类目 | Agent 与 AI 工程 |
| 二级类目 | Harness Engineering |
| 核心问题 | 如何给 Agent 建一个可执行、可评估、可回滚、可观测的运行时约束层 |
| 不解决什么 | 不承接具体工具教程；不把所有提示词技巧都叫 Harness；不把普通工作流自动化当作 Harness |
| 用户当前认知假设 | L2：知道 Agent 不可靠，当前重点是运行时控制、状态、环境、门禁、评估和交付闭环 |

## 排重准则

问题指纹：

```text
运行时对象 + 控制层机制 + 质量门禁 + 评估/回滚方式 + 适用边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都讲 Harness | 继续比较环境、状态机、评估、监控、回滚、知识沉淀哪个是主问题 |
| 只是工具发布 | 没有运行时机制和质量证据时降为略读 |
| 只是 Prompt 模板 | 归 Prompt Engineering |
| 以 Hermes 为主角 | 归 AI 编程工具 / Hermes |
| 有真实项目闭环 | 优先沉淀为核心知识点候选 |
