## 二级定位

`0209_Harness Engineering` 只承接 Agent 运行控制面的长期知识：环境、状态、工具、权限、上下文装配、评测、回滚、监控、知识沉淀和多角色治理。

不把 Harness 当成“所有 AI 工程文章”的收纳筐。标题出现 Harness、Agent、Claude、LangGraph、Browser、Prompt、Skill、Data Engineering 时，仍按正文主问题判断。

| 项 | 内容 |

## 排重指纹

```text
Harness 对象 + 控制面模块（环境/状态/上下文/工具/权限/评测/知识/编排）+ 核心机制 + 解决的问题 + 可执行约束 + 证据链 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是状态机 | 区分任务级、阶段级、会话级、工具调用级、评审级状态。 |
| 都是环境/沙箱 | 区分文件隔离、进程隔离、网络隔离、凭证隔离、资源配额和日志回放。 |
| 都是工具加载 | 区分 System Prompt、Skill 延迟加载、MCP 声明、代码模式、SubAgent 隔离。 |
| 都是验证 | 区分测试、Lint、Trace、浏览器现场、接口回包、人工审批、独立 Judge。 |
| 具体工具文章 | 主问题是 Claude Code、Codex、Cursor、DeerFlow、LangChain 等工具/框架本体时迁入对应目录；只有抽象出通用控制面机制才留在 0209。 |
| 项目清单/资讯 | 只做生态锚点，不新建知识点；除非改变平台能力边界、选型判断或私域自建边界。 |
