## 核心问题

Agent 上生产前，必须先回答“它在哪里执行、能访问什么、失败后如何复现、凭证如何隔离、资源如何限制”。

## 判断准则

| 模块 | 判断准则 | 反例 |
|---|---|---|
| 环境即代码 | 依赖、工具链、启动命令、代理、浏览器、数据库连接和测试入口可复现 | 只在对话里告诉模型“本地环境大概是这样” |
| 执行隔离 | 读写文件、进程、网络、浏览器、凭证、资源配额有边界 | 把开发机、生产凭证和 Agent 执行直接混在一起 |
| Brain/Runner 分离 | 推理控制面与执行环境分离，执行侧可换、可回放、可审计 | 模型进程直接持有所有执行权限 |
| 真实观察面 | 浏览器、日志、接口回包、控制台、网络请求、截图能作为证据 | 只让模型根据代码猜测结果 |
| 资源限制 | 时间、并发、文件范围、网络范围、成本预算可被系统强制执行 | “希望模型不要乱跑”但无硬限制 |

## 认知偏差

- Docker、Kubernetes、浏览器自动化都只是实现手段；沙箱的本质是文件、进程、网络、凭证、资源和审计边界。
- Browser Use 类文章可作为“真实网页是前端 Agent 的观察面”来源，但主问题若是 GUI/Computer Use，应优先归入 `020201_ComputerUse`。
- DeerFlow、LangChain、DeepAgents 文章如果主问题是框架本体，应该进入 `0201_Agent框架`；在 0209 只抽取运行底座机制。

## 验收清单

- Agent 启动前能自动确认项目类型、包管理器、测试入口和运行命令。
- 每个工具调用都有权限级别、入参约束、输出记录和失败处理。
- 任务执行在可丢弃、可重建的工作目录或容器中运行。
- 浏览器或外部系统操作有页面状态、控制台、网络、截图或接口回包作为证据。
- 高危操作需要审批或白名单，凭证不进入模型上下文。

## 待验证缺口

- 需要补一份“最小 Agent 沙箱”实现对比：容器沙箱、浏览器沙箱、远程执行沙箱、本地受限工作区。
- 需要官方补证各平台对凭证隔离、文件隔离、网络隔离、日志留存的真实边界。
