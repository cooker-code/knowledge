## 用户认知重点

| 项 | 内容 |
|---|---|
| 已知基础 | 知道 context window 有限，了解 token 成本与上下文长度的关系 |

## 排重指纹

```text
上下文管理对象 + 阶段（发现/选择/切分/压缩/缓存/注入/验证）+ 核心机制 + 解决问题 + token成本/精度边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是上下文压缩 | 区分摘要压缩、重要性筛选、结构化提取和递归压缩 |
| 都是 KV Cache | 区分 Prefix Cache、跨请求复用、位置敏感性和显存占用 |
| MCP 协议文章 | 不因 "Context Protocol" 名称归入本目录，按工具调用处理 |
| 只说"减少 token" | 没有具体方法和效果验证时不新建知识点 |
