# 020800_Context Engineering知识地图

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 02_Agent与AI工程 / 0208_Context Engineering |
| 解决的问题 | 让 Agent 在有限上下文窗口中拿到正确、足量、低噪声、可验证的信息 |
| 典型输入 | 用户任务、项目文件、规则文档、检索结果、工具输出、历史记忆、会话摘要 |
| 典型产出 | 上下文装载策略、压缩策略、缓存策略、Chunking 规则、失败诊断信号 |
| 用户阅读目标 | 判断文章补的是上下文发现、切分、压缩、缓存、注入还是验证 |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| C00 任务边界 | 当前任务到底需要哪些信息 | 不先界定任务，就无法判断上下文是否足够 | 只讲“加更多资料”的文章降权 |
| C01 上下文发现 | 从文件、工具、知识库、记忆中找什么 | 上下文发现是检索问题，也是权限问题 | 搜索代理、Context7、文件索引文章进入 |
| C02 切分与选择 | 文档和代码片段怎么切、怎么选 | Chunking 要按语义和任务验收，而不是固定长度 | Chunking、语义切分、选择策略文章进入 |
| C03 压缩与摘要 | 何时压缩，压缩后保留什么 | 压缩必须保留决策、证据、待办和风险 | 会话压缩、摘要、长任务续航文章进入 |
| C04 缓存与稳定前缀 | 哪些上下文稳定，哪些动态 | Prompt Cache 收益来自稳定前缀和动态注入边界 | Prompt Cache、工具定义加载文章进入 |
| C05 注入与回收 | 上下文放进 system、developer、tool 还是 memory | 注入位置决定可见性、优先级和污染风险 | prompt/context 组合、工具上下文文章进入 |
| C06 质量验证 | 如何知道上下文真的帮到模型 | 需要命中率、回答质量、失败复现和成本证据 | 验证 Chunking、长上下文陷阱文章进入 |
