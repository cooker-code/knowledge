Cherry Studio 的所有用户和开发者朋友们，大家好。

我们想和大家分享一个项目的进展：近日，Cherry Studio 被 Coinbase 正式列为其新发布的“Payments MCP”系统的官方推荐客户端之一。在相关文档中，我们与 Anthropic (Claude)、Google (Gemini) 和 OpenAI (Codex) 的工具并列。

这是一个值得我们和社区分享的时刻。我们想借此机会，坦诚地聊聊 Cherry Studio 这一年多来的演进，以及我们对未来的思考。

### 我们的演进：从聚合工具到 Agent 工作台

坦率地说，Cherry Studio 的起点是一个多模型聚合工具。

在2024年项目启动时，我们最初的目标是解决一个非常实际的问题：让用户能在一个地方方便、快捷地切换和使用不同的大语言模型 (LLM)。

但在与社区的深入交流和对 AI 未来的持续探索中，我们逐渐清晰地意识到，对话（Chat）只是 AI 能力的第一层。AI 的真正潜力在于“行动”（Action）。

因此，我们的项目重心发生了战略性的转移，转向了我们非常看好的一个定位：AI 代理工作台 (Agent Workbench) 。

我们开始重构底层架构，不再满足于做一个“聊天壳”，而是为支持更复杂的 AI 代理（Agents）、插件和自动化工作流打下基础。我们相信，未来 AI 的核心价值在于它能作为我们的“代理”，自主地理解、规划并执行任务。

### Payments MCP：一次重要的技术契合

Coinbase 此次发布的 Payments MCP 系统，对我们来说，是行业对我们技术路线的一次重要验证。

该系统基于 MCP (Model ContextProtocol) 和 x402 开放支付标准，其核心是让 AI 代理第一次能够自主地拥有钱包、使用稳定币（USDC）安全地在链上进行支付 。

这意味着 AI 代理将能够自主付费调用 API、购买所需数据或服务资源，这是 AI 迈向“代理商业” (Agentic Commerce) 的关键一步。

正是因为 Cherry Studio 前期在 Agent Workbench 架构上的投入，使得我们能够自然且快速地支持这一新标准。我们很荣幸能与 Anthropic、Google 这样令人尊敬的公司的工具一同被列为首批推荐客户端。

### 我们的思考：AI 代理经济与 Web3

我们认为，这次接入的意义远不止于增加一个“支付功能”。它揭示了一个更宏大的未来图景：AI 代理经济 (Agentic Commerce) 的崛起。

在这个图景中，AI 不再只是工具，它们是能自主执行任务、创造价值、并参与经济流转的“数字实体”。而 Cherry Studio 的定位，就是希望成为这个新时代的 ‘AI 代理经济基础设施’ 之一，为所有开发者和用户提供一个安全、可靠、高效的 Agent Workbench 。

Coinbase 选择用加密钱包和稳定币作为支付手段，这并非偶然。

这预示着 AI 代理经济很可能天生就是 Web3 原生 的。一个无需许可、透明、低摩擦的全球支付网络（即区块链），是未来无数 AI 代理进行高频、自动化微交易的最佳土壤。我们对探索 AI 与 Web3 结合的各种可能性充满期待。

### 写在最后

从一个多模型聊天工具，到今天在 AI 代理支付领域被 Coinbase 这样的行业领导者认可，Cherry Studio 的每一步演进都离不开开源社区的智慧和早期用户的包容。

坦诚说，我们深知这只是一个开始。AI 代理的基础设施建设依然任重道远。

感谢大家的一路同行。我们期待与大家一起，继续构建和见证这个未来。
