一、经典问题

一个推荐经典问题：用户看了更多样的内容，用户的播放时长和留存率会不会提升？

这是一个典型的**因果推断问题**——不是简单的"看多样内容的用户更活跃"（相关），而是"因为看了多样内容，所以第二天更愿意回来"（因果）。

抱着试试心态，拿universal-data-analyst skill 测试，看它是否可解决较为复杂的数科项目，没想到，结果还不错。

二、数据和业务场景简介

**数据集：KuaiRec 是快手发布的公开推荐数据集，包含：****7176** 个用户，**10728** 个视频，1200万+ 条交互记录，时间跨度：2020年7-9月（28天）

**研究假设**

**H1（主效应）：** 用户t日观看视频的类别多样性（Shannon指数），与t+1日播放时长呈正相关

**H2（长期效应）：** t日多样性与t+1至t+7日活跃天数呈正相关

**因果挑战：这里最大的难点是****混淆因素**：高活跃用户本身就更愿意看多样内容，也更愿意第二天继续使用。如何证明是"多样性"带来的效应，而不是"用户本来就是高活跃用户"？

这需要用到**因果推断方法**：固定效应模型、事件研究法等。

三、实例分析对话记录

**第一轮：明确分析需求**

分析KuaiRec 数据集：

1) 先进行探索性分析，了解用户行为特征、内容消费模式、视频类别分布；

2) 重点验证多样性假设：用户t日观看视频的三级类别香农多样性指数，是否与t+1日播放时长呈正相关，以及与t+1至t+7日活跃天数呈正相关。

数据清洗要求：同一用户同一天同一视频累计播放时长超过视频时长则截断为视频时长

**要点**：用业务语言描述，不必懂技术实现。AI会自动翻译为Shannon Entropy计算、pearson/spearman检验等。

第二轮、第三轮：修改HTML报告产出风格（这时候skill还没有完全记住我让它固定的报告风格，后来我又加强了一下）

**第四轮、第五轮：补充专业内容**

因果推断内容，如果有对应的公式，把公式也写在文档中，并且把公式、变量、参数展现样式优化一下。

四、给产品经理的实操建议

1. 你可以通过提问就完成的分析类型

2. 补坑指南

|  |  |
| --- | --- |
| **误区1：一次****prompt****写太长** | **误区2：不敢追问细节** |
| **❌错误：试图在一个消息里描述所有细节** | ❌**错误：AI给什么就接受什么** |
| **✅正确：先出框架，再迭代优化** | ✅**正确：像审外包方案一样审AI输出，不满意就改** |

**3. 恰当的分析流程**

Step 1: 数据理解：上传数据 → 让AI描述字段分布 → 确认数据质量

Step 2: 探索性分析：指定关键维度（用户/内容/时间）→ 生成EDA图表

Step 3: 假设验证：明确假设 → 选择统计方法 → 执行检验

Step 4: 因果推断（如需要）：识别混淆因素 → 选择识别策略 → 稳健性检验

Step 5: 报告输出：生成初版 → 迭代优化格式 

4. 这里给产品经理一些追问技巧：从业务问题出发

❌错误说法："帮我跑个相关性" 

✅正确说法："我想验证推荐多样性对用户留存的影响，用Shannon指数计算多样性，检验与次日播放时长的相关性"

❌错误说法："看看用户活跃度" 

✅正确说法："计算用户单日播放时长分布，识别异常值，分活跃度层级（高/中/低）统计"

**原则**：描述"要什么结果"，而非"用什么方法"。
