使用 Claude 时，有一个习惯比任何其他因素都更浪费时间——反复编写相同的指令。

每次新对话，你都要重新解释自己的角色、描述写作风格、声明格式偏好、粘贴公司背景、指定输出形式。

第二天再来一次，第三天又一次。

一个月下来，你在已经写过的指令上浪费了数小时。没有新思考，没有新需求，只是无休止地重复相同的设置。

Claude Projects 和 Skills 能彻底解决这个问题。

Projects 让你保存一次上下文，然后自动应用到每次对话。 Skills 让你把整套工作流保存为可复用的命令，一句话就能触发。

两者结合， Claude 从"每次从零开始的工具"变成了"已经知道一切、只等你提具体需求的系统"。

下面是从零搭建的完整方法。

## 什么是 Claude Projects

Claude Project 是一个容器，用于存放共享相同上下文的对话。

创建 Project 时，你上传知识文件并编写系统提示词。该 Project 内的每次对话都会自动获得这些文件的访问权限，并遵循那些指令。

无需重新解释、重新粘贴、重新描述。上下文始终在线。

举例：你创建一个名为"内容营销"的 Project ，上传品牌指南、编辑日历、表现最好的文章样本、受众画像。然后写一条系统提示词：

"你是我的内容策略师。你了解我们的品牌声音、受众和内容策略。每篇内容都应符合指南并针对我们定义的受众。"

现在这个 Project 内的每次对话——头脑风暴标题、起草文章、分析竞品——都从完整上下文开始。 Claude 已经知道你的声音、受众和标准。

只需一次设置，无限次对话，零重复。

## 第一步：创建你的第一个 Project

打开 Claude.ai ，侧边栏点击"Projects"，点击"Create Project"。

取一个明确的名称。不要叫"工作资料"，要具体一些："Q3 营销策略""客户提案""产品文档"。

编写系统提示词。这是最重要的部分。包括：角色定义、工作背景、输出标准、针对重复性需求的具体指令。

上传知识文件：
- 品牌指南
- 写作风格指南
- 受众画像文档
- 你最好的内容样本（ 2-5 篇代表质量标准的作品）
- 产品功能文档
- 竞品分析摘要
- 编辑日历

Claude 会在该 Project 每次对话开始时读取这些文件。上传的相关材料越多，输出越符合你的需求。

## 第二步：构建你的 Project 库

一个 Project 有用，一个 Project 库就是系统。

**内容创作**：品牌指南、风格指南、受众画像、最佳内容样本。用于任何内容相关任务。

**客户沟通**：客户档案、过往提案、邮件模板、定价信息。用于起草邮件、提案和客户材料。

**研究与分析**：行业报告、竞品数据、市场调研。用于分析任务、趋势报告和战略建议。

**会议准备**：会议议程模板、关键利益相关者简介、项目简报。用于准备谈话要点、演示和后续跟进。

**个人发展**：职业目标、学习笔记、技能评估。用于职业规划、学习计划和自我提升。

每个 Project 设置需要 15-20 分钟。时间节省立即开始，随每次对话复利增长。

## 第三步：理解 Claude Skills

Skill 是一个可复用的工作流， Claude 可以按需执行。

如果说 Project 是上下文（ Claude 是谁、它知道什么），那么 Skill 就是能力（你要求时 Claude 能做什么）。

创建 Skill 的方法是编写一条详细的提示词，定义：
- Skill 做什么
- 需要你提供什么输入
- 遵循什么步骤
- 输出是什么样子
- 应满足什么质量标准

然后保存它。现在你可以用一句话触发这个 Skill ，而不用每次都写完整提示词。

## 第四步：构建你的前五个 Skills

**Skill 1 ：会议纪要生成器**

触发时，你粘贴原始会议笔记。 Skill 提取关键决策、列出所有行动项（含负责人和截止日期）、注明未解决的问题、写一段 3 句话的执行摘要、用干净的 Markdown 格式化所有内容。 500 字以内，行动项用项目符号，负责人加粗。

不再手动写总结，你只需粘贴笔记并说"运行会议纪要"。

**Skill 2 ：邮件起草器**

触发时，你描述情况和收件人。 Skill 用你的品牌声音撰写专业邮件， 200 字以内，包含清晰的主题行，以具体的 CTA 结尾，根据收件人匹配正式程度。提供两个版本：直接版和柔和版。

**Skill 3 ：内容重组器**

触发时，你粘贴一篇长文。 Skill 生成： 5 条独立的 X 帖子（每条都有强力钩子）、 1 篇总结关键洞察的 LinkedIn 帖子、 3 条邮件主题行、 1 段 2 句话的 Slack 摘要。每条内容独立成文。

**Skill 4 ：提案构建器**

触发时，你描述客户、项目和预算范围。 Skill 生成结构化提案：执行摘要、问题陈述、建议方案（含交付物）、时间线（含里程碑）、投资（含定价）、下一步行动。 2 页以内，用标题便于扫读。

**Skill 5 ：周报生成器**

触发时，你粘贴原始周记。 Skill 生成：分类的成就摘要、进行中的项目状态、关键学习、下周的优先事项清单、需要关注的风险或障碍。一页。

## 第五步：结合 Projects 和 Skills

真正的威力来自 Projects 和 Skills 的结合。

你的"内容创作"Project 拥有所有上下文——品牌声音、受众、风格指南、样本。你的"内容重组器"Skill 拥有工作流——生成什么、如何格式化。

当你在"内容创作"Project 内触发"内容重组器"Skill 时， Claude 同时拥有上下文和指令。它既知道你的品牌声音，又知道确切的输出格式。输出不是泛化的——它完美贴合你的标准。

这种结合彻底消除了重复性工作。你设置一次上下文（ Projects ），设置一次工作流（ Skills ），然后随时触发它们。

## 实际使用是什么样子

周一早上。你打开 Claude ，进入"内容创作"Project ，粘贴上周的博客文章，说"运行内容重组"。

60 秒内，你得到 5 条 X 帖子、 1 篇 LinkedIn 帖子、 3 条邮件主题行、 1 段 Slack 摘要。全部符合你的品牌声音。全部可以直接发布。

下午，客户发来一封棘手的邮件。你进入"客户沟通"Project ，描述情况，说"起草回复"。 Claude 写出两个版本，你选一个，编辑 30 秒，发送。

周五，你需要写周报。打开"个人发展"Project ，粘贴周记，说"生成周报"。一分钟后得到结构化总结，准备发给老板。

每个任务，从 30 分钟降到 2 分钟。质量不变，速度提升 15 倍。

## Projects 和 Skills 对比普通对话

普通对话每次都从零开始。你粘贴说明、提供背景、请求输出。 Claude 不记得昨天的内容，不知道你的风格，不了解你的标准。输出是通用的。

Projects 和 Skills 版本的 Claude 已经知道一切。你只提供具体请求——这次的主题、这次的客户、这次的文章。其他一切都已就位。输出看起来像你自己写的。

## 没人谈论的系统

大多数 AI 建议关注单个提示词。"用这个提示词写更好的邮件""试试这个提示词获取内容点子"。

那就像给人一根钓竿却不配钓具箱。一个工具，用一次，产出一个结果。

Projects 和 Skills 是钓具箱。它们是让每次交互更快、更一致、更高质量的系统。

2026 年从 Claude 获得最大价值的人，不是写出最好单个提示词的人，而是构建了最好系统的人。

今天就设置你的第一个 Project 。本周构建你的第一个 Skill 。到下周一，你将拥有一个自动处理重复性工作的系统——你再也不会写两次相同的提示词。
