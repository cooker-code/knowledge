> 不用每次跟 AI 重复「按我们公司的文档规范来」——把规范装进技能包，一次配置，长期生效。

---

你是不是有过这种经历：让 AI 写一份 API 文档或技术说明，结果格式每次都不一样，要么缺章节，要么风格对不上团队规范，只好自己再改一遍。Document Skills（文档类技能）要解决的，就是这个问题：把「怎么写好文档」变成 AI 可复用、可版本管理的能力，让输出既通俗易懂，又保持专业度。

---

# 一、Document Skills 到底是什么？

用一句话说：Document Skills 是专门教 AI「怎么写文档」的技能包。它跑在 Cursor 等支持Agent Skills的编辑器里，把写 API 文档、技术指南、产品说明、会议纪要等能力，打包成一个个「技能」：每个技能里既有说明（AI 什么时候用、按什么结构写），也可以带脚本、模板和参考文档，AI 按说明执行，你只管提需求。

可以把它理解成「给 AI 装的文档插件」：你装的是「技术文档插件」还是「产品发布说明插件」，AI 就按哪套规范来写，不用你每次口述一遍格式和结构。

---

# 二、为什么需要它？

写文档最烦的两件事，一是格式不统一——今天一个样明天另一个样；二是规范说不清——团队有模板和规范，但很难每次都对 AI 讲完整。Document Skills 把这两点都解决了：

格式统一：技能包里写死了章节结构、标题层级、表格和代码块规范，AI 每次都用同一套，输出风格一致。

规范可沉淀：你们团队的文档规范、示例模板，可以写进SKILL.md和references/，新人、新项目都能复用，不用反复口头传达。

对个人来说，省掉的是「重复说格式」的时间；对团队来说，得到的是「文档即规范」——文档怎么写，规范就在技能包里，可追溯、可迭代。

---

# 三、和「随便让 AI 写文档」有啥区别？

| 对比项 | 普通让 AI 写文档 | 使用 Document Skills |
| --- | --- | --- |
| 规范与模板 | 每次都要口头说明格式、结构 | 技能里已写好，一次配置长期生效 |
| 一致性 | 容易每次风格不一 | 同一技能下输出风格统一 |
| 可复用 | 难以在团队或项目间复用 | 技能包可放进仓库或从 GitHub 安装，多人共用 |
| 触发方式 | 只能靠你主动说「写文档」 | 可自动识别场景，也可用`/`手动唤起 |

换句话说：普通用法是「每次现教」，Document Skills 是「一次教好，到处用」。适合文档需求多、对格式和规范有要求的个人和团队。

---

# 四、它是怎么运作的？

Document Skills 基于Agent Skills这套开放标准，和 Cursor 自带的技能体系是一套东西，有四个特点：

可移植：只要支持 Agent Skills 的 AI 或工具，都能用同一份技能包，不锁死在某个产品里。

版本可控：技能就是一堆文件（文件夹 +SKILL.md+ 可选的脚本和参考），可以放进 Git，谁改了什么、什么时候改的，一清二楚。

可执行：技能里不仅能写「怎么写」的说明，还能放脚本、模板，AI 会按说明去执行脚本或引用资源，而不只是「说说而已」。

按需加载：详细说明、长文档可以放在references/里，AI 用到再读，不占满上下文，响应更快。

每个技能就是一个文件夹，里面至少有一个SKILL.md。这个文件用 YAML 头写清楚技能的名字和简介，正文里写「什么时候用、按什么步骤写、注意什么」。AI 会根据你当前在做的事和对话内容，自动判断要不要用这个技能；你也可以在输入框里输入/技能名主动指定用哪个文档技能。

# 五、怎么安装和使用？

这是目前讨论度最高的用法：把 Document Skills 当作Claude Code 的插件，主要面向个人开发者或办公用户，让 AI 直接读取、编辑 PDF、Word、Excel、PPT 等文档。核心就是三步：安装 → 对话 → 完成。

1. 安装步骤

在 Claude Code 环境中执行：

```
/pluginmarketplaceaddanthropics/skills/plugininstalldocument-skills@anthropic-agent-skills
```

安装完成后，document-skills 包里会包含处理PDF、DOCX、PPTX、XLSX等格式的能力，无需再单独配置。

2. 使用方法（核心）

安装后完全不需要记忆复杂命令，有三种方式可以调用：

方式一：自然语言自动触发（最常用）

像平常一样提需求，Claude 会自动判断是否需要调用技能。例如：

「读取这个 Excel 文件的内容」

「帮我总结一下这篇 PDF 的要点」

「基于这份会议记录 PPT，生成一份详细的 Word 文档」

方式二：斜杠命令显式调用（精确控制）

明确想用某个功能时，可以输入斜杠命令：

```
/document-skills:pdf
```

帮我合并这两个文档

```
/document-skills:xlsx
```

把这个表格转换成 Markdown 格式

```
/document-skills:pptx
```

给所有幻灯片套用商务主题

方式三：编写 Agent / Command（批量自动化）

如果有高频重复任务（比如每周分析简历），可以在.claude/agents/目录下配置好 Agent。之后只需输入/resume-analyze 李明\_简历.pdf，就会自动完成读取简历、提取信息、填充模板等一系列操作。

3. 实际场景演示

假设你需要处理一份年终总结 PPT：

直接对 Claude 说：「用 document-skills 的 pptx 模块读取我的 2025总结.PPTX，提取核心成果和数据，生成一份 Word 年终总结。」

Claude 自动加载 pptx 技能，读取文件内容。

几秒钟后，Claude 直接输出一份结构清晰的 Word 格式总结。

全程不需要记命令，说清楚「读什么、要什么」即可。

# 六、小结

Document Skills把「写好文档」从「每次口头教 AI」变成「一次配置、长期复用」；作为Claude Code 的插件使用时，又能让 AI 直接读、编 PDF、Word、Excel、PPT，安装后只需用自然语言或斜杠命令说出需求即可完成。无论是写规范文档，还是日常处理 Office 与 PDF，都可以从安装 document-skills 开始，用「安装 → 对话 → 完成」三步体验个人提效。
