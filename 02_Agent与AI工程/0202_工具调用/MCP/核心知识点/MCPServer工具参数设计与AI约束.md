## 纵向理解

| 维度 | 判断 |
|---|---|
| 全局架构 | Agent/MCP Client 读取工具定义，模型选择工具并生成参数，MCP Server 校验和执行 |
| 本文位置 | 只讲 Tools 参数设计和模型约束 |
| 核心机制 | 最小参数、枚举、白名单、JSON Schema、分阶段提取、默认值 |
| 使用链路 | 定义工具 -> 限定 Schema -> 模型提参 -> Server 校验 -> 执行或拒绝 |
| 前置条件 | 工具边界清楚、危险操作有权限策略、返回结构稳定 |
| 边界 | 参数设计不能替代鉴权、审计、限流和人工审批 |

## 横向对标

| 对标技术 | 实现方式 | 优势 | 劣势 | 适合场景 |
|---|---|---|---|---|
| 宽参数 MCP Tool | 暴露大量筛选字段 | 看似完整 | 模型容易臆造参数 | 不建议默认使用 |
| 最小参数 MCP Tool | 只暴露必要字段，其他由服务端默认或二次过滤 | 稳定、可控 | 需要更多服务端逻辑 | 面向 LLM 的工具设计 |
| Function Calling Schema | 模型 API 侧工具定义 | 简单直接 | 跨客户端复用弱 | 单应用工具调用 |
| REST API | 传统参数接口 | 工程成熟 | 不面向模型理解和选择 | 后端服务接口 |
