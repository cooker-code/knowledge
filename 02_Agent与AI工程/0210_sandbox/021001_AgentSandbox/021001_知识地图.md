# 021001_AgentSandbox 知识地图

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 02_Agent与AI工程 / 0210_sandbox / 021001_AgentSandbox |
| 解决的问题 | Agent 执行代码、浏览器、文件、命令和网络访问时，如何把副作用放进可隔离、可回收、可审计的环境 |
| 典型输入 | Agent 任务、代码片段、网页自动化任务、文件工作区、短期凭证、网络访问请求 |
| 典型产出 | 沙箱实例、执行结果、输出文件、审计日志、回收事件、异常信号 |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| S00 威胁边界 | 这次 Agent 执行会触达哪些副作用 | 先定义隔离对象，再选运行时 | 只有能说明风险对象和边界的文章进入 |
| S01 运行时选型 | 用本地沙箱、Docker、K8s、微虚拟机还是浏览器沙箱 | 运行时决定隔离强度、成本和可运维性 | 产品资讯必须能落到选型维度才保留 |
| S02 环境生命周期 | 沙箱如何创建、复用、快照、超时和销毁 | 长任务和并发需要明确生命周期 | 只讲安装命令的文章降权 |
| S03 文件/网络边界 | 工作区、挂载、出站、代理和内网怎么限制 | 文件与网络是最常见外流面 | 有配置、默认值、失败模式时吸收 |
| S04 凭证与审计 | 凭证如何不进入模型可写环境，执行如何可复盘 | 隔离必须配合短期凭证和结构化日志 | 有审计字段、权限分层或密钥代理时吸收 |

## 待补缺口

| 流程节点 | 缺什么 | 后续文章进入时怎么判断 |
|---|---|---|
| S00 威胁边界 | 缺少按任务类型划分的威胁矩阵 | 看文章是否明确区分代码执行、浏览器操作、文件写入、网络访问和外部 API |
| S01 运行时选型 | 缺少官方、同口径的性能和隔离能力对比 | 看文章是否提供可复验的冷启动、并发、网络和文件策略证据 |
| S02 环境生命周期 | 缺少多 Agent 并发下的池化、回收和配额策略 | 看文章是否讲实例池、空闲超时、快照、资源回收和残留清理 |
| S03 文件/网络边界 | 缺少宿主敏感路径和内网元数据服务防护清单 | 看文章是否有默认 deny、白名单、代理和挂载配置 |
| S04 凭证与审计 | 缺少结构化审计字段和凭证代理实践 | 看文章是否给出日志字段、密钥代理、短期凭证和失败回滚路径 |

## 使用方式

- 读 Sandbox 文章时先问：它讲的是执行环境隔离，还是泛泛安全治理。
- 如果文章只讲 Prompt 注入、Skill 审查、供应链投毒、RBAC/ABAC 或合同审查，不进入本节点。
- 如果文章只给安装命令，不讲隔离边界和验收方式，只作为低价值教程处理。
