> 本文件只维护 Sandbox 领域的认知重点、排重指纹、已覆盖三级节点和待补缺口。

## 三级节点入口

| 节点 | 用途 |
|---|---|
| [021001_AgentSandbox](021001_AgentSandbox/AGENTS.md) | Agent 执行环境隔离、代码/浏览器/文件/网络沙箱、凭证隔离、审计和运行时选型 |

## 用户认知重点

| 项 | 内容 |
|---|---|
| 已知基础 | 用户知道高权限 Agent 调用工具、执行命令和访问文件存在风险 |
| 待补边界 | Sandbox 是执行环境隔离，不等同于 Prompt 安全、Skill 审查、通用 RBAC/ABAC 或供应链安全 |
| 易偏差点 | 文章常把“安全”“权限”“沙箱”“Agent 平台”混用，容易把风险治理大框架误写成 Sandbox 本体 |
| 优先抽取 | 隔离对象、运行时形态、文件/网络/进程/凭证边界、生命周期、审计字段、降级策略和成本 |

## 排重指纹

```text
Sandbox 对象 + 执行负载 + 隔离维度 + 运行时形态 + 生命周期控制 + 可验证边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是 Agent 沙箱 | 继续区分代码执行、浏览器执行、桌面/GUI、文件工作区、网络出站和多 Agent 隔离 |
| 都是 Docker/K8s | 继续比较生命周期、资源限制、网络策略、挂载策略、冷启动和观测能力 |
| 都是安全文章 | 只有讨论执行环境隔离时进入本目录；Prompt 注入、供应链投毒、通用权限模型迁出或删除 |
| 只有产品清单 | 没有隔离机制、边界、失败模式或选型准则时降权，不新建知识点 |
| 有可运行配置 | 只有同时说明输入、输出、边界和验证方式时才标为实践候选 |

## 已覆盖问题（按三级节点）

| 节点 | 已覆盖 | 还缺 |
|---|---|---|
| AgentSandbox | 沙箱威胁边界、控制面分层、Docker/K8s/微虚拟机/浏览器运行时、凭证与网络隔离、审计和选型准则 | 官方能力补证、真实冷启动/资源成本基准、浏览器沙箱的登录态边界、多租户配额与回收策略 |

## 待补缺口

| 优先级 | 项 | 为什么补 |
|---|---|---|
| 高 | 官方能力矩阵 | OpenSandbox、AgentRun、CubeSandbox、BoxLite、Codex sandbox 等文章多为二手资料，需要官方文档校准 |
| 高 | 浏览器 Sandbox 边界 | BrowserUse/Computer Use 会触达登录态和网页数据，隔离边界不能只看容器 |
| 中 | 多 Agent 并发隔离 | 多沙箱生命周期、资源配额、会话复用和审计聚合还缺实践证据 |
| 中 | 成本和性能基线 | 冷启动、并发、镜像体积、网络策略和日志采集会影响落地选型 |
