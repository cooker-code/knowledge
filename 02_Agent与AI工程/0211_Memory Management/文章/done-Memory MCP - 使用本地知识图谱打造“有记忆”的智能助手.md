# Memory MCP - 使用本地知识图谱打造“有记忆”的智能助手

众所周知，大模型没有**没有长期记忆**，每一次对话对它们来说都是全新开始。最近发现一款 MCP - Memory MCP（https://github.com/modelcontextprotocol/servers/blob/main/src/memory/README.md），它通过\*\*本地知识图谱\*\*，记住关于你的偏好信息。更重要的是，这些\*\*记忆\*\*存储在本地，保证了隐私和安全。

## 什么是知识图谱

知识图谱（Knowledge Graph）是一种结构化的语义知识库，用于描述现实世界中的实体（如人、地点、事物、概念等）及其之间的关系。它以图的形式组织知识：节点表示实体或概念，边表示实体之间的语义关系。

核心组成：

1. 1. **实体（Entities）**：现实世界中的对象或概念，例如“孙悟空”、“唐僧”、“白龙马”。
2. 2. **关系（Relations）**：连接实体的语义联系，例如“孙悟空—师傅—唐僧”、“孙悟空—搭档—白龙马”。
3. 3. **属性（Attributes）**：实体自身的特征信息，例如“孙悟空—武器—金箍棒”。在 Memory MCP 中称之为 **观察 (Observation)**。

## 快速开始

### 安装

```
{
  "mcpServers":{
    "memory":{
      "command":"npx",
      "args":[
        "-y",
        "@modelcontextprotocol/server-memory"
      ]
    }
}
}
```

### 系统提示词

官方 README 给了一个提示词，这里我翻译为中文：

```
请按照以下步骤进行每次交互：

1. 用户识别：
   - 您应该假设您正在与 default_user 交互
   - 如果您还没有识别 default_user，请主动尝试识别

2. 记忆检索：
   - 始终以只说"正在记忆..."开始您的聊天，并从您的知识图谱中检索所有相关信息
   - 始终将您的知识图谱称为您的"记忆"

3. 记忆
   - 与用户交谈时，请注意任何属于以下类别的新信息：
     a) 基本身份 (年龄、性别、位置、职位、教育水平等)
     b) 行为 (兴趣、习惯等)
     c) 偏好 (沟通风格、首选语言等)
     d) 目标 (目标、指标、愿望等)
     e) 关系 (个人和职业关系，最多 3 度分离)

4. 记忆更新：
   - 如果在交互过程中收集了任何新信息，请按以下方式更新您的记忆：
     a) 为重复出现的组织、人员和重要事件创建实体
     b) 使用关系将它们连接到当前实体
     c) 将关于它们的事实存储为观察
```

将这段提示词作为全局提示，比如 `.clinerules/memory.md`。

### 使用

后续在和模型的聊天过程中，模型会从对话中识别提示词中提到的“基本身份、行为、偏好、目标、关系”等内容，进行记忆和检索。在问答中不断地收集你的行为和偏好，慢慢的，你可能获得一个“数字分身”。

上述提示词不适合编码场景的话，如果需要用于编码场景，更新一下3. 记忆这一部分，比如前端开发，尝试记录 **代码片段、工具函数、编码习惯**等等，这样无需每次更新 Rule 就能获得全局的编码辅助，或者干脆手动记录一些重要信息。

## 总结

Memory MCP 实现上也很精悍，只有一个 TS 文件，很轻量。记忆内容完全实现在本地，在保护隐私的同时，也让模型有了“记忆”，聊的越多 AI 越懂你。

另外，本地文件可以拷贝可以复制，多个设备多个 Agent 都可以使用这份记忆。再次体现了**一切皆文件**的重要性。
