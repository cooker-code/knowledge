# 021100_Memory Management知识地图

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 02_Agent与AI工程 / 0211_Memory Management |
| 解决的问题 | 管理 Agent 的短期上下文、长期事实、用户偏好、经验教训和任务状态，让记忆可写入、可检索、可验证、可遗忘 |
| 典型输入 | 对话历史、任务日志、工具结果、用户偏好、项目规则、失败复盘、知识库条目 |
| 典型产出 | 记忆分类、写入策略、检索策略、压缩策略、污染处理、遗忘机制和评估指标 |
| 用户阅读目标 | 判断文章补的是记忆类型、写入、存储、检索、注入、压缩、遗忘还是评估 |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| M00 记忆分类 | 这条信息属于事实、偏好、经验、任务状态还是技能 | 不分类就无法判断写入和遗忘 | 记忆类型综述、工具记忆文章进入 |
| M01 写入触发 | 什么时候应该记住 | 写入门槛要高于聊天记录保存 | 自动沉淀、Learning Loop、MEMORY.md 文章进入 |
| M02 存储与索引 | 记忆放在哪里、用什么结构查 | 存储底座服务生命周期，不等于记忆本身 | Memori、LanceDB、Cognee、Memory MCP 文章进入 |
| M03 压缩与归档 | 长期任务如何压缩历史 | 压缩要保留决策、证据、错误和待办 | 记忆压缩、Subagent 记忆过载文章进入 |
| M04 检索与注入 | 什么时候把记忆放回上下文 | 记忆检索要受任务相关性和权限约束 | system_prompt、四层记忆、工具记忆文章进入 |
| M05 污染与遗忘 | 错误、过期、隐私信息如何处理 | 记忆必须能删除、降权和标记置信度 | 记忆治理、隐私和安全文章进入 |
| M06 评估 | 如何证明记忆有用 | 看任务成功率、重复错误减少和检索命中 | 评估、对照实验文章进入 |
