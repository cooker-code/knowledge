## 用户认知重点

| 项 | 内容 |
|---|---|
| 已知基础 | 知道 Agent 需要在对话间保留信息，了解向量存储可以用作记忆 |
| 待补边界 | 短期/长期/外部记忆的分层设计；记忆污染和遗忘机制 |
| 易偏差点 | 把 RAG 当作 Memory；把 context window 管理当作 Memory Management |
| 优先抽取 | 记忆类型分层、写入时机、检索策略、污染检测和版本管理 |

## 排重指纹

```text
记忆类型 + 生命周期阶段（写入/存储/检索/压缩/遗忘/评估）+ 核心机制 + 解决问题 + 准确性/污染/成本边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是向量存储 | 区分作为记忆存储 vs 作为 RAG 知识库的边界 |
| 都是记忆压缩 | 区分摘要式、键值式、实体式压缩的适用场景 |
| 普通 RAG 建库 | 主问题是知识检索而非 Agent 跨会话记忆时，归入 `0203_RAG与知识库` |
| 记忆生命周期类文章 | 从 RAG 目录迁入本目录 |
