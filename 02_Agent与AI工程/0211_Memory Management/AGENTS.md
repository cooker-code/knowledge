## 类目定位

| 项 | 内容 |
|---|---|
| 一级类目 | Agent 与 AI 工程 |
| 二级类目 | Memory Management |
| 核心问题 | Agent 如何采集、压缩、归档、检索、更新、遗忘和验证长期记忆、短期记忆、经验记忆与任务状态 |
| 不解决什么 | 不承接操作系统/数据库内存；不把普通 RAG 建库都叫 Memory；不把具体工具的记忆功能和通用记忆机制混为一谈 |
| 用户当前认知假设 | L2：知道长期记忆能提升连续任务，但需要补记忆类型、生命周期、污染、评估和治理 |

## 排重准则

问题指纹：

```text
记忆类型 + 写入触发 + 存储/压缩机制 + 检索/注入方式 + 污染治理 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都讲长期记忆 | 比较事实、偏好、经验、程序性记忆、任务状态是否不同 |
| 都讲向量库 | 如果主问题是索引和召回，归 RAG；如果主问题是记忆生命周期，留在本目录 |
| 都讲工具记忆 | 如果以 Hermes/Claude Code 等具体工具为主角，要判断是否能抽象成通用机制 |
| 只有“越用越聪明” | 没有写入、检索、评估和遗忘机制时降权 |
| 有记忆污染案例 | 优先沉淀为认知校准点 |
