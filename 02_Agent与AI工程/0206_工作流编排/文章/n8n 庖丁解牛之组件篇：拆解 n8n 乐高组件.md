# n8n 庖丁解牛之组件篇：拆解 n8n 乐高组件

大家好，我是 Asher 同学，

上篇文章我们介绍了 n8n 的界面布局，这篇文章介绍 n8n 的核心节点及其功能。

n8n 的工作流（workflow）由两种核心节点组成：

1， 触发节点 (Trigger Nodes)：这是工作流的起点，决定了工作流何时以及如何被激活。一个工作流有且只有一个触发节点。

2， 动作节点 (Action Nodes)：在触发节点被激活后，执行具体任务的节点，比如处理数据、发送请求、调用 AI 模型等。一个工作流可以包含任意多个动作节点。

## 第一部分：所有 n8n 节点的思维导图

在开始详细介绍之前，先来看一张全景图。这张思维导图展示了 n8n 的所有核心节点类型，帮你快速建立整体认知。

主要分为两大类型：

1， 8 种触发节点

2， 6 种动作节点

有了这张图，你就知道 n8n 能做什么、不能做什么了。接下来我们逐个拆解。

## 第二部分：8 种触发节点详细介绍

### 脑图

### 1，手动触发 - 最简单的开始按钮

**一句话理解：** 就像遥控器上的播放键，想用就按一下。

**典型场景：** 测试工作流、临时执行任务。

**小贴士：** 一个工作流只能有一个触发节点哦，就像一个遥控器只有一个开关。

### 2，App 事件 - 应用的"小侦探"

**一句话理解：** 盯着 GitHub、Gmail、Slack 等应用，一有风吹草动就立刻行动。

**典型场景：**

* • GitHub 有新 PR → 自动发 Slack 通知
* • Gmail 收到特定邮件 → 自动保存附件到网盘
* • Google Sheets 数据更新 → 触发数据处理

**小贴士：** 需要先给 n8n "钥匙"（API 认证），才能监听对应应用。

### 3，定时任务 - 准时的闹钟

**一句话理解：** 到点就执行，可以是每天早上 9 点，也可以是每隔 5 分钟。

**典型场景：**

* • 每天早上 8 点生成昨日报表
* • 每小时检查一次服务器状态
* • 每月 1 号自动备份数据

**小贴士：** 支持 Cron 表达式，想怎么定时就怎么定时。

### 4，Webhook - 你的专属 API 接口

**一句话理解：** n8n 给你一个 URL，别人往这个 URL 发请求，工作流就启动。

**典型场景：**

* • 用户支付成功 → 支付平台回调你的 Webhook → 发货
* • 传感器检测到异常 → POST 数据到 Webhook → 发送告警

**小贴士：** 可以设置密码保护，防止被恶意调用。

### 5，表单提交 - 秒变问卷大师

**一句话理解：** n8n 帮你生成一个网页表单，用户填完提交就触发工作流。

**典型场景：**

* • 活动报名表 → 自动发确认邮件
* • 意见反馈表 → 自动记录到数据库
* • 客户咨询表 → 自动创建工单

**小贴士：** 表单样式可定制，支持多种字段类型。

### 6，子工作流触发 - 工作流界的"函数调用"

**一句话理解：** A 工作流可以调用 B 工作流，实现模块化复用。

**典型场景：**

* • 多个工作流都需要"数据清洗"这个步骤 → 做成子工作流，谁需要谁调用

**小贴士：** 可以互相传数据，就像函数有参数和返回值。

### 7，聊天消息 - 打造你的 AI 助手

**一句话理解：** 生成一个聊天界面，用户发消息就触发工作流，通常接 AI Agent。

**典型场景：**

* • 智能客服机器人
* • 私人 AI 助理
* • 知识库问答系统

**小贴士：** 支持上下文记忆，可以"记住"之前聊过什么。

### 8， Other ways...

除了以上常见触发方式，n8n 还支持其他特殊触发方式：

* • **Email Trigger（邮件触发）：** 通过 IMAP 监听邮箱，收到邮件时触发
* • **File Trigger（文件触发）：** 监听文件夹变化，有新文件时触发

这些触发方式覆盖了从简单的手动执行到复杂的事件驱动自动化的各种场景，让 n8n 能够适应不同的业务需求。

## 第三部分：6 种动作节点详细介绍

### 脑图

### 1，AI - 让工作流变聪明

**核心能力：** 调用各种 AI 模型（GPT、Claude、Gemini 等）处理智能任务。

**能做什么：**

* • 智能客服机器人
* • 文档自动总结
* • 数据智能分析
* • 内容创作助手

**明星节点：** AI Agent（会自己思考和规划的智能体）、Basic LLM（调用大模型）、Vector Store（构建 RAG 应用）

### 2，应用操作 - 500+ 应用随便连

**核心能力：** 连接第三方应用，让它们互相配合工作。

**热门组合：**

* • Gmail + Notion：新邮件自动存到数据库
* • GitHub + Slack：代码更新自动通知团队
* • Google Sheets + 微信：表格更新推送到企业微信

**支持应用：** Google 全家桶、微软全家桶、Slack、GitHub、数据库等 500+ 个应用

### 3，数据转换 - 数据处理的瑞士军刀

**核心能力：** 清洗、转换、格式化数据。

**常用操作：**

* • 修改字段值（Edit Fields）
* • 写代码处理复杂逻辑（Code 节点支持 JS/Python）
* • 日期时间处理
* • 数组操作（拆分、合并、去重）

**典型场景：** API 返回的数据太乱？用这些节点整理成你想要的格式。

### 4，流程控制 - 工作流的交通指挥

**核心能力：** 控制数据流向，实现复杂业务逻辑。

**核心节点：**

* • **If：** 满足条件走 A 路，不满足走 B 路
* • **Switch：** 多条分支，根据不同值走不同路
* • **Loop：** 循环处理每条数据
* • **Merge：** 把分叉的路合并回来

**典型场景：** 订单金额 > 1000 元走人工审核，≤ 1000 元自动发货

### 5，核心功能 - 基础但强大

**核心能力：** 提供工作流的基础功能，是构建自动化的基石。

**核心节点：**

* • **HTTP Request：** 调用任何 API 的万能节点，没有专门节点的应用就用它
* • **Execute Workflow：** 调用其他工作流，实现模块化

**典型场景：**

* • 调用公司内部 API 获取数据
* • 调用第三方服务（天气、汇率、地图等）
* • 把通用逻辑封装成子工作流，多处复用

### 6，人工干预 - 关键时刻需要人

**核心能力：** 在自动化流程中加入人工审核。

**典型场景：**

* • 报销申请：自动收集材料 → 等待领导审批 → 审批通过自动打款
* • 内容发布：AI 生成文章 → 人工审核润色 → 自动发布

**为什么需要：** 有些决策机器做不了，必须人来拍板。

## 内容小结

看完这篇，你应该明白了：**n8n 就像一盒乐高积木**，触发节点是"启动开关"，动作节点是"功能模块"。

**8 种触发方式，总有一款适合你：**

* • **手动触发：** 最简单，点一下就跑
* • **App 事件：** 盯着第三方应用，有动静就行动
* • **定时任务：** 到点就干活，准时得很
* • **Webhook：** 给你一个 URL，别人来敲门
* • **表单提交：** 做问卷、做报名，分分钟的事
* • **子工作流：** 模块化复用，程序员最爱
* • **聊天消息：** 打造 AI 助手
* • **其他方式：** Email、File 等特殊需求

**6 类动作节点，搭建自动化的必备工具：**

1. 1. **AI：** 让工作流会思考
2. 2. **应用操作：** 500+ 应用随便连
3. 3. **数据转换：** 数据处理的瑞士军刀
4. 4. **流程控制：** If、Switch、Loop，想怎么跳转就怎么跳
5. 5. **核心功能：** HTTP Request 走天下，没有连不上的 API
6. 6. **人工干预：** 关键时刻，人说了算

**核心记忆点：**

* • 1 个工作流 = 1 个触发节点 + N 个动作节点
* • 节点就是乐高积木，简单模块组合出复杂功能
* • 从"什么时候启动"到"启动后干什么"，这两个问题想清楚了，工作流就成了
