本节探讨RAG。

RAG (Retrieval Augmented Generation)

在大语言模型（LLMs）的落地应用过程中，领域知识储备不足、信息输出准确性欠佳以及虚假内容生成等问题，始终是亟待攻克的核心挑战。而 RAG 通过整合外部知识库等额外信息源，为这些痛点提供了切实可行的缓解路径。尤其在知识需持续迭代的知识密集型场景，或是特定领域的应用中，RAG 的优势更为突出 —— 相较于其他解决方案，它无需针对具体任务对大语言模型进行重新训练，极大降低了应用成本。近年来，凭借在对话助手等场景中的出色表现，RAG 迅速走红并获得广泛关注。

本文将聚焦于最新的调查研究成果 ——《Retrieval-augmented generation for large language models: A survey.》（Gao et al., 2023），系统梳理其中的核心发现与实用见解。内容将重点围绕 RAG 系统的关键组成部分展开，具体涵盖现有技术方法、前沿技术成果、主流评估手段、典型应用案例，以及检索、生成、增强三大核心模块的相关技术细节。

RAG概念及定义

究竟什么是RAG？简单来说，它能将用户的输入内容与一组相关的支撑文档相结合 —— 这些文档通常来自维基百科等权威信息源，随后系统会把这些文档作为补充内容加入原始输入提示词中，一同送入文本生成器，最终产出精准输出。而它的工作原理，可以通过与传统大语言模型（LLM）的对比来理解：没有 RAG 时，LLM 仅依据自身训练数据（即已知信息）对用户输入做出响应；有了 RAG 后，系统会先通过信息检索组件，利用用户输入从新数据源中提取信息，再将用户查询与相关信息一同提供给 LLM，让其结合新知识与训练数据生成更优响应。

具体来看，这一过程包含几个关键步骤。

首先是创建外部数据，也就是 LLM 原始训练数据集之外的新数据，它们可能来自 API、数据库、文档存储库等多个来源，形式包括文件、数据库记录、长篇文本等。借助嵌入语言模型这一人工智能技术，这些数据会被转换为数字表示形式并存储在向量数据库中，形成生成式 AI 模型可理解的知识库。

接下来是检索相关信息，系统会将用户查询转换为向量表示形式，与向量数据库中的数据进行匹配。比如，当员工询问 “我有多少年假？” 时，智能聊天机器人会检索年假政策文档和该员工的休假记录，通过数学向量计算筛选出高度相关的特定文档。

然后是增强 LLM 提示，RAG 模型会把检索到的相关数据添加到用户输入的上下文中，借助提示工程技术与 LLM 有效沟通，帮助其生成准确答案。

此外，还需要考虑外部数据的更新问题。为了保证检索信息的时效性，需异步更新文档及其嵌入表示形式，可通过自动化实时流程或定期批处理实现，这一过程可运用不同数据科学方法进行变更管理。下图展示了 RAG 与 LLM 配合使用的概念流程。

RAG 机制有哪些好处？

这一机制的核心价值，在于完美解决了大语言模型（LLM）的 “知识滞后” 难题。要知道，LLM 的参数知识本质上是静态的，无法自动跟上信息更新的节奏。而 RAG 无需对模型进行重新训练，就能让它直接获取最新信息，从而生成基于检索证据的可靠内容。

从实际效用来看，RAG不仅能通过引入检索到的权威证据，显著提升大语言模型（LLM）响应的准确性、可控性和相关性，在信息快速迭代的场景中展现出极高的应用价值 —— 有效减少错误信息生成并避免模型因知识陈旧导致的性能下降，还具备经济高效、实时更新、增强信任以及赋予开发者更多控制权等多重优势。在实施层面，相较于成本高昂的基础模型（FM）重新训练，RAG 能以更经济的方式将新数据引入 LLM，让生成式 AI 技术的获取和使用变得更广泛；在信息时效性上，它可将 LLM 直接连接到实时社交媒体提要、新闻网站等常更新的信息来源，为模型提供最新研究、统计数据或新闻，助力其向用户输出当前信息；在用户信任构建方面，RAG 支持 LLM 通过来源归属呈现内容，输出中可包含引文或引用，方便用户追溯源文档，进而提升对生成式 AI 解决方案的信任度；而对开发者而言，借助 RAG 能更高效地测试和改进聊天应用程序，灵活控制和更改模型的信息来源以适应需求变化，还能对敏感信息检索设置授权级别限制，同时针对模型引用错误信息来源的问题进行故障排除与修复，让组织能更有信心地将生成式 AI 技术应用于更广泛的场景中。

RAG 的演变

RAG的发展历程包括预训练（pre-training）、微调（fine-tuning）及推理（inference）。最初，RAG的诞生恰逢Transformer架构的兴起，最开始专注于给预训练（pre-training）模型增加额外的知识来提高模型效果；随后ChatGPT的出现展现了大语言模型强大的上下文学习（in-context learning, ICL）能力，RAG的研究也转向为给LLMs提供更丰富、及时的信息，从而能够回答出更复杂的知识密集型的推理（inference）任务。再往后，RAG也不仅限于推理阶段，转而走向更具挑战的微调（fine-tunin）。

值得关注的是，RAG 领域的研究一直在持续深化：不仅优化了模型的预训练方法，还逐渐实现了技术融合 —— 将 RAG 的优势与 ChatGPT、Mixtral 等高性能微调模型的特性相结合，进一步拓展了技术的应用边界。

近年来，RAG系统围绕性能、成本与效率的核心挑战，完成了从初级到高级再到模块化的三阶进化，每一步都在前一阶段的基础上突破局限、拓展能力。

其中，初级 RAG 率先搭建起 “索引建立 - 文档检索 - 内容生成” 的基础技术框架：系统接收用户输入后，会先检索匹配的相关文档，将其与提示语结合后传递给模型生成回答；若涉及多轮对话，还会把对话历史融入提示语，让交互更连贯。不过，初级 RAG 的短板也较为突出，检索环节常面临精确度低（信息不准）与召回率低（漏检相关信息）的问题，且容易获取过时信息，这直接导致模型可能生成脱离事实的 “幻觉” 内容，影响回答可靠性。此外，引入额外信息时的内容重复冗余、多篇检索文档的优先级排序、生成内容的风格统一性，以及避免模型过度依赖外部信息而缺乏创新等，都是初级阶段亟待解决的难题。

正是为了攻克这些痛点，高级 RAG 将优化重点聚焦于检索质量的提升，通过检索前、检索中、检索后三个环节的全流程打磨实现突破。在检索前的准备阶段，系统会细化数据细节、优化索引结构、添加元数据、改进对齐方式，并采用混合检索方法，从源头提升数据质量；进入检索阶段，要么通过微调嵌入模型提高检索相关性，要么采用能更好理解上下文的动态嵌入模型（如 OpenAI 的 embeddings-ada-02 模型），让上下文片段的匹配更精准；而在检索后，针对上下文窗口限制和噪音信息干扰的问题，会通过重新排列文档优先级、重新计算语义相似度、压缩提示信息等手段，进一步净化输入模型的信息，确保生成内容的准确性。

随着高级 RAG 解决了核心的检索优化问题，RAG 系统的发展逐渐走向更灵活的模块化阶段。模块化 RAG 以 “功能模块增强” 为核心，不仅引入相似性检索搜索模块、精细调整检索工具来提升基础性能，更关键的是打破了固定流程的限制 —— 可根据具体任务需求，自由添加、替换模块或调整模块间的工作流程。这种设计不仅涵盖了初级与高级 RAG 的核心能力，还拓展出搜索、记忆、融合、路由、预测、任务适配等多元模块，能应对更复杂的场景需求。也正因模块化 RAG 的灵活度大幅提升，一系列针对性的优化技术应运而生：混合式搜索探索结合关键词搜索与语义搜索，适配多样化查询需求；递归式检索与查询引擎从语义片段逐步拓展至更大内容块，平衡检索效率与信息丰富度；StepBack-prompt 提示技术引导模型进行概念抽象与深度推理，突破具体事例的局限；子查询策略将复杂查询拆分为多个小问题，利用不同数据源分别解答；假设性文档嵌入技术（HyDE）则通过生成假设性回答并嵌入检索，替代直接使用原始查询，进一步优化检索效果，让 RAG 系统的能力边界持续拓展。

拆解 RAG 框架核心：检索、生成与增强的技术进阶之道

RAG系统由检索、生成、增强三大核心环节构成，这三大组件的协同运作不仅奠定了 RAG 的技术基础，更持续通过技术迭代提升系统性能与应用灵活性。

作为 RAG 系统的 “信息筛选器”，检索环节的核心目标是从海量数据中精准抓取高价值信息，其效能提升可从三个维度推进。

首先是强化语义理解能力，这需要制定适配数据内容与应用需求的数据分块策略 —— 比如句子转换器更适配单句处理，文本嵌入 - ada-002 模型在 256 或 512 词元的文本处理中表现更优，同时还需结合用户提问长度、词元数量上限、应用需求等因素反复测试优化。针对专业领域场景，还需对嵌入模型进行微调，像 BAAI 开发的 BGE-large-EN 模型，通过领域知识或下游任务微调显著提升检索相关性，避免系统对专业查询的理解偏差。

其次是实现查询与文档的精准匹配，面对语义缺失或措辞模糊的查询，可借助 Query2Doc、HyDE 等工具进行查询重写，或通过优化查询的嵌入表示，使其与任务潜在空间更好对齐。

最后是优化检索器与大语言模型（LLM）的协同效果，一方面可通过 AAR、REPLUG 等技术分析 LLM 的反馈来完善检索系统，另一方面可引入 PRCA、PKG 等外部工具辅助信息对齐。

生成环节则扮演着 “信息转化器” 的角色，负责将检索到的碎片化信息转化为流畅自然的最终输出。这一过程面临着输入信息复杂多变的挑战，需通过两类方法突破瓶颈：一是检索后处理与模型固定策略，在不改动 LLM 的前提下，通过信息简化剔除冗余内容、解决长文本处理限制，同时对检索结果进行优先排序，确保高相关信息优先被利用；二是针对 RAG 场景的 LLM 微调，通过精细化调整生成过程，让输出既符合语言表达习惯，又能充分融合检索到的文档核心信息。

增强技术是 RAG 系统的 “效能放大器”，核心在于将检索信息高效融入生成流程，其应用贯穿预训练、微调、推理全阶段。在不同阶段，增强技术有着不同的落地路径。

预训练阶段可借鉴 RETRO 模型的思路，引入外部知识编码器实现检索增强；微调阶段结合 RAG 可进一步提升系统性能；推理阶段则需根据任务需求灵活融入检索内容。在数据源选择上，非结构化数据、结构化数据以及 LLM 生成数据是三大核心类型，其选择直接影响模型效果。针对多步骤推理类复杂问题，单一检索难以满足需求，因此迭代检索、递归检索、自适应检索等进阶方法应运而生 —— 迭代检索通过多轮交互丰富信息，递归检索以 “前序输出作为后序输入” 的方式深度挖掘信息，适配学术、法律等专业场景，自适应检索则可根据需求动态调整检索时机与内容，代表性实践包括 FLARE、Self-RAG 等。

RAG 的工作流程

RAG的典型工作流程主要包含四个核心步骤，通过这套标准化流程，能有效弥补大语言模型（LLM）在实时信息获取上的短板。

第一步是 “输入”，也就是用户提出的、需要 LLM 系统解答的问题。如果不采用 RAG ，这个问题会直接传递给 LLM，由其仅凭自身训练数据生成回答。

第二步为 “索引”，这是 RAG 的基础准备环节。系统会先将相关文档进行分块处理，再为每个文档块生成对应的嵌入向量，最后把这些向量存储到向量库中建立索引；后续接收用户查询时，也会以相同的方式将查询内容转化为嵌入向量。

第三步是 “检索”，系统通过计算用户查询嵌入向量与向量库中已索引向量的相似度，精准筛选出与问题高度相关的文档内容。

第四步即 “生成”，系统将检索到的相关文档作为额外上下文，与用户的原始提问相结合，一同传递给 LLM，由模型整合这些信息生成最终的回答。

举个直观的例子，若用户询问最新发生的事件，直接使用 LLM 可能因训练数据未涵盖该内容而无法作答；但借助 RAG，系统能快速检索到该事件的相关信息并提供给模型，从而产出准确回应。

RAG 与微调：大模型优化的两大路径，该如何选？

在大语言模型的性能优化领域，RAG与微调的差异对比及场景适配问题，始终是行业内热议的焦点话题。这两种技术路径虽都以提升模型表现为目标，但核心逻辑与适用场景却各有侧重，且并非相互对立，反而能形成有效互补。

具体来看，RAG 的核心优势在于对新知识的快速融合能力。在信息迭代频繁的场景中，比如实时获取行业动态、整合最新研究成果等，RAG 无需对模型进行复杂的重新训练，只需通过检索组件从外部知识库中提取最新信息，就能让模型生成符合当前事实的响应，完美解决了传统模型参数知识静态化的痛点。而微调技术则更偏向于模型的 “内功修炼”，通过对模型内部参数的调整优化，不仅能深化模型对已有知识的理解与运用，还能精准适配特定的输出格式需求，同时提升模型对复杂指令的执行效率，让模型在特定任务场景中展现出更稳定的专业性能。

值得注意的是，RAG 与微调并非二选一的关系，二者的协同使用能释放更大的技术价值。在处理复杂的知识密集型任务时，RAG 负责保障信息的时效性与全面性，微调则聚焦于优化模型的输出质量与任务适配度；而在需要快速适应新知识、同时对反馈内容有定制化要求（如特定格式、语调和风格）的可扩展应用场景中，二者的结合能实现 “高效迭代 + 精准输出” 的双重目标，进一步拓宽大语言模型的应用边界。此外，提示工程作为另一种重要的优化手段，通过精准设计提示词充分发挥模型的原生能力，也能在提升输出效果方面发挥关键作用，与 RAG、微调技术形成互补，共同推动大语言模型的性能升级。

RAG在与其他模型优化方法相比时的独特特性

RAG 模型测评

和大语言模型（LLM）的多维度评估逻辑类似，对RAG模型的科学评估，是深入理解其性能边界、推动技术在各类场景落地优化的关键环节。传统评估方式多聚焦下游任务表现，通过 F1 分数、准确率（EM）等特定任务指标来衡量效果，比如在知识密集型任务中，RaLLe 框架就是评估检索增强型大语言模型性能的经典工具。

完整的 RAG 模型评估需兼顾 “检索质量” 与 “生成质量” 两大核心维度。在检索效果评估上，会借鉴推荐系统、信息检索等领域的专业指标，例如用归一化折扣累计增益（NDCG）衡量排序合理性，用命中率判断相关信息的召回能力；而生成质量的评估则需分场景细化 —— 针对未标记内容，重点考察相关性与有害内容筛选能力；针对已标记内容，则聚焦答案准确性。从评估方法来看，整体可分为手动评估与自动评估两类，具体可根据需求灵活选择。

更系统的 RAG 框架评估会围绕 “三大质量指标” 与 “四大核心能力” 展开。三大质量指标分别是：上下文相关性（衡量检索信息的精确度与相关度）、答案忠实度（评估答案对检索上下文的还原度）、答案相关性（判断答案与用户问题的契合程度）。四大核心能力则聚焦系统的适应性与效率，包括对噪声的鲁棒性、负面信息的排除能力、信息整合能力以及面对假设情况的鲁棒性。

为实现全面且高效的评估，行业内已形成成熟的工具与基准体系。一方面，RGB、RECALL 等基准测试为评估提供了标准化场景；另一方面，RAGAS、ARES、TruLens 等工具的出现，大幅简化了评估流程的自动化实现。此外，部分系统还会借助大语言模型的能力，对上述质量指标进行智能判定，进一步提升评估的便捷性与精准度。

RAG 研究：在挑战中解锁未来发展新可能

随着 RAG 系统的持续迭代完善，一系列关键挑战亟待攻克，而这些挑战的突破过程，也暗藏着技术未来的发展方向。

首先是上下文长度的适配难题。当前大语言模型的上下文处理范围不断拓展，这就要求 RAG 系统随之调整优化，确保在海量上下文信息中精准捕获最核心、最相关的内容，避免因信息过载导致检索与生成效率下降。

其次，系统鲁棒性的提升迫在眉睫。在复杂的实际应用场景中，RAG 系统难免会接触到与事实相悖的错误信息或带有对抗意图的干扰信息，如何有效甄别并处理这类信息，直接关系到系统输出结果的可靠性，是提升 RAG 性能的关键一环。

混合方法的深度探索也成为研究热点。研究者们正致力于破解 RAG 与定制化微调模型的最优结合模式，通过两者的协同互补，充分发挥各自技术优势，实现 1+1>2 的应用效果。

与此同时，如何进一步释放大语言模型在 RAG 系统中的潜力，也是行业高度关注的方向。目前大语言模型在 RAG 架构中尚未完全发挥其能力，未来需通过技术创新拓展其应用边界，让模型在信息整合、推理决策等方面发挥更大作用。

大语言模型的规模化法则在 RAG 系统中的适配问题，至今仍未得到彻底解决。如何让规模化训练的模型在 RAG 场景中稳定发挥性能，平衡模型规模与应用效率，是需要持续攻关的课题。

对于生产级 RAG 系统的落地，工程层面的严苛要求构成了重要挑战。要实现商业化应用，系统不仅要在核心性能上达标，还需在运行效率、数据安全防护、用户隐私保护等多个维度达到工程卓越标准，兼顾技术先进性与应用安全性。

多模态领域的拓展为 RAG 开辟了新赛道。此前 RAG 研究多聚焦于文本任务，如今随着技术发展，研究者开始探索将其应用于图像、音频、视频、编码等多模态场景，这一方向虽充满潜力，但也面临跨模态信息检索与融合的技术难题。

最后，评价机制的完善是 RAG 成熟的重要支撑。随着 RAG 被用于构建更复杂的应用，传统评价方式已难以满足需求，亟需开发能精准评估上下文相关性、内容创新性、多样性及准确性等多维度的专业工具。同时，加强 RAG 的解释性研究并开发相关工具，也能帮助用户更好地理解系统决策逻辑，提升技术的可信度与应用价值。

这些挑战既是 RAG 研究道路上的 “绊脚石”，也是推动技术突破的 “催化剂”。未来，随着这些关键问题的逐步解决，RAG 有望在更多领域实现深度落地，绽放更大的技术价值。

快速构建 RAG 系统的工具

在 RAG系统的开发领域，丰富多样的工具为开发者提供了多元化选择，不同工具凭借独特优势适配各类场景需求，助力高效搭建智能应用。

综合性工具是构建 RAG 系统的核心支撑。LangChain 作为模块化框架，专为复杂大语言模型（LLM）应用打造，其涵盖链式操作、代理、内存管理等丰富功能，能轻松支持多步骤工作流程与工具集成；文档加载器可兼容多种格式，高效提取文本内容并保留原始结构，搭配评估框架 LangSmith，还能全程追踪用户查询到最终回答的流程，帮助开发者精准定位问题。而 LlamaIndex 则聚焦 RAG 场景深度优化，秉持 “数据先行” 的核心理念，凭借强大的数据连接器与分块策略，在数据摄取、查询速度及质量上表现突出，通常优于 LangChain，同时提供多种索引类型与灵活检索策略，适配不同数据类型与查询需求。此外，DSPy 虽未获取详细信息，但同样具备支撑 RAG 系统构建的强大功能，也是开发者可考虑的选择之一。

低代码平台的出现大幅降低了 RAG 应用的开发门槛，Flowise AI 便是其中的典型代表。它通过简化开发流程，让开发者无需深入掌握复杂技术，就能快速完成 RAG 应用的搭建与部署，加速项目落地。

在细分领域，多款技术提供了针对性解决方案。由德国 Deepset 公司维护的 Haystack，定位为企业级 NLP 框架，以模块化与可部署性为核心优势，通过 Pipeline 编排机制将 Retriever、Reader、Ranker 等组件以 DAG 方式组合，完美适配对稳定性、可审计性、可监控性有高要求的企业级 RAG 系统。而 Meltano 与 Cohere Coral 虽未明确具体应用场景与优势，但同样能为特定需求提供专属解决方案。

云服务提供商也纷纷布局 RAG 领域，推出专业化服务。Weaviate 旗下的 Verba 工具，依托 Weaviate 向量数据库的强大数据摄取能力，成为构建个人助理应用的优质选择；亚马逊的 Kendra 则专注于智能企业搜索服务，精准满足企业在知识管理与信息检索方面的核心需求。

这些工具与服务的持续迭代，不仅拓宽了 RAG 的应用边界，更为开发者与研究人员提供了更多创新可能。开发者可根据项目实际需求，灵活选择单一工具或组合使用多种工具，打造更复杂、高效的 RAG 应用。随着技术的不断突破，未来有望涌现出更多创新应用，为用户带来更丰富、深入的智能交互体验。
