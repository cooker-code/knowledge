## 知识点组织

| 知识点类型 | 应沉淀什么 | 来源吸收方式 |

## 排重准则

问题指纹：

```text
智能问数对象 + 语义层/Schema Linking/生成/校验/权限模块 + 核心机制 + 解决问题 + 准确性/追责边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是 Text-to-SQL | 按 Schema Linking、业务术语、SQL 修复、评估、训练数据拆分 |
| 都是智能问数产品 | 没有口径、权限和校验链路则降权 |
| 都是模型榜单 | 不进入核心知识点，除非改变工程准则 |
| 有语义层和权限 | 优先沉淀到可追责问数链路 |
