## 核心问题

AI 数据分析助手的价值不在“替人生成一张图或一段报告”，而在把业务问题、数据连接、执行过程、图表配置、结论证据和人工复核保留下来。

## 判断准则

| 环节 | 准则 |
|---|---|
| 问题澄清 | 先确认指标、维度、时间窗口、对比对象和行动目标 |
| 数据连接 | 必须知道字段含义、表关系、权限和口径来源 |
| 执行工作区 | SQL、Python、表格变换和图表配置要能重跑 |
| 结果解释 | 区分事实、推断、建议和不确定性 |
| 质量控制 | 检查缺失值、异常值、重复、单位、样本偏差和口径漂移 |
| 人工复核 | 高风险结论必须能回到输入数据和计算过程 |
| 产品边界 | IDE、Excel 插件、问数工具和图表助手必须区分：只封装入口不等于具备分析闭环 |

## 认知校准点

| 偏差 | 处理 |
|---|---|
| “AI 会用 Excel”就是分析能力 | 只有能处理口径、关联、验证和解释，才算分析助手 |
| 自动生成图表就能替代分析师 | 图表是表达层，业务假设和质量控制仍需人工把关 |
| 内部数据分析自动化比例可直接采信 | 需要看任务类型、失败率、人工复核和权限边界 |
