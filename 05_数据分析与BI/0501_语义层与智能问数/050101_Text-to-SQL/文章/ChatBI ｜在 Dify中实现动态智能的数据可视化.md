# 1. 痛点：传统 ChatBI 可视化方案的“僵化”与“繁琐”

在典型的 ChatBI 工作流中，我们面临一个普遍的挑战：用户的提问是开放的、不可预测的，而系统输出的数据结构也因此是动态变化的。然而，传统的图表展示方案却要求我们“提前”确定图表类型。

为了应对这一矛盾，业界通用的做法是“LLM路由分发”模式：

1. 用户提问，系统查询数据库，返回原始数据。

2. LLM 路由节点：使用一个大模型（LLM）来判断返回的数据“适合”哪种图表（例如，判断为“趋势数据”）。

3. 分支流转：工作流根据 LLM 的判断，走向不同的分支，每个分支上都配置了一个固定类型的图表节点（如折线图节点、柱状图节点等）。

这种模式存在三大核心痛点：

\* 缺乏灵活性：一旦出现新的数据组合或用户问法，就需要工程师手动添加新的路由分支和图表节点。例如，系统能处理“各品牌销量”，但处理不了“各品牌销量及达成率”，除非你专门为之创建一个新的组合图分支。

\* 维护成本高：随着业务深入，工作流会变得异常复杂，形成一个难以维护的“蜘蛛网”。每次调整都需要检查整个链路，牵一发而动全身。

\* 能力天花板低：很难实现如“双Y轴组合图”这类需要根据数据量纲、指标数量进行精细判断的复杂图表，因为简单的路由判断无法承载如此复杂的逻辑。

# 2. 新方案的效果：从“规则判断”到“智能生成”的飞跃

新方案的核心思想是：放弃复杂的路由，将图表生成的自主权完全交给一个经过精心设计的“数据可视化专家”LLM节点。

通过该方案，我们可以实现以下效果：

|  |  |  |
| --- | --- | --- |
| 对比维度 | 传统方案（路由分发） | 新方案（动态生成） |
| 灵活性 | 低 | 极高 |
| 维护成本 | 高 | 极低 |
| 用户体验 | 一般 | 优秀 |
| 实现复杂度 | 高 | 低 |

最终，系统变得更加“智能”，能够像一个真正的数据分析师那样，拿到数据后，思考并创造出最恰当的可视化呈现。

# 3. 实现思路：三步构建动态可视化引擎

此方案的实现，依赖于前端（如 Dify 的对话界面）具备直接渲染 ECharts JSON 配置的能力。我们的任务就是引导 LLM 生成这段配置。

核心原理：将“分析数据、选择图表、编写配置”这一系列人类分析师的思考过程，完整地编码到一个 Prompt 中，让 LLM 模拟这个过程。

实现步骤如下：

## 第一步：获取原始数据

用户提问后，通过数据库查询（如 Text2SQL）得到结果。将这个结果（通常是 JSON 格式的数组）作为原始数据，准备传入 LLM。

## 第二步：构建核心 Prompt，让 LLM 成为 ECharts 专家

这是整个方案的灵魂。我们需要创建一个强大的 Prompt，它包含以下五个关键要素：

1. 角色定义 (Role)：为 LLM 分配一个专家身份。

> 你是一名顶级的、精通 ECharts 的数据可视化专家。

2. 任务指令 (Task)：清晰地描述它的工作内容和步骤。

> 你的任务是分析用户问题和原始数据，并将其转换为一个严格符合 echart 要求的 JSON 对象，用于驱动可视化图表。

3. 逻辑规则 (Rules)：提供明确的图表选择和数据处理指南，这是 LLM 进行智能判断的基础。

\* 图表选择：如果数据包含时间序列，优先选择 line (折线图)。 / 如果数据是类别和数值的对比，优先选择 bar (柱状图)。

\* 数据清洗：如果数据中包含"合计"和分项的内容，只需要呈现分项内容。 / 如果数据结果为0或者NULL，就不展示。

4. 示例参考 (Few-Shot Examples)：提供高质量、多样的 ECharts JSON 示例。这是让 LLM 理解并生成复杂、正确配置的最关键一步。示例应覆盖常用及高级场景。

1 // 提供一个基础柱状图示例...

2 // 提供一个多系列折线图示例...

3 // 提供一个带双Y轴的组合图（Bar & Line）示例...

4 // 提供一个饼图示例...

这些示例教会了 LLM 如何处理多系列数据、如何配置双Y轴 (`yAxisIndex`)、如何格式化提示框 (`tooltip`) 等高级技巧。

5. 输出约束 (Output Constraint)：强制要求 LLM 输出纯净、可直接使用的代码块，防止任何多余的解释性文本污染最终结果。

> 你必须严格按照以下结构输出，不能包含任何额外的解释、注释或 markdown 标记。输出内容如下标志包裹在里：echarts ... ``

## 第三步：调用 LLM 并渲染

将核心 Prompt 和 第一步获取的原始数据 一同发送给 LLM。

LLM 将返回一个被 echarts ... 包裹的、干净的 JSON 字符串。

在前端，提取这个 JSON 字符串，并将其直接传递给 ECharts 的 setOption() 方法进行渲染。

通过以上三步，我们就构建了一个强大且灵活的动态可视化引擎，彻底摆脱了传统方案的束缚。
