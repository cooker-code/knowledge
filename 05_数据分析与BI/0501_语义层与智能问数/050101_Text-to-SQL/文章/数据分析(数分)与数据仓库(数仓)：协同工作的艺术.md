# 数据分析(数分)与数据仓库(数仓)：协同工作的艺术

3. 只对数分了解的同学
4. 只对做BI\大屏了解的同学
5. 只对做传统ETL了解的同学
6. 分不清楚数据分析师和数据开发工程师的关系的！

## unsetunset数仓工程师和数据分析师的关系？unsetunset

1. 数仓工程师是数据分析师的上游，数据分析师就是数仓工程师的下游
2. 数仓工程师建立好数仓之后数据分析师才能通过数仓进行数据的提取、可视化、以及数据分析报告的产出
3. 分析师无法独立面临比如说没有做过整合的，没有做过清洗的，没有做过主题划分的一些数据
4. 这样会大大增加数据分析的一个难度，所以呢 数据仓库工程师就构成了数据分析的上游
5. 数据分析师和数仓（Data Warehouse）是密切相关的两个概念
6. 数据分析师需要使用数据仓库中的数据进行分析，而数据仓库则是为数据分析师提供数据支持的基础设施
7. 数据仓库和数据分析师之间的协作非常重要，他们的合作可以帮助企业更好地利用数据进行决策和创新
8. 数据仓库的建设需要考虑到数据分析师的需求，提供可靠的数据支持
9. 而数据分析师则需要了解数据仓库的架构和数据模型，以便更好地进行数据分析和决策支持

## unsetunset数分需要具备哪些技能，怎么和数仓协作？unsetunset

1. 数据分析师通常需要使用各种数据分析工具和技术，如Python,SQL,Excel,可视化,BI,数据分析模型,统计学,AB测试、R等，来对数据进行处理和分析
2. 这些数据分析工具需要从数据仓库中获取数据进行分析
3. 数据仓库通常包括多个数据源的数据集成，数据清洗、转换和加载等过程，以及数据模型设计和数据存储等方面的内容
4. 在数据仓库中，数据通常是按照主题或业务流程进行组织，以便更好地支持数据分析和决策。
5. 其实这些技能点呢，优秀的数仓工程师也同样具备，但是没有数据分析师的思路，逻辑思维，对业务的一个理解，一个深度的业务整合。
6. 一个优秀的数据分析师，他同样也是半个数仓工程师也会了解一些hadoop生态体系，数仓分层，数据建模，指标建设
7. 数据分析师需要了解数据仓库的架构和数据模型，以便更好地理解和使用数据。
8. 他们需要了解数据仓库中的数据内容、数据质量和数据粒度等方面的信息，以便更好地准备和清洗数据，进行数据分析和建模。

### 数开和数仓有啥区别（史的宁loser）

1. 数开：相对比较泛，可能是做一些数据清晰、数据加工、数据集成、也会包含数仓，这是数据开发的一个统称
2. 数仓：相对比较固定，一般就是主流的以hadoop hive 为主的数仓建设，主要为企业提供数据分析、决策的支持。
