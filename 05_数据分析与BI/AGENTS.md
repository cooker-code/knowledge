## 排重指纹

```text
问题域 + 技术/方法本体 + 核心机制 + 解决问题 + 适用边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 只是入门概念 | 用户大概率已知，不新建知识点 |
| 只是工具宣传 | 没有口径、权限、校验、行动闭环或机制证据时降权 |
| 只是 SQL 题解 | 迁入 SQL 书写目录，作为表达模式或边界样例 |
| 能补判断准则 | 合并到对应核心知识点，不按文章标题建页 |
| 与目录冲突 | 先按正文主问题重路由，再决定是否吸收 |

## 文章处理规则

1. 一级目录不设 `文章/`。
2. 二级 `文章/` 只保留已吸收到知识地图、核心知识点或规则缺口的 `done-*.md`。
3. `done-` 表示已经贡献到长期结构，不表示仅仅读过。
4. 没有技术或方法增量的营销、资讯、投资新闻，只作为降权样例写入规则，不新建知识点。
5. 后续新增文章先看本文件边界，再看对应二级 `AGENTS.md` 和知识地图。
