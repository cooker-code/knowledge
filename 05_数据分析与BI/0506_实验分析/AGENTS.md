# 实验分析
## 知识点入口

- 本模块先看宏观流程，再看文章：[知识地图](050600_核心知识点/知识地图.md)。
- 新文章必须先归入流程节点，再判断是补充、冲突、不同层次还是降权。
- `文章/` 只保留原文锚点，长期知识必须沉淀到 `050600_核心知识点/`。


## 类目定位

| 项 | 内容 |
|---|---|
| 一级类目 | 数据分析与 BI |
| 二级类目 | 实验分析 |
| 核心问题 | 如何设计、运行、评估和解释实验，让数据结论能支撑决策 |
| 不解决什么 | 不把所有统计方法都放进来；统计概念只有进入实验设计、判断和解释时才引用 |
| 用户当前认知假设 | L3：知道 AB 实验基础，重点补样本量、显著性、分层、污染、长期效应和决策边界 |

## 知识点组织

| 知识点类型 | 应沉淀什么 | 来源吸收方式 |
|---|---|---|
| 实验设计 | 假设、分流、样本量、指标选择 | 多篇文章按设计问题合并 |
| 实验评估 | 显著性、置信区间、效应量、多重检验 | 统计文章只吸收对实验判断有用的部分 |
| 实验解释 | 异质性、长期效应、边际 ROI、反直觉结果 | 案例必须写清适用边界 |
| 实验平台 | 分流、埋点、指标计算、灰度和回滚 | 与数据工程交叉时只保留分析决策相关内容 |

## 排重准则

问题指纹：

```text
实验对象 + 设计/评估/解释/平台模块 + 统计机制 + 决策边界 + 认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是 AB 实验入门 | 已知基础不新建 |
| 补充了失败模式 | 合并进相关知识点 |
| 只讲统计公式 | 转到统计方法，除非直接服务实验判断 |
| 只讲平台功能 | 没有实验判断增量则降权 |
