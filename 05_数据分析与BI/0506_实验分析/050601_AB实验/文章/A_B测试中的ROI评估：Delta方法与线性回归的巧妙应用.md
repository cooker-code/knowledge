# A/B测试中的ROI评估：Delta方法与线性回归的巧妙应用

## 引言

在上一篇文章中，我们明确了在A/B测试中评估投资回报率（ROI）时，应该关注“回报的平均”（总收入增长/总成本增长），而非“平均的回报”。我们也提出了计算ROI置信区间的两种方法：自助法和Delta方法。本文将聚焦于**Delta方法**，并展示一个非常巧妙的技巧，如何利用简单的**线性回归**来完成对ROI这一复杂比率指标的统计推断。

---

## Delta方法：从复杂代数到简单回归

Delta方法是一个基于泰勒展开的强大统计工具，它能帮助我们估算一个或多个随机变量的函数的方差。对于ROI（即 ）这个比率函数，经过一系列的数学推导，Delta方法给出了其渐近方差的表达式。

虽然推导过程涉及较多的矩阵代数，但最终的结论却异常简洁和富有洞察力。结论是：**估计ROI估计量  的方差，等价于估计一个新构造的辅助变量  的均值差异的方差。**

### 这个公式告诉我们什么？

1. 1. **直观解释**：变量  可以被看作是“调整后”的收入，它剔除了由我们估计的ROI所解释的成本部分。我们可以把它理解为一种“基线收入”。将这个基线收入除以总的平均成本差异，意味着我们投资的规模越大（即越大），我们对ROI估计的精确度就越高（即方差越小）。
2. 2. **实用方法**：这个结论最妙的地方在于，它将一个复杂的比率方差问题，转化为了一个我们非常熟悉的**均值差异估计**问题。而对于A/B测试中的均值差异问题，我们有一个最直接、最简单的工具——**线性回归**。

---

## 四步线性回归法，搞定ROI推断

现在，我们可以通过以下四个简单的步骤，使用线性回归来计算ROI的置信区间：

**第一步：估计ROI的点估计值**

这我们在上一篇文章已经做过，通过计算处理组和对照组的平均收入差，除以平均成本差得到。

```
rho_hat = 2.0698
```

**第二步：计算平均成本差异的绝对值**

```
abs_Delta_C = np.abs(df.loc[df.new_machine==1, "cost"].mean() - df.loc[df.new_machine==0, "cost"].mean())
```

**第三步：构建辅助变量**

根据上面的公式，为我们的数据框 `df` 添加新的一列 `revenue_tilde`。

```
df['revenue_tilde'] = (df['revenue'] - rho_hat * df['cost']) / abs_Delta_C
```

**第四步：对辅助变量  进行线性回归**

现在，我们只需要将这个新构造的辅助变量 `revenue_tilde` 对处理变量 `new_machine` 做一个简单的OLS回归。

```
smf.ols('revenue_tilde ~ new_machine', df).fit().summary().tables[1]
```

### 最终决策

有了ROI的点估计值（2.07）和它的标准误差（0.917），我们就可以计算出95%的置信区间：

这个置信区间告诉我们什么？

* • 它不包含0，这很好。
* • 但更重要的是，它**包含了1**。

在ROI分析中，真正的盈亏平衡点是1（每投入1元，收回1元）。由于我们的置信区间包含了1，这意味着我们**无法在统计上拒绝“这项投资不盈利（即ROI≤1）”的原假设**。

尽管我们的点估计值2.07看起来很诱人，但考虑到其不确定性，我们还不能充满信心地说这项服务器投资一定是划算的。我们可能需要收集更多的数据，或者寻找降低成本、提高收入的进一步策略。

## 结论

本文展示了在A/B测试中评估投资回报率（ROI）时，如何从一个看似复杂比率推断问题，通过巧妙的统计方法（Delta方法），最终转化为一个简单的线性回归问题。

**核心要点：**

1. 1. **明确目标**：评估ROI时，应使用“回报的平均”，而非“平均的回报”。
2. 2. **选择工具**：Delta方法为比率型指标的方差估计提供了强大且高效的理论基础。
3. 3. **简化执行**：通过构建一个辅助变量，可以将复杂的方差计算问题转化为一个简单的OLS回归，快速得到ROI的标准误差和置信区间。

这个方法不仅大大提升了计算效率，也为我们理解和分析比率型指标提供了一个清晰、严谨的框架，是数据科学家工具箱中不可或缺的一项技能。
