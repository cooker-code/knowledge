# 告别“相关性”谬误：用贝叶斯网络发现数据背后的因果关系（下）

在上一篇文章中，我们探讨了因果推断的基本概念，并了解了如何通过**结构学习**从数据中发现变量间的因果关系图（DAG）。我们介绍了基于评分和基于约束的两大类学习方法。

现在，让我们进入实战环节。本文将以经典的“洒水器”案例为例，手把手教你如何使用Python的`bnlearn`库，完成从数据到因果模型的完整流程，包括**结构学习**、**参数学习**和最终的**因果推断**。

---

## 4. `bnlearn`库简介与案例实战

`bnlearn`是一个强大的Python库，它封装了创建、学习、分析和可视化贝叶斯网络的完整工具链。其核心功能包括：

* • **结构学习：** 从数据中自动学习网络结构。
* • **参数学习：** 估计网络中的条件概率。
* • **推断：** 在给定证据的情况下，查询网络以计算概率。
* • **数据处理与可视化：** 提供数据离散化、模型比较、交互式绘图等实用功能。

### 4.1 小型案例分析：洒水器问题

让我们从一个简单直观的例子开始。想象一个场景，我们有过去1000天的数据，记录了四个二进制变量：

* • **Cloudy (是否多云):** 是/否
* • **Sprinkler (洒水器是否开启):** 是/否
* • **Rain (是否下雨):** 是/否
* • **Wet\_Grass (草地是否湿润):** 是/否

根据生活常识，我们大概能猜出它们之间的因果关系。现在，让我们看看`bnlearn`能否从数据中自动发现这个结构。

**第一步：安装并导入库，加载数据**

```
# 安装bnlearn  
# pip install bnlearn  
  
import bnlearn as bn  
  
# 加载内置的洒水器示例数据集  
df = bn.import_example('sprinkler')  
print(df.head())  
#    Cloudy  Sprinkler  Rain  Wet_Grass  
# 0       0          0     0          0  
# 1       1          0     1          1  
# 2       0          1     0          1  
# 3       1          1     1          1  
# 4       1          0     1          1
```

**第二步：结构学习**

使用`bnlearn`进行结构学习非常简单。默认情况下，它会使用高效的`hillclimbsearch`（爬山法）和`bic`（贝叶斯信息准则）评分来寻找最优的DAG。

```
# 从数据中学习DAG结构  
DAG = bn.structure_learning.fit(df)  
  
# 打印邻接矩阵并可视化图形  
print(DAG['adjmat'])  
bn.plot(DAG)
```

学习到的洒水器DAG

  
图3：从数据中学习到的洒水器因果关系图。

仅需一行代码，`bnlearn`就为我们揭示了变量间的因果依赖关系：

* • **草地湿润 (Wet\_Grass)** 是由 **下雨 (Rain)** 和 **洒水器 (Sprinkler)** 共同导致的。
* • **下雨 (Rain)** 和 **洒水器 (Sprinkler)** 的开启都受到 **是否多云 (Cloudy)** 的影响（例如，天阴更可能下雨，天晴炎热更可能开洒水器）。
* • **多云 (Cloudy)** 是这个系统中的“根因”，它不被其他任何变量所影响。

这个结果完全符合我们的直觉！

## 5. 参数学习：量化因果关系

我们已经有了因果图的“骨架”，但要让它能进行推理和预测，我们还需要为这个骨架填充“血肉”——也就是量化每个因果关系的强度。这个过程称为**参数学习（Parameter Learning）**，其目标是估计**条件概率表（Conditional Probability Tables, CPTs）**。

CPT描述了在给定其父节点状态的情况下，一个节点取不同状态的概率。例如，我们需要知道在“多云”和“晴天”两种情况下，“下雨”的概率分别是多少。

`bnlearn`支持两种主要的参数学习方法：

* • **最大似然估计 (Maximum Likelihood Estimation, MLE):** 直接使用数据中的频率来计算概率。例如，P(下雨=是 | 多云=是) = (多云且下雨的天数) / (多云的总天数)。这种方法简单直观，但在数据稀疏时容易过拟合。
* • **贝叶斯估计 (Bayesian Estimation):** 从一个“先验”的概率表开始，然后用观测数据来更新它。这种方法在数据量较少时更为稳健，因为它引入了先验知识（或平滑），避免了零概率的出现。

**第三步：参数学习**

```
# 我们可以使用上一步学习到的DAG，也可以基于专家知识手动创建一个DAG  
edges = [('Cloudy', 'Sprinkler'),  
         ('Cloudy', 'Rain'),  
         ('Sprinkler', 'Wet_Grass'),  
         ('Rain', 'Wet_Grass')]  
DAG_manual = bn.make_DAG(edges)  
  
# 使用贝叶斯估计进行参数学习  
model = bn.parameter_learning.fit(DAG_manual, df, methodtype='bayes')  
  
# 打印学习到的CPTs  
bn.print_CPD(model)
```

执行后，`bnlearn`会为每个节点生成一个CPT，如下所示：

```
[bnlearn] >[Conditional Probability Table (CPT)] >[Node Rain]:  
+---------+--------------------+-------------------+  
| Cloudy  | Cloudy(0)          | Cloudy(1)         |  
+---------+--------------------+-------------------+  
| Rain(0) | 0.807              | 0.177             |  
+---------+--------------------+-------------------+  
| Rain(1) | 0.192              | 0.822             |  
+---------+--------------------+-------------------+
```

这张表告诉我们：

* • 在非多云（Cloudy=0）的日子里，下雨（Rain=1）的概率是19.2%。
* • 在多云（Cloudy=1）的日子里，下雨（Rain=1）的概率是82.2%。

## 6. 因果推断：回答“如果...会怎样”的问题

现在，我们拥有了一个完整的贝叶斯网络模型（包含结构和参数），终于可以用来进行\*\*推断（Inference）\*\*了。推断允许我们根据观测到的“证据”，来计算其他变量的概率，从而回答各种“what if”问题。

**第四步：因果推断**

`bnlearn`的推断引擎可以轻松回答我们关心的问题。

**问题1：如果洒水器没开，草地湿的概率有多大？**

```
# 'evidence'参数用于设定我们已知的条件  
q1 = bn.inference.fit(model, variables=['Wet_Grass'], evidence={'Sprinkler': 0})  
print(q1)  
# +----+-------------+----------+  
# |    |   Wet_Grass |        p |  
# +====+=============+==========+  
# |  0 |           0 | 0.486917 |  
# +----+-------------+----------+  
# |  1 |           1 | 0.513083 |  
# +----+-------------+----------+
```

**答案：** P(Wet\_Grass=1 | Sprinkler=0) = **51.3%**。即使洒水器没开，草地仍有大约一半的可能是湿的（因为可能下雨了）。

**问题2：如果洒水器没开，而且天是多云的，那么下雨的概率有多大？**

```
q2 = bn.inference.fit(model, variables=['Rain'], evidence={'Sprinkler': 0, 'Cloudy': 1})  
print(q2)  
# +----+--------+----------+  
# |    |   Rain |        p |  
# +====+========+==========+  
# |  0 |      0 | 0.336957 |  
# +----+-------------+------+  
# |  1 |      1 | 0.663043 |  
# +----+-------------+------+
```

**答案：** P(Rain=1 | Sprinkler=0, Cloudy=1) = **66.3%**。多云这个证据，大大提升了我们对“下雨”的信念。

## 7. 讨论与总结

通过这个简单的例子，我们完整地体验了使用贝叶斯网络进行因果发现和推断的强大流程。相较于传统的机器学习模型，贝叶斯网络提供了几个独特的优势：

1. 1. **可解释性强：** 因果图直观地展示了变量间的依赖关系，让我们能理解模型的决策逻辑。
2. 2. **融合领域知识：** 我们可以将专家的先验知识融入到图的构建和参数学习中。
3. 3. **处理不确定性：** 结果以概率分布的形式给出，让我们能对预测的置信度进行判断。
4. 4. **进行因果推断：** 能够回答“干预”和“反事实”问题，这是许多预测模型无法做到的。

当然，贝叶斯网络也有其挑战，尤其是在处理大量变量时，结构学习的计算成本会非常高。但对于理解中等规模系统的核心驱动因素而言，它无疑是一个极其强大的工具。

希望本文能为你打开一扇通往因果科学的大门。告别在“相关性”的迷雾中徘徊，开始用贝-耶斯网络去探索数据背后真正的因果链条吧！
