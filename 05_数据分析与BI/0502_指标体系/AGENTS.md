## 用户认知重点

| 项 | 内容 |
|---|---|
| 已知基础 | 了解 KPI、DAU/MAU 等常用指标，知道指标需要统一口径 |
| 待补边界 | 北极星指标的选取依据；指标树如何与业务动作对齐 |
| 易偏差点 | 指标越多越好；指标数字正确 ≠ 指标有行动指导意义 |
| 优先抽取 | 指标分层设计、口径统一方法、指标平台建设准则 |

## 排重指纹

```text
指标对象 + 设计维度（定义/分层/口径/平台/治理）+ 核心机制 + 解决问题 + 业务可行动性边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是指标定义 | 区分口径、分子分母、时间维度、分组维度 |
| 都是指标平台 | 区分元数据管理、计算引擎、自助查询、订阅和口径校验 |
| 指标 + Text-to-SQL | 按语义层归入 `0501_语义层与智能问数` |
| 指标 + 归因分析 | 归入 `0505_归因分析` |
| 通用 SQL 题解 | 归入 `03_数据工程与数仓/0302_离线数仓/030204_SQL书写` |
