## 开篇

在业务数据分析中，环比和同比永远是最核心的两个趋势指标：

* • 环比：今天相比昨天，最近一个月相比上一个月……看短周期内的变化趋势
* • 同比：今年这个月相比去年同月……看长周期的增长健康度

但每次写日报、月报、年报，很多人依然在纠结：

* • 怎么写 SQL 能写得又短又准？
* • 怎么搞定日期偏移？
* • 怎么高效支持日/月/年多层级对比？

其实在 Hive 里，只要掌握了窗口函数 LAG()，配合简单的时间格式化，5个指标逻辑可以全部一次性搞定！

## 指标逻辑拆解

| 指标 | 计算逻辑 | 核心函数 |
| --- | --- | --- |
| 日环比 (Daily MoM) | 今日 vs 昨日 | `LAG(gmv, 1)` |
| 日同比 (Daily YoY) | 今日 vs 去年今日 | `LAG(gmv, 365)` |
| 月环比 (Monthly MoM) | 本月 vs 上月 | `LAG(total_gmv, 1)` |
| 月同比 (Monthly YoY) | 本月 vs 去年同月 | `LAG(total_gmv, 12)` |
| 年环比 (Yearly MoM) | 本年 vs 上年 | `LAG(total_gmv, 1)` |
