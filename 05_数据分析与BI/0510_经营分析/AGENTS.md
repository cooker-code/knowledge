## 用户认知重点

| 项 | 内容 |
|---|---|
| 已知基础 | 了解 DAU/MAU/留存/转化等核心经营指标，接触过业务复盘流程 |
| 待补边界 | 经营分析 vs 指标体系 vs 归因分析的职责划分；增长模型如何驱动动作 |
| 易偏差点 | 经营分析停留在"数字看板"；行动建议缺乏可验证的闭环 |
| 优先抽取 | 经营模型设计、用户分层方法、增长杠杆识别、分析→行动→验证的闭环 |

## 排重指纹

```text
经营对象（用户/业务/产品）+ 分析维度（增长/留存/转化/复盘）+ 核心机制 + 解决问题 + 业务行动可行性边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是用户增长 | 区分获客、激活、留存、收益、裂变的分析方法和指标 |
| 都是用户分层 | 区分 RFM、行为标签、价值分层和分层运营策略 |
| 指标定义和口径 | 归入 `0502_指标体系` |
| 实验验证增长效果 | 归入 `0506_实验分析` |
