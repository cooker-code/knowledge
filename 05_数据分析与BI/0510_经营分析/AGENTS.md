# 经营分析
## 知识点入口

- 本模块先看宏观流程，再看文章：[流程化知识点总览](knowledge/05_数据分析与BI/0510_经营分析/核心知识点/流程化知识点总览.md)。
- 新文章必须先归入流程节点，再判断是补充、冲突、不同层次还是降权。
- `文章/` 只保留原文锚点，长期知识必须沉淀到 `核心知识点/`。


## 类目定位

| 项 | 内容 |
|---|---|
| 一级类目 | 数据分析与 BI |
| 二级类目 | 经营分析 |
| 核心问题 | 如何把业务目标、过程指标、分析结论和行动复盘连接起来 |
| 不解决什么 | 不收纳泛数据分析工具、Excel 技巧、AI 工具资讯；只有能沉淀经营判断框架时才进入 |
| 用户当前认知假设 | L3：基础分析模型已知，重点补业务目标拆解、行动闭环、分析 Agent 的可控边界和复盘方法 |

## 知识点组织

| 知识点类型 | 应沉淀什么 | 来源吸收方式 |
|---|---|---|
| 经营模型 | 收入、成本、转化、留存、供给需求等结构 | 多篇模型文章合并为业务拆解图 |
| 分析流程 | 提问、取数、拆解、解释、行动、复盘 | 工具文章只吸收流程能力和风险 |
| 增长分析 | 新增、活跃、留存、转化、ROI | 与实验/归因交叉时分别引用 |
| AI 数据分析 | 数据分析 Agent、ChatExcel、自动报告 | 必须写权限、口径、验证和可追责边界 |

## 排重准则

问题指纹：

```text
业务目标 + 分析模型/流程/工具模块 + 行动闭环 + 风险边界 + 认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 只是分析模型清单 | 不新建，除非形成可复用拆解框架 |
| 只是 AI 工具演示 | 没有口径、校验、权限和复盘边界则降权 |
| 与智能问数重叠 | 问数链路归语义层与智能问数，经营决策框架归这里 |
| 与归因/实验重叠 | 原因定位归归因，实验验证归实验，业务决策闭环归经营分析 |
