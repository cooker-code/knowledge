## 核心问题

报表看板的核心不是“把图表摆出来”，而是让稳定指标、筛选器、权限、数据集、解释路径和复用方式形成低成本决策入口。

## 认知校准点

| 校准点 | 处理 |
|---|---|
| 开源 BI 文章常偏产品宣传 | 必须降到数据集、权限、查询、看板治理等模块判断 |
| 告别宽表不等于不要建模 | DQL 或语义查询能缓解宽表爆炸，但仍依赖指标口径和数据模型 |
| 看板不是可视化目录的全部 | 看板治理归报表看板，图表编码和图形语法归可视化 |

## 判断准则

| 维度 | 准则 |
|---|---|
| 数据集 | 是否沉淀复用数据集，而不是每张图单独写 SQL |
| 筛选器 | 是否能表达业务维度，且不会造成指标口径漂移 |
| 权限 | 是否区分数据权限、看板权限、行列权限和分享边界 |
| 生命周期 | 是否有创建、复制、废弃、口径变更和负责人机制 |
| 使用体验 | 首屏是否展示关键结论，详情是否能下钻到原因 |
| 二次开发 | 开源 BI 只有在权限、数据集、查询、插件和部署治理明确时才有长期价值 |
