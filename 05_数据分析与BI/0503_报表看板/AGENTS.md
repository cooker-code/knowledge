# 报表看板
## 知识点入口

- 本模块先看宏观流程，再看文章：[流程化知识点总览](knowledge/05_数据分析与BI/0503_报表看板/核心知识点/流程化知识点总览.md)。
- 新文章必须先归入流程节点，再判断是补充、冲突、不同层次还是降权。
- `文章/` 只保留原文锚点，长期知识必须沉淀到 `核心知识点/`。


## 类目定位

| 项 | 内容 |
|---|---|
| 一级类目 | 数据分析与 BI |
| 二级类目 | 报表看板 |
| 核心问题 | 分析结果如何被展示、交互、解释和复用，让使用者更快发现问题并采取动作 |
| 不解决什么 | 不收纳普通可视化工具资讯、Git 可视化工具、项目看板工具，除非它们直接改变 BI 报表设计判断 |
| 用户当前认知假设 | L3：基础图表和看板概念已知，重点补信息架构、图表选择、交互筛选、治理和误导风险 |

## 知识点组织

| 知识点类型 | 应沉淀什么 | 来源吸收方式 |
|---|---|---|
| 看板信息架构 | 指标层级、页面结构、阅读路径 | 多篇案例按结构原则合并，不按模板逐篇保存 |
| 图表选择 | 图表和分析问题的匹配关系 | 同类图表文章合并为判断规则 |
| 交互与筛选 | 维度下钻、联动、权限、默认筛选 | 只吸收能减少误读或提升分析效率的设计 |
| 报表治理 | 口径、刷新、权限、使用率、废弃 | 与指标体系交叉时保留报表侧治理动作 |

## 排重准则

问题指纹：

```text
展示对象 + 图表/交互/治理模块 + 解决的分析问题 + 使用者场景 + 误读风险或行动增量
```

| 判断项 | 排重规则 |
|---|---|
| 只是工具推荐 | 不进入核心知识点 |
| 只是好看的模板 | 不沉淀，除非能抽象成信息架构准则 |
| 与智能问数重叠 | 自然语言问数链路放语义层与智能问数，结果呈现规则可引用到这里 |
| 与经营分析重叠 | 经营动作归经营分析，展示承载归报表看板 |
