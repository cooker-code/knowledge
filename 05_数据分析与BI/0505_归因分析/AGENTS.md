## 用户认知重点

| 项 | 内容 |
|---|---|
| 已知基础 | 了解指标波动时需要找原因，知道漏斗分析和维度下钻的基本方法 |
| 待补边界 | 相关分析 vs 因果推断的边界；归因结论如何转化为行动建议 |
| 易偏差点 | 维度下钻找到相关性就断言因果；归因结论缺行动可行性验证 |
| 优先抽取 | 归因流程设计、维度选择策略、因果推断适用场景、行动闭环 |

## 排重指纹

```text
归因对象（漏斗/指标/用户）+ 分析方法 + 核心机制 + 解决问题 + 相关/因果/行动边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是漏斗分析 | 区分转化率计算、断点定位、用户行为路径和归因模型 |
| 都是维度拆解 | 区分加法分解、乘法分解、同比环比和贡献度计算 |
| 因果推断方法 | 若服务建模训练（DID、合成控制）归入 `06_机器学习`；若服务业务分析归本目录 |
| 实验与归因混淆 | 有随机分配就是 AB 实验，归入 `0506_实验分析` |

## 已覆盖问题（按三级节点）

| 节点 | 已覆盖 | 还缺 |
|---|---|---|
