# 归因分析
## 知识点入口

- 本模块先看宏观流程，再看文章：[知识地图](050500_核心知识点/知识地图.md)。
- 新文章必须先归入流程节点，再判断是补充、冲突、不同层次还是降权。
- `文章/` 只保留原文锚点，长期知识必须沉淀到 `050500_核心知识点/`。


## 类目定位

| 项 | 内容 |
|---|---|
| 一级类目 | 数据分析与 BI |
| 二级类目 | 归因分析 |
| 核心问题 | 指标变化发生后，如何拆解原因、定位贡献因素、识别可行动路径 |
| 不解决什么 | 不把所有因果推断、统计建模或运营复盘都放进来；只有服务“为什么变了、下一步做什么”的内容才归入 |
| 用户当前认知假设 | L3：知道常见分析框架，重点补归因边界、维度拆解、因果误判和行动闭环 |

## 知识点组织

| 知识点类型 | 应沉淀什么 | 来源吸收方式 |
|---|---|---|
| 维度拆解 | 时间、渠道、人群、产品、区域等拆解路径 | 多篇案例合并为拆解准则和反例 |
| 漏斗归因 | 转化链路各环节的变化定位 | 与实验/经营分析文章交叉时，只吸收归因机制 |
| 业务归因 | 策略、价格、活动、供给、需求变化 | 要写清证据链，不把相关性当原因 |
| 因果边界 | 归因分析和因果推断的区别 | 冲突文章必须写明证据不足或待实验验证 |

## 排重准则

问题指纹：

```text
指标变化对象 + 拆解维度/漏斗/因果边界 + 证据链 + 行动建议 + 认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是波动分析 | 按拆解路径、证据链和行动闭环判断是否新增 |
| 只是列举分析方法 | 不进入核心知识点 |
| 只给结论不给证据 | 降权，写入冲突或边界 |
| 与实验分析重叠 | 实验设计和显著性归实验分析，实验后解释可引用到归因分析 |
