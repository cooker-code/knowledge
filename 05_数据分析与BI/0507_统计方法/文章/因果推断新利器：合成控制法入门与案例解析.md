# 因果推断新利器：合成控制法入门与案例解析

在评估一项处理（如新药、广告、产品功能）的因果效应时，A/B测试（随机实验）被公认为黄金标准。然而，在许多现实场景中，严格的A/B测试并不可行。特别是当处理组和对照组之间存在**污染**（如网络效应、溢出效应）时，或者当处理成本高昂、只能在少数单位（如城市、国家）上实施时，传统方法便会失效。

在这些情况下，一种名为\*\*合成控制（Synthetic Control）\*\*的方法应运而生，并迅速成为过去十年政策评估和业界因果推断领域最重要的创新之一。其核心思想是，当我们只有一个或少数几个处理单元时，我们可以利用多个未处理单元的数据，**合成**一个“虚拟”的对照组，这个对照组在处理前的行为轨迹与处理单元高度相似，从而为我们提供一个理想的反事实参照物。

本文将通过一个生动的案例——**评估自动驾驶汽车对网约车平台收入的影响**——来深入浅出地介绍合成控制法的基本思想和应用。

> **因果推断（Causal Inference）**：致力于探索变量之间“因果关系”的统计学分支。与仅仅发现变量之间“相关关系”不同，因果推断试图回答“如果X发生改变，Y会如何变化？”这类问题。

## 一、问题的提出：自动驾驶汽车案例

假设你是一个网约车平台，想测试在你的车队中引入自动驾驶汽车的效果。你会发现，传统的A/B测试在这里困难重重：

1. 1. **溢出效应**：如果自动驾驶汽车更高效，它们会服务更多客户，从而减少普通司机（对照组）的客源，这种溢出效应会**污染**实验结果。
2. 2. **成本高昂**：在单个城市大规模部署自动驾驶汽车是一项极其昂贵的干预。

因此，我们决定只选择一个城市——迈阿密——作为处理单元，在2013年引入自动驾驶汽车。我们的数据集包含了2002-2019年间美国46个大城市的时间面板数据，包括收入、人口密度、GDP等社会经济变量。

迈阿密

### 1.1 传统方法的局限

首先，我们检查一下处理组（迈阿密）和对照组（其他城市）在处理前是否具有可比性。

| 特征 | 对照组 (平均) | 处理组 (平均) |
| --- | --- | --- |
| **density** | 2062.24 | 4585.34 |
| **employment** | 0.65 | 0.62 |
| **gdp** | 63231.13 | 54964.71 |
| **population** | 2.65e+06 | 5.83e+06 |
| **revenue** | 100.91 | 99.88 |

可以看到，两组在人口密度、GDP、人口规模等方面存在显著差异，**不具有可比性**。简单地比较处理前后两组的平均收入差异，或者只比较迈阿密自身处理前后的收入变化，都无法得出可靠的因果结论，因为我们无法排除其他混杂因素的影响。

从下图我们可以看到，虽然迈阿密的收入在2013年后有所增长，但其他城市的收入也呈现增长趋势。我们很难从图中直接判断自动驾驶汽车的引入是否带来了额外的增长。

## 二、合成控制法的基本思想

答案是肯定的。合成控制法的核心思想是：**我们可以通过对多个未处理单元（其他城市）进行加权平均，来构造一个“合成迈阿密”。这个合成迈阿密在引入自动驾驶汽车之前的各种特征（特别是收入趋势）都与真实的迈阿密高度相似。**

这样，在引入处理之后，真实的迈阿密与这个“合成迈阿密”之间的差异，就可以被归因为处理（即自动驾驶汽车）带来的因果效应。

### 2.1 与线性回归的类比

你可能会觉得这与线性回归非常相似。没错！

* • 在线性回归中，我们通常有**多个观测单元**和**少量特征**，我们试图用特征的线性组合来预测结果。
* • 在合成控制中，我们反过来，有**多个时间段**和**少量控制单元**，我们试图用控制单元在各个时间段上的线性组合，来拟合处理单元在处理前的行为轨迹。

本质上，我们可以通过**转置数据**，将时间段视为“观测单元”，将城市视为“特征”，然后运行一个回归模型，来找到最优的权重组合。

数据转置

### 2.2 初步尝试：无约束的线性回归

我们首先尝试使用一个简单的线性回归来找到权重，并构建“合成迈阿密”。

```
from sklearn.linear_model import LinearRegression  
  
# 假设 synth_predict 是一个封装好的函数，用于训练模型并预测  
coef = synth_predict(df, LinearRegression(), treated_city, treatment_year).coef_
```

然后，我们绘制出真实迈阿密的收入、合成迈阿密的收入以及它们之间的差异。

初步结果

从上图看，效果似乎非常显著！引入自动驾驶汽车后，真实迈阿密的收入（蓝色实线）远高于合成迈阿密（橙色虚线）的预测，表明处理带来了巨大的积极影响。

**但先别高兴得太早！** 这个模型存在两个严重的问题：

1. 1. **过拟合**：在处理前，预测线与实际线几乎完美重合。考虑到迈阿密收入本身的高波动性，这显然是模型过拟合了数据中的噪声。
2. 2. **负权重与可解释性**：检查回归系数（即权重），我们会发现许多负值。

负权重

我们可以理解“合成迈阿密”是“0.2个圣路易斯 + 0.15个俄克拉荷马”的组合，但“-0.15个密尔沃基”是什么意思？这在现实中没有直观的解释，也违背了我们构建一个“加权平均”反事实的初衷。

为了解决这两个问题，我们需要对权重施加一些限制。

在下一篇文章中，我们将探讨如何通过**约束权重**来改进合成控制模型，得到一个更稳健、更可解释的结果，并介绍如何进行统计推断，以判断我们观察到的效应是否显著。
