# 050700_统计方法知识地图

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 05_数据分析与BI / 07_统计方法 |
| 解决的问题 | 先定义 `07_统计方法` 的应用流程，再把文章吸收到流程节点 |
| 典型输入 | 本地文章、已有 AGENTS、已有核心知识点、后续新增资料 |
| 典型产出 | 流程节点知识、判断准则、风险边界、待补证据 |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| B00 业务问题 | 业务问题如何转成指标、维度、口径和动作 | 分析不是出图，而是回答业务问题并能行动 | 经营、指标、用户画像文章进入 |
| B01 指标语义 | 指标、维度、时间窗口、权限和语义层如何定义 | 语义层解决口径一致、可追责和可复用 | 指标、语义层、Text-to-SQL 文章进入 |
| B02 数据校验 | 分析数据从哪里来，如何验证准确性和完整性 | 问数系统必须能解释 SQL、校验结果并暴露不确定性 | SQL、校验、问数文章进入 |
| B03 分析方法 | 描述、对比、归因、实验、统计或因果如何选择 | 方法服务问题类型，不能把相关性当因果 | 归因、实验、统计文章进入 |
| B04 展示决策 | 看板、可视化、报告如何让结论被使用 | 报表价值在稳定监控和决策触发 | 看板、可视化、报告文章进入 |

## 文章进入时的对比规则

| 判断项 | 处理方式 |
|---|---|
| 补充 | 文章提供新的模块边界、失败模式、实现步骤、验证指标或工程权衡时，写入对应流程节点 |
| 冲突 | 文章观点和当前节点准则不一致时，先记录冲突原因，不直接覆盖已有沉淀 |
| 不同层次 | 入门教程、源码机制、生产实践、产品发布分属不同层次，必须标注层次后再吸收 |
| 降权 | 只有标题党、工具清单、无证据收益、泛泛最佳实践时，不进入核心知识点 |

## 使用方式

- 先读本文件的宏观流程，再读具体主题知识点。
- 新文章不要单独写摘要，先归入流程节点，再判断补充、冲突、不同层次或降权。
- 如果某个节点已有多篇文章，后续应拆成更细的 `050700_核心知识点/<主题>.md`，而不是继续扩展文章列表。
