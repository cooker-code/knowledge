# 统计方法
## 知识点入口

- 本模块先看宏观流程，再看文章：[流程化知识点总览](knowledge/05_数据分析与BI/0507_统计方法/核心知识点/流程化知识点总览.md)。
- 新文章必须先归入流程节点，再判断是补充、冲突、不同层次还是降权。
- `文章/` 只保留原文锚点，长期知识必须沉淀到 `核心知识点/`。


## 类目定位

| 项 | 内容 |
|---|---|
| 一级类目 | 数据分析与 BI |
| 二级类目 | 统计方法 |
| 核心问题 | 统计概念如何帮助分析判断、解释不确定性、避免把相关当因果 |
| 不解决什么 | 不收纳纯数学推导大全；不服务分析判断的公式细节不进入核心知识点 |
| 用户当前认知假设 | L3：基础概念大多可略过，重点补适用条件、误用场景、业务解释和与实验/归因的关系 |

## 知识点组织

| 知识点类型 | 应沉淀什么 | 来源吸收方式 |
|---|---|---|
| 不确定性表达 | 置信区间、置信水平、效应量 | 多篇解释文章合并为业务判断规则 |
| 相关与因果 | 相关、回归、贝叶斯网络、因果推断边界 | 冲突观点要写清证据要求 |
| 回归与估计 | 回归、最大似然、分布假设 | 只吸收影响分析解释的部分 |
| 准实验方法 | 双重差分、事件研究、合成控制 | 与实验分析交叉时互相引用 |

## 排重准则

问题指纹：

```text
统计概念 + 服务的分析判断 + 前提假设 + 误用风险 + 认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 只是公式推导 | 不进入核心知识点 |
| 能解释业务判断 | 合并到对应统计知识点 |
| 与实验分析重叠 | 方法概念放这里，实验流程和决策放实验分析 |
| 与归因分析重叠 | 因果边界放这里，业务拆解路径放归因分析 |
