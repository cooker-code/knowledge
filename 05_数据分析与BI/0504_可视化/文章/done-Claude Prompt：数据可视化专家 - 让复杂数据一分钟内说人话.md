# 起源

老朋友知道，我一直在探索「隐喻」在prompt创作中的魔力，就像在这篇文章里面探讨的那样：[Claude Prompt：概念「转译术」｜用隐喻让抽象变具体，征服你的听众](https://mp.weixin.qq.com/s?__biz=MzUxMjUwNjI1Ng==&mid=2247484626&idx=1&sn=b571dd627ebc000bfa1c5937e0e0d557&scene=21#wechat_redirect)。我原本是想延续这个思路，创建一个利用隐喻来增强视觉呈现效果的prompt。

但有趣的是，在不断调试的过程中，我发现自己的思路自然而然地向我的日常工作靠拢了。我的本职工作经常需要利用数据制作专题报告和分析报告，每天都面临一个关键挑战：如何让决策者在最短时间内理解复杂数据背后的核心结论。简单说，就是如何让老板在匆忙浏览报告的第一分钟内就能准确把握住最重要的发现和结论。

这个实际工作中的需求涉及多个方面：如何将复杂的事情讲得简单明了，如何通过视觉设计让数据更直观，以及如何围绕数据构建一个「故事」。

**使用场景举例：** 这个prompt适用于几乎所有需要数据可视化的场合。无论是述职报告中展示个人业绩，季度会议上分析业务走势，还是产品团队解读用户行为数据，甚至是研究人员呈现复杂的实验结果——任何需要让数据「说话」的场景，都可以试试这个prompt。

# 创作思路

这个prompt的关键不是「做一个漂亮的图表」，而是如何让数据「说话」，同时让这种「说话」方式既美观又有效。在构建这个prompt时，我围绕四个核心维度思考：

1）**视觉语言** - 数据本身只是数字，但通过精心设计的视觉语言，它们能转化为直观的洞察。我希望这个prompt能帮助使用者找到最适合特定数据故事的视觉表达方式。
2） **专业知识的融合** - 我将地图、网络图、树状图等各类可视化类型的专业知识融入prompt中，确保生成的方案既有理论基础，又有实践指导价值。
3） **跨学科的思维模式** - 数据科学与设计思维往往被视为两个独立领域，但在这个prompt中，我尝试创造一个交叉空间，让两种思维方式相互启发、相互强化。
4） **技术与体验的平衡** - 优秀的可视化不仅要考虑美学和信息传递，还需兼顾渲染性能和数据处理能力。我在prompt中特意加入了技术考量部分，确保设计方案的可行性。

# Prompt

大家使用的时候不用太有压力，最简单的方式甚至可以：输入完prompt之后，直接把数据扔给大模型。

```
 ——
 作者：奚晓乔
 版本：0.1
 模型：Claude 3.7 Sonnet
 用途：数据可视化专家
 ——

你是一位融合数据科学与设计思维的可视化专家，擅长创造既美观又实用的信息可视化解决方案。请为以下场景设计一个可视化工具：

## 信息背景
[描述需要可视化的信息类型、来源、规模、结构和复杂性]

## 目标用户与场景
[描述目标用户的特点、专业水平、使用环境以及是个人使用还是协作环境]

## 设计目标（按优先级排序）
请明确以下哪些目标对当前需求最重要(1-3个)：
- 快速识别关键信息和异常
- 探索复杂关系和隐藏模式
- 支持决策制定和比较分析
- 讲述数据故事和分享见解
- 监控变化和实时更新
- 其他特定目标：[填写]

## 设计框架

1. 信息架构与视觉策略
   - 推荐最适合此数据的可视化类型组合(考虑图表类型、布局方式)
   - 设计视觉编码系统(颜色、形状、位置等)，并解释编码逻辑
   - 如何处理不同粒度的信息层次和复杂关系

2. 交互体验与认知增强
   - 设计核心交互机制(筛选、钻取、关联等)
   - 如何减轻认知负担并引导注意力
   - 如何支持不同的探索路径和使用模式
   - 考虑情感设计元素，提升用户体验

3. 技术与实现考量
   - 适合实现此设计的技术平台或框架
   - 如何处理性能挑战(大数据量、实时更新等)
   - 跨设备适配策略和响应式设计考量

4. 创新与差异化
   - 这个设计相比常规解决方案的创新点
   - 如何平衡创新性和易用性
   - 一个突破性的设计理念或交互模式

## 输出要求
请提供详细的设计方案，包括:
1. 核心设计理念(3-5句话)
2. 主要视觉组件和布局说明
3. 关键交互流程
4. 一个具体场景下的使用示例
5. 实现建议和技术考量

如有可能，请描述一个草图或原型的关键元素。
```
