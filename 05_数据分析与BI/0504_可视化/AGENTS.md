## 用户认知重点

| 项 | 内容 |
|---|---|
| 已知基础 | 了解常用图表类型，接触过 ECharts 或 AntV 库 |
| 待补边界 | 何时用 ECharts vs AntV vs D3；大数据量渲染的性能边界 |
| 易偏差点 | 复杂图表 ≠ 清晰表达；动画效果 ≠ 分析价值 |
| 优先抽取 | 图表选择决策树、视觉编码准则、大数据量渲染策略、交互设计模式 |

## 排重指纹

```text
可视化对象 + 技术层级（图表库/渲染引擎/交互）+ 核心机制 + 解决问题 + 数据量/渲染性能/认知负担边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是 ECharts | 区分配置项使用 vs 渲染优化 vs 自定义扩展 |
| 都是图表选择 | 区分数据关系类型（比较/分布/关联/流向/时间序列）和用户任务类型 |
| 报表看板工具使用 | 归入 `0503_报表看板` |
| 图表在前端应用中的实现 | 按前端框架归入 `07_工程与架构/0702_前端工程` |
| MCP/Skill/Claude Code 工具机制 | 归入 `02_Agent与AI工程` 对应目录 |
| AI Excel/自动分析助手 | 归入 `0501_语义层与智能问数/050102_智能数据分析助手` |
