## 核心问题

图表工程生态要区分图形语法、渲染引擎、移动端图表、图布局、智能推荐和上层业务看板，不能把所有 AntV 文章都当作同一个知识点。

## 判断准则

| 层级 | 关注点 |
|---|---|
| 图形语法 | 数据映射、坐标、标记、编码、交互 |
| 渲染引擎 | Canvas、SVG、WebGL、性能、跨端能力 |
| 图布局 | 节点关系、布局算法、可读性、交互探索 |
| 智能图表 | 数据语义识别、图表推荐、解释和校验 |
| 业务看板 | 数据集、筛选器、权限、发布和治理 |
| 开发提效工具 | 只能作为生态了解，不能替代图表设计和性能判断 |
| 社区与版本发布 | 只记录职责边界和生态变化，不能把活动宣传当选型结论 |

## 认知校准点

| 偏差 | 处理 |
|---|---|
| 把库发布当成选型结论 | 必须补版本、场景、性能和生态约束 |
| 只看 ECharts vs AntV | 还要区分 D3、Vega、Plot、BI 内置图表和自研图形引擎 |
| Skill 能画图不等于可用于决策 | 仍要检查数据口径、图表语义和输出可复核性 |
