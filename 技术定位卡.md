# 技术定位卡模板

维护输出模板

本文件定义三类输出：

- 二级类目 `AGENTS.md`：某个技术认知领域的认知入口、排重指纹和三级节点索引。
- 三级技术/主题节点 `AGENTS.md`：具体技术或稳定专题的纵向知识地图。
- `编号_核心知识点/` 下的主题 `.md` 文件：从有价值文章中提取出的核心知识点。

## 实践门槛

只有阅读投入建议为 `实践` 时必须填写。其他文章可以写"不适用"。

| 门槛 | 判断 | 证据 |
|---|---|---|
| 可运行 | 是 / 否 | 最小例子、命令、配置、SQL、代码路径 |
| 可验证 | 是 / 否 | 输入、输出、指标、对照组、验收标准 |
| 可排障 | 是 / 否 | 失败模式、日志信号、定位路径 |
| 可迁移 | 是 / 否 | 能否迁移到用户已有场景 |
| 结论 | 实践 / 降为精读 | 如果任一关键门槛不满足，不能判为实践 |

## 跨域判断

如果一个技术可被多个领域使用，必须明确"本体"和"场景"的区别。

| 问题 | 判断 |
|---|---|
| 它本体属于哪里 | 按 `目录划分.md` 的归类优先级判断 |
| 这篇文章为什么可能跨域 | 例如 Airflow 被用于训练流水线、Flink 被用于实时特征、Redis 被用于缓存或状态 |
| 当前文章主问题是否改变分类 | 按 `目录划分.md` 的跨域示例和边界规则判断 |
| 应避免的误归类 | 不因标题出现模型、Agent、数据、BI 等词就直接归类 |
