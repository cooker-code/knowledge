**建立三级节点的触发条件：**

| 条件 | 说明 |
|---|---|
| 某类管理沟通主题积累 ≥3 篇精读文章 | 如向上管理、跨团队协作、汇报表达各自 3 篇 |

## 用户认知重点

| 项 | 内容 |
|---|---|
| 已知基础 | 有实际工作中的沟通协作经验，接触过向上汇报和跨团队合作场景 |
| 待补边界 | 技术人员 vs 管理者的沟通风格差异；跨职能协作中的信息对齐方法 |
| 易偏差点 | 把沟通问题当性格问题；忽视受众认知水平的差异 |
| 优先抽取 | 结构化表达方法、利益对齐策略、冲突处理框架、汇报准则 |

## 排重指纹

```text
沟通场景（向上/向下/跨团队/对外）+ 方法类型 + 核心机制 + 解决问题 + 适用组织环境边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是向上汇报 | 区分日常同步、项目汇报、问题升级和影响力建设 |
| 都是跨团队协作 | 区分信息共享、决策对齐、冲突处理和资源争取 |
| 职业发展相关内容 | 归入 `0801_职业发展`；管理技巧和沟通方法归本目录 |
| 只是励志故事 | 无可操作框架时略读 |
