## 核心问题

沟通边界不是冷漠，而是在尊重他人的同时维护自己的时间、责任和需求，避免讨好、过度解读、完美主义和责任心过剩带来的长期内耗。

## 判断准则

| 问题 | 判断准则 | 行动 |
|---|---|---|
| 讨好 | 是否为了避免冲突而压抑真实需求 | 清晰、直接、诚恳地表达需求和边界 |
| 过度敏感 | 是否把感受当事实、把他人表情当结论 | 分离事实、解释和情绪 |
| 完美主义 | 是否用细节打磨逃避交付反馈 | 先交 60 分版本，再迭代 |
| 责任过剩 | 是否替别人承担本不属于自己的课题 | 做课题分离，把责任归还给责任人 |
| 无效社交 | 是否为了合群牺牲长期目标和精力 | 保持有边界、有质量的低糖关系 |

## 认知偏差

| 常见错误认知 | 正确理解 |
|---|---|
| 拒绝会破坏关系 | 清楚边界能降低猜测和怨气，反而更利于长期关系 |
| 强硬才是不讨好 | 自信表达在讨好和攻击之间，不卑不亢 |
| 性格问题不能改 | 沟通风格可以通过训练和反馈逐步调整 |
| 多社交一定有价值 | 无效社交也是 token 消耗，要看是否带来真实支持、信息或合作 |

## 可执行句式

| 场景 | 句式 |
|---|---|
| 拒绝额外任务 | “我理解这件事重要，但我当前优先级是 X。如果要接，需要调整 Y 的截止时间。” |
| 表达不同意见 | “我的担心是 X，依据是 Y。我建议先用 Z 验证。” |
| 处理情绪倾倒 | “我愿意听你说 10 分钟，但今天没有精力继续深入讨论。” |
| 归还责任 | “这部分我可以提供建议，但最终取舍需要你来定。” |
