## 知识点入口

| 主题 | 用途 |
|---|---|
| [学习方法-AI辅助项目制学习](0802_核心知识点/学习方法-AI辅助项目制学习.md) | AI 时代如何用知识地图、项目、最小行动和输出闭环学习技能。 |
| [学习方法-输出驱动与知识产品化](0802_核心知识点/学习方法-输出驱动与知识产品化.md) | 如何把输入、经验和写作转成可复制、可执行、可反馈的知识产物。 |
| [学习方法-决策与问题解决框架](0802_核心知识点/学习方法-决策与问题解决框架.md) | 决策期限、负空间、系统思维、逆向设计、奥卡姆剃刀等问题解决框架。 |

**建立三级节点的触发条件：**

| 条件 | 说明 |
|---|---|
| 某类学习方法主题积累 ≥3 篇精读文章 | 如刻意练习、知识管理、读书方法各自 3 篇 |
| 能形成稳定的可复用方法论 | 不是只有鸡汤和激励 |
| 有与用户实际工作场景的强结合 | 否则保持原文锚点层 |

候选三级节点：`刻意练习`、`知识管理`、`技术学习策略`

## 用户认知重点

| 项 | 内容 |
|---|---|
| 已知基础 | 有一定的自学经验，知道学习效率有高低之分 |
| 待补边界 | 碎片化学习 vs 系统学习的适用场景；知识管理工具和方法的选择 |
| 易偏差点 | 收藏 ≠ 学习；看懂 ≠ 掌握；学习材料多 ≠ 学习效果好 |
| 优先抽取 | 主动学习方法、知识内化验证、间隔重复策略、学习周期管理 |

## 排重指纹

```text
学习方法类型 + 适用场景 + 核心机制 + 解决问题 + 时间成本/迁移效果边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是知识管理 | 区分捕获、整理、内化和输出的工具与流程 |
| 都是刻意练习 | 区分技能识别、反馈机制、刻意设计和进度评估 |
| 工具使用方法 | 归入 `09_电脑工具`；思维方法归本目录 |
| 只是励志故事 | 无可操作方法时略读 |
