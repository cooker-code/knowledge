Lenny's Podcast 本期访谈的对象是 Max Schoening。Max 做过 Google 产品经理，管过 Heroku 的设计团队，在 GitHub 既做设计负责人，也做工程相关工作，后来两次创业，现在负责 Notion 的产品。 Lenny's Podcast 这期讨论的问题是当 AI 把很多技能普惠之后，人和人的差距会会体现哪里？

Max 给出的答案是 agency。

这个词很难用一个中文词完全对应。它不是简单的主动性，也不是执行力，更像一个人是否相信周围世界可以被改变，并且愿意亲手去改。过去，一个人很容易说自己做不到，因为不会写代码，不懂设计，不会做原型，不熟悉某个工具。现在模型已经把很多技能喂到嘴边了，技能本身不再是硬门槛，门槛变成了你会不会真的去做。

但，agency 在世界上的分布并不均匀。

现在还是有人会问产品经理到底该做什么，设计师到底该做什么，工程师的边界在哪里。另一些人会直接把角色定义改掉，先做一个版本出来，再让别人判断。

Notion 内部的变化就是从一个很小的 playground 开始的。

Max 刚加入 Notion 时，团队在做很多聊天界面。问题是，聊天界面用 Figma 画出来很像 Brett Victor 那个著名的死鱼比喻。AI 产品的交互感不在静态图里，必须真的跑起来，才能感受到模型什么时候慢、什么时候笨、什么时候突然给人惊喜。

于是 Max 和两位设计师搭了一个很粗糙的小工程。这是模型很擅长的领域，和 Notion 主代码库也是隔开的。这样设计师不用一开始就面对十年迭代留下的复杂工程结构，也不用先被终端吓退。进入这个 playground 之后，原型设计变成了一种聊天式的构建。

后来，模型能力继续变强，同一批设计师和产品经理开始少量贡献生产代码。但 Max 一直强调，重点不在于让非工程师多开发几个功能。

他在意人们是否写代码的原因是，这会迫使他们理解自己正在设计的东西。

这句话对产品团队很关键。设计师写代码的价值，不是把工程师的活抢过来。产品经理做原型的价值，也不是把 PRD 省掉。真正的变化是，代码正在成为产品判断的东西本身。如果你在设计一个 agent loop，却完全不理解 agent loop 是怎么工作的，你的判断会停在很表层的位置。

Max 举了一个例子。如果一个设计师会用 Claude Code 或 Codex 调几个界面细节，但不理解 agent loop，他宁愿选择另一个人。后者也许不太会写传统软件，但真正理解 agent loop 怎么运转，并能围绕它设计体验。

这也是为什么他鼓励 Notion 的产品和设计同事用终端，而不是只用图形界面。终端一开始吓人，但一旦进到 Claude Code 或 Codex 里，很多人会慢慢对下面的计算机底层产生好奇，会多理解一点计算机是怎么工作的。

**前 10% 免费了**

AI 对产品工作的第一个改变，Max 概括得很漂亮。

现在每个项目的前 10% 都免费了。

以前你可能要写 PRD、开会、画流程图、争论方案。现在很多时候可以先做一个粗糙版本，把想法摊在桌面上。GitHub 以前有一句话，demos not memos。还有一句话，give me something to react to。别先写备忘录，先给我一个能互动的东西。

AI 让这件事变得便宜。你可以让 10 个 agent 同时探索 10 条路径，看看自己到底判断对了没有。一个创业产品的 0.8 版，过去可能需要一个小团队跑很久，现在一个周末就能有个样子。

但 Max 没有把这件事说成乌托邦。他马上补了一句，即便你慷慨一点，说前 90% 都能被 AI 加速，最后 10% 仍然可能是 90% 的工作。

这就是很多 AI 产品讨论里容易被忽略的部分。原型越来越便宜，探索越来越便宜，功能数量越来越容易膨胀。可真正把东西做可靠、做顺手、做到一亿人能用，仍然是很难的。

Max 用硬件做比喻。如果两个人做一家硬件创业公司，第一版外壳可以用 3D 打印。你能看到层纹，一眼就知道这不是应该卖给用户的最终产品。后面还有很长的路要走，从原型走到制造，从制造走到良率、精度和规模。

软件行业现在的问题是，大家太容易把 3D 打印原型当成产品。讨论里充满了花了多少 token、发了多少功能、一天写了多少代码。Max 反问工程在哪里？一个东西要服务一亿人、十亿人时，谁来保证它真的可靠？

他说自己很喜欢现在的 AI 编程工具，也几乎住在 CLI 里。但他也不客气地批评这些工具的质量。每隔两周，一个三周前修过的问题又回来了。TUI 的渲染帧率也不像一个足够打磨过的产品。软件行业需要找回某种 Apple 式的、像整块铝切出来一样的工程质感。

Max 的观点是需要做好的很显著的东西。

这句话看起来像废话，但他觉得判断标准并不神秘。第一代 iPhone 出来时，很多人不需要解释就知道它好。ChatGPT 刚出现时，也有这种非常显著的好的感觉。

难的是，团队不能躲进山洞里闭门等一个完美版本。Max 更相信 incremental correctness，先增加 shots on goal，增加射门次数，再把长出来的分支收束回那个真正简单、能留下来的核心。

Notion 自己也有这种问题。Max 说用户经常吐槽 Notion 里面有六种 automation primitives，算上 agent 更多。他承认，这就是不同想法都先长出来了。后面必须重新整合。**可塑软件会回来**

Max 早年提了一个概念可塑软件（malleable software）。现在 AI 让这个概念突然变得可感知。

他对可塑软件的定义很简单。软件应该更贴近使用者的利益，而不只是贴近生产软件那家公司的利益。

今天很多 app 像一个胶死的小方块。界面、数据所有权、行为逻辑都粘在一起。你喜欢一个 app 的 90%，只想改掉其中 10%，通常做不到。

另一种是自己维护开源软件，类似 Linux 这当然可塑，但你可能不想从头研究为什么触控板不能用。

AI 让很多人第一次意识到，原来我可以自己做工具。Lenny 也提到，自己现在会做播客准备、录制相关的小工具。这当然是一种可塑软件。但 Max 提醒，如果每个人只是做一堆孤立小工具，那还不够。真正有价值的可塑软件，需要建在鼓励协作、权限、安全、实时协作的平台上。

这也是他为什么不认同简单的 SaaS apocalypse。

很多人说，AI 之后我们不需要 SaaS 了，不需要 Salesforce，不需要 Notion，每个人都能周末自己做一个。但 Max 说，大多数人并不想维护完整的软件栈。

他用了一个很有画面感的比喻。大多数人并不想去打猎，只想去 Costco 买一块已经放在泡沫托盘里的牛排，甚至假装它本来就不是一只动物。软件也是一样。你付费买 as a service，买的很大一部分是维护，是有一群专业的人持续照看这座软件花园。

Max 预计，软件会更多回到 90 年代那种通用工具的形态，像文字处理器、电子表格、FileMaker Pro。只不过这些工具仍然会以服务形式存在。专门领域也还会有专业工具，尤其是安全、权限、合规这类真的需要专家持续走到最后一公里的问题。

这对今天做产品的人是个提醒。不要只问用户能不能自己做一个替代品。要问的是，他愿不愿意长期维护它，愿不愿意处理权限、安全、同步、协作、反馈、边缘情况。很多时候答案是否定的。

Notion AI agent 的价值也在这里。Max 说 agent 需要上下文，不喜欢隔着很窄的孔去访问另一个数据仓库。Notion 作为 connected workspace，里面本来就放着公司大量文档、任务、知识和权限关系。agent 可以在这个环境里漫游，处理企业搜索和权限，这让 Notion 更像一个操作系统，而不只是一个文档工具。

**品味是一台虚拟机**

AI 时代另一个被反复提到的词是**品味**。

很多人说，AI 负责生成，人类负责品味。Max 对这个说法保留意见。他不觉得品味是某种神秘的最后堡垒。相反，他给了一个很工程化的定义。

品味意味着，你能在脑子里跑一台虚拟机。给定一个想法，你能预测某个群体会不会喜欢它。

这里的关键是某个群体。一个东西如果全世界只有你一个人觉得好，那很难说它真的好。但你也不一定要为 80 亿人做产品。你要先决定自己的 in-group 是谁，再训练自己预测他们会怎么反应。

训练方法也不神秘。做东西，给人看，拿反馈，再做。Max 说这几乎像训练模型。输入一个想法，观察人们怎么反应，误差回来，再调整。最有品味的软件设计师，往往有自己的 side project，负责端到端结果，也总是在试新工具，接触别人的想法。那种总跑来问团队要不要试第 49 个新工具的人，虽然有时烦人，但他在积累暴露量。

他还说，应该让自己待在有品味的东西旁边。Notion 的会议室以打字机、Macintosh、Porsche 911 这类经典物件命名。Max 说，坐在这些房间里，如果你真的看一眼那些东西，就会觉得自己手里的工作还不够好。

这段话和他对产品成功的看法连在一起。

Max 不相信多加一个功能，产品就终于会变好。很多团队会掉进这个循环，觉得再加一个功能就好了，再加一个就好了。可真正伟大的产品，通常有一个微小却像超能力一样的核心。

iPhone 是 multi-touch。GitHub 是 Pull Request。Heroku 是 git push heroku master，那一行命令把本地代码变成一个 URL。Notion 是 blocks 和 slash commands。Dropbox 是菜单栏里那个可靠同步的小图标，甚至可靠到你可以用它判断自己有没有网。

一个产品如果没有那个 tiny core superpower，再多功能也救不了。Max 2014 年做过一个接近 Notion 的文档协作产品，团队花了很多时间打磨编辑体验，做 Markdown folding，很多今天 Obsidian 里有的东西他们当时已经做了。可他们没有找到真正的核心。相比之下，早期 Notion 的编辑器很糟，连跨两个 block 选中文本都做不好。但 blocks 这个核心更重要。

这是很痛的产品教训。你可能非常勤奋，非常认真，也可能只是在错误的地方勤奋太久。

**别 FOMO**

访谈后半段，Max 讲了一个很反常识的观点。我们已经有全民基本收入了，它叫知识工作。

他说这句话一半是玩笑，一半认真。很多知识工作者已经坐在有空调的房间里，对着电脑输入一些正确的词和字母，拿到不错的报酬。AI 来了以后，人类还会继续找到，把自己插到 agent loop 中的理由，但人类总会给自己找到位置。

Max 觉得现在硅谷少见地充满了一批并不真正热爱计算机的人。他们当然也想赚钱，Max 说自己也喜欢赚钱。但他警惕一种心态，仿佛这是最后一班车，错过就会掉进永久底层。

这种焦虑会影响一个人怎么度过自己的一生。

他的建议不是躺平。相反，他认为 18 到 25 岁时非常努力工作是一件好事。问题不在努力，而在那种狂躁的 FOMO。历史并不会完全重启，很多事只是换了样子又来一遍。读历史，读计算机科学史，会让人不那么容易被眼前的速度吓住。

那应该如何应对呢？Max 给了一个很适合这篇文章收尾的回答。

去你所在的城市或森林走一走。最好是人造环境。仔细看看周围所有东西，它们都是由并不比你聪明的人造出来的。然后意识到，对于身边大多数东西，只要给你 6 到 9 个月，你都可以弄清楚怎么从零做出来。

这就是 agency 的起点。

不是先等一个正式授权，也不是先等一个岗位说明告诉你该做什么。世界是人造出来的，软件也是人造出来的。AI 只是把很多材料递到了你手边。**真正稀缺的，是你愿不愿意动手去做。**
