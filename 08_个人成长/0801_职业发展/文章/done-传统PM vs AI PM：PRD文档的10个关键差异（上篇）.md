## 核心差异概览：确定性 vs 概率性

### 📌传统产品 vs AI产品的本质区别

**核心差异对比**

| 维度 | 传统产品 | AI产品 |
| --- | --- | --- |
| **逻辑基础** | 确定性规则 | 概率性预测 |
| **需求表达** | 精确的if-then | 期望的效果 |
| **实现方式** | 代码逻辑 | 训练模型 |
| **结果** | 100%可预测 | 有准确率概念 |
| **优化方式** | 改代码逻辑 | 优化模型/增加数据 |
| **可解释性** | 完全可解释 | 部分黑盒 |

**这种本质差异，导致PRD写法完全不同！**

# 差异1：需求定义方式 - 从功能到问题类型

## 传统PRD的需求定义

需求：用户可以搜索商品

功能描述：

1. 1. 用户在搜索框输入关键词
2. 2. 点击搜索按钮
3. 3. 系统返回包含关键词的商品列表
4. 4. 支持按价格、销量排序

搜索逻辑：

* • 商品名称包含关键词（模糊匹配）
* • 或商品分类包含关键词
* • 或商品标签包含关键词
  SQL: SELECT \* FROM products WHERE name LIKE '%keyword%'

## AI PRD的需求定义

需求：智能商品搜索（理解用户意图）

问题类型：

* • NLP文本理解 + 语义搜索
* • 需要理解同义词、相关词、用户意图

输入：

* • 用户搜索词："便宜的手机"
* • 用户历史行为数据
* • 当前上下文（浏览分类、价格范围偏好）

期望输出：

* • 理解意图："用户想要性价比高的手机"
* • 返回相关商品（不仅匹配"手机"，还要"性价比高"）
* • 置信度排序

模型能力要求：

1. 1. 同义词识别："手机" = "智能手机" = "移动电话"
2. 2. 属性提取："便宜" → 价格筛选<2000元
3. 3. 语义理解："学生党必备" → 推荐性价比手机
4. 4. 纠错能力："苹果shouji" → "苹果手机"

效果目标：

* • 搜索结果点击率 > 15%（传统搜索8%）
* • 搜索后下单转化率 > 5%（传统搜索2.5%）

## 差异总结表

| 对比维度 | 传统PRD | AI PRD |
| --- | --- | --- |
| **需求表达** | 精确功能描述 | 问题类型+效果目标 |
| **逻辑定义** | if-then规则 | 模型输入输出 |
| **实现方式** | SQL查询 | 模型推理 |
| **结果可控性** | 100%可预测 | 有准确率，需要测试验证 |
| **PM需要懂** | 业务规则、SQL基础 | AI问题分类、模型能力边界 |

## 🔧 AI产品的6大问题类型

**PM需要在PRD中明确**：

| 问题类型 | PRD必须说明 | 示例 |
| --- | --- | --- |
| **分类** | 有哪些类别，每个类别的定义 | 内容审核：色情/暴力/违规/正常 |
| **预测** | 预测什么指标，预测周期 | 预测未来7天销量 |
| **生成** | 生成什么内容，生成规则 | 生成商品描述，100-200字 |
| **推荐** | 推荐什么，推荐策略 | 推荐商品，基于协同过滤 |
| **检测** | 检测什么目标，检测精度 | 检测图片中的商品，准确率>95% |
| **理解** | 理解什么内容，输出格式 | 理解用户意图，输出结构化数据 |

```
## 功能描述
### 3.1 搜索功能
**入口**：首页顶部搜索框

**交互流程**：
1. 用户输入关键词
2. 点击搜索按钮或按Enter
3. 跳转搜索结果页
4. 展示商品列表

**搜索规则**：
- 匹配商品名称（模糊匹配）
- 匹配商品分类
- 匹配商品品牌
- 匹配商品标签

**排序规则**：
- 默认：综合排序（销量×权重 + 评分×权重）
- 可选：价格升序/降序、销量、最新
```

### AI PRD需求定义

#### 需求背景

当前基于关键词匹配的搜索无法理解用户真实意图，导致：

* • 用户搜"便宜的手机"，无法识别"便宜"这个属性
* • 搜索"学生党必备"，不知道推荐什么
* • 同义词无法识别："手机" ≠ "智能手机"
  数据支撑：搜索无结果率15%，搜索后跳出率68%

#### AI能力定义

##### 问题类型

**NLP语义理解 + 智能搜索**

##### 模型输入（Input）

**必需输入**：

* • query：用户搜索词（String，1-50字符）
* • user\_id：用户ID（用于个性化）

**可选输入**（用于个性化）：

* • user\_profile：用户画像（年龄、性别、消费能力）
* • browse\_history：最近浏览的30个商品ID
* • search\_history：最近10次搜索记录
* • context：上下文（当前浏览分类、价格范围）

* • ✅ 保留：有点击或购买的搜索
* • ✅ 保留：真实用户的搜索行为

* • 设备特征：设备类型、操作系统（5维）
* • 场景特征：搜索入口、当前页面（10维）

```
# 角色
你是XX电商的智能客服助手"小智"，为用户提供专业、友好的咨询服务。

# 能力边界
你可以回答：
1. 订单相关：订单查询、发货进度、物流跟踪
2. 退换货：退货流程、退款进度、换货政策
3. 商品咨询：商品参数、使用方法、搭配推荐

你不能回答：
1. 超出范围的问题（天气、新闻、闲聊）
2. 需要隐私信息的问题（不能直接说密码、支付密码）
3. 投诉类问题（直接转人工）

# 回答原则
1. 【准确性优先】基于知识库和实时数据回答，不确定时转人工
2. 【简洁明了】答案控制在100字以内，分点说明
3. 【提供操作路径】不只回答"是什么"，还要告诉"怎么做"
4. 【情绪识别】如果用户情绪负面（催促/不满/投诉），降低AI置信度阈值，优先转人工
5. 【追问确认】关键信息缺失时（订单号/商品名），主动追问

# 回答格式
- 称呼：您
- 自称：小智
- 开头：根据情况使用"您好"、"好的"、"明白了"等
- 结尾：根据情况使用"还有其他问题吗？"、"如还有疑问，随时找我~"

# 思维链（Chain of Thought）
在生成答案前，按以下步骤思考：
1. 理解用户问题（意图+实体）
2. 检索知识库/查询数据库
3. 组织答案结构
4. 生成自然语言回答
5. 检查答案质量（准确性、完整性、友好度）
```
