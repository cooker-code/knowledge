## 用户认知重点

| 项 | 内容 |
|---|---|
| 已知基础 | 用户熟悉数据工程、AI 工具和组织协作，不需要重复“要学习、要努力、要拥抱 AI”这类口号。 |
| 待补边界 | 数据岗位的可替代部分与不可替代部分、懂业务与替业务负责的边界、AI 产品与传统产品的评估差异。 |
| 易偏差点 | 职业文章容易把焦虑、营销、个案晋升、公司文化神话包装成通用规律。 |
| 优先抽取 | 能转成判断准则、边界条件、反例、行动检查表和复盘问题的内容。 |

## 排重指纹

```text
职业对象 + 能力模块 + 核心机制 + 解决问题 + 适用边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 角色相同 | 继续比较能力模块和组织语境，不直接判重。 |
| 解决问题相同 | 没有新增边界、反例、证据或行动检查时合并。 |
| 只多了案例 | 案例不改变准则时，只追加原文锚点，不新建知识点文件。 |
| 多了观点冲突 | 必须记录为认知校准点，不能直接吸收为结论。 |
| 明显跨域 | 技术本体强于职业问题时，标记迁出候选，不在本模块正式沉淀。 |

## 已覆盖问题

| 主题 | 已覆盖 | 还缺 |
|---|---|---|
| 数据岗位能力模型 | AI 时代数仓/数开/数据分析的能力分层、数据团队边界、数据 x AI 新能力 | 需要后续补真实招聘 JD、团队职责和项目代表作证据。 |
| AI时代技术人稀缺能力 | AI 对原型、编码、取数的平权，以及品味、判断、系统工程、动手性仍然稀缺 | 需要后续补高质量实测和用户自身工作流证据。 |
| 晋升影响力与组织交换 | 影响力不是说服，晋升不是胜任；要有信任、信息、组织贡献、目标感 | 需要后续把个人实际项目复盘映射到这些准则。 |
| 产品角色与AI产品工作方式 | AI PM 的 PRD 从确定性功能转为概率系统、数据需求、评估、降级和监控 | 下篇/更多案例进入时补测试验收、数据闭环和安全合规。 |
| 职业环境与机会判断 | 裁员、行业降温、通勤、时代机会等外部材料只作风险校准，不直接变成人生结论 | 需要后续用更稳定的数据源或个人约束做验证。 |
