一个简单的实战案例：如何让AI帮你自动巡检Web系统，并实时推送结果到微信

在日常运维工作中，定期检查系统状态、监控关键指标是必不可少的工作。传统方式要么人工登录各个系统查看，要么配置复杂的监控告警系统。

今天分享一个更简单、更智能的方案——利用AC-AIBot构建微信通知机器人，让AI成为你的7×24小时运维助手。

**一、认识AC-AIBot**

AC-AIBot是一个基于AI的智能助手框架，本次案例中使用到的核心能力包括：

1. 自动执行预设的任务脚本

2. 访问网页、截取屏幕

3. 通过微信发送消息和文件

4. 实现自动化的工作流程

**二、实战案例背景**

我们团队有两个高频巡检任务：

|  |  |  |
| --- | --- | --- |
| **巡检项目** | **监控地址** | **检查内容** |
| BI作业运维 | databi-xxxx.xxxx.com | 查看作业执行状态 |
| 大模型网关基础设施 | dataai-xxxx.xxxx.com | 检查模型调用成功率，错误率>10%触发告警 |

**三、四步实现智能巡检**

**第一步：定义巡检规则**

在AC-AIBot的规则配置中，创建巡检清单文件规则（如日常运维列表清单规则.md）：

日常运维列表清单  
1. BI作业运维页面：https://databi-xxx.com/page/xxxxx  
2. 大模型网关基础设施：http://dataai-xxx.com/.../ServiceList  
  
操作说明：  
- 查看时需要点击“调用详情”  
- 检查分模型接口调用的成功率情况  
- 若错误率大于10%，则触发告警

注意：对于需要登录的后台管理页面，需提前登录或在规则中配置账号密码，确保登录正常进行。

配置完成后，只需在对话框输入：“帮我进行下系统巡检”，AI就会自动打开浏览器并访问相关页面。注意初次登录时候会提示未登录，需要手工登录相关页面，或者在规则中配置相关账号密码

登录完毕后，重新输入“请帮我进行下业务系统巡检”

**第二步：配置微信通知**

AC-AIBot内置了微信通知机器人（基于 wechat\_agent），配置非常简单，考虑到这个规则使用概率极大，推荐打开AI必须阅读按钮：

规则配置示例（wework.md）：  
注意我让你发微信通知的话，就通过weixin\_agent发给'[好友昵称]'这个微信号。  
注意如果其中需要发文件，需要同步发送一句话的文件概要，便于区分不同任务的临时文件作用，使用md格式进行发送。

快速测试：配置完成后，发送消息“微信发个你好给我”即可验证。

首次使用需扫码登录微信agent，推荐登录小号微信+主要使用号为好友的模式，安全又方便。

**第三步：远程执行巡检脚本**

手机端也能轻松触发！参考入门案例，在手机登录终端后，发送指令：

*“请帮我进行相关任务巡检，巡检结果报告信息通过文件形式发我微信”*

1. 参考入门教程，手机端访问助手，并输入我的描述，等待若干分钟后完成回答

2. 微信收到的小助手的回复文字及对应的巡检报告

3. 查看巡检报告的文件内容

AC-AIBot会自动完成：

✅ 执行系统巡检

✅ 生成巡检报告

✅ 推送报告到微信（文字摘要 + Markdown文件）

**第四步：配置定时任务（进阶）**

在定时任务模块，可以设置自动循环执行我询问的特定问题，以一个简单的公众号结果巡检为例：

|  |  |
| --- | --- |
| **配置项** | **示例** |
| 任务名称 | 每天看下公众号的情况 |
| 执行间隔 | 24小时 |
| Prompt | 帮我看一下今天公众号的浏览情况，并生成一份报告。并将报告通过文字摘要和文件的形式发到微信。 |

设置完成后，系统会按照设定周期自动执行，无需人工干预。

**四、实际运行效果**

**场景一：BI作业巡检**

AC-AIBot自动执行流程：

1.打开BI运维页面

2.截取关键数据区域

3.分析作业状态

4.推送结果到微信

**巡检结果示例：**

✅ BI作业运维页面 - 正常  
关键指标：任务运行分析、ETL排队时长正常  
  
✅ 大模型网关基础设施 - 正常  
关键指标：错误率0.64%，低于10%告警阈值

**场景二：大模型网关巡检（告警场景）**

当检测到某个模型错误率**超过10%**时：

⚠️ 告警通知  
请及时处理！  
  
自动发送告警信息到微信，包含具体异常模型和错误详情。
