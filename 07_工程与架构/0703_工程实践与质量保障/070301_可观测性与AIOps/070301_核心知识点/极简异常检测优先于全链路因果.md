## 纵向理解

| 维度 | 判断 |
|---|---|
| 全局架构 | 日志采集 -> 模板化 -> 时间窗口统计 -> 历史基线对比 -> 异常候选排序 -> 人工确认 |
| 本文位置 | 只讲异常检测和根因候选起步，不覆盖完整监控平台 |
| 核心机制 | 时间窗口、历史对照、暴涨规则、出现时间排序 |
| 使用链路 | 清洗日志模板 -> 当前窗口统计 -> 过去 24 小时对照 -> 标记异常 -> 输出候选 |
| 前置条件 | 日志模板质量、时间戳准确、历史数据足够、维度不过度爆炸 |
| 边界 | 不能证明因果，无法替代拓扑、变更记录和人工复核 |

## 横向对标

| 对标技术 | 实现方式 | 优势 | 劣势 | 适合场景 |
|---|---|---|---|---|
| 静态阈值 | 超过固定阈值告警 | 简单稳定 | 误报多，无法适应周期变化 | 基础稳定指标 |
| 极简异常检测 | 当前窗口和历史窗口对比 | 快速落地、解释明确 | 对复杂因果链支持弱 | AIOps 起步、日志异常候选 |
| 全链路因果推断 | 构建服务拓扑和因果关系 | 理论完整 | 数据要求高、成本高、建设慢 | 成熟可观测平台 |
| LLM RCA | 用模型汇总日志、变更、指标 | 表达和整合能力强 | 易幻觉，需证据约束 | 辅助诊断和复盘报告 |
