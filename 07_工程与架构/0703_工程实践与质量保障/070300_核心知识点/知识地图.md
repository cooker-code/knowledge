# 070300_工程实践与质量保障知识地图

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 07_工程与架构 / 0703_工程实践与质量保障 |
| 解决的问题 | 先定义系统交付保障流程，再把文章吸收到接口、测试、安全、部署、观测和运行时节点 |
| 典型输入 | 本地工程实践文章、测试/安全/观测/部署/运行时资料、后续新增资料 |
| 典型产出 | 工程判断准则、验证指标、失败模式、排障路径、回滚动作和待补证据 |
| 用户阅读目标 | 读到一篇工程文章时，先知道它补了上线前后的哪个保障环节 |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| G00 需求边界与风险分级 | 哪些接口、任务、权限、部署和运行路径有高风险 | 工程保障从风险分级开始，不从工具开始 | 只吸收能说明风险、影响面和验收标准的内容 |
| G01 接口契约与副作用边界 | API、网关、协议、幂等、错误语义和副作用如何定义 | 接口不是调用通道，而是契约和副作用边界 | 接口、网关、API 测试和接口优化文章进入 |
| G02 测试设计与质量门禁 | 单测、集成、E2E、回放、压测、报告和门禁怎么建立 | 测试价值在可复现输入、断言和失败信号 | 测试、压测、流量回放、质量平台文章进入 |
| G03 安全权限与审计 | 认证、授权、2FA、LDAP、凭证和审计如何控制 | 权限边界必须可验证、可追踪、可撤销 | 安全、认证、权限、审计文章进入 |
| G04 构建、部署与环境可复现 | 镜像、配置、依赖、部署和回滚如何稳定 | 环境问题要能复现、回滚和隔离 | Docker、部署、发布、配置文章进入 |
| G05 运行观测与故障定位 | 日志、指标、链路、告警、RCA 如何定位问题 | 观测先给证据，不把相关性写成因果 | Prometheus、日志、AIOps、RCA 文章进入 |
| G06 运行时容量与稳定性 | 线程池、调度、资源隔离和容量边界如何影响稳定性 | 运行时机制要落到容量、拒绝策略和失败模式 | 并发、线程池、运行时文章进入 |
| G07 复盘、回滚与规则沉淀 | 故障和缺陷如何转成可复用规则 | 复盘要沉淀到门禁、监控、回滚或开发规则 | 事故复盘、质量改进文章进入 |

## 当前已吸收知识点

| 流程节点 | 知识点 | 关系 | 来源路线 | 吸收结论 |
|---|---|---|---|---|
| G01 | 接口文章必须区分契约、网关、优化和测试工具，不再混为“API” | 纠偏 | 接口契约与网关 | 后续精读优先抽副作用边界和错误语义 |
| G02 | 测试文章按测试层级和失败信号拆，不按工具名堆叠 | 纠偏 | 测试质量 | 质量路线是 P0，候选实践较多 |
| G03 | 安全文章必须显式记录权限边界、审计和可撤销性 | 补充 | 安全权限 | 文章少但风险高，后续补证优先 |
| G04 | Docker 资讯只有形成环境复现和发布判断时才沉淀 | 降权 | 部署发布与环境 | 版本/收费资讯默认略读 |
| G05 | AIOps 不能把相关性当因果，异常检测优先于臆测式 RCA | 已沉淀 | 可观测性与 AIOps | 已有 [极简异常检测优先于全链路因果](../070301_可观测性与AIOps/070301_核心知识点/极简异常检测优先于全链路因果.md) |
| G06 | 运行时文章要落到容量、拒绝策略、隔离和排障信号 | 补充 | 并发运行时 | ThreadPoolExecutor 候选精读 |

## 待补缺口

| 流程节点 | 缺什么 | 后续文章进入时怎么判断 |
|---|---|---|
| G01 | 契约测试、错误语义、幂等、副作用边界 | 是否说明输入输出、错误码、重试、超时和幂等风险 |
| G02 | 单测/集成/E2E/回放/压测之间的分层决策 | 是否有可复现输入、断言、隔离、清理和失败信号 |
| G03 | 威胁模型、凭证隔离、权限审批、审计 | 是否能说明攻击面、权限边界和撤销路径 |
| G04 | 环境可复现、灰度、回滚、配置治理 | 是否能形成上线前后动作和失败回滚 |
| G05 | 日志模板化、指标基线、告警降噪、变更关联 | 是否能区分证据、相关性和因果 |
| G06 | 线程池以外的运行时机制 | 是否能说明调度模型、容量边界、资源隔离和排障指标 |
| G07 | 复盘如何改写门禁和规则 | 是否能把事故/缺陷沉淀成可执行检查项 |

## 使用方式

- 先读本文件的宏观流程，再进具体路线目录。
- 新文章不要单独写摘要，先归入路线和流程节点。
- 精读候选必须先和本目录已有流程节点、工程保障路线和核心知识点的问题指纹对比。
- 如果某条路线文章变多，优先抽 `070300_核心知识点/<主题>.md`，不要继续扩大文章列表。
