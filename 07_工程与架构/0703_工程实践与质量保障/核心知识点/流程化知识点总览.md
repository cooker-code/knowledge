# 工程实践与质量保障流程化知识点总览

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 07_工程与架构 / 0703_工程实践与质量保障 |
| 解决的问题 | 把接口、安全、测试、观测、发布、并发运行时等横切文章吸收到同一条工程保障流程 |
| 典型输入 | 本地文章、已有 AIOps 知识点、后续新增工程实践材料 |
| 典型产出 | 判断准则、验证动作、排障路径、风险边界、可复盘证据 |

## 宏观流程

| 节点 | 节点问题 | 当前吸收位置 | 待补缺口 |
|---|---|---|---|
| G00 接口契约 | API、协议、网关、幂等、错误语义如何设计 | `AI网关`、接口优化、SPI/API、UnoAPI 等原文 | 需要拆出“接口契约与副作用边界”主题知识点 |
| G01 质量验证 | 单测、集成测试、回放、压测、测试报告如何形成质量门禁 | 测试技能、ReportPortal、流量回放、AB 测试可信度、自动化测试原文 | 需要区分测试方法、测试平台、测试生成工具 |
| G02 安全权限 | 认证、授权、审计、供应链、非法访问如何控制 | GitHub 2FA、Hermes 登录、连续登录 SQL 等原文 | 需要补“安全模型”和“权限边界”判断，不把普通 SQL 题误吸收 |
| G03 发布运维 | 构建、配置、镜像、发布、回滚、健康检查如何闭环 | DockerHub、版本发布、VS Code/JupyterLab/Flutter 发布等原文 | 发布资讯需要大量降权，只保留迁移风险和发布判断 |
| G04 运行观测 | 日志、指标、链路、告警、AIOps、RCA 如何定位问题 | Prometheus、structlog、日志实践、AIOps 子目录 | RCA 必须标证据等级，不能把相关性当因果 |
| G05 并发运行时 | 线程池、协程、WAL、MemTable、压测容量如何影响稳定性 | ThreadPoolExecutor、WAL/MemTable、压测工具、Python 动态导入等原文 | 需要把运行机制和工具宣传分开 |

## 文章进入时的判断表

| 判断项 | 必答问题 |
|---|---|
| 流程节点 | 文章补的是 G00-G05 的哪个环节 |
| 可验证性 | 是否提供输入、输出、指标、日志、失败样例或验收标准 |
| 可排障性 | 是否能留下排障信号、定位路径或回滚动作 |
| 可迁移性 | 是否能迁移到用户已有的数据工程、Agent 工程或后端服务场景 |
| 降权理由 | 如果只是发布资讯、工具清单、标题党或无基线收益，为什么不进入核心知识点 |
