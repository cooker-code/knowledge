## 排重指纹

```text
工程路线 + 流程节点（契约/测试/安全/部署/观测/运行时）+ 核心机制 + 解决问题 + 生产边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是 API Gateway | 按契约、副作用、错误语义、性能边界拆分 |
| 都是测试 | 按设计、流量回放、E2E、压测、报告、平台拆分 |
| 都是部署 | 按 GitOps、CI/CD、灰度、回滚、环境一致性拆分 |
| 只是工具清单 | 缺失败模式或验证指标时不新建 |
| 跨路线交叉 | 容器化本体进 0704；Agent 安全沙箱进 0210 |
