## 宏观流程

| 节点 | 节点问题 | 文章吸收规则 |
|---|---|---|
| G00 接口契约 | API、协议、网关、幂等、错误语义、输入输出边界怎么设计 | 只吸收能说明契约边界、副作用、一致性或验证方式的内容 |
| G01 质量验证 | 单测、集成测试、契约测试、回放、压测、验收指标怎么建立 | 只吸收可复现的测试设计、质量门禁和失败信号 |
| G02 安全权限 | 认证、授权、审计、依赖、供应链、沙箱边界怎么控制 | 只吸收有威胁模型、权限边界或审计证据的内容 |
| G03 发布运维 | 构建、配置、部署、灰度、回滚、健康检查怎么闭环 | 版本发布资讯默认略读，只有形成发布判断或回滚动作才沉淀 |
| G04 运行观测 | 日志、指标、链路、告警、RCA、AIOps 如何定位问题 | 不把相关性写成因果；必须保留证据等级 |
| G05 并发运行时 | 线程池、协程、事件循环、WAL、内存结构等运行机制如何影响稳定性 | 优先沉淀机制、容量边界、失败模式和排障指标 |

## 排重准则

问题指纹：

```text
工程节点 + 技术对象 + 核心机制 + 解决问题 + 验证/排障证据 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是接口优化 | 按契约设计、性能优化、网关治理、协议语义拆分 |
| 都是测试文章 | 按测试层级、输入构造、断言方式、质量门禁拆分 |
| 都是可观测性 | 按日志、指标、链路、异常检测、根因候选拆分 |
| 都是发布资讯 | 没有迁移准则或风险边界时只保留原文锚点 |
| 都是并发运行时 | 比较调度模型、状态管理、原子性、容量边界和失败模式 |
