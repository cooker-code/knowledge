# 070104_Rust知识地图

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | `07_工程与架构/0701_后端架构/070104_Rust` |
| 解决的问题 | Rust 作为后端与系统工程语言时，如何组织语言语义、项目结构、并发 I/O、服务框架、系统集成和生产化治理 |
| 典型输入 | Rust 文章、Cargo 工程实践、Tokio/Axum/Actix 案例、流式处理和系统安全实践 |
| 典型产出 | Rust 工程判断准则、文章锚点、后续精读候选、可验证实验动作 |
| 用户阅读目标 | 判断文章补的是 Rust 哪个工程流程节点，而不是只看标题是否出现 Rust |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| R00 语言语义与安全边界 | 所有权、借用、生命周期、unsafe 如何约束系统边界 | 内存安全、资源生命周期、并发安全、边界隔离 | 只吸收能影响架构判断的语义；基础教程降权 |
| R01 Cargo、crate 与项目结构 | 包、库、二进制、workspace、feature 如何组织工程 | 依赖治理、模块边界、构建产物、供应链 | 能对比 Maven、Python packaging、Node package 管理时优先吸收 |
| R02 类型建模与错误处理 | 类型、trait、Result、错误层级如何表达业务与系统边界 | 类型边界、错误传播、可测试性、可演进性 | 只讲语法清单降权；能落到服务分层则精读 |
| R03 并发、异步运行时与任务调度 | Tokio/async 如何管理任务、取消、超时、资源限制 | 调度、背压、取消、超时、任务泄漏 | 必须看失败模式和资源边界，不能只看性能口号 |
| R04 I/O、零拷贝、流式处理与背压 | 大文件和网络流如何平衡吞吐、内存与延迟 | 零拷贝、buffer、背压、批量、内存上限 | 有输入/输出/指标/基线/排障信号才可能判为实践 |
| R05 Web/API 服务与中间件 | Axum/Actix/Rocket/Tower 如何组织 API 服务 | 路由、中间件、状态、鉴权、限流 | 需要能对标 FastAPI、SpringBoot、Node 服务结构 |
| R06 数据访问、缓存与外部系统集成 | Rust 服务如何接数据库、缓存、队列和第三方接口 | 连接池、事务、一致性、超时、重试 | 必须说明外部依赖边界和故障处理 |
| R07 FFI、系统编程、嵌入式与安全关键场景 | Rust 什么时候适合替代 C/C++ 或进入安全关键领域 | FFI、unsafe 边界、实时性、认证、安全标准 | 行业采用文章只作线索，不直接写成选型结论 |
| R08 测试、观测、构建、发布与供应链 | Rust 工程如何验证、发布、观测和治理依赖 | 测试、基准、日志、指标、镜像、供应链安全 | 有可执行命令、指标和回滚路径才进入核心知识点 |

## 待补缺口

| 流程节点 | 缺什么 | 后续文章进入时怎么判断 |
|---|---|---|
| R02 | 类型建模、错误处理、trait 边界 | 能不能减少状态泄漏、错误吞没和边界污染 |
| R03 | Tokio/async 生产实践 | 有没有取消、超时、背压、任务泄漏和资源指标 |
| R05 | Axum/Actix Web/API 服务结构 | 能不能对标 FastAPI、SpringBoot、Node 的路由、状态、中间件 |
| R06 | 数据访问与外部系统集成 | 有没有连接池、事务、一致性、重试和限流边界 |
| R08 | 测试、观测、构建和发布 | 有没有可执行命令、基准、日志指标、镜像和回滚动作 |
