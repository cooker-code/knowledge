## 核心问题

Java 文章池同时包含语言特性、AOP、MapStruct、HTTP 客户端和工具库。重蒸馏的核心不是把这些都写成“Java 很强”，而是区分业务编排、扩展点、数据转换和客户端边界。

## 判断准则

Java 小技巧文章只有在能改变分层、扩展点、缓存、异常或数据转换准则时才进入核心知识；纯语法清单和收益口号只作排重锚点。

| 判断项 | 处理 |
|---|---|
| 精读候选 | 需要有机制、边界、失败模式、实测或官方证据，才继续拆成更细核心知识点 |
| 略读/降权 | 保留为排重来源或补证线索，不把标题收益、工具清单、产品资讯直接写成长期结论 |
| 错放负例 | 写入本准则提醒后续路由，不因关键词继续扩散目录 |

## 认知校准点

| 校准点 | 处理 |
|---|---|
| 标题不是分类依据 | 标题里的框架名、产品名、收益数字只作为入口，实际按文章主问题和流程节点吸收 |
| 资讯不是版本记录 | 版本、发布、Beta、生态新闻必须等官方文档或实测证据，不能在核心知识里写死结论 |
| 清单不是选型准则 | 第三方库清单和工具推荐只有在说明边界、失败模式和替代方案时才升级为知识点 |

## 待验证缺口

- 对精读候选来源补官方文档、版本、实测环境或失败模式。
- 对降权来源只在后续出现相同问题指纹时用于排重，不继续扩写摘要。
- 若某类来源连续提供高质量机制证据，再拆独立核心知识点。
