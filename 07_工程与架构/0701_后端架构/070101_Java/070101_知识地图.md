## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| J00 语言版本与基础语义 | Java 版本和基础特性是否影响后端设计 | 基础语法不重复沉淀，只保留会影响设计的规则 | Java 8、新特性文章进入 |
| J01 抽象边界与函数式接口 | Function/lambda 如何组织可组合逻辑 | 函数式接口要服务边界、复用和测试，不是语法炫技 | Function、lambda、策略组合文章进入 |
| J02 反射、SPI 与扩展点发现 | 如何在运行时发现类型、加载实现和构建扩展点 | 反射/SPI 要显式记录类型安全、权限、性能和生命周期风险 | ReflectionUtils、SPI 文章进入 |
| J03 缓存、并发与资源边界 | 本地缓存和分布式缓存如何组合 | 缓存价值来自命中率、失效、一致性、容量和降级 | Caffeine、Guava、Redis 文章进入 |
| J04 Native 互操作与运行时扩展 | Java 与原生库如何互操作，代价是什么 | 原生互操作需要部署、内存、安全和可观测边界 | Panama、JNI/JNA 文章进入 |
| J05 动态模块与插件生命周期 | 动态模块如何隔离、装配、卸载 | 插件化必须管理类加载、依赖、状态和资源释放 | Celix、模块化文章进入 |
| J06 后端业务编排模式 | 复杂业务步骤如何拆成可维护流程 | Pipeline 是业务编排模式，不等于工作流或规则引擎 | Pipeline、责任链、状态机文章进入 |
| J07 与 SpringBoot 衔接 | Java 机制如何进入 Spring Boot 应用 | 框架专项不在 Java 重复沉淀 | Spring Boot 文章转入 `SpringBoot` |
