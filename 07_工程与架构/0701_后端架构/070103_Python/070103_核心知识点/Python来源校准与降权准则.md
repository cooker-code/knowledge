## 核心问题

Python 文章池混有 FastAPI、Pydantic、异步、脚本自动化和语言小技巧。重蒸馏要把后端服务边界和个人脚本效率文章拆开，避免把零散技巧当成 Python 后端架构。

## 判断准则

FastAPI/Pydantic/asyncio 进入服务边界；一行代码、日常脚本、文件工具、字符串小库等只保留为降权来源，不升级为后端架构准则。

| 判断项 | 处理 |
|---|---|
| 精读候选 | 需要有机制、边界、失败模式、实测或官方证据，才继续拆成更细核心知识点 |
| 略读/降权 | 保留为排重来源或补证线索，不把标题收益、工具清单、产品资讯直接写成长期结论 |
| 错放负例 | 写入本准则提醒后续路由，不因关键词继续扩散目录 |
