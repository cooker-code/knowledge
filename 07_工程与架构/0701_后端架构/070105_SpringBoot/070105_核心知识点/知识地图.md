## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| S00 版本与运行时基线 | Spring Boot 版本、容器、JVM/Native Image、启动方式有什么变化 | 版本变化只有影响迁移、部署和资源成本才值得吸收 | 版本发布、Undertow 移除、Native Image 文章进入 |
| S01 应用边界与分层 | 轻量分层、六边形、DDD、低代码平台如何选择 | Spring Boot 不是架构本身，分层强度要按业务复杂度选 | 六边形、低代码平台、项目结构文章进入 |
| S02 Web 入口与接口契约 | Controller、DTO、参数校验、幂等、外部 API 调用如何设计 | 入口层只做协议适配和校验，业务规则和副作用要下沉 | 校验、幂等、外部接口文章进入 |
| S03 数据访问与文件集成 | 数据源、连接池、Excel、对象存储、文档解析、多源查询怎么接入 | 集成文章重点看资源生命周期、事务、容量和失败模式 | HikariCP、EasyExcel、MinIO、Tika、Calcite 文章进入 |
| S04 外部依赖与韧性治理 | 第三方 API 慢、失败、超时时如何止损 | TimeLimiter/fallback 只是韧性治理的一层，不等于完整稳定性方案 | Resilience4j、重试、隔离、限流文章进入 |
| S05 批处理、规则与插件扩展 | 批处理、规则引擎、动态 jar 如何可追踪和可回滚 | 插件化不是加载 jar，必须管 ClassLoader、Bean、任务和配置生命周期 | Spring Batch、LiteFlow、动态 jar 文章进入 |
| S06 性能、启动与资源优化 | 连接池、异步、缓存、Native Image、自动配置怎么优化 | 性能数字没有基线、环境和指标时必须降权 | 性能优化、连接池、启动优化文章进入 |
| S07 安全、测试、观测与部署 | Spring Boot 服务上线后如何验证、监控和回滚 | 当前缺口大，后续高质量文章优先补这里 | Security、Actuator、Micrometer、Testcontainers、部署文章进入 |
| S08 低代码/AI/业务案例 | 业务案例是否能形成可迁移规则 | 只展示功能的案例默认略读，不直接写成准则 | JeecgBoot、DL4J 业务例子进入 |
