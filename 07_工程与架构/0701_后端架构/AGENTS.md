## 排重指纹

```text
后端路线 + 流程节点（边界/入口/校验/编排/数据/可靠性）+ 核心机制 + 解决问题 + 生产边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是 FastAPI | 按应用入口、路由、Pydantic、认证、中间件、性能、镜像拆分 |
| 都是 Spring Boot | 按自动配置、Controller、校验、幂等、外部依赖、批处理、插件拆分 |
| 都是 Java 语言机制 | 按反射、SPI、函数式接口、Panama、动态模块、Pipeline 拆分 |
| 只是教程或资讯 | 没有失败模式或可迁移准则时不新建 |
| 跨路线对标 | 同一流程节点的差异沉淀到对应路线，不混入单一路线 |

## 已覆盖问题（按三级节点）

| 节点 | 已覆盖 | 还缺 |
|---|---|---|
| Python | FastAPI 应用入口、路由、Pydantic/SQLModel、SQLAlchemy 会话、异步、WebSocket、镜像、教程降权 | 认证、后台任务、观测、发布回滚证据 |
