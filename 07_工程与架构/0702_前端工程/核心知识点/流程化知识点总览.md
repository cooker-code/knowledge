# 前端工程流程化知识点总览

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 07_工程与架构 / 0702_前端工程 |
| 解决的问题 | 先定义前端应用的工程流程，再把文章吸收到具体技术路线和流程节点 |
| 典型输入 | 本地文章、已有 AGENTS、已有核心知识点、后续新增资料 |
| 典型产出 | 流程节点知识、判断准则、风险边界、待补证据 |
| 用户阅读目标 | 读到一篇前端文章时，先知道它补的是框架、组件、状态、数据、质量、AI 接入还是发布链路 |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| F00 用户场景与产品边界 | 面向仪表板、内容站、管理后台、AI 应用、CLI UI 还是数字孪生 | 前端不是从框架开始，而是从交互对象、渲染位置和交付约束开始 | 趋势、架构、产品形态文章进入，只有口号则降权 |
| F01 框架与运行时选择 | React、Vue、Nuxt、Next/TanStack Start、浏览器、终端或视频渲染怎么选 | 先判断运行时和渲染边界，再判断框架栈 | React/Vue/Nuxt/TypeScript/运行时文章进入 |
| F02 组件模型与交互状态 | 组件 API、组合模式、状态、表格、编辑器、三维交互如何组织 | 组件设计要减少状态耦合和不可控变体 | 组件库、状态管理、高交互组件文章进入 |
| F03 数据获取与服务交互 | API、认证、缓存、Server Function、Tool Calling、流式响应怎么接 | 前端数据交互要同时考虑类型、权限、副作用和水合边界 | React Query、NuxtAuth、Nitro、AI SDK 文章进入 |
| F04 渲染、性能与体验质量 | SSR/CSR、水合、性能、可访问性、视觉稳定性如何保证 | 体验问题要有可验证信号，不写泛泛最佳实践 | 性能、质量扫描、渲染进阶、可访问性文章进入 |
| F05 构建、测试与发布闭环 | 构建、CLI E2E、质量门禁、发布、回归、升级如何闭环 | 工程化价值来自可重复验证和失败定位 | 前端工程化、测试、趋势、版本升级文章进入 |
| F06 AI 与自动化能力接入 | 生成式 UI、模型 Provider、Tool Calling、AI 可操作页面如何进入前端 | AI 能力必须有组件注册、工具权限、Schema 和可回滚边界 | Tambo、TanStack AI、AI Elements、AI 浏览器接口文章进入 |

## 待补缺口

## 使用方式

- 先读本文件的宏观流程，再进具体路线目录。
- 新文章不要单独写摘要，先归入路线和流程节点。
- 精读候选必须先和 [文章拆分路由表](文章拆分路由表.md) 里的问题指纹对比。
- 如果某条路线文章变多，优先抽 `核心知识点/<主题>.md`，不要继续扩大文章列表。
