### 核心判断

生成式 UI 的核心不是“让模型画界面”，而是把模型输出约束到组件白名单、Schema、权限、用户确认和回滚路径里。

| 环节 | 准则 |
|---|---|
| 组件注册 | 只能暴露白名单组件，组件入参必须有 Schema，不能让模型自由拼任意前端代码 |
| 流式渲染 | Markdown/消息流优先做增量解析和局部更新，避免每个 token 全量重渲染 |
| Tool Calling | 工具调用是副作用边界，读操作、写操作、高风险操作要分级 |
| 用户确认 | 下单、删除、改配置、发消息等动作必须有人类确认或可撤销 |
| 观测评估 | 记录模型输出、组件选择、工具入参、失败原因和用户纠正 |
| 降级回滚 | 生成式 UI 失败时要能退回普通表单、固定页面或人工流程 |
