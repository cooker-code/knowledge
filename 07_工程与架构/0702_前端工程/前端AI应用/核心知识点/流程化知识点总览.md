# 前端 AI 应用流程化知识点总览

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 07_工程与架构 / 0702_前端工程 / 前端AI应用 |
| 解决的问题 | 先定义前端 AI 应用流程，再把文章吸收到模型接入、生成式 UI、工具调用、权限和评估节点 |
| 典型输入 | Tambo、TanStack AI、AI Elements Vue、AI 可操作网页相关文章 |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| A00 用户任务与 AI 交互边界 | AI 是回答、生成 UI、执行动作还是操作页面 | 先定义任务，不先选 SDK | AI 应用形态文章进入 |
| A01 模型 Provider 与流式通道 | 模型、多 Provider、流式响应怎么接入 | 模型接入要可替换、可观测、可降级 | TanStack AI 等文章进入 |
| A02 组件注册与生成式 UI | AI 如何选择和渲染组件 | 组件注册需要白名单和 Schema | Tambo、AI Elements 文章进入 |
| A03 Tool Calling 与业务动作 | AI 如何调用业务逻辑 | 工具调用是副作用边界，不是 UI 小功能 | Tool Calling、MCP 文章进入 |
| A04 Schema、权限与用户确认 | 输出和动作如何被约束 | 参数、权限、确认、审计必须前置 | 安全、权限、Schema 文章进入 |
| A05 浏览器/页面可操作语义 | 网页如何暴露给 AI 操作 | 页面给 AI 的接口要像 API 一样治理 | AI 浏览器接口文章进入 |
| A06 评估、观测与回滚 | 失败怎么发现和恢复 | AI 前端要能退回普通 UI 或人工确认 | 评估、埋点、回滚文章进入 |
