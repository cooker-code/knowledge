## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| R00 运行时与产品边界 | React 运行在浏览器、服务端、终端还是视频渲染中 | 先判断运行时，再谈框架能力 | 特殊渲染文章进入，不能覆盖 Web 主线 |
| R01 应用结构与框架选择 | Next.js、TanStack Start、普通 React 怎么选 | 选型看路由、数据、认证、构建、部署 | 框架栈文章进入，标题党先降权 |
| R02 组件 API 与组合模式 | 组件如何暴露 API、组合能力和状态边界 | 避免布尔地狱，优先显式状态和组合 | 组件模式、架构文章进入 |
| R03 状态、副作用与数据获取 | 客户端状态、服务状态、缓存和失效怎么区分 | React Query 是服务状态层，不是请求工具 | Hook、React Query、缓存文章进入 |
| R04 服务交互、认证与缓存 | API、Server Function、认证、缓存如何闭环 | 交互边界要考虑副作用和权限 | 全栈栈、认证、API 文章进入 |
| R05 性能、质量扫描与可访问性 | 如何发现代码质量、性能和体验问题 | 质量门禁要可扫描、可复现、可修复 | React Doctor、Million.js、性能文章进入 |
| R06 非浏览器渲染 | React 用于 CLI、终端、视频时如何降权处理 | 这是渲染模型迁移，不是 Web 主流程 | Ink、Remotion、TUI 文章进入 |
| R07 构建、测试与发布 | 如何构建、测试、发布和回滚 React 项目 | 质量必须进入 CI 和回归链路 | 测试、发布、包体文章进入 |
