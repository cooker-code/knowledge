## 用户认知重点

| 项 | 内容 |
|---|---|
| 已知基础 | 了解 Docker 容器化基本概念，知道 K8s 用于容器编排 |
| 待补边界 | 镜像供应链安全；生产 K8s 集群的稳定性保障和滚动发布 |
| 易偏差点 | 容器化 ≠ 云原生；K8s YAML 复杂不代表架构合理 |
| 优先抽取 | 镜像分层和安全扫描、健康检查设计、资源限制、滚动发布和回滚策略 |

## 排重指纹

```text
容器对象（镜像/Pod/Deployment/Service）+ 治理阶段（构建/运行/调度/监控/回滚）+ 核心机制 + 解决问题 + 安全/资源/可观测性边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是 Dockerfile | 区分基础镜像选择、分层优化、安全扫描和多阶段构建 |
| 都是 K8s 部署 | 区分 Deployment、StatefulSet、Job 和 DaemonSet 的适用场景 |
| 都是资源管理 | 区分 Request/Limit 设计、HPA/VPA 和 Node Affinity |
| 只给 YAML 配置 | 没有设计原则和失败模式时不新建知识点 |
