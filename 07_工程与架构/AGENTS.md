## 工程主流程

| 节点 | 要回答的问题 | 主要落点 |
|---|---|---|
| E00 边界确认 | 系统职责、输入输出、非功能约束和不做什么是什么 | 后端架构、前端工程 |
| E01 实现组织 | 模块、分层、依赖、状态、数据访问和扩展点如何组织 | 后端架构、前端工程 |
| E02 交互契约 | API、协议、网关、幂等、鉴权和错误语义如何设计 | 工程实践与质量保障 |
| E03 质量验证 | 单测、集成测试、契约测试、回放、压测和验收标准如何覆盖 | 工程实践与质量保障 |
| E04 运行保障 | 日志、指标、链路、告警、部署、回滚、容量和故障处置如何闭环 | 工程实践与质量保障、容器化 |
| E05 复盘沉淀 | 缺陷、事故、性能瓶颈和架构偏差如何变成可复用准则 | 各三级节点核心知识点 |

## 本轮纠偏规则

| 错放模式 | 处理 |
|---|---|
| 只讲工具安装、产品推荐、API 客户端使用 | 迁出到电脑工具，或在原节点降权为工具锚点 |
| 讲 BI 可视化、报表表格、智能问数、查询优化 | 按主问题迁出到 `05_数据分析与BI` 或 `04_OLAP与数据库` |
| 讲 AI 编程、AI 设计工具、Agent/ReAct、Agent 安全工具 | 迁出到 `02_Agent与AI工程` |
| 讲机器学习模型、多模态能力 | 迁出到 `06_机器学习` 或 `01_LLM与大模型` |
| 标题含框架名但正文主问题是横切质量、测试、观测、部署 | 归入 `0703_工程实践与质量保障` |

## 文章处理规则

- `07_工程与架构/文章` 顶层文章池不再使用。
- 有三级节点时，文章只能进入三级节点的 `文章/`；二级目录不新增 `文章/`。
- `done-` 只表示文章已被路由、蒸馏或降权处理；长期知识必须写入核心知识点或 `AGENTS.md`。
- 性能、版本、发布资讯类文章没有官方证据、环境、基线和指标时，只能作为后续补证锚点，不直接写成结论。
- 新文章先回答“它优化哪个工程流程节点”，再决定补充、冲突、不同层次或降权。
