## 用户认知重点

| 项 | 内容 |
|---|---|
| 已知基础 | 用户知道数据质量影响数仓可信度，无需重复"数据质量很重要" |
| 待补边界 | 准确性、一致性、完整性、唯一性、及时性、有效性的差异；与指标治理的边界 |
| 易偏差点 | 质量文章易用概念解释代替可执行规则；易与指标口径混淆 |
| 优先抽取 | 规则、反例、验收口径和责任边界 |

## 排重指纹

```text
数据质量对象 + 质量维度 + 校验机制 + 解决问题 + 业务/技术边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是质量维度解释 | 按准确性、一致性、完整性、唯一性、及时性、有效性拆分 |
| 都是质量平台 | 按规则配置、调度执行、告警、阻断、修复闭环拆分 |
| 只有概念 | 没有反例和规则边界时只略读 |
| 有生产闭环 | 优先沉淀为实践候选 |
| 与指标治理混淆 | 数据值质量归本类，指标口径和业务动作归数据分析与 BI |

## 已覆盖问题（按三级节点）

| 节点 | 已覆盖 | 还缺 |
|---|---|---|
| 数据质量 | 准确性/一致性、离线质量闭环、流式质量监控、主动元数据触发质量动作 | 完整性、唯一性、及时性、有效性的规则模板；质量失败与调度补跑联动；动态探查和平台规则配置验证 |

## 待补缺口

| 优先级 | 项 | 为什么补 |
|---|---|---|
| 高 | 数据质量规则平台 | 质量维度需要落到规则、调度、告警和阻断 |
| 高 | 数据质量与补跑联动 | 质量失败后需要自动恢复和验收 |
| 中 | 数据质量指标体系 | 需要区分表级、字段级、指标级质量 |
| 中 | 数据质量责任边界 | 上游系统、数仓加工、指标消费的责任不同 |
