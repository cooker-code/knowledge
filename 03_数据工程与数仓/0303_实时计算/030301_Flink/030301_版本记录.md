# 030301_Flink 版本记录

Flink 各大版本的关键变化。蒸馏知识点时如涉及特定版本行为，在知识点文件中标注版本并回查此表。

官方 Changelog：[GitHub Releases](https://github.com/apache/flink/releases)

## 命名变更对照

文章中常见旧名与当前官方名的对应关系（1.13+ 以后）：

| 旧名（文章中常见） | 当前官方名 |
|---|---|
| MemoryStateBackend | HashMapStateBackend |
| FsStateBackend | HashMapStateBackend（运行时）+ FileSystemCheckpointStorage |
| RocksDBStateBackend | EmbeddedRocksDBStateBackend |

## 版本关键变化

| 版本 | 关键变化 | 影响知识域 |
|---|---|---|
| 1.11 | Application Mode 正式引入 | F SQL提交 |
| 1.13 | State Backend 与 Checkpoint Storage 解耦；旧 Backend 类名废弃 | C 状态 |
| 1.14 | State TTL `NeverReturnExpired` 语义明确；旧版 SQL 引擎相关接口及代码全部移除 | C 状态 |
| 1.15 | Changelog State Backend MVP；Generic Incremental Checkpoint；SQL Gateway alpha；PyFlink 新增"线程"模式（通过 JNI 在 JVM 执行 Python 函数，而非独立 Python 进程） | C、D、F |
| 1.15.1 | bug fix 版本，无 API/配置变更；注意 FLINK-28322（DataStreamScanProvider 新方法不兼容）留至 1.15.2 修复 | - |
| 1.16 | SQL Gateway GA（FLIP-91），支持 REST API 和 HiveServer2 协议；Changelog State Backend 生产可用（支持状态迁移、本地恢复、从 Checkpoint 切换）；新 Backend 命名正式生效；RocksDB 日志默认写入 Flink 日志目录；新增透支缓冲区 `taskmanager.network.memory.max-overdraft-buffers-per-gate`（默认 5）；新增 `USING JAR` / `CREATE TABLE AS SELECT` / `ANALYZE TABLE` DDL；Hive 2.3 语法兼容性达 94.1%；PyFlink"线程"模式扩展至 DataStream API 和 TVF；Blocking Shuffle 新增 LZO、ZSTD 压缩算法 | C、D、F、G |
| 1.17 | 自适应批调度器成为默认调度器（FLIP-283）；Scala API 标记为 deprecated（FLIP-265），未来将移除；引入 `DELETE`/`UPDATE` 行级 API（FLIP-282）；PLAN ADVICE 功能可在 EXPLAIN 时检测非确定性与优化建议；通用增量 Checkpoint (GIC) 生产可用（WordCount Checkpoint 耗时减少 ~90%）；Unaligned Checkpoint 生产可用；TPC-DS vs 1.16 整体提升 26% | A、C、F、G |
| 1.18 | 支持 `REPLACE TABLE AS SELECT` / `CREATE OR REPLACE TABLE AS SELECT` 原子 DDL；`TRUNCATE TABLE`（批模式）；Table API/SQL 支持算子级 State TTL（之前只能全局配置 `table.exec.state.ttl`）；通过 REST API / Web UI 动态调整 Task 并行度；SQL Gateway 提供 JDBC Driver；SQL Hint 支持水印对齐和数据源空闲检测 | C、F |
| 1.20 | 物化表功能（Materialized Table）MVP 引入；UDF API 完善；SQL 语法扩展；JDK 11 软要求 | A、F |
| 2.0 | 存算分离状态管理（ForSt State Backend，远程存储）；流批 API 统一；物化表与湖格式集成（Paimon）生产就绪；批作业自适应执行（动态 Broadcast Join、倾斜 Join 优化）；`state.backend` 配置键变化（`forst` 新类型）；异步执行模型（状态访问和 Checkpoint 可异步） | A、C、F、G |
| 2.1 | AI 模型 DDL（`CREATE MODEL`）和 `ML_PREDICT` TVF，支持 OpenAI 兼容接口；Process Table Functions (PTFs) 正式支持（直接访问托管状态/事件时间/定时器）；新增 `VARIANT` 半结构化数据类型和 `PARSE_JSON` 函数；`DeltaJoin` 算子（依赖 Apache Fluss，默认开启，Join 无状态化）；`MultiJoin` 算子（需 `table.optimizer.multi-join.enabled=true`，仅支持 INNER/LEFT JOIN）；Keyed State SQL 连接器（`connector='savepoint'`，直接查询 Checkpoint/Savepoint 状态）；PyFlink 移除 Python 3.8 支持，新增 Python 3.12；Parquet 升级至 1.15.3 修复 CVE-2025-30065 | C、F、G |
| 2.2 | JDK 11 强制要求（不再支持 JDK 8） | A 运行环境 |

## Flink Table Store / Paimon 版本对照

| 版本 | 关键变化 |
|---|---|
| Table Store 0.2.0 (2022-09) | 引入独立 Catalog（跨 session 持久化元数据，支持 DFS 或 Hive Metastore）；支持 Flink 1.14 和 1.15；新增 Append-only 表模式（`write-mode=append-only`）；支持 Bucket 扩缩容（ALTER TABLE + INSERT OVERWRITE）；Hive/Spark/Trino 读取支持；存储格式稳定 |
| Paimon（从 Table Store 孵化为独立 Apache 项目，1.17 时代） | 支持流批读写、Time Travel、Lookup Join、OLAP 列存（ORC/Parquet/Avro）、ACID、Zorder/Hilbert 数据布局优化 |
