# Flink 2.0 关键变化与选型影响

> 验证版本：Flink 2.0（基于 FFA 2024 官方演讲，尚处于规划/早期发布阶段）

## 来源
- [Flink2.0未来趋势中需要注意的一些问题](../文章/done-Flink2.0未来趋势中需要注意的一些问题.md)

## 核心问题
Flink 2.0 的哪些变化会影响现有作业的升级路径和架构选型决策？普通开发者需要关注哪些趋势？

## 判断准则

### 存算分离（ForSt DB）
- **变化**：引入 ForSt DB 替代（或补充）RocksDB，支持状态存储到远程云存储，实现计算与存储解绑
- **影响**：带大状态的作业迁移和扩缩容将变得更方便（当前 RocksDB 大状态迁移是高频痛点）
- **选型建议**：大状态任务（GB 级以上）关注 ForSt DB 成熟度；小状态任务影响不大

### Materialized Table（物化表）
- **变化**：流批一体统一抽象，一份代码支持流式持续刷新、批式全量刷新、增量刷新三种模式；通过修改 FRESHNESS 参数切换
- **作者评价**：谨慎怀疑——代码层面统一解决的是"代码兼容性"，是流批一体很小的一部分；大部分业务场景无法做到完全的批流一体
- **适用场景**：时效性要求可弹性调整的报表类任务（如允许在实时/延迟之间切换）；不适合对语义一致性要求极高的场景

### Streaming Warehouse（Flink + Paimon 深度集成）
- **定位**：Paimon 作为批流一体存储引擎，解决传统离线/实时数仓的部分痛点，而非构建"纯流式数仓"
- **价值边界**：
  - 有价值：流式写入 + 批式查询统一存储；upsert 语义；Schema 演化
  - 无革命性进步：不能替代所有离线架构；复杂聚合仍需 OLAP 引擎

### 版本升级兼容性注意点
- 配置文件格式可能变化（需平台处理，开发者关注）
- 状态兼容性：大版本跨越需通过 Savepoint 迁移
- 常见 API 可能有 breaking change（1.x → 2.x）

## 认知偏差
| 常见错误认知 | 正确理解 |
|---|---|
| Materialized Table 解决了批流一体所有问题 | 仅解决了代码层面的统一，业务语义层面的批流一体仍然复杂 |
| Flink 2.0 存算分离立即可用于生产 | 处于演进阶段，生产采用前需等待社区成熟度验证 |
| Paimon 会取代现有数仓架构 | Paimon 解决部分痛点（upsert/Schema 演化/流批统一存储），不是全替换 |

### 流批一体的本质：存储是核心（字节跳动实践视角）
来源：字节跳动 FFA 2022 实践

**核心判断**：流批一体的关键不在计算层的 API 统一，而在**存储层的统一**。

理由：
- 流数据 = 随时间推进的无界变化序列（Binlog）
- 批数据 = 某一时刻的有界快照（Snapshot）
- 同一份存储同时支持 Binlog（流读）+ Snapshot（批读）= 真正的流批一体

**Paimon（前身 Flink Table Store）的核心能力**：
| 能力 | 说明 |
|---|---|
| Snapshot + Log 双模式 | 支持批读（Snapshot）和流读（Log/Change） |
| 存算分离 | 基于 DFS/对象存储，无须本地盘 |
| Merge Tree | 支持高效点查（全局有序 + Data Skipping）和快速更新 |
| 全面支持 Flink SQL | DDL/DML/Change 类型/点查全支持（业界大多数外部存储不能同时支持所有 Flink SQL 能力） |

**端到端一致性三层**（字节实践）：
1. 数据源 → ETL：Flink Exactly-Once
2. ETL → 单表写入：Flink Exactly-Once + Paimon 事务写
3. 多中间表关联一致性：通过流批一体自动调度 + 中间表事务写入

**字节 Flink OLAP 规模**：11 个集群 6000+ Core，每天 Query 50w+；简单作业 QPS 从 30 → 1000+；复杂作业 QPS 从 10 → 100+。

## 待验证缺口
- ForSt DB 在生产环境的稳定性和性能对比 RocksDB 的实测数据
- Materialized Table 的 FRESHNESS 切换是否有状态兼容性问题
- Paimon 在高 QPS 点查场景的实际延迟表现（字节内部使用自研实时服务分析引擎）
- Paimon Merge Tree 在写入密集型场景的小文件合并策略（汽车之家实践：Iceberg v2 小文件合并需注意 sequence number 语义）
