# Fluss

## 技术定位

| 项 | 内容 |
|---|---|
| 技术名 | Fluss |
| 一级类目 | 数据工程与数仓 |
| 二级类目 | 实时计算 |
| 技术本体 | 面向流分析的实时流存储，给 Flink 等实时计算链路提供可分析、可更新、可点查的数据层 |
| 全局架构位置 | 位于消息队列、实时计算引擎和湖仓表格式之间，承担实时流数据的存储与分发 |
| 主要使用者 | 实时数仓工程师、实时平台工程师、数据平台工程师 |
| 主要产出 | 实时流存储表、变更日志、主键表、日志表、湖仓实时数据层 |

## 归类边界

| 相邻对象 | 不归入这里的情况 |
|---|---|
| Flink | 文章主问题是状态、窗口、Checkpoint、SQL 计算语义时，归 Flink |
| Kafka | 文章主问题是消息队列、分区、副本、消费位点时，归 Kafka |
| Paimon / Iceberg / Hudi / LakeSoul | 文章主问题是湖仓表格式的快照、事务、Schema 演进和多引擎互操作时，归湖仓表格式 |
| Doris / StarRocks / ClickHouse | 文章主问题是交互式查询、索引、物化视图和 OLAP 性能时，归 OLAP 与数据库 |
