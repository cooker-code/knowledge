# 030205_Spark 知识地图

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 03_数据工程与数仓 / 0302_离线数仓 / 030205_Spark |
| 解决的问题 | 离线数仓计算如何用 Spark 完成批处理、SQL 优化、Shuffle/Join 治理、资源管理和观测排障 |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| P00 作业入口 | Spark 是批任务、SQL 服务、Connect 客户端还是流任务 | 先判断 Spark 承担计算引擎还是服务入口 | Spark SQL、Spark Connect、PySpark、Spring 集成文章进入 |
| P01 计划与执行 | Catalyst、逻辑计划、物理计划和 AQE 如何工作 | 理解计划才能定位慢 SQL 和 Hint 生效边界 | 执行计划、源码、AQE、指标文章进入 |
| P02 Join 与 Shuffle | Join、Shuffle、宽依赖、倾斜和中间数据如何治理 | 大部分性能问题落在数据重分布和中间数据 | Join、Shuffle、Celeborn、宽窄依赖文章进入 |
| P03 资源与内存 | Executor、内存、CPU、本地盘、网络和调度如何配置 | 资源参数必须服务数据规模和作业形态 | 硬件、内存、YuniKorn、并发、动态资源文章进入 |
| P04 观测排障 | History Server、SQL Metrics、EventLog 和日志如何定位问题 | 排障要从 Stage、Task、数据量、指标和日志入手 | UI、指标、日志、Agent 工具文章进入 |
| P05 版本与生态 | Spark 3.x/4.x、Scala/JDK、Hive Catalog、Native Engine 如何演进 | 版本资讯只在改变迁移判断时沉淀 | 3.3/3.4/4.0、Native、Rust/Sail、暗黑模式文章进入 |

- Spark 文章先问“它改变的是执行计划、资源、观测还是版本判断”。
- 性能数字没有数据规模、执行计划、指标和对照组时，一律降权。
