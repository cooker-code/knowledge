# 数据仓库｜标签流转模型设计

> 本文继续`揭露大厂数据模型实际落地应用`，上一篇介绍了`留存数据模型设计`，本文讲讲`标签流转模型设计`。

## 1. 背景

`最近在做用户标签流转的需求`，用户运营团队一般将用户分为新用户、活跃用户、沉默用户和流失用户等不同群体。针对不同群体，制定有针对性的运营策略。例如，对新用户进行引导和激励，对沉默用户进行唤醒，对流失用户进行挽回。`而运营动作效果的监控和分析，则需要标签流转数据支持，那今天就介绍一下实际生产场景标签流转模型`。

## 2. 模型设计思路

### a.模型定位

* `收敛：收口所有实体（可以是C端用户，也可以B端商家...）标签流转指标`
* `解耦：灵活设计分区字段，不同场景使用自己的例行加工任务，互不影响`
* `通用：除支持大盘标签流转，也支持特定维度上的流转`

### b.模型结构

```
create table if not exists tag_change (  
    `history_tag` string comment '历史标签',  
    `current_tag` bigint comment '当前标签',  
    `date_cycle` string comment '流转天数，1/3/7/14/30',  
    `change_num` string comment '流转数量',  
    `extend_params` string comment '拓展参数，可以加入特定维度参数'  
) comment '标签流转模型' PARTITIONED BY (  
    `date` string comment ' 日期分区'  
    `tag_code` string comment '标签分区，标签+实体池子'  
)
```

### c.分区字段

`日期分区date、标签分区tag_code`

* 标签分区：`标识是哪个标签及实体池子范围`，如：用户生命周期标签+当日活跃用户

### d.主键

* 对应主键：`日期分区date、标签分区tag_code、流转天数date_cycle、历史标签history_tag、当前标签current_tag、扩展字段extend_params`

### e.基本信息字段

* 历史标签：`history_tag，实体的历史标签`
* 当前标签：`current_tag，实体的当前标签`
* 流转天数：`用于标识流转天数`，如T-1到T日的流转，那流转天数等于1。
* 流转数量：`change_num，假如date_cycle=1，实体从T-1的历史标签流转到T的当前标签的数量，`

### f.如何使用

* 特别说明下：`扩展字段extend_params`，`有些场景不满足于只看标签流转的整体情况，也希望看特定维度下流转`。如希望结合用户年龄分层维度交叉的去看流转情况，这些额外信息都可以放在extend\_params中，一般写成json格式。

## 3. 实际案例

### a.数据情况

* 以下为一张全量用户标签表user\_tag，`包含用户ID、生命周期标签tag、时间分区date`。主键为user\_id

| user\_id | tag | date |
| --- | --- | --- |
| 101 | 未转化用户 | 2024-06-06 |
| 102 | 活跃用户 | 2024-06-06 |
| 101 | 新用户 | 2024-06-07 |
| 102 | 流失用户 | 2024-06-07 |
| ... | ... | ... |

### b.需求详情

* 计算用户在`2024-06-06到2024-06-07的生命周期标签流转`

### c.代码示例

* 示例代码如下，供参考。
* 对应示例数据的结果：在2024-06-06到2024-06-07，从`未转化用户`流转到`新用户`有1个人，从`活跃用户`流转到`流失用户`有1个人。

```
select  
  nvl(b.tag, '未知')  as history_tag   --历史标签  
  ,nvl(a.tag, '未知') as current_tag   --当前标签  
  ,1                 as date_cycle    --流转周期  
  ,count(a.user_id)  as change_num    --流转数量  
  ,''                as extend_params --扩展字段  
  ,'2024-06-07'      as date          --日起分区  
  ,user_tag_all_user as tag_code      --标签分区，用户生命周期标签+全量用户  
from  
  (  
    select  
      user_id  
      ,tag  
    from user_tag  
    where date='2024-06-07'  
  ) a  
left join  
  (  
    select  
      user_id  
      ,tag  
    from user_tag  
    where date='2024-06-06'  
  ) b on a.user_id=b.user_id  
group by   
  nvl(a.tag, '未知')  
  ,nvl(b.tag, '未知')
```
