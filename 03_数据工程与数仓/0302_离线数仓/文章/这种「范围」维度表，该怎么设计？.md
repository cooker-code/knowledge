在我的项目里，有这么一份维度数据，存储着「ip」跟 这个 ip 对应的「地理位置」(国家、省份、城市)，以及「运营商」信息。

但这份数据有个特点，它并不是跟普通数据那样，一个 ip 对应一个地理位置跟运营商信息的，取而代之的，是一个「IP 范围」。

比如，像这样：

前面两列(用"|"分割)，分别是「起始 ip」、跟「终止 ip」，总数据条数 60w+。

就是这样一份维度数据，涵盖了全世界所有 IPv4 的地理位置跟运营商信息。

也就是说，你任意的一个 IPv4 地址，都可以在这份数据里面，找到其对应的位置跟运营商信息。

0. 数据结构分析

看过我历史文章的同学应该知道，之前介绍过关于这份维度数据「基于 Redis 跳表」的玩法，很多同学表示，看球不太懂。

之所以有难度，是因为这个维度数据，跟普通数据不一样的地方在于——你没有办法对它做普通的「等值关联」查询。

也就是说，你不能用 SQL 语句通过普通的 join，来关联到这个维度信息，只能想别的招。

那今天，我们就来设计个大家都能看懂、且也理论上可行的方案。

先说一个热知识：

> IP(不管是 ipv4 还是 ipv6)，都是可以转成对应的十进制数字的，只不过这个数字一般比较大。
>
> 对于所有的 IPv4 来说，32 位的数值类型，也就是一般的 Long 类型，就可以把它完全框住；
>
> 但对于 IPv6 来说，32 位的数值肯定就不行了，得更大，但这不在本次内容的讨论范围内。

了解到这个知识点后，这个存储的方案设计，就相对简单了。

那就是，先把这份维度数据里面的每条「起始 IP」跟「终止 IP」提取出来，然后分别转成十进制数值，就像这样：

之后，就把这个「起始 IP」跟「终止 IP」作为两个单独的字段，给拎出来存储到数据库里。

然后，对于要查询的 IPv4，通过将其转成 long 类型后，在 SQL 的 where 条件里，用「大于等于起始ip，小于等于终止 ip」就能把这条维度数据给精确定位出来。

2. 数据入库

这里面唯一需要注意的，就是怎么利用 api，先把这个 IP 转成数值类型。

比如，我用的 Spark 入库(Scala 来写)，对应的转换方式就是这样的：

```
/**  
     * @DESC: 将 IP 转成 Long  
     * */  
    private def getIPLong(ip:String):Long = {  
        val ipByteArray = InetAddress.getByName(ip).getAddress  
        var ipLong:Long = 0  
        for (ipByte <- ipByteArray) {  
            ipLong <<= 8  
            ipLong |= ipByte & 0xff  
        }  
        ipLong  
    }
```

大概对比了一下，这份 60w+的数据，通过 Spark 分别写 CK 跟 Doris 的用时，都差不多——通过本地 idea 运行、不超过 10 秒。
