## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| H00 表与分层 | Hive 表如何承接 ODS/DWD/DWS/ADS 和公共维度 | Hive 价值不只是 SQL，而是离线表模型和元数据生态 | 时间维表、拉链、快照、分区和建模文章进入 |
| H01 Metastore | 元数据如何存储、查询、共享和扩展 | Metastore 是 Spark/Flink/Kyuubi 等共享 Hive 表的中枢 | ObjectStore、DirectSQL、Thrift、元数据库瓶颈文章进入 |
| H02 查询入口 | JDBC/ODBC/Beeline 请求如何进入 Hive | HiveServer2 负责会话、连接、认证和稳定性 | HS2 内存、连接、认证、并发、Operation 日志文章进入 |
| H03 SQL 执行 | Hive SQL 如何解析、优化并交给执行引擎 | Explain、分区裁剪、Join 计划和引擎差异决定执行风险 | Explain、PawSQL、排序、Join、CBO 文章进入 |
| H04 性能治理 | 慢 SQL、倾斜、小文件、存储格式如何治理 | 优化要先定位瓶颈，再选择 SQL 改写或参数调整 | 倾斜、参数、ORC/Parquet、全局排序、小文件文章进入 |
| H05 表结构演进 | 新增字段、格式差异、历史分区如何兼容 | `alter table` 只是元数据变更，还要验证旧数据和下游读取 | 新增字段、Schema 演进、兼容读文章进入 |
| H06 生态互通 | Hive 如何与 HBase、Spark、湖仓表格式互通 | Hive 常是 SQL/Catalog 入口，不替代相邻系统本体 | HBase 互通、Spark 读写、湖仓迁移文章进入 |
