# 030201_Hadoop&HDFS 版本记录

Hadoop/HDFS/YARN 的版本判断必须区分 Apache 上游版本、企业发行版组件版本和文章中的经验版本。涉及版本行为时，先查官方文档或发行版 Release Notes，再写入知识点。

## 官方锚点

- Apache Hadoop Releases: https://hadoop.apache.org/releases.html
- HDFS Architecture: https://hadoop.apache.org/docs/current/hadoop-project-dist/hadoop-hdfs/HdfsDesign.html
- HDFS Erasure Coding: https://hadoop.apache.org/docs/current/hadoop-project-dist/hadoop-hdfs/HDFSErasureCoding.html
- YARN Architecture: https://hadoop.apache.org/docs/current/hadoop-yarn/hadoop-yarn-site/YARN.html
- Cloudera Runtime 7.3.1 Overview: https://docs.cloudera.com/cdp-private-cloud-base/7.3.1/private-release-notes/topics/rt-runtime-overview.html
- Cloudera Runtime 7.3.1 Component Versions: https://docs.cloudera.com/cdp-private-cloud-base/7.3.1/private-release-notes/topics/rt-runtime-component-versions_731.html

## 关键能力版本边界

| 能力 | 官方依据 | 判断边界 |
|---|---|---|
| HDFS 架构 | 当前 HDFS Architecture 文档 | HDFS 面向大文件、高吞吐、批处理式访问；放宽部分 POSIX 语义换吞吐和数据本地性。 |
| HDFS EC | 当前 HDFS Erasure Coding 文档 | EC 面向温/冷数据降存储成本；复制因子对 EC 文件无意义，EC 策略按目录继承且已创建文件不能靠 rename 改策略。 |
| YARN | 当前 YARN 文档 | YARN 将资源管理和作业调度/监控拆分为 ResourceManager、NodeManager、ApplicationMaster；Scheduler 负责按队列/容量等约束分配 Container。 |

## Cloudera Runtime 7.3.1

| 项 | 官方记录 | 对知识库的处理 |
|---|---|---|
| 版本号格式 | 7.3.1.0 起采用 Base.Major.Minor.Patch | 不把公众号中的“7.3.1”简单理解为 Apache Hadoop 版本。 |
| Hadoop 组件 | Apache Hadoop（含 YARN/HDFS）为 `3.1.1.7.3.1.0-197` | CDP 7.3.1 的 Hadoop 能力基线和 Apache 上游 3.5.0 不同，排障和配置必须看发行版文档。 |
| Hive / Iceberg | Hive `3.1.3000.7.3.1.0-197`，Iceberg `1.3.1.7.3.1.0-197` | `0929-Cloudera 7.3.1正式GA` 中的 Hive + Iceberg 信息应进入 Hive/湖仓节点时再细化。 |
| Spark | Spark `3.4.1.7.3.1.0-197` | Spark 版本变化不在本目录展开，只保留发行版对离线生态的影响。 |
| Ozone | 7.3.1.0 标注 Ozone 1.3.0 等价于 Apache Ozone 1.4.0，7.3.1.100 起为 1.4.0 | Ozone 与 HDFS/对象存储边界后续应独立补证，不混入 HDFS 结论。 |

## 使用规则

- 写知识点时，若结论依赖 Apache 上游版本，在文件顶部标注 `> 验证版本：Apache Hadoop x.x`。
- 写知识点时，若结论依赖 CDP/Cloudera 发行版，在文件顶部标注 `> 验证版本：Cloudera Runtime x.x`。
- 发布公告、发行版资讯和组件版本变化优先进入本文件，不新建普通知识点。
- 文章提供的版本信息只能作为线索，必须用官方文档或 Release Notes 核对。
