# 030201_Hadoop&HDFS知识地图

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 03_数据工程与数仓 / 0302_离线数仓 / Hadoop&HDFS |
| 解决的问题 | 离线数仓底层数据如何以 Hadoop 生态完成分布式存储、资源调度、批处理承载、容错扩展和排障 |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| H00 场景定位 | 当前文章是在讲 Hadoop 生态、HDFS 存储、MapReduce 计算，还是 YARN 调度 | Hadoop 是生态，HDFS 是文件系统，YARN 是 Hadoop 生态资源层 | 只出现 Hadoop 关键词不直接迁移，必须看主问题 |
| H01 存储语义 | 数据以什么命名空间、元数据、Block 和副本模型组织 | HDFS 是目录树 + NameNode + Block 映射，对象存储是 bucket/key 对象语义 | HDFS vs 对象存储、NameNode、Block、rename/list/append 文章进入 |
| H02 容错与成本 | 如何在可靠性、存储成本、读写性能之间权衡 | 副本适合热数据和简单语义，EC 适合冷/温数据降成本但有 CPU/网络代价 | EC、副本、故障恢复、冷温数据分层文章进入 |
| H03 元数据扩展 | NameNode 如何扩展、隔离和高可用 | Federation、RBF、HA 解决不同维度瓶颈，不等于无成本扩展 | NameNode 内存、HA、Federation、RBF 文章进入 |
| H04 计算模型对标 | MapReduce 与 Spark 等计算模型的边界是什么 | MapReduce 可作为 Hadoop 批计算历史对照，Spark 运行时问题仍归 Spark | MapReduce/Spark 对比只吸收对 Hadoop 生态边界有用的部分 |
| H05 排障运维 | DataNode、磁盘、网络、副本和 NameNode 发生故障如何定位 | 先看心跳和节点状态，再看网络、进程、磁盘、日志和副本恢复 | 有明确命令、日志信号、恢复验证的文章进入 |
| H06 YARN 资源调度 | Hadoop 生态任务如何排队、分配 Container、聚合日志、重试和对标 Kubernetes | YARN 负责队列、Application、Container 和日志，不负责 Flink/Spark 算子语义 | YARN 调度器、YARN vs K8s、Flink/Spark on YARN、Application 日志文章进入 |

## 使用方式

- 先看流程节点，再看具体核心知识点。
- Hadoop 生态文章必须先拆主问题：HDFS 存储、YARN 调度和 MapReduce 对标进本目录；Spark/Flink 算子、状态、Checkpoint、Shuffle 等运行时机制进对应技术目录。
- HDFS 文章只有提供机制、边界、失败模式或验证动作时才正式沉淀；纯面试题清单只进入来源索引。
