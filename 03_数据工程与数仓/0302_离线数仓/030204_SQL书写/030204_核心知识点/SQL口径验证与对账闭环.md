## 核心问题

SQL 写对不是“跑出结果”。离线数仓 SQL 必须先把业务口径、时间范围、去重范围、表粒度和边界样例说清楚，再用样例数据或对账结果证明 SQL 符合口径。

## 判断准则

| 步骤 | 准则 |
|---|---|
| 口径拆解 | 明确统计对象、时间口径、去重键、过滤条件和异常值处理 |
| 样例构造 | 至少覆盖正常、重复、缺失、边界日期、多状态变化 |
| SQL 实现 | 每个 CTE 只解决一个中间口径，避免在一层里混合过滤、聚合、去重和排序 |
| 对账验收 | 输出总量、分组样例、极端样例和历史结果对比 |

## 认知偏差

| 常见错误认知 | 正确理解 |
|---|---|
| 面试 SQL 没有工程价值 | 可作为边界样例池，但不能直接当生产准则 |
| SQL 跑通就正确 | 还要验证粒度、重复计数、时间边界和空值 |
| 复杂 SQL 越短越好 | 可维护性和可验证性优先于技巧压缩 |
