# 030204_SQL书写知识地图

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 03_数据工程与数仓 / 0302_离线数仓 / SQL书写 |
| 解决的问题 | 离线数仓 SQL 如何写得正确、可维护、可验证，并尽量提前规避执行风险 |
| 典型输入 | 指标口径、业务过程、Hive/Spark 表、分区、维表、明细表、历史快照 |
| 典型产出 | SQL 写法准则、验证样例、改写规则、慢 SQL 排查入口 |
| 用户阅读目标 | 读到 SQL 文章时，先判断它补的是表达能力、性能边界还是验证方法 |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| S00 口径确认 | 这条 SQL 要回答什么业务问题，粒度是什么 | 指标口径、过滤条件、时间范围、去重范围先定清楚 | 只讲题解但不讲口径边界的文章降权 |
| S01 表关系设计 | 用哪些表，Join 键和粒度是否匹配 | 先防重复计数、笛卡尔积和多对多放大 | Join、路径分析、日期交叉文章进入 |
| S02 表达实现 | 采用窗口、聚合、行列转换还是递归式改写 | 语法价值必须落到可复用表达模式 | 窗口函数、lateral view、explode、滚动逻辑文章进入 |
| S03 执行前置 | SQL 写法会触发什么执行计划和资源风险 | 关注分区裁剪、Shuffle、长尾、全局排序、Count Distinct 膨胀 | 调优技巧和慢 SQL 文章进入 |
| S04 验证对账 | 如何证明结果正确且可回归 | 必须有样例、边界数据、Explain、任务指标或对账方式 | 只有经验结论无证据的文章只保留为候选 |
