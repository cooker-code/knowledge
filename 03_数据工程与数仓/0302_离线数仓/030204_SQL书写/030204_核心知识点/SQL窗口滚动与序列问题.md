## 核心问题

窗口、滚动、路径、日期区间和缺失补齐类 SQL 的关键不是记函数，而是稳定定义序列：分区键是什么、排序键是否唯一、窗口边界是否闭合、缺失日期如何补齐、区间是否重叠。

## 判断准则

| 场景 | 判断准则 |
|---|---|
| 排名/TopN | 排序字段必须稳定，处理并列和 Null |
| 累计/滚动 | 明确窗口范围、时间粒度和是否包含当前行 |
| 路径分析 | 先定义事件序列和去重规则，再拼接路径 |
| 日期交叉 | 先拆区间、合并重叠，再统计覆盖天数 |
| 缺失补齐 | 先生成日期维表或序列，再左关联事实 |

## 认知偏差

| 常见错误认知 | 正确理解 |
|---|---|
| 会窗口函数就会写序列 SQL | 序列 SQL 的难点是边界定义和样例验证 |
| 一次开窗越少越好 | 少一层窗口不能牺牲正确性和可读性 |
| 日期题只是面试题 | 生产中补齐、交叉和滚动口径很常见 |
