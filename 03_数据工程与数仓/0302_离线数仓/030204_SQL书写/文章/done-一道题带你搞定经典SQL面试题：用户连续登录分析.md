> 现有有一张记录用户登录信息的表，表名为user\_login，有两个字段 uid、login\_date，记录的是账号id和账号对应的登录日期

#### 问题 ：编写 SQL查出连续登录3天的账号

###### 示例数据

表名：`user_login`

| uid | login\_date |
| --- | --- |
| 1011 | 2025-08-01 |
| 1011 | 2025-08-02 |
| 1011 | 2025-08-03 |
| 1011 | 2025-08-05 |
| 1022 | 2025-08-01 |
| 1022 | 2025-08-03 |
| 1033 | 2025-08-01 |
| 1033 | 2025-08-02 |
| 1033 | 2025-08-03 |
| 1033 | 2025-08-04 |

#### 解法：

##### 方法一：表自连接（最容易想到，也最直观）

> 土哥Tips：以 MySQL为代表的关系型数据库，数据记录是以行的形式存储的，本质上不同行之间的数据并无对应关系，同一行数据中的不同列才存在对应关系，这一点很多初学者没理解，所以在面对统计"连续"类的问题时没有思路。

思路分析：要查找连续 3 天登录的用户，**本质上是对单个用户自己的数据进行比较**，上面 tips中提到了不同行之间的数据没有对应关系，那么我们就需要想办法将同一个用户的不同行数据转变为可以对比的同一行数据，即**表的自管理(自连接)**。

1. 1. 首先用uid做连接键，就能够将同uid的不同数据关联到同一行
2. 2. 然后再利用date计算日期差值关联，进一步将同账号但日期递增的数据限制到同一行
3. 3. 因为用的是 inner join，所以不需要再用where筛选空值。当然使用 left join也可以，只是需要在后面用where筛选关联到的空值
4. 4. 因为同一个账号可能有多个连续 3 天，例如账号1033，连续天有（2025-08-02，2025-08-02、2025-08-03） 和（2025-08-02，2025-08-03、2025-08-04）；但我们只需要账号uid，所以加DISTINCT去重

关联效果如下：

示例 SQL 如下

```
SELECT 
    DISTINCT a.uid
FROM
    user_login as a
INNERJOIN
    user_login as b 
ON a.uid = b.uid 
AND b.login_date = DATE_ADD(a.login_date, INTERVAL1DAY)
INNERJOIN
    user_login as c 
ON a.uid = c.uid 
AND c.login_date = DATE_ADD(a.login_date, INTERVAL2DAY)
```

**结果：**

> * • 用户 1011（连续登录的日期：2025-08-01 ~ 2025-08-03）
> * • 用户 1033（连续登录的日期：2025-08-01 ~ 2025-08-03、2025-08-02 ~ 2025-08-04）

## 进阶解法

我们再来捋捋这道题，最开始我们的分析核心是要统计单个用户的日期连续情况，那么除了将单用户数据合并到同一行，还有其他办法吗？

**有的，盆油！**

我们先忘记这道题，来看一个简单的示例，有这么一列数 :

> 2、3、4、5、6、8，我们怎么判断其中有多少数据是连续的呢？

聪明的你应该发现规律了，数据之间的间隔是 1、1、1、1、2，前 5 个数的间隔是一样的！这样就可以通过相同的间隔将2、3、4、5、6分为一组，将 6、8 分为一组，这个逻辑就带来了一个新的方法：**差值分组**

> 差值分组的核心逻辑：相邻数据的差值相同，就可以分为同一组。

那么问题来了，我们怎么构造这个差值呢？因为我们要计算连续数据，那么就需要将数据转换为一列连续递增的数据，即给数据排序，然后再计算差值，示例如下

| 数值 | 排序 | 差值 |
| --- | --- | --- |
| 3 | 1 | 2 |
| 4 | 2 | 2 |
| 5 | 3 | 2 |
| 6 | 4 | 2 |
| 8 | 5 | 3 |

可以看到**差值**相同的数据就是连续的，3、4、5、6

接下来我们运用同样的逻辑来解题，相较于这个简单的示例，要计算连续日期，我们可以先给同uid的日期做排序

```
    SELECT 
        uid,
        login_date,
        ROW_NUMBER() OVER (PARTITION BY uid ORDER BY login_date) AS rn
    FROM user_login
```

排序结果如下

| uid | login\_date | rn |
| --- | --- | --- |
| 101 | 2025/8/1 | 1 |
| 101 | 2025/8/2 | 2 |
| 101 | 2025/8/3 | 3 |
| 101 | 2025/8/5 | 4 |
| 102 | 2025/8/1 | 1 |
| 102 | 2025/8/3 | 2 |
| 103 | 2025/8/1 | 1 |
| 103 | 2025/8/2 | 2 |
| 103 | 2025/8/3 | 3 |
| 103 | 2025/8/4 | 4 |

然后再计算日期和排序的差值 （不用临时表 with table写法也可以，正常子查询一样的）

```
WITH base AS (
-- 分组排序
    SELECT
        uid,
        loin_date,
        ROW_NUMBER() OVER (PARTITIONBY uid ORDERBY login_date) AS rn
    FROM user_login
)
    -- 计算日期和排序的差值
SELECT
   uid,
   login_date,
   DATE(login_date) -INTERVAL rn DAYAS gap_id
FROM
    base
```

结果如下：

| uid | login\_date | rn | gap\_id |
| --- | --- | --- | --- |
| 101 | 2025/8/1 | 1 | 2025/7/31 |
| 101 | 2025/8/2 | 2 | 2025/7/31 |
| 101 | 2025/8/3 | 3 | 2025/7/31 |
| 101 | 2025/8/5 | 4 | 2025/8/1 |
| 102 | 2025/8/1 | 1 | 2025/7/31 |
| 102 | 2025/8/3 | 2 | 2025/8/1 |
| 103 | 2025/8/1 | 1 | 2025/7/31 |
| 103 | 2025/8/2 | 2 | 2025/7/31 |
| 103 | 2025/8/3 | 3 | 2025/7/31 |
| 103 | 2025/8/4 | 4 | 2025/7/31 |

最后再计算根据 uid 和 gap\_id 筛选要查询的连续日期天数，N是要筛选的连续天数

```
WITH base AS (
    SELECT
        uid,
        login_date,
        ROW_NUMBER() OVER (PARTITIONBY uid ORDERBY login_date) AS rn
    FROM user_login
)
SELECT
    uid, 
    COUNT(*) AS days
FROM
(
    SELECT
        uid,
        login_date,
        DATE(login_date) -INTERVAL rn DAYAS gap_id
    FROM
        base
) as t
GROUPBY
    uid, gap_id
HAVING
    COUNT(*) >= N
```

怎么样，这种写法是不是优雅很多？

### 这就是第二种方法：窗口函数 + 差值分组

其核心是通过计算统计目标字段之间的差值，新增一个排序参考列，用排序列去标识“日期”在数据列中的**位置**，再通过做差值方法给数据分组。

第二种方法是解决**连续问题**的通用方法！
