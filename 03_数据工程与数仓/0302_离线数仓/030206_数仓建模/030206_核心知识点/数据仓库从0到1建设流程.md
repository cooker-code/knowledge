## 核心问题

从0到1建数仓不是先建很多表，而是按需求边界、源系统盘点、主题域、分层、模型、调度、质量、权限、交付和运维验收逐步收敛。

## 判断准则

| 阶段 | 必须回答 |
|---|---|
| 需求边界 | 服务哪些业务问题，不服务哪些 |
| 源系统盘点 | 数据来源、口径冲突、质量风险 |
| 主题域规划 | 业务过程、实体、公共维度 |
| 分层设计 | 每层输入、输出、粒度、消费者 |
| 模型设计 | 事实表、维表、宽表、汇总表 |
| 物理落地 | 分区、命名、存储、调度 |
| 治理验收 | 质量、血缘、权限、对账、SLA |

## 认知偏差

| 常见错误认知 | 正确理解 |
|---|---|
| 先把 ODS/DWD/DWS/ADS 建起来就是数仓 | 分层名词只是形式，关键是业务过程、口径和消费者 |
| 初建数仓越全越好 | 先围绕核心业务闭环，逐步扩展公共模型 |
| 平台先行能解决建模问题 | 平台只固化流程，不能替代需求和口径判断 |

## 待验证缺口
