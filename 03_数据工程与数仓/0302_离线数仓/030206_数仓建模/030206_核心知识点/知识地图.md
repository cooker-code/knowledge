# 030206_数仓建模知识地图

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 03_数据工程与数仓 / 0302_离线数仓 / 数仓建模 |
| 解决的问题 | 业务数据如何被组织成稳定、可复用、可治理、可演进的离线数仓模型 |
| 典型输入 | 业务过程、业务实体、交易事件、日志行为、指标需求、下游报表和服务需求 |
| 典型产出 | 主题域、分层模型、事实表、维表、宽表、模型规范、模型评估准则 |
| 用户阅读目标 | 读到建模文章时，先判断它优化的是哪个模型设计节点，而不是按标题新建孤立笔记 |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| M00 需求边界 | 当前模型服务什么业务过程和分析问题 | 先确定业务过程、消费者和口径边界 | 只讲概念、不讲业务边界的文章降权 |
| M01 主题域与实体 | 业务对象如何划分主题域和实体边界 | 主题域服务复用、治理和跨系统协作 | 主题域、数据总线、Data Vault 文章进入 |
| M02 分层与粒度 | ODS/DWD/DWS/ADS 各层职责和粒度是什么 | 粒度先于字段，层次先于宽表 | 分层、ODS/DWD/DWS/ADS、ADS 契约文章进入 |
| M03 事实与维度 | 事实表、维表、拉链、快照如何选 | 模型类型要匹配业务变化方式 | 事实表、维度建模、拉链表、累积快照文章进入 |
| M04 物理落地 | 表如何命名、分区、存储、调度和生命周期管理 | 物理设计承接性能、成本、治理和运维 | 命名规范、物理模型、平台建设文章进入 |
| M05 演进与复用 | 新维度、粒度变化、口径变更如何降低返工 | 模型稳定性要用下游影响和复用性评估 | 宽表、模型复用、动态字段、平台化文章进入 |
| M06 治理接口 | 质量、血缘、权限、指标和语义层如何接入模型 | 建模要暴露治理接口，但不替代治理系统 | 准入准出、指标、血缘文章按主问题外路由 |
