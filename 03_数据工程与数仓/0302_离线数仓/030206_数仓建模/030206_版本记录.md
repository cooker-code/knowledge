# 030206_数仓建模 版本记录

数仓建模不是单一软件产品，本文件记录的是建模范式、平台形态和 Data+AI 语境下的演进边界。遇到具体软件版本时，应转到对应技术目录；本目录只记录会影响建模判断的范式变化。

## 范式与平台演进

| 主题                    | 演进线索                                    | 对本目录的意义                             |
| --------------------- | --------------------------------------- | ----------------------------------- |
| Kimball 维度建模          | 业务过程、事实表、维度表、总线矩阵、一致性维度                 | 仍是多数离线数仓建模文章的基础参照。                  |
| Inmon/企业数仓            | 主题域、集成、历史稳定数据集合                         | 用于理解企业级数据架构和主题域边界。                  |
| Data Vault 2.0        | Hub/Link/Satellite、敏捷扩展、审计追踪            | 作为维度建模之外的横向范式，不默认替代所有场景。            |
| OneData / 数据中台        | 统一口径、公共模型、指标复用、治理平台化                    | 需要区分方法论和平台产品营销。                     |
| Data+AI / Agent Ready | 元数据、语义、Schema Evolution、审计和 Agent 查询上下文 | 只记录对建模流程和治理接口的影响，不采信“几天替代几个月”类效率宣传。 |

## 使用规则

- 方法论文章要写清适用组织规模、团队能力、治理成熟度和代价。
- 平台文章只能沉淀可复用流程、元数据/语义/治理接口，不沉淀营销收益。
- AI 建模文章必须保留人工评审、口径确认、样例验证和下游影响评估。
