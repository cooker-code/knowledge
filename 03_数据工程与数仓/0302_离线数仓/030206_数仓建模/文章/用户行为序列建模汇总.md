# 2. UBM的研究背景

随着各类互联网应用的快速发展，推荐系统（RS）在个性化服务提供和缓解信息过载方面的作用日益凸显。当前推荐系统研究的主要痛点在于缺乏用户偏好的显式反馈。用户兴趣偏好更多隐含记录在粗糙且有噪音的行为日志中。这时候用户行为兴趣建模就开始成为一个专门的研究点，主要目的就是：从历史行为中来挖掘用户的兴趣表征，并用于推荐任务中；

# 3. UBM的分类体系

为推进用户隐式兴趣建模的探索，学术界已提出多种用户行为建模（UBM）方法。为深入理解 UBM 的发展脉络，本文梳理了具有代表性和影响力的研究工作，分析其研究动机并总结核心技术贡献。如表 1 所示，本综述构建了全新的 UBM 分类体系，将现有研究划分为四大类。研究从关注简单用户行为的传统 UBM 出发，逐步拓展至考虑用户行为序列的长度增长、类型多样性提升和异构性增强三大趋势（如图 1 所示）。以下简要阐述这三个新兴研究方向的核心思想：

1. 传统 UBM：从简单历史行为序列中学习用户兴趣表征(**简单的行为历史：主要就是单一行为类型 single type (比如点击，下载)历史**)。首先将用户行为历史Hu处理为具有固定长度的时序物品序列(一般现需要进行padding，然后使用sequence\_mask)，随后从多个维度分析用户复杂行为模式，包括**session结构**、行为粒度(粗粒度，marco-behavior，micro-behavior)与影响范围 以及物品间的依赖关系(比如i2i相似度计算，就是通过)。
2. 长序列 UBM：通过学习更长的用户行为序列（至少千级规模，原则上如此，还是要看业务及系统能处理的长度）。保留更多行为记录使**模型能够挖掘更丰富的长期用户兴趣，但****深度模型****处理超长序列面临计算瓶颈**。为此，研究者提出两种解决方案：

1. 采用存储增强方法来**存储长距离行为依赖**
2. 行为检索方法筛选与目标相关的行为历史，也就是GSU，有hard-GSU和soft-GSU；

3. 多类型 UBM：进一步研究用户行为的多样性。在统一推荐模型中**显式建模不同行为类型（如点击、购买**），为细粒度理解行为模式和关联关系提供了新视角，比如ESSM2就是通过分析不同的用户行为，并找到不同行为之间的转化率；

各种大佬致力于解决行为多样性带来的一系列新挑战，包括多行为定义、多行为融合 和多行为预测。

4. UBM with Side Information—带side-information的 UBM

我们都知道每个item有各种各样的属性信息，因此可以把itemId 泛化到属性序列上；因此我们进一步考虑与用户ActionUnit相关的异构特征；引入这些item的side-information有什么作用呢？

这些辅助信息可以帮助还原 用户行为发生时的上下文交互逻辑，为理解复杂的用户兴趣提供重要的补充；目前风头正盛的NLP和CV模型都会将item丰富的多模态信息来转化为用户兴趣表征空间；

这4个研究方向紧密关联在一起，共同驱动UBM的研究工作快速发展；

在后续的章节中，我们将遇到的**研究难点**，**对应的代表性解决方案**和**发展趋势**三个维度来对UBM的研究方向进行详细阐述；

## 4.1 RNN-based 方法

循环神经网络（RNNs）能够自然捕捉序列的长短期依赖关系（比如DIEN等）。

* 作为用户行为建模领域的早期尝试之一，**GRU4Rec  采用****门控循环单元****（GRUs）学习session内用户行为的演变模式，相当于序列建模和session建模；**
* 后续工作通过**数据增强技术或改进排序****损失函数** 对 GRU4Rec 进行了扩展。
* 后面基于 RNN 的encoder-decoder模型，**能够融合session中的全局兴趣-global与局部兴趣-local**。
* Quadrana 等人进一步提出了层次化的**双-GRUs 结构**，分别建模session内和跨session间用户行为演变模式，一个GRUs建模session内的信息，一个GRUs建模session间的兴趣，其实这里的细节还是挺多的；

## 4.2 CNN-based方法

基于循环神经网络（RNN）的方法可能无法有效捕捉跳跃行为，即下一步行为受前面几步的影响，**而非相邻行为**。

（其实，这些用户行为路径建模中非常重要）；他们提出了 Caser 模型，将近期行为从时间维度与latent维度视为一幅 "图像"，并通过horizontal and vertical 卷积核 来 **捕捉序列模式和跳跃行为**。NextItNet 提出了一种带residual block 结构的生成式CNN来捕捉长期和短期的用户行为依赖关系；

## 4.3 Attention-based 方法

注意力机制在用户行为建模（UBM）中得到广泛应用，其在建模任意行为对之间的交互关系时，具有编码距离不衰减的优势。下面列举出几个常见的Attention-based的方法：

1. 通过利用更多监督信息，SASRec在auto-regressive 预测框架中采用self-attention识别历史行为的重要性。
2. DIN 通过target注意力机制，针对target-item从历史行为中自适应学习用户兴趣表征。
3. DIN 忽略了用户兴趣的动态演变， DIEN  中通过结合注意力机制与门控循环单元（GRU）得以解决。
4. 考虑用户行为在单次session内的行为模式，DSIN分别使用self-attention和Bi-LSTM提取session内兴趣表征和session间的兴趣表征。

尽管取得上述进展，研究者也注意到传统attention mechanisms在探索复杂行为模式时的缺陷，因此转向更复杂的注意力架构，例如 BST中的 Transformer encoder和 BERT4Rec 中的堆叠bidirectional Transformer encoder。

## 4.4 讨论

传统用户行为建模（UBM）的研究探索了多种网络结构，用于从简单行为序列中提取依赖模式。除上述网络结构外，其他深度网络结构（如多层感知机 [MLP 和图神经网络 [GNN，）或多网络组合结构也在传统 UBM 中得到研究。例如，SURGE通过度量学习从行为序列中构建物品 - 物品兴趣图。

这样，long-sequence可能会变的非常长(至少千级规模),对所有历史行为进行建模的**计算成本过高（一个挑战）**。同时更长的序列也**可能包含更多的噪声**，而严格的在线延迟要求进一步限制了在大规模工业生产中引入长行为序列。为缓解上述问题，学术界和工业界百花齐放，主要分为两大方向：

1. 存储增强方法
2. 基于用户行为检索的方法。

## 5.1 存储增强的技术—Memory-Augmented Methods

存储增强 网络 最初是在自然语言处理（NLP）领域中提出的，其将提取的知识 显式存储于外部存储器中(把某些层存储在一个map中or其他中？) 。

在推荐任务中，提取的知识=用户的兴趣表征，将用户的兴趣偏好存储于外部的存储器，并通过特定的神经网络根据用户行为记录进行读写更新。

1. NMRN(神经记忆网络): 通过为每一个用户维护一个  **external memory bank**，这个兴趣库里面存放的是该用户的若干个兴趣向量(其实就是把用户的多兴趣向量存储起来了)，这个兴趣库 根据用户新产生的行为进行更新。兴趣表征与 反应用户不同偏好的 个性化权重有一定的关系；
2. RUM（循环更新记忆）： 显式存储用户历史行为。  它采用 first-in-first-out mechanism 维护用户存储单元中的最新交互物品。
3. KSR（知识存储与检索） 将item属性作为key，从外部存储器中的索引来提取知识，增强序列推荐中的物品表示。

为更高效地处理long-sequence用户行为，大佬们设计了专用内存增强网络，将长行为序列压缩为dense 向量。

1. HPMN（分层周期性记忆网络，Hierarchical Periodic Memory Network）：是一个life-long建模序列框架，通过multi-layer GRU架构捕捉multi-scale sequential 模式，每一层GRU按照不同的更新频率来 更新外部的用户兴趣表征；
2. UIC(用户兴趣中心，User Interest Center)， 通过Multi-channel user Interest Memory Network (MIMN) 来存储用户的多个维度的兴趣；最大的改善在于推理阶段，直接从UIC 获取用户行为表征，无需在线计算，这样就对比较耗时的离线训练过程 和 real-time prediction 相互解耦；UIC 从工业视角为长序列 UBM 提供系统性解决方案，并已成功落地于实际推荐系统中。

## 5.2  用户行为检索的方法—User Behavior Retrieval Methods

除了通过外部存储单元中存储用户的兴趣表征之外，从**超长序列中检索****最相关、最重要****的行为**是另一种解决方案。

基于检索的方式的优点是：检索过程能够高效执行，且检索步骤可减少序列中的潜在噪声。

1. UBR（用户行为检索模型）：采用搜索引擎技术，针对target-item检索最相关的历史行为，UBR并没有使用用户连续的长序列，仅仅将检索得到的检索结果集合用于下游计算(比如序列模型抽取)；
2. Qin 进一步研究如何优化检索函数，提出基于L2R的优化方法;
3. SIM(序列兴趣匹配模型): 就是阿里提出的大名鼎鼎的SIM模型；如何去检索用户的相关行为呢？ 这里提出了两种检索方式，hard search 和 soft search

1. hard search：通过user ID 和 item的类别来构建 two-layer 索引
2. Soft search：通过局部敏感hash的技术，基于用户行为embedding来检索相关的行为；

上面提到的这两种方法：UBR和SIM，都是two-phase，基于两阶段的，第一阶段的检索函数 和后面的prediction module 是相互独立的，这就存在链路不一致的问题；

1. ETA模型（end-to-end target attention）：提出了一种end-end训练检索函数的方法；这个模型使用 SimHash 算法将 将用户行为 embedding 和 target items映射到二进制签名，并基于binary signatures检索与 target items  Hamming distance最小的用户行为(后面要看看这个怎么操作的)。由于检索过程中，只有item embedding作为参数，因此整个流程可以通过端到端部署操作；
2. SDIM(Sampling-based Deep Interest Modeling, 基于采样的深度兴趣模型): 通过对hash采样进一步优化，提出新的hash采样方法进一步优化改进ETA，直接收集与target-item共享 相同hash signature 历史行为item，进一步优化耗时；
3. ADFM(对抗过滤模型)： 引入对抗过滤机制（adversarial filtering mechanism），对检索到的行为序列进行压缩，并且可以从检索到的结果中去除噪声行为；

## 5.3 讨论

memory-augmented methods 的设计理念，通过引入user/item相关的更多特定参数(i.e., external memory)，从而存储更多的信息，存储增强的方法尽管很有效，但是整个流程比较复杂，并且不容易部署到真实的生产环境中。

相比而言，基于检索的框架在大规模的推荐系统中具有更好的效率和可扩展性，这两个优势使得其更适合在处理工业界长用户行为序列。

行为检索过程不可避免地会导致信息损失；因此，如何在两者间进行权衡并充分发挥各自优势，仍是一个有待解决的开放性问题。

## 6.1 行为类型界定—Behavior Type Definition

multi-type用户行为建模（UBM）的首要问题是行为类型的定义。由于推荐场景差异极大，这一问题仍悬而未决。研究人员通常通过专业性分析定义不同行为类型，大致可分为以下三类：

### 6.1.1 宏观行为- Macro behaviors

**宏观行为指具有明确动机的交互行为，由推荐场景的逻辑设计所定义**。例如，点击、加入购物车、收藏和购买行为共同构成了电子商务服务的基本功能。用户执行macro behavior以达成特定目的，这类行为日志记录完整（包含行为类型），可直接用于后续多类型用户行为建模（UBM）方法。

### 6.1.2 微观行为 Micro behaviors

微观行为是基于expert knowledge从宏观行为中提取的 ，为用户行为分析提供a fine-grained视角。例如，点击行为可根据click source 划分为不同微观行为(**还是根据当前用户行为来源**)，有助于解释用户对特定items的兴趣。然而，微观行为的定义并非易事，**需要结合特定推荐场景和目标应用的专业知识。**

### 6.1.3 Behaviors from different domains or scenarios

跨领域 / 场景行为也可整合以提供用户兴趣挖掘的补充信息(从这个角度讲，是不是和上面的micro-behavior定义重复)。例如：

1. ZEUS 模型将搜索场景中的query行为与推荐场景中的点击行为相结合，挖掘用户的隐式反馈。
2. 同一观看行为在两个领域中被视为不同行为，以突出domain差异。

根据应用需求，需通过专家指导从相关领域或场景中选择互补行为。

### 5.4 讨论

多类型 UBM 的三大核心问题（行为定义、行为融合、行为预测）紧密关联：

* ZEUS 模型 整合跨场景行为类型（**如搜索查询与推荐点击**），会存在导致不同的标签分布（如搜索行为的高稀疏性），并对复杂的多行为预测模块提出需求
* DIPN 模型： 同时建模不同层级的宏观 / 微观行为，传统早期融合设计因粒度差异难以有效整合。

结论：多类型 UBM 解决方案需在行为定义粒度、融合方式选择和预测任务设计之间进行综合权衡，实现特定场景的定制化建模。

## 7.1 辅助信息来源—Side Information Sources

在用户行为建模（UBM）中可利用的辅助信息主要分为三类：时间信息、物品属性和多模态信息。

#### 1. 时间信息建模

传统 UBM 方法（如 SASRec、BERT4REC ）仅用时间戳tk对行为记录排序，仅影响**序列建模前**的位置编码。近期进展包括：

* TiSASRec：发现不同item对之间的时间间隔time gap 包含关键知识，提出time interval aware mechanism 用于注意力权重计算，根据权重来融合用户的行为序列；
* TISSA首创time interval-based的 GRU提取会话级行为表征，再通过multi-scale 时间窗口切片捕捉时序依赖（temporal dependencies）

#### 2. 物品属性建模—item attributes

item属性提供关键上下文信息，可以更加显著影响用户行为：

* FDSA：将item ID 与category, brand, 以及 description text等属性结合，用于序列推荐
* Trans2D：为了追踪item的属性随着时间的变化，对item ID 和属性进行特征变换，学习复杂的item-attribute时序模式

#### 3. 多模态信息建模—multi-modal side information

不同于以上的信息，多模态信息的利用更为复杂：

* p-RNN： 从视频缩略图(video thumbnails)提取图像特征，从商品描述提取文本特征，分别使用 GoogLeNet（CV模型） 和词袋模型（NLP方法）独立建模
* SEMI：受多模态预训练启发，直接利用CV/NLP 领域 SOTA 模型（如 CLIP、BERT）获取视频和文本表征

## 7.2 Side Information Utilization

在获取辅助信息表征后，如何有效利用这些信息成为另一核心问题。

#### 1. 早期工作 融合方法

早期研究通常通过简单加法或拼接算子融合不同模态表征向量，再将融合后的item embedding 输入下游学习模块：

* p-RNN：提出**在****输入层****通过特征拼接**或 **输出层****通过加权求和**融合物品 ID 与辅助信息，使用 RNN 进行序列处理
* SC-CNN：将辅助信息视为 different views 数据，拼接起来作为一个 3D 立方体后采用semi-causal CNN捕捉跨视图关系
* Trans2D：同样将item - attributes转换为 3D 立方体，不过这里使用改进的 2Dself-attention Transformer处理三维数据

#### 2. 动态融合架构

CARCA提出双分支多头自注意力框架：two-branch multi-head selfattention-based framework 捕捉动态偏好 和 属性感知的画像；

* 左分支-left branch：提取历史行为与属性 profile 的依赖关系
* 右分支：捕捉行为序列对target-item的影响

#### 3. 表征解耦方法

上面简单融合方法 可能破坏原始物品 ID 表征，改进方法包括：

* NOVA-BERT:将辅助信息作为自注意力模块的辅助信号来学习优化注意力分布，而非直接融合到item表征
* DIF-SR：提出解耦注意力计算：在注意力计算前分离不同辅助信息，避免限制注意力矩阵学习

#### 4. 自监督学习（SSL）方法—Self-Supervised Learning

灵感来自于 CV/NLP 领域启发，SSL 技术被用于辅助信息建模：

* S3Rec：为了挖掘item-attribute，设计属性相关自监督目标：① 关联属性预测-associated attribute prediction ② 掩码属性预测-masked attribute prediction
* MISS：采用CNN 特征提取器捕捉兴趣表征、依赖关系及相关性

## 7.3 讨论

由这一章节的描述可以发现，异构辅助信息源在后续的提取与融合过程中起着决定性作用。像**时间和属性**这类简单信息，能够润滑地融入行为建模之中；而图像和文本等复杂信息，则需要借助现有的专门技术来进行处理。所以，在探索如何高效且有效地将辅助信息整合到兴趣表征空间方面，还有很大的研究空间。

# 9. Summary and Future Prospects

这篇文章总结了用户行为建模领域的近期进展。通过分析行为日志可挖掘丰富的用户偏好，传统 UBM 方法借助循环神经网络（RNN）、卷积神经网络（CNN）和注意力机制，能够从行为序列中捕捉隐含反馈，显著提升推荐性能。**通过引入长期行为历史、多类型行为和用户行为伴随的辅助信息，相关研究取得了进一步突破**。尽管成就显著，UBM 仍面临来自学术和工业界的双重挑战：

1. 更深度的信息融合 现有研究多单独探索长行为序列、多类型行为或辅助信息，三者的联合建模尚未充分开展。例如，长多类型行为序列建模可能带来新突破，但需要新型type-aware检索 / 采样技术。因此，多源信息的广泛与深度融合仍有巨大探索空间。
2. 更高效的学习方法 作为应用导向型研究，在线服务需在效果与效率间权衡。随着行为长度增加、行为类型丰富化和辅助信息多样化，UBM 模型的计算复杂度和存储负担显著上升。在维持性能的前提下，轻量级 UBM 解决方案始终是研究重点。
3. 更可解释的用户表征 现有方法虽提升了推荐效果，但学习到的用户兴趣表征缺乏可解释性，限制了 UBM 在用户画像、因果分析等领域的应用。因此，可解释 UBM是未来重要研究方向。
4. 更先进的技术适配 预训练、大模型等先进深度学习技术已用于辅助信息学习的学术探索，但距离工业落地仍有较大差距。需解决模型轻量化、实时推理优化等关键问题。
