# 030203_Kyuubi 知识地图

## 模块定位

| 项 | 内容 |
|---|---|
| 所属目录 | 03_数据工程与数仓 / 0302_离线数仓 / 030203_Kyuubi |
| 解决的问题 | 多个 SQL 引擎如何以统一、多租户、可治理的服务入口交付给用户和任务 |
| 典型输入 | JDBC/ODBC/Thrift/REST 请求、用户配置、Spark/Flink/Hive SQL、批任务 |
| 典型产出 | Session、Engine、SQL 执行记录、审计血缘、指标、资源隔离策略 |
| 用户阅读目标 | 读到 Kyuubi 文章时，先判断它补的是网关控制面、引擎接入、迁移治理还是社区资讯 |

## 宏观流程

| 节点 | 节点问题 | 关键知识点 | 文章吸收规则 |
|---|---|---|---|
| K00 入口定位 | Kyuubi 是 SQL 网关、控制面还是计算引擎 | Kyuubi 不替代 Spark/Flink/Hive，只统一入口和治理 | 介绍、架构、SQL Proxy 文章进入 |
| K01 Server 与 Session | 请求如何接入、认证、建 Session 和路由 | Server 管会话和策略，不直接执行计算 | Server、Session、HA、认证文章进入 |
| K02 Engine 生命周期 | Engine 如何启动、共享、隔离和回收 | Engine 共享策略决定隔离、资源和启动成本 | Engine、Spark/Flink/Hive 接入文章进入 |
| K03 多租户与权限 | 如何做资源隔离、权限、审计和血缘 | 多租户不是口号，要落到配置注入、AuthZ、Event Handler | AuthZ、SessionConfAdvisor、审计文章进入 |
| K04 SQL 迁移治理 | Hive SQL 迁移 Spark/Flink 时如何验证和治理 | Kyuubi 承接双跑、配置标签、审计和兼容性适配 | Hive 迁移 Spark、小文件治理文章进入 |
| K05 版本与兼容 | 版本发布改变了哪些控制面能力和引擎支持 | 版本资讯进版本记录，不直接新建普通知识点 | 1.6、1.8、1.11 等发布文章进入 |
| K99 社区动态 | 社区成员、活动、Playground、文档贡献 | 仅作来源存档，不沉淀为知识点 | 社区资讯、礼品、活动回顾文章重命名 |

## 使用方式

- 先看 K00 判断文章是不是 Kyuubi 本体；不是本体就转 Spark/Flink/Hive。
- 版本发布看 [030203_版本记录.md](030203_版本记录.md)，不要把发布新闻写成普通知识点。
- 企业实践只保留能复用到统一入口、多租户、审计、成本治理、迁移验证的部分。
