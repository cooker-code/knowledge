Paimon本身自带的系统表会记录用户创建每一张表的各类信息，比如快照、文件、tag、引用了哪些聚合函数等，接下来我们详细看一下各类系统表的用法。

01

**Snapshots Table**

可以通过快照表查询表的快照历史信息，其中包括快照中发生的记录数。举例中的my\_table就是自己创建的任何一张表。

通过查询快照表，可以了解该表的提交和过期信息，并可以通过数据进行时间旅行（即回溯到某个历史时间点的数据状态）。

```
SELECT * FROM my_table$snapshots;
/*+--------------+------------+-----------------+-------------------+--------------+-------------------------+--------------------------------+------------------------------- +--------------------------------+---------------------+---------------------+-------------------------+----------------+|  snapshot_id |  schema_id |     commit_user | commit_identifier |  commit_kind |             commit_time |             base_manifest_list |            delta_manifest_list |        changelog_manifest_list |  total_record_count |  delta_record_count |  changelog_record_count |      watermark |+--------------+------------+-----------------+-------------------+--------------+-------------------------+--------------------------------+------------------------------- +--------------------------------+---------------------+---------------------+-------------------------+----------------+|            2 |          0 | 7ca4cd28-98e... |                 2 |       APPEND | 2022-10-26 11:44:15.600 | manifest-list-31323d5f-76e6... | manifest-list-31323d5f-76e6... | manifest-list-31323d5f-76e6... |                   2 |                   2 |                       0 |  1666755855600 ||            1 |          0 | 870062aa-3e9... |                 1 |       APPEND | 2022-10-26 11:44:15.148 | manifest-list-31593d5f-76e6... | manifest-list-31593d5f-76e6... | manifest-list-31593d5f-76e6... |                   1 |                   1 |                       0 |  1666755855148 |+--------------+------------+-----------------+-------------------+--------------+-------------------------+--------------------------------+------------------------------- +--------------------------------+---------------------+---------------------+-------------------------+----------------+2 rows in set*/
```

**Schemas Table**

可以通过 schemas 表查询表的历史模式（schema）。

```
SELECT * FROM my_table$schemas;
/*+-----------+--------------------------------+----------------+--------------+---------+---------+-------------------------+| schema_id |                         fields | partition_keys | primary_keys | options | comment |       update_time       |+-----------+--------------------------------+----------------+--------------+---------+---------+-------------------------+|         0 | [{"id":0,"name":"word","typ... |             [] |     ["word"] |      {} |         | 2022-10-28 11:44:20.600 ||         1 | [{"id":0,"name":"word","typ... |             [] |     ["word"] |      {} |         | 2022-10-27 11:44:15.600 ||         2 | [{"id":0,"name":"word","typ... |             [] |     ["word"] |      {} |         | 2022-10-26 11:44:10.600 |+-----------+--------------------------------+----------------+--------------+---------+---------+-------------------------+3 rows in set*/
```

可以通过连接快照表和模式表来获取指定快照的字段信息。

```
SELECT s.snapshot_id, t.schema_id, t.fields     FROM my_table$snapshots s JOIN my_table$schemas t     ON s.schema_id=t.schema_id where s.snapshot_id=100;
```

**Options Table**

可以通过 options 表查询从 DDL 中指定的表选项信息。对于未显示的选项，将使用默认值。

```
SELECT * FROM my_table$options;
/*+------------------------+--------------------+|         key            |        value       |+------------------------+--------------------+| snapshot.time-retained |         5 h        |+------------------------+--------------------+1 rows in set*/
```

04

**Audit log Table**

如果需要审核表的变更日志，可以使用 audit\_log 系统表。通过 audit\_log 表，当获取表的增量数据时，可以获得 rowkind 列。可以使用这一列来进行过滤和其他操作，以完成审核工作。

rowkind 列有四个可能的值：

• +I: 插入操作。

• -U: 包含更新行之前内容的更新操作。

• +U: 包含更新行之后新内容的更新操作。

• -D: 删除操作。

```
SELECT * FROM my_table$audit_log;
/*+------------------+-----------------+-----------------+|     rowkind      |     column_0    |     column_1    |+------------------+-----------------+-----------------+|        +I        |      ...        |      ...        |+------------------+-----------------+-----------------+|        -U        |      ...        |      ...        |+------------------+-----------------+-----------------+|        +U        |      ...        |      ...        |+------------------+-----------------+-----------------+3 rows in set*/
```

05

**Read-optimized Table**

如果需要极高的读取性能，并且可以接受读取稍微旧一些的数据，可以使用读优化（ro, read-optimized）系统表。读优化系统表通过仅扫描那些不需要合并的文件来提升读取性能。

对于主键表，ro 系统表只会扫描最高层的文件。这意味着，ro 系统表只会生成最近一次完整压缩后的结果。

由于不同的桶可能在不同的时间点执行完整压缩，因此不同键的值可能来源于不同的快照。对于追加表，因为所有文件都能在不合并的情况下被读取，所以 ro 系统表的行为类似于常规的追加表。

```
SELECT * FROM my_table$ro;
```

07

**Tags Table**

可以通过 tags 表查询表的标签历史信息，其中包括标签基于的快照及这些快照的历史信息。也可以获取所有的标签名称，并通过标签名称进行时间旅行，即回溯到特定标签对应的数据状态。

```
SELECT * FROM my_table$tags;
/*+----------+-------------+-----------+-------------------------+--------------+--------------+| tag_name | snapshot_id | schema_id |             commit_time | record_count |   branches   |+----------+-------------+-----------+-------------------------+--------------+--------------+|     tag1 |           1 |         0 | 2023-06-28 14:55:29.344 |            3 |      []      ||     tag3 |           3 |         0 | 2023-06-28 14:58:24.691 |            7 |  [branch-1]  |+----------+-------------+-----------+-------------------------+--------------+--------------+2 rows in set*/
```

08

**Branches Table**

可以查询表的分支信息。这允许查看表的不同开发或测试路径，每个分支可能包含不同的数据版本或实验性更改。

```
SELECT * FROM my_table$branches;
/*+----------------------+---------------------------+--------------------------+-------------------------+|          branch_name |          created_from_tag |    created_from_snapshot |             create_time |+----------------------+---------------------------+--------------------------+-------------------------+|              branch1 |                    tag1   |                        2 | 2024-07-18 20:31:39.084 ||              branch2 |                    tag2   |                        5 | 2024-07-18 21:11:14.373 |+----------------------+---------------------------+--------------------------+-------------------------+2 rows in set*/
```
