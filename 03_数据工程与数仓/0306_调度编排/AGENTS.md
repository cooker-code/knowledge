## 用户认知重点

| 项 | 内容 |
|---|---|
| 已知基础 | 用户知道 Airflow、DolphinScheduler、补数、依赖等概念，不重复解释 |
| 待补边界 | 调度编排与计算引擎、数据质量、资源调度、工作流 Agent 之间的边界 |
| 易偏差点 | 标题易用"自动化、智能、AI"包装调度问题；不能按关键词抢分类 |

## 排重指纹

```text
调度编排 + DAG/依赖/补跑/重试/资源/告警模块 + 核心机制 + 解决问题 + 适用边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是补跑 | 按失败触发、策略选择、资源隔离、幂等断点、观测复盘拆分 |
| 都是调度系统介绍 | 没有 DAG、调度器、执行器、资源和失败恢复机制时只略读 |
| 都是 Airflow/DolphinScheduler | 按技术本体归调度编排，除非主问题转为 MLOps 或通用系统架构 |
| 只讲参数配置 | 没有失败模式和验证指标时不新建核心知识点 |
| 有生产恢复链路 | 优先沉淀为实践候选，仍需可运行/可验证/可排障/可迁移四个门槛 |

## 已覆盖问题（按三级节点）

| 节点 | 已覆盖 | 还缺 |
|---|---|---|
| DolphinScheduler | 参数语义、内置参数、全局参数、上下游传参、参数优先级、开源数据平台中的编排边界 | Master/Worker 架构、补数、失败策略、租户队列、资源中心、实时任务调度状态语义 |
| 调度补跑 | 自动化补跑的触发、决策、执行、观测、复盘闭环 | Airflow/DolphinScheduler 的具体实现、补跑状态模型、补跑资源池实操 |

## 待补缺口

| 优先级 | 项 | 为什么补 |
|---|---|---|
| 高 | Airflow | 调度本体常见工具，需明确 DAG、Scheduler、Executor、Worker、Backfill 边界 |
| 高 | DolphinScheduler 补数与失败策略 | 已有技术 index，仍缺生产恢复和失败治理 |
| 高 | 补数与重跑治理 | 数仓生产高频问题，影响 SLA、资源和数据一致性 |
| 高 | 调度资源隔离 | 补跑、正班任务、临时任务的资源冲突需要准则 |
| 中 | 任务依赖与影响分析 | 与血缘、质量、SLA 联动，是调度治理核心 |
