## 排重准则

问题指纹：

```text
数据集成工具 + 全量/增量/Schema/一致性/下游写入模块 + 核心机制 + 解决问题 + 适用边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是 CDC | 按全量快照、增量日志、无锁切换、Schema 演进、下游一致性拆分 |
| 都是 MySQL -> 目标端 | 主问题是采集就归数据集成，主问题是目标端查询/索引才归 OLAP |
| 都是教程 | 没有失败模式、版本边界和验收指标时只精读或略读 |
| 有生产故障链路 | 优先沉淀为实践候选 |
| 只讲工具安装 | 不新建核心知识点 |

## 待补技术和问题
