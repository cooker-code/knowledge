## 三级节点入口

暂无三级节点；当前以二级 `0106_核心知识点/` 承接多模态通用准则。

**建立三级节点的触发条件：**

| 条件 | 说明 |
|---|---|
| 同一模态方向积累 ≥3 篇精读文章 | 如视觉语言模型、语音模型、视频理解各达 3 篇时可独立 |
| 有稳定的工程接入模式可沉淀 | 如图像输入的编码方式、音频分块策略等 |
| 用户有持续使用某类多模态能力的场景 | 否则保持原文锚点层 |

候选三级节点：`视觉语言模型（VLM）`、`语音模型`、`视频理解`

## 用户认知重点

| 项 | 内容 |
|---|---|
| 已知基础 | 了解 LLM 可以处理图像输入（如 GPT-4V、Claude Vision），基本会调 API |
| 待补边界 | 图像分辨率对 token 成本的影响；视频帧采样策略；音频转文字后处理 |
| 易偏差点 | 视觉能力"支持"不等于"可靠"；图文混合输入的 token 消耗常被低估 |
| 优先抽取 | 多模态输入的 token 估算、失效边界、与纯文本任务的成本对比 |

## 排重指纹

```text
模态类型（图/音/视频）+ 处理模块 + 核心机制 + 解决问题 + 成本/精度/延迟边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是 VLM 图像理解 | 区分 OCR/图表解析/场景理解/文档解析的适用边界 |
| 都是 token 成本 | 必须有分辨率、帧数、具体模型版本的实测基线 |
| 都是新模型发布 | 仅留影响选型的模态能力差异，不保留营销内容 |
| 多模态 + RAG 结合 | 归入 `02_Agent与AI工程/0203_RAG与知识库` |
