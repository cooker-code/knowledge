## 一级目录排重准则

| 文章主问题 | 正确落点 |
|---|---|
| 模型为什么会幻觉、长上下文为什么贵、结构化输出为何不稳定 | `0101_模型能力` |
| 如何训练、微调、对齐或用数据改变模型行为 | `0104_微调与训练` |
| 如何部署、压测、路由、缓存或管理模型推理服务 | `0105_模型服务` |
| 模型如何理解图片、语音、视频或跨模态生成 | `0106_多模态` |
| 如何比较模型、设计评测集、做业务选型 | `0107_模型评测` |
| 具体 Agent 框架、工具调用、AI 编程工具或上下文工程 | 转入 `02_Agent与AI工程` |

## 收口规则

- 一级目录不设 `文章/`。
- 有二级目录时，新文章先进入对应二级；二级已有三级节点时，再由二级 `AGENTS.md` 决定是否下沉。
- 模型版本、能力数字、榜单和价格类信息必须有官方文档、论文、发布说明或可复现实测支撑；无证据时只作为待验证锚点。
- 不把“AI 产品体验”直接放进 01，除非正文主问题是模型能力本体。
