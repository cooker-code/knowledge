现在使用Ai生成前端项目也是越来越强大了，只要有想法就可以很轻松的通过Ai去生成一个功能完善的前端项目，但是当我们想把生成的代码真正的发布线上的时候就会变得很麻烦，尤其是对于一些并没有接触过这方面的朋友，这会极大的影响了我们如何去把我们的想法真正的去落地。

为此我在 GitHub 上找到了一个很有意思的工具 PinMe。它真的只需要一条命令就可以把AI 生成的网页或 Vue、React 项目快速、免费部署到 IPFS 上发布

### 一个目标：把部署流程做到不能再简单

PinMe 的定位非常直接：  
用最轻的方式，把前端项目部署到 IPFS 去中心化网络。

项目本身用 TypeScript 写的，整体比较干净，算是小而精的类型。而且部署流程几乎没有学习成本：

* • 不需要服务器

  • 不需要配置 CDN
* • 不需要写 yaml、配环境变量
* • 不需要绑定域名（它自动帮你搞）  
  一句 pinme upload 就能把整个目录丢进 IPFS。

### 内容永久保存

传统部署方式多少都依赖服务提供方，平台挂了、额度用完、收费了，链接就失效了。PinMe 把数据直接丢进 IPFS 网络，这意味着：只要 IPFS 还在，你上传的站点就不会掉。  
对于一些想长期留存的内容，比如工具站点、公开资料、活动页面、作品展示，这种“去中心化存储”比普通托管稳得多。

### 自动生成 ENS 子域名

虽然是部署在IPFS，但是PinMe 上传后会自动生成一个 ENS 子域名，通过 eth.limo 之类的网关直接访问，不需要记那串长哈希。让部署出来的网站可以通过地址优雅访问。

### 交互模式

PinMe 的 CLI 有两种使用方式：

1. 1. 交互模式：一步步选择上传目录、设置配置，新手用起来无压力
2. 2. 命令模式：直接上传指定路径，更符合开发者习惯  
   除此之外，它还自带上传历史记录，所有发过的版本都能查到，删错了也可以补救。小细节做得不错。

### 简单上手，五分钟搞定

安装就一句话：

```
npm install -g pinme
```

或者：

```
yarn global add pinme
```

常用命令也很直观：

• 交互式上传：

```
pinme upload
```

• 上传指定目录：

```
pinme upload ./dist
```

• 查看上传列表：

```
pinme list
```

• 删除文件：

```
pinme rm <IPFS_hash>
```

如果你用 Vite 搭项目，有一个小坑要注意，把路径基准改成相对路径：

```
export default {  
  base: "./",  
}
```

否则构建出来的资源路径可能访问不到。
