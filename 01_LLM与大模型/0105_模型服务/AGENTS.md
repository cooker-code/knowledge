## 用户认知重点

| 项 | 内容 |
|---|---|
| 已知基础 | 了解模型可以本地部署或通过 API 调用，知道 vLLM/Ollama 的基本用途 |
| 待补边界 | 各推理引擎的吞吐/延迟/显存边界；推理网关 vs 直接调用的成本对比 |
| 易偏差点 | 单机基准不等于生产吞吐；"支持 xxx 模型"不等于生产可用 |
| 优先抽取 | 显存估算、批处理策略、量化边界、故障恢复和成本控制 |

## 排重指纹

```text
推理引擎 + 部署/吞吐/显存/成本模块 + 核心机制 + 解决问题 + 规模/版本/量化边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是 vLLM | 区分 PagedAttention、连续批处理、量化（GPTQ/AWQ/FP8）、多 GPU 分片 |
| 都是 Ollama | 区分本地使用和 API 服务化场景，不按"简单好用"合并 |
| 都是推理网关 | 区分负载均衡、模型路由、Key 管理和成本聚合 |
| 只有安装启动命令 | 不新建知识点，只保留原文锚点 |
| Docker/K8s 部署模型服务 | 归入本目录，主问题是模型服务运行形态 |
