即使像Claude这样强大的AI,在面对真实世界复杂多变的任务时,也像一个刚走出校门的毕业生——潜力巨大,但缺乏特定领域的实战知识和流程经验。为了解决这个问题,Anthropic公司推出了一种名为"智能体技能"(Agent Skills)的全新范式,其核心思想简单而深刻:**像为一位新员工准备入职指南一样,通过一个精心组织的文件夹,将专业知识和工具打包传授给AI。**

这套方法旨在将AI从一个什么都懂一点的"通才",转变为能够胜任特定岗位、融入特定团队工作流的"专家",并且整个过程无需为每个场景都进行复杂的定制开发。

### 使用方式

Claude、Claude code、Claude api现在都支持了，最简单的方式就是使用Claude，在设置里面打开并使用系统预置的一些skills，也可以通过内置的skills-creator创建自己的skills。

### 像入职新员工一样,为AI装备"技能包"

所谓"技能",本质上就是一个包含了说明文件、参考资料、甚至可执行代码的文件夹。它的核心是一个名为 `SKILL.md` 的Markdown文件,这构成了AI学习和使用该技能的基础。

一个技能就是一个包含SKILL.md文件的目录,其中包含指令、脚本和资源,为AI智能体提供额外能力。

这个 `SKILL.md` 文件必须包含两项关键元数据:`name`(技能名称)和 `description`(技能描述)。当AI启动时,它会加载所有已安装技能的这两个信息。这就像一个工具箱的目录,让AI对自己的能力有一个初步的了解,知道在面对"填写PDF表单"这类任务时,可以去调用"PDF处理"这个技能。

SKILL.md文件的结构,包含名称、描述等元数据,以及与技能具体操作相关的上下文。

### 核心设计:用"渐进式披露"让AI按需学习

"技能"机制最巧妙的设计,在于它采用了"渐进式披露"(Progressive Disclosure)的原则,避免了在任务开始时就将所有信息一股脑塞进AI有限的"记忆"(即上下文窗口)中。这个过程分为三个层次,如同阅读一本组织良好的手册:

1. **第一层(目录):** AI启动时只加载所有技能的`name`和`description`。这提供了最基本的信息,让AI知道"我有哪些技能可用",以及"每个技能大概是干什么的"。
2. **第二层(章节):** 当用户提出的任务与某个技能的描述匹配时,AI会触发该技能,完整读取其核心文件 `SKILL.md` 的内容。此时,AI才开始学习这个技能的具体操作指南和核心知识。
3. **第三层(附录):** 如果 `SKILL.md` 中的内容过于复杂,或者某些信息仅在特定子任务中才需要,开发者可以将其拆分到其他文件中(如 `reference.md` 或 `forms.md`),并在主文件中引用。AI只有在处理到相关步骤时,才会去读取这些更深层次的参考文件。

技能中上下文的渐进式披露机制,AI仅在需要时加载信息。

以一个具体的PDF处理技能为例,该技能旨在赋予Claude原生不具备的PDF表单填写能力。当用户要求填表时:

1. AI的上下文窗口首先包含了系统提示、所有技能的元数据以及用户的请求。
2. AI识别出"PDF技能"是相关的,于是读取 `pdf/SKILL.md` 文件。
3. 在执行过程中,AI发现需要填写表单,根据 `SKILL.md` 的指引,它接着读取了专门讲解表单填写的 `forms.md` 文件。
4. 在获取了所有必要指令后,AI才开始执行用户的填表任务。

此图描绘了当用户消息触发技能时,上下文窗口的变化过程。

这种"按需加载"的机制,意味着一个技能可以包含几乎无限量的上下文信息,因为它不会一次性挤占AI宝贵的注意力资源,从而实现了极高的可扩展性。

### 不止于说明书:当技能包里有了代码工具

"技能"不仅可以包含文本指令,还可以打包预先编写好的代码脚本。这解决了大语言模型的一个固有弱点:对于某些任务,传统代码远比生成式AI更高效、更可靠。例如,让AI通过生成文本来排序一个长列表,远不如直接运行一段排序算法来得快和准。

在PDF技能的例子中,开发者可以包含一个Python脚本,专门用于从PDF文件中提取所有表单字段。当需要时,AI可以直接调用并执行这个脚本,而无需将脚本内容或整个PDF文件加载到上下文中。这种方式保证了结果的确定性和可重复性,为AI的工作流引入了传统代码的严谨与高效。

### Agent vs Skills vs MCP对比

| 特征 | 🤖 AI Agent (智能体) | 🛠️ Agent Skills (智能体技能) | 📄 Model-Context-Protocol (MCP) |
| --- | --- | --- | --- |
| 核心定义 | **执行者 (The Executor)** ：一个能感知、规划并行动以完成目标的AI系统。 | **能力包 (The Capability)** ：一个封装了特定领域知识、指令和工具的、可复用的模块。 | **通信格式 (The Format)** ：一个结构化的信息协议，用于组织和呈现所有提供给模型上下文，确保清晰和可靠。 |
| 核心作用 | **“思考和行动”** 是整个系统的核心，负责决策和执行。 | **“提供专业的工具和知识”** 是Agent可以调用的、标准化的外部能力。 | **“组织和传递信息”** 是Agent与开发者/系统之间沟通的语言和规则，是构建Agent的基础设施。 |
| 实体类比 | **一位建筑师** ，负责规划和指挥整个工程。 | **预制好的高科技模块** （如智能厨房、中央空调系统），可以直接安装到建筑中。 | **建筑蓝图和施工规范** ，它规定了所有信息（如材料清单、电路图）应该如何组织和呈现。 |
| 解决的问题 | 如何让AI**自主完成**复杂任务？ | 如何让Agent**高效、可靠地掌握**特定领域的专业能力？ | 如何让提供给模型的上下文**不混乱、不冲突、易于理解和管理**？ |

**MCP是构建Agent的“骨架和神经系统”，它确保了信息能被清晰地传递。Agent是“大脑”，负责思考和决策。而Skills则是Agent可以调用来完成具体任务的、标准化的“四肢或外部工具”。**

---

展望未来,Anthropic希望AI不仅能使用技能,更能自主创建、编辑和评估技能,将自己成功的行为模式固化为可重用的能力。

"智能体技能"这个看似简单的概念,为我们揭示了AI走向实用化的一个关键路径:与其追求一个无所不能的超级大脑,不如构建一个开放的生态,让无数的专家知识和工作流程能够以标准化的"技能包"形式,被AI轻松学习和调用。

当个人和组织开始大规模创建和分享这些AI"技能"时,一个去中心化的、服务于AI的程序化知识库正在形成。这不禁引出一个更深层的问题:我们未来需要什么样的治理模式,来确保这个新兴的AI能力生态系统的质量、可靠性与安全?
