## 核心问题

这组文章多数是训练基础、模型压缩、框架介绍和工具链能力解释，价值在于补边界，不适合直接当“微调方案”。它们应服务于判断：训练成本来自哪里、压缩会牺牲什么、框架能解决什么、工具调用训练应归到哪里。

## 判断准则

| 主题 | 可吸收结论 | 降权点 |
|---|---|---|
| 参数量与模型大小 | 参数量主要影响权重存储、显存和推理成本，是部署约束字段 | 不能用参数量直接判断能力 |
| PyTorch/Jupyter 基础 | 自动求导、动态图、张量和优化器是训练入门基础 | 对用户长期知识只保留最小链路，不重复教程 |
| ViT/DiT/Transformer | Transformer 可迁移到视觉和扩散建模，但任务机制不同 | 不把视觉/扩散教程硬塞进 LLM 微调 |
| 知识蒸馏 | 软标签保留类别间暗知识，可用于模型压缩和学生模型训练 | 缺任务基线和蒸馏温度时只作机制说明 |
| 训练框架/计图 | 框架价值是算子、分布式、国产硬件适配和复现链路 | 新闻稿式性能数字需官方或实测补证 |
| 工具链训练 | 连续工具调用训练更接近 Agent 工具调用能力 | 长期落点应在 `02_Agent与AI工程/0202_工具调用` |

## 认知校准点

| 校准点 | 处理建议 |
|---|---|
| “训练基础”不等于“微调实践” | 只有能形成数据、训练、评测、部署闭环的内容才进入微调方法准则。 |
| “知识蒸馏”与本仓库的知识蒸馏不同 | 前者是模型压缩训练技术，后者是文章吸收流程，不能混用概念。 |
| “连续调用工具”是 Agent 能力 | 如果后续精读，应迁到工具调用训练或 Agent 评估，不在 0104 扩写。 |
| 新闻稿指标要降权 | 计图、框架、国产硬件适配类文章没有复现环境时只保留追查锚点。 |

## 待验证缺口

- 知识蒸馏需要补温度、损失函数、教师/学生模型规模和任务指标的最小实验。
- 训练框架选型需要对比 PyTorch、DeepSpeed、FSDP、Megatron、Jittor 的实际适用边界。
- 工具链训练若继续积累，应迁到 Agent 工具调用专题，补样本格式和评测方式。
