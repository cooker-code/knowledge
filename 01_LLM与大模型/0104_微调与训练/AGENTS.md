## 排重指纹

```text
微调方法 + 训练阶段（SFT/RLHF/DPO/LoRA）+ 核心机制 + 解决问题 + 数据/成本/效果边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是 LoRA 实现 | 区分 rank、alpha、目标层选择、量化边界和推理对齐 |
| 都是 RLHF | 区分 PPO、DPO、GRPO 等算法差异和奖励模型训练方式 |
| 都是训练框架 | 按 DeepSpeed、FSDP、Megatron-LM 分开，不按"加速"笼统归并 |
| 只给代码片段 | 无机制和失败模式时不新建知识点 |
| 微调 vs RAG 对比 | 归入模型能力或 RAG 节点，不在本目录扩写 |

## 待补缺口

| 优先级 | 项 | 为什么补 |
|---|---|---|
| 高 | SFT 数据构造与质量控制 | 微调效果的核心瓶颈，缺可复用的数据格式和清洗准则 |
| 高 | LoRA 超参选择与效果验证 | 最常用参数高效微调方法，需要 rank/alpha/目标层的判断准则 |
| 高 | 微调 vs RAG vs Prompt 决策树 | 高频选型问题，需要基于成本/效果/维护性的决策框架 |
| 中 | DPO/GRPO 对比 PPO | RLHF 简化路线，适用场景和稳定性差异 |
| 中 | 训练框架选型 | DeepSpeed、FSDP 适用规模和配置边界 |
