# 机器学习知识入口
## 知识点入口

- 本模块先看宏观流程，再看文章：[知识地图](0600_核心知识点/知识地图.md)。
- 新文章必须先归入流程节点，再判断是补充、冲突、不同层次还是降权。
- `文章/` 只保留原文锚点，长期知识必须沉淀到 `0600_核心知识点/`。


## 目的

本目录沉淀机器学习项目从业务问题到模型上线的完整流程。文章不是按来源保存，也不是单篇摘要；新文章进入时，先判断它优化的是哪一个流程节点，再和已有准则对比：补充、冲突、不同层次，或只是低价值重复。

## 应用流程

| 节点 | 问题 | 对应目录 | 文章吸收方式 |
|---|---|---|---|
| ML00 问题定义 | 这是预测、分类、排序、聚类，还是因果/统计分析问题 | 本目录 | 先判断是否真属于机器学习；统计分析和经营分析优先归到数据分析与 BI |
| ML01 数据与标签 | 样本、标签、时间窗口、泄漏、偏差怎么定义 | 0601_特征工程 | 重点吸收标签构造、样本选择、训练/线上一致性 |
| ML02 特征工程 | 特征从哪里来，如何清洗、编码、聚合和选择 | 0601_特征工程 | 只沉淀可复用的特征准则，不保留教程式代码片段 |
| ML03 模型选择 | 规则、统计模型、树模型、深度模型如何选择 | 0602_传统模型 | 按数据量、特征形态、可解释性、延迟和维护成本判断 |
| ML04 训练与调参 | 如何训练、验证、调参、避免过拟合 | 0602_传统模型 | 关注验证集划分、超参搜索、基线模型，而不是只记模型名 |
| ML05 评估 | 指标是否匹配业务目标，离线指标能否解释线上收益 | 0603_模型评估 | 重点吸收指标选择、校准、阈值、A/B 验证和错误分析 |
| ML06 可解释性 | 为什么模型这么判断，是否可被业务接受 | 0604_可解释性 | 吸收 SHAP、特征重要性、局部解释、稳定性风险 |
| ML07 上线与监控 | 模型如何部署、监控、回滚和重训 | 0605_MLOps | 关注数据漂移、特征一致性、模型版本、灰度和回滚 |
| ML08 推荐与预测 | 推荐、时序预测、销量预测等专项场景怎么落地 | 0606_推荐与预测 | 只沉淀场景特有的建模流程和失败模式 |

## 吸收准则

| 情况 | 处理 |
|---|---|
| 只讲模型名或库用法 | 降权为了解，除非补齐训练、评估和失败模式 |
| 只给准确率提升 | 标为证据不足，必须追问数据集、基线、指标和验证方式 |
| 讲统计推断或实验设计 | 优先检查是否应归入数据分析与 BI |
| 讲模型上线、漂移、监控 | 优先进入 MLOps 节点 |
| 讲业务预测或推荐 | 必须写清样本、标签、反馈延迟和线上验证 |
