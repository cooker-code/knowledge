## 一级目录排重准则

| 文章主问题 | 正确落点 |
|---|---|
| 原始业务数据如何变成模型特征 | `0601_特征工程` |
| 选择、训练和调参传统机器学习模型 | `0602_传统模型` |
| 如何度量模型效果、比较模型和避免评估偏差 | `0603_模型评估` |
| 如何解释模型决策、审计风险和沟通业务原因 | `0604_可解释性` |
| 模型如何上线、监控、回滚、追踪版本和处理漂移 | `0605_MLOps` |
| 推荐、排序、用户预测、时序预测等业务建模 | `0606_推荐与预测` |
| 大模型能力、微调、推理服务或多模态模型本体 | 转入 `01_LLM与大模型` |
| 指标分析、实验、归因和经营复盘方法 | 转入 `05_数据分析与BI` |

## 收口规则

- 一级目录不设 `文章/`。
- 只保留能形成算法选择、特征边界、评估陷阱、解释准则、上线风险或业务建模判断的内容。
- 单纯模型清单、榜单、营销性能数字和无数据集基线的经验文章降权处理。
- 预测类文章先判断是机器学习建模问题，还是经营分析/指标分析问题；不要只因出现“预测”就放入 06。
