# 可解释性流程
## 知识点入口

- 本模块先看宏观流程，再看文章：[知识地图](060400_核心知识点/知识地图.md)。
- 新文章必须先归入流程节点，再判断是补充、冲突、不同层次还是降权。
- `文章/` 只保留原文锚点，长期知识必须沉淀到 `060400_核心知识点/`。


## 目的

本目录记录模型判断如何被解释、审计和业务接受。重点不是解释工具本身，而是解释是否能帮助排错、合规、沟通和迭代。

## 流程节点

| 节点 | 要回答的问题 |
|---|---|
| EX01 全局解释 | 哪些特征整体影响最大，是否符合业务常识 |
| EX02 局部解释 | 单个样本为什么被这样预测，是否能定位异常 |
| EX03 稳定性 | 解释在时间、人群和重训后是否稳定 |
| EX04 冲突处理 | 解释结果和业务认知冲突时如何验证 |
| EX05 使用边界 | 解释是辅助排查，不等于因果结论 |

## 文章吸收准则

SHAP、特征重要性等内容必须写清解释对象、适用模型、误用风险和能支持的决策，不能把相关性解释成因果。
