# MLOps 流程
## 知识点入口

- 本模块先看宏观流程，再看文章：[知识地图](060500_核心知识点/知识地图.md)。
- 新文章必须先归入流程节点，再判断是补充、冲突、不同层次还是降权。
- `文章/` 只保留原文锚点，长期知识必须沉淀到 `060500_核心知识点/`。


## 目的

本目录记录模型从训练产物到生产服务的工程流程。重点是版本、特征一致性、监控、漂移、回滚和重训闭环。

## 流程节点

| 节点 | 要回答的问题 |
|---|---|
| MO01 训练流水线 | 数据、特征、模型、参数和指标如何可复现 |
| MO02 模型注册 | 模型版本、元数据、评估结果和审批如何记录 |
| MO03 部署服务 | 批预测、在线服务、嵌入式推理分别如何选择 |
| MO04 监控 | 延迟、错误率、特征分布、预测分布和业务指标如何监控 |
| MO05 漂移与重训 | 数据漂移、概念漂移和性能衰减如何触发处理 |
| MO06 回滚 | 模型异常时如何快速降级、回滚或切换规则 |

## 文章吸收准则

只讲工具平台功能不够；必须把工具映射到训练、注册、部署、监控、重训和回滚节点，才能成为长期知识。
