## 用户认知重点

| 项 | 内容 |
|---|---|
| 已知基础 | 了解模型需要上线部署和持续监控，知道 MLflow 等工具 |
| 待补边界 | 数据漂移 vs 概念漂移 vs 模型退化；实验自动化 vs 生产 MLOps |
| 易偏差点 | 模型上线了就结束；自动跑实验等于可生产；监控只看服务响应 |
| 优先抽取 | 实验追踪、特征版本、模型注册、漂移检测、重训触发、版本回滚 |

## 排重指纹

```text
MLOps 对象 + 生命周期阶段（实验/部署/监控/重训/回滚）+ 核心机制 + 解决问题 + 数据漂移/版本/成本边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是实验自动化 | 只有补充复现、审计、版本、并行搜索或验证集纪律时吸收 |
| 都是模型部署 | 区分批处理、实时推理、A/B 灰度和 Canary 发布 |
| 都是漂移监控 | 区分输入分布漂移、预测分布漂移、概念漂移和标签漂移 |
| 都是 MLflow | 区分实验追踪、模型注册、版本管理和部署 |
| 只讲平台功能 | 无生产边界和失败处理时降权 |

## 待补缺口

| 优先级 | 项 | 为什么补 |
|---|---|---|
| 高 | 漂移检测与重训触发 | 模型上线后持续可用的核心保障 |
| 高 | 模型版本管理与回滚 | 新版本出问题时快速恢复的机制 |
| 中 | 灰度发布与 AB 对比 | 新旧模型并行验证的实施方案 |
