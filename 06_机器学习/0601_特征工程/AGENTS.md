## 三级节点入口

暂无三级节点。当前文章只支撑二级核心知识点，还不足以拆成稳定三级技术节点。

候选三级节点：`样本与标签`、`特征变换与选择`、`特征平台`。

建立三级节点的触发条件：

| 条件 | 说明 |
|---|---|
| 某类特征工程主题积累 >=3 篇精读文章 | 如样本构造、特征变换、特征平台各自 3 篇 |
| 能形成稳定的可复用准则 | 不是只有代码片段或配图教程 |
| 有训练/线上一致性或泄漏防止的可验证经验 | 否则保持二级核心知识点 |

## 用户认知重点

| 项 | 内容 |
|---|---|
| 已知基础 | 了解特征工程的基本概念，知道 OneHot、归一化、特征选择等常用方法 |
| 待补边界 | 样本泄漏、训练/线上特征一致性、目标编码泄漏、时序窗口泄漏 |
| 易偏差点 | 特征越多越好；可视化图替代验证；训练集和测试集分别筛选特征 |
| 优先抽取 | 标签构造、时间窗口、特征泄漏检测、线上线下一致性验证、特征版本 |

## 排重指纹

```text
特征对象 + 处理阶段（样本/标签/变换/选择/存储）+ 核心机制 + 解决问题 + 泄漏/一致性/可维护性边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是预处理清单 | 只有补充泄漏、口径、验证或 Pipeline 边界时才吸收 |
| 都是特征选择 | 区分过滤法、包裹法、嵌入法、交叉验证、线上可得性 |
| 都是实时推荐特征 | 区分短窗口、长序列、状态下沉、在线服务 SLA 和训练/推理一致性 |
| 都是相关性/重要性图 | 没有验证指标和业务解释时只作为降权来源 |
| 只有 sklearn 代码 | 没有设计准则和失败模式时不新建知识点 |

## 待补缺口

| 优先级 | 项 | 为什么补 |
|---|---|---|
| 高 | 训练/线上特征一致性 | 特征计算口径不同是模型落地失败的高频原因 |
| 高 | 目标编码和时序窗口泄漏案例 | 影响离线指标可信度 |
| 中 | 特征平台与特征版本 | 从一次性建模走向可复用生产链路 |
| 中 | 实时特征存储与在线服务边界 | Paimon、KV、Feature Store 的职责不能混在一起 |
