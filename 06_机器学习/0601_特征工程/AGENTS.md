# 特征工程流程
## 知识点入口

- 本模块先看宏观流程，再看文章：[知识地图](060100_核心知识点/知识地图.md)。
- 新文章必须先归入流程节点，再判断是补充、冲突、不同层次还是降权。
- `文章/` 只保留原文锚点，长期知识必须沉淀到 `060100_核心知识点/`。


## 目的

本目录记录从业务数据到可训练特征的流程。核心不是“有哪些特征处理函数”，而是样本、标签、时间窗口、泄漏和训练/线上一致性如何管理。

## 流程节点

| 节点 | 要回答的问题 |
|---|---|
| FE01 样本定义 | 一条训练样本代表什么对象、什么时间点、什么业务状态 |
| FE02 标签构造 | 标签从哪里来，正负样本窗口如何定义，是否存在未来信息泄漏 |
| FE03 特征来源 | 特征来自交易、行为、内容、画像、外部数据还是统计聚合 |
| FE04 特征变换 | 缺失、异常、标准化、分桶、编码、聚合如何处理 |
| FE05 特征选择 | 如何判断特征有效、冗余、泄漏或不稳定 |
| FE06 一致性 | 训练特征和线上特征是否同源、同口径、同时间窗口 |

## 文章吸收准则

新文章先判断补的是哪个节点。只给代码 API 的内容不直接进入核心知识点；能补标签构造、泄漏识别、特征稳定性或线上一致性的内容才值得沉淀。
