## 三级节点入口

候选三级节点：`推荐系统`、`时序预测`。

建立三级节点的触发条件：

| 条件 | 说明 |
|---|---|
| 某类场景积累 >=3 篇精读文章 | 如推荐系统、时序预测各自 3 篇 |
| 能形成稳定的场景特有建模流程 | 不是只有算法介绍或工具清单 |
| 有样本、标签、反馈延迟和线上验证的完整知识 | 否则保持二级核心知识点 |

## 用户认知重点

| 项 | 内容 |
|---|---|
| 已知基础 | 了解协同过滤、Embedding、时序预测和基本建模流程 |
| 待补边界 | 推荐系统召回/排序/重排分层；时序预测的外生变量、回测和反馈延迟 |
| 易偏差点 | 推荐只看点击率；时序预测忽视分布变化；趋势吻合就认为可上线 |
| 优先抽取 | 召回/排序分层、冷启动策略、时序特征构造、线上线下一致性、预测区间 |

## 排重指纹

```text
场景类型（推荐/预测/排序）+ 建模阶段（召回/排序/特征/评估/上线）+ 核心机制 + 解决问题 + 样本/反馈/冷启动/时间切分边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是推荐系统 | 区分召回、粗排、精排、重排、冷启动和反馈延迟 |
| 都是时序预测 | 区分时间窗口、外生变量、多步预测、概率预测、回测和分布漂移 |
| 只讲算法名 | 无场景特有建模流程时不新建知识点 |
| 数据工程趋势预测 | 不属于本目录，降权或路由到数据工程相关目录 |
