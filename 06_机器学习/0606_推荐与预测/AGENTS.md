# 推荐与预测流程
## 知识点入口

- 本模块先看宏观流程，再看文章：[流程化知识点总览](knowledge/06_机器学习/0606_推荐与预测/核心知识点/流程化知识点总览.md)。
- 新文章必须先归入流程节点，再判断是补充、冲突、不同层次还是降权。
- `文章/` 只保留原文锚点，长期知识必须沉淀到 `核心知识点/`。


## 目的

本目录记录推荐、排序、销量预测、时间序列预测等专项机器学习场景。重点是业务目标、样本反馈、时间窗口和线上验证。

## 流程节点

| 节点 | 要回答的问题 |
|---|---|
| RP01 场景定义 | 是推荐、排序、召回、销量预测、需求预测还是风险预测 |
| RP02 反馈信号 | 点击、购买、留存、复购、销量等标签是否可靠 |
| RP03 时间窗口 | 训练窗口、预测窗口、反馈延迟如何定义 |
| RP04 模型与基线 | 规则、统计模型、树模型、深度模型哪个先做基线 |
| RP05 线上验证 | 指标是否能被实验或业务回放验证 |

## 文章吸收准则

业务预测文章必须先写清目标变量、预测粒度、时间窗口和验证指标。没有这些信息的文章只能作为背景，不进入核心知识点。
