# 决策树分裂准则与剪枝边界

## 来源

- [决策树（机器判断依据）学后感](../文章/done-决策树（机器判断依据）学后感.md)

## 核心问题

决策树的价值不在于“像规则一样好懂”这个表层印象，而在于它把特征选择、样本纯度和可解释规则连接起来。使用决策树时要同时看分裂准则、过拟合风险和剪枝策略。

## 判断准则

| 判断项 | 准则 |
|---|---|
| 信息熵 | 度量类别混乱程度，越低表示节点越纯 |
| 信息增益 | 衡量某个特征分裂后减少了多少不确定性，ID3 偏好高增益特征 |
| 信息增益率 | 修正信息增益偏好多取值特征的问题，是 C4.5 的核心思想 |
| 基尼系数 | 衡量随机抽两个样本类别不同的概率，CART 常用；计算简单，工程上常见 |
| 过拟合 | 树过深会记住训练集噪声，测试集表现下降 |
| 预剪枝 | 建树时限制最大深度、最小样本数、最小增益，适合控制复杂度 |
| 后剪枝 | 先长树再删除低价值分支，适合在验证集上比较泛化收益 |

## 认知偏差

| 常见错误认知 | 正确理解 |
|---|---|
| 决策树天然可解释，所以不用验证 | 可解释的是路径规则，不代表规则稳定或泛化 |
| 信息增益最高的特征一定最好 | 信息增益偏好多取值特征，需要用增益率或其他约束修正 |
| 树越深越准确 | 深树通常训练集更准，但更容易过拟合 |
| 剪枝会损失模型能力 | 合理剪枝是在解释性和泛化之间做取舍，常常提升测试集效果 |

## 使用边界

- 小样本、强规则、业务需要解释路径时，决策树适合做基线或规则发现。
- 高维稀疏、复杂非线性且追求效果时，单棵树通常不如随机森林、GBDT、XGBoost 或 LightGBM。
- 如果业务关注稳定规则，必须检查不同训练切分下根节点和主要分支是否稳定。

## 待验证缺口

- 需要补一篇树模型和线性模型、GBDT 的选型对比，尤其是数据量、特征形态和解释性要求如何影响选择。
