## 用户认知重点

| 项 | 内容 |
|---|---|
| 已知基础 | 了解树模型基本原理，知道 XGBoost/LightGBM 是工业界主流 |
| 待补边界 | 树模型 vs 线性模型 vs 深度模型的适用场景；何时调参、何时补特征 |
| 易偏差点 | 默认超参就是最优；验证集 AUC 高等于线上效果好；单棵树规则天然稳定 |
| 优先抽取 | 模型选型决策树、超参调优策略、过拟合诊断、基线建立方法 |

## 排重指纹

```text
模型类型 + 训练阶段（选型/训练/调参/验证/部署）+ 核心机制 + 解决问题 + 数据量/特征形态/延迟/可解释性边界 + 对用户的认知增量
```

| 判断项 | 排重规则 |
|---|---|
| 都是决策树 | 只有补充分裂准则、剪枝、稳定性或业务解释边界时吸收 |
| 都是逻辑回归 | 只有补充二分类标签、概率校准、阈值、Odds Ratio 或业务动作闭环时吸收 |
| 都是 XGBoost/LightGBM | 区分超参含义、树结构、并行策略、内存和速度 |
| 都是模型调参 | 区分网格搜索、随机搜索、贝叶斯优化和早停策略 |
| 只给准确率 | 无数据集、基线、验证方式时降权 |
