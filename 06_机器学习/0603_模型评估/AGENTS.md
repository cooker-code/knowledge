# 模型评估流程
## 知识点入口

- 本模块先看宏观流程，再看文章：[流程化知识点总览](knowledge/06_机器学习/0603_模型评估/核心知识点/流程化知识点总览.md)。
- 新文章必须先归入流程节点，再判断是补充、冲突、不同层次还是降权。
- `文章/` 只保留原文锚点，长期知识必须沉淀到 `核心知识点/`。


## 目的

本目录沉淀模型从离线指标到线上效果的验证流程。评估不是只看准确率，而是判断指标是否服务业务目标，错误是否可解释，线上是否可验证。

## 流程节点

| 节点 | 要回答的问题 |
|---|---|
| ME01 指标选择 | 分类、回归、排序、推荐分别该看什么指标 |
| ME02 阈值与校准 | 阈值如何影响成本收益，概率是否需要校准 |
| ME03 错误分析 | 错误集中在哪些人群、场景、样本类型 |
| ME04 稳定性 | 不同时间、渠道、人群下指标是否稳定 |
| ME05 线上验证 | A/B、灰度、回放或人工审核如何验证离线结论 |

## 文章吸收准则

只报告单个指标提升的文章不能直接采信。必须记录数据集、基线、验证方式、业务指标映射和失败样本。
